# Справочник модуля `pathnorm`

> **Модуль:** `std/pathnorm`  
> **Стабильность:** API нестабилен  
> **Назначение:** Нормализация путей ОС — схлопывание лишних разделителей, разрешение компонентов `.` и `..`, итерация по сегментам пути без промежуточных аллокаций.

---

## Общее описание

Пути к файлам нередко содержат лишний «мусор». Пользовательский ввод, конкатенация строк и кросс-платформенные инструменты систематически порождают пути вида `./foo//bar/../baz`. Перед сравнением путей, передачей их в системные вызовы или выводом в сообщениях об ошибках важно привести их к канонической, чистой форме.

`pathnorm` предоставляет два уровня для этого:

- **Высокий уровень:** `normalizePath` — принимает «грязную» строку пути, возвращает чистую. Это функция на каждый день.
- **Низкий уровень:** `PathIter` + `next` + `hasNext` — итератор без аллокаций, возвращающий индексные границы каждого отдельного компонента пути (сегмента) внутри исходной строки. Используется внутри `normalizePath` и `os.joinPath`, но доступен и тем, кто нуждается в тонкой обработке пути без материализации промежуточных строк.

### Что означает «нормализация»

| Входной паттерн | Что происходит |
|---|---|
| Несколько подряд идущих разделителей (`foo//bar`) | Схлопываются в один (`foo/bar`) |
| Точка текущего каталога (`./foo`) | Компонент `.` отбрасывается (`foo`) |
| Двойная точка родительского каталога (`foo/bar/../baz`) | Предыдущий компонент удаляется (`foo/baz`) |
| Двойная точка в корне (`/..`) | Остаётся как есть; выше корня подняться нельзя |
| Двойная точка без компонента для отмены (`../foo`) | Остаётся как есть |
| Весь путь сводится к пустой строке | Заменяется на `"."` |
| Префикс диска Windows (`C:\…`) | Сохраняется дословно в начале |

Нормализация работает исключительно со строкой — она **не обращается** к реальной файловой системе. Символические ссылки не разрешаются.

### Платформенность

Модуль использует `DirSep` и `AltSep` из `std/private/osseps` — платформенно-нативные константы разделителей. На POSIX `DirSep = '/'`. На Windows (и DOS-подобных системах) `DirSep = '\\'` и `AltSep = '/'`. Все публичные процедуры принимают параметр `dirSep`, позволяющий переопределить разделитель — например, при обработке путей другой платформы.

## Экспортируемые типы

---

### `PathIter`

```nim
type PathIter* = object
  i, prev: int
  notFirst: bool
```

#### Что это такое

Лёгкий объект-курсор для итерации по компонентам строки пути, сегмент за сегментом. `PathIter` хранит всего три целых числа и работает исключительно с оригинальной строкой — он не копирует и не аллоцирует подстроки. Каждый вызов `next` продвигает курсор и возвращает байтовые индексные границы следующего компонента в исходной строке.

#### Поля

Все поля публичны (экспортированы), однако рекомендуется воспринимать их как непрозрачное состояние, управляемое процедурами `next` и `hasNext`. Не устанавливайте их вручную.

| Поле | Тип | Внутренний смысл |
|---|---|---|
| `i` | `int` | Текущая позиция сканирования в строке пути. |
| `prev` | `int` | Позиция до последнего вызова `next`; используется для вычисления границ сегмента. |
| `notFirst` | `bool` | `false` до первого вызова `next`; после устанавливается в `true`, позволяя итератору отличать ведущий разделитель абсолютного пути от последующих разделителей. |

#### Как использовать

Всегда инициализируйте через `default(PathIter)` (или просто `var it: PathIter`, что даёт нулевую инициализацию), затем итерируйте с помощью `hasNext` / `next`:

```nim
import std/pathnorm

let path = "/usr/local/bin"
var it = default(PathIter)
while hasNext(it, path):
  let (a, b) = next(it, path)
  echo path[a..b]
# Вывод:
# /
# usr
# local
# bin
```

## Экспортируемые процедуры

---

### `hasNext`

```nim
proc hasNext*(it: PathIter; x: string): bool
```

#### Что делает

Возвращает `true`, если в строке пути `x` ещё остались необработанные символы — то есть вызов `next` вернёт ещё один компонент. Это условие цикла, используемое совместно с `next`.

Проверка просто сводится к `it.i < x.len`. Содержимое строки не анализируется: хвостовой ряд разделителей без последующего контента всё равно даст `true`, пока `next` не пройдёт их.

#### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `it` | `PathIter` | Курсор итератора (только для чтения в этом вызове). |
| `x` | `string` | Строка пути, по которой итерируемся. |

#### Пример

```nim
import std/pathnorm

var it = default(PathIter)
let p = "a/b"
echo hasNext(it, p)  # → true (ещё не начинали)
discard next(it, p)  # потребляем "a"
echo hasNext(it, p)  # → true  (остаётся "b")
discard next(it, p)  # потребляем "b"
echo hasNext(it, p)  # → false (исчерпан)
```

### `next`

```nim
proc next*(it: var PathIter; x: string): (int, int)
```

#### Что делает

Продвигает курсор `PathIter` к следующему компоненту пути в `x` и возвращает его границы в виде кортежа `(startIndex, endIndex)` — оба индекса включительно — в исходной строке. Чтобы получить текст компонента, используйте `x[startIndex..endIndex]`.

Процедура обрабатывает особый случай ведущего разделителя в абсолютных путях: при самом первом вызове, если путь начинается с разделителя, функция возвращает `(0, 0)`, указывая на сам разделитель (представляет корень `/`). Далее она возвращает каждый сегмент без разделителей. Серии подряд идущих разделителей между сегментами автоматически пропускаются после выдачи каждого компонента.

#### Возвращаемое значение

`(a, b)`, где `x[a..b]` — текст компонента. Для корневого компонента абсолютного пути это `"/"` (или `"\\"` в Windows). Для обычного сегмента — текст сегмента без разделителей.

#### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `it` | `var PathIter` | Курсор итератора; продвигается на месте. |
| `x` | `string` | Строка пути, по которой итерируемся. |

#### Примеры

```nim
import std/pathnorm

# Относительный путь
var it = default(PathIter)
let p = "foo/bar/baz"
while hasNext(it, p):
  let (a, b) = next(it, p)
  echo p[a..b]
# foo
# bar
# baz
```

```nim
import std/pathnorm

# Абсолютный путь — первый компонент является корневым разделителем
var it = default(PathIter)
let p = "/etc/hosts"
while hasNext(it, p):
  let (a, b) = next(it, p)
  echo repr(p[a..b])
# "/"
# "etc"
# "hosts"
```

```nim
import std/pathnorm

# Несколько подряд идущих разделителей схлопываются при итерации
var it = default(PathIter)
let p = "a//b///c"
while hasNext(it, p):
  let (a, b) = next(it, p)
  echo p[a..b]
# a
# b
# c
```

```nim
import std/pathnorm

# Пользовательская обработка: подсчёт глубины пути без аллокаций
proc pathDepth(p: string): int =
  var it = default(PathIter)
  while hasNext(it, p):
    let (a, b) = next(it, p)
    if p[a..b] notin ["/", ".", ".."]:
      inc result

echo pathDepth("usr/local/lib")   # → 3
echo pathDepth("/usr/local/lib")  # → 3 (корень "/" не считается)
echo pathDepth("./foo/../bar")    # → 2 (точка и двойная точка считаются как есть)
```

### `normalizePath`

```nim
proc normalizePath*(path: string; dirSep = DirSep): string
```

#### Что делает

Главная высокоуровневая функция модуля. Принимает «грязную» строку пути и возвращает новую, нормализованную строку. Применяет все правила очистки, описанные в обзоре:

- Схлопывает серии подряд идущих разделителей в один.
- Удаляет компоненты `.` (текущий каталог).
- Разрешает компоненты `..` (родительский каталог), удаляя предшествующий компонент пути, если он есть.
- Если весь путь сводится к пустой строке (например, одиночный `"."`), возвращает `"."` вместо пустой строки.
- На Windows/DOS сохраняет букву диска (например, `C:`) неизменной.

Нормализация полностью лексическая — без обращения к файловой системе, без разрешения символических ссылок.

#### Параметры

| Имя | Тип | Умолч. | Описание |
|---|---|---|---|
| `path` | `string` | *(обязателен)* | «Грязный» путь для нормализации. |
| `dirSep` | `char` | `DirSep` | Символ разделителя в выходной строке. По умолчанию — платформенный. Переопределите, чтобы получить POSIX-пути на Windows или наоборот. |

#### Возвращаемое значение

Новая строка с нормализованным путём. Входная строка не изменяется.

#### Примеры

```nim
import std/pathnorm

# Схлопывание двойных слэшей
echo normalizePath("foo//bar")        # → foo/bar
```

```nim
import std/pathnorm

# Удаление точек текущего каталога
echo normalizePath("./foo/./bar")     # → foo/bar
echo normalizePath(".")               # → .
```

```nim
import std/pathnorm

# Разрешение ссылок на родительский каталог
echo normalizePath("foo/bar/../baz")  # → foo/baz
echo normalizePath("/foo/../bar")     # → /bar
```

```nim
import std/pathnorm

# Двойная точка, которую нечем отменить, остаётся на месте
echo normalizePath("../foo")          # → ../foo
echo normalizePath("../../a/b")       # → ../../a/b
```

```nim
import std/pathnorm

# Путь, полностью отменяющий сам себя, становится "."
echo normalizePath("foo/..")          # → .
```

```nim
import std/pathnorm

# Комбинация всего вместе
echo normalizePath("./foo//bar/../baz")  # → foo/baz
```

```nim
import std/pathnorm

# Кросс-платформенно: получить POSIX-путь даже на Windows
echo normalizePath("C:\\Users\\..\\Public", dirSep = '/')
# → C:/Public  (буква диска сохранена, слэши заменены)
```

```nim
import std/pathnorm, std/os

# Практический пример: нормализация пути от пользователя перед передачей в ОС
proc openSafe(userPath: string) =
  let clean = normalizePath(userPath)
  echo "Открываем: ", clean
  # передаём `clean` в open(), readFile() и т. д.

openSafe("./data/../config/app.cfg")
# Открываем: config/app.cfg
```

### `addNormalizePath`

```nim
proc addNormalizePath*(x: string; result: var string; state: var int;
    dirSep = DirSep)
```

> **Примечание:** Документация модуля помечает эту процедуру как *«Low level proc. Undocumented.»* Тем не менее она экспортирована и используется в `os.joinPath`. Описание ниже основано на наблюдаемом поведении.

#### Что делает

Инкрементальный, с сохранением состояния вариант `normalizePath`. Вместо возврата новой строки он **дописывает** нормализованную форму `x` в уже существующую строку `result`, обновляя целое число `state`, которое переносит контекст нормализации между вызовами. Это позволяет соединять и нормализовывать несколько сегментов пути за один проход без повторных аллокаций.

Сама `normalizePath` реализована как однострочник, вызывающий `addNormalizePath` с пустой строкой и нулевым состоянием.

#### Когда использовать

Используйте `addNormalizePath`, когда нужно конкатенировать несколько фрагментов пути и получить нормализованный результат целиком, а не фрагмент за фрагментом. Именно так устроен `os.joinPath` внутри. В прикладном коде предпочитайте `os.joinPath` или `normalizePath`, если только не нужна именно инкрементальная сборка.

#### Кодировка состояния

Целое число `state` — это упакованный битовый флаг:

| Биты | Смысл |
|---|---|
| Бит 0 (`state and 1`) | Установлен в 1, если накопленный путь является абсолютным (начинается с разделителя). |
| Биты 1+ (`state shr 1`) | Счётчик не-корневых, не-dotdot компонентов, накопленных к этому моменту. Каждый обычный компонент добавляет 2; каждый `..`, успешно отменивший предыдущий компонент, вычитает 2. |

`state` должен быть инициализирован в `0` в начале нового пути. Один и тот же `result` и `state` должны использоваться во всех вызовах `addNormalizePath` для одного логического пути.

#### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `x` | `string` | Фрагмент пути для дозаписи и нормализации. |
| `result` | `var string` | Буфер накопления. Передайте пустую строку для начала нового пути. |
| `state` | `var int` | Состояние нормализации. В начале должен быть `0`; передаётся между вызовами. |
| `dirSep` | `char` | Символ разделителя вывода; по умолчанию `DirSep`. |

#### Примеры

```nim
import std/pathnorm

# Воспроизводим то, что normalizePath делает внутри:
var res = ""
var st = 0
addNormalizePath("./foo//bar/../baz", res, st)
echo res  # → foo/baz
```

```nim
import std/pathnorm

# Инкрементальная сборка: дозаписываем несколько фрагментов
var res = ""
var st = 0
addNormalizePath("/usr",   res, st)
addNormalizePath("local",  res, st)
addNormalizePath("../lib", res, st)  # ".." отменяет "local"
echo res  # → /usr/lib
```

```nim
import std/pathnorm

# Состояние сохраняется: флаг абсолютного пути переносится между вызовами
var res = ""
var st = 0
addNormalizePath("/",     res, st)
addNormalizePath("etc",   res, st)
addNormalizePath("hosts", res, st)
echo res        # → /etc/hosts
echo (st and 1) # → 1  (путь абсолютный)
```

## Замечания по дизайну и типичные ловушки

**1. API помечен как нестабильный**  
Заголовок модуля предупреждает: *«Unstable API.»* Типы и сигнатуры процедур могут измениться между релизами Nim. Для кода, который должен оставаться стабильным, рассмотрите копирование или оборачивание только функции `normalizePath`.

**2. Нормализация лексическая, а не семантическая**  
`normalizePath("foo/../bar")` даёт `"bar"` даже если `foo` является символической ссылкой на `/completely/different/directory` — в этом случае реальный разрешённый путь был бы `/completely/different/directory/../bar` = `/completely/different/bar`. Если нужно разрешение с учётом симлинков — используйте `os.expandFilename`.

**3. Состояние `PathIter` привязано к конкретной строке**  
Значение итератора имеет смысл только в паре с той строкой, для которой он был создан. Не переиспользуйте один `PathIter` с другой строкой и не изменяйте строку в процессе итерации.

**4. `next` можно вызывать только когда `hasNext` возвращает `true`**  
Вызов `next` на исчерпанном итераторе (когда `it.i >= x.len`) приведёт к выходу за границы строки. Всегда проверяйте `hasNext` перед вызовом.

**5. Состояние `addNormalizePath` нужно сбрасывать между несвязанными путями**  
Повторное использование переменной `state` от предыдущего пути с новым `result` и новой строкой пути даст неверный результат, потому что счётчик компонентов и флаг абсолютного пути перенесутся некорректно.

**6. Граничный случай пустого пути**  
`normalizePath("")` возвращает `"."`. Если пустой путь имеет особый смысл в вашей предметной области (например, «использовать текущий каталог»), обработайте его до вызова `normalizePath`.

**7. Параметр `dirSep` влияет только на вывод, но не на разбор**  
И `/`, и `\` (и платформенный `AltSep`) всегда распознаются как разделители при разборе, независимо от аргумента `dirSep`. Аргумент управляет только тем, какой символ появляется в возвращаемой строке.
