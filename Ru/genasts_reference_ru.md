# genasts — справочник по модулю

## Обзор

Модуль `std/genasts` предоставляет более чистую и безопасную альтернативу `quote do:` для генерации AST-узлов (Abstract Syntax Tree, дерево синтаксического разбора) внутри макросов Nim. Его центральная идея — **явный захват переменных**: вместо того чтобы полагаться на механизмы гигиены, которые могут быть хрупкими или удивительными, вы явно объявляете, какие рантаймовые значения из тела макроса должны быть встроены в генерируемый код, и под какими именами они там появятся.

### Зачем это нужно — проблема `quote do:`

При написании макросов самым распространённым инструментом для создания AST является `quote do:`. Он хорошо работает для простых случаев, но имеет известную ловушку: любая локальная переменная в теле макроса, упомянутая внутри блока `quote do:`, захватывается *неявно* через синтаксис обратных апострофов (`` `имяПеременной` ``). Забытый апостроф означает, что компилятор ищет символ с этим именем в области видимости *вызывающего кода* — что может молча завершиться успехом с неправильным символом или дать запутанную ошибку.

`genAst` делает захват явным и обязательным. Нельзя случайно обратиться к локальной переменной макроса внутри тела шаблона, не включив её предварительно в список захвата. Это упрощает рассуждения о коде макросов и снижает риск тонких ошибок гигиены.

### Как работает `genAst` внутри

`genAst` — это макрос, который по списку захвата и телу блока строит **локальный шаблон**, параметры которого соответствуют захваченным переменным, немедленно вызывает его с помощью `getAst` и возвращает полученный `NimNode`. Подход через шаблон обеспечивает захват значений без хрупкости `quote do:`.

## `GenAstOpt`

```nim
type GenAstOpt* = enum
  kDirtyTemplate
  kNoNewLit
```

### Описание

Тип-перечисление, используемый как `static set[GenAstOpt]` — аргумент `genAstOpt`. Каждое значение включает определённое изменение поведения. По умолчанию множество пустое (через `genAst`), что является рекомендуемым для большинства случаев.

### Значения

**`kDirtyTemplate`**

По умолчанию `genAst` генерирует *гигиеничный* шаблон (умолчание для шаблонов в Nim). Это означает, что символы, введённые внутри тела шаблона, автоматически получают `gensym` и не могут случайно затенить имена в области видимости вызывающего кода — безопасный выбор.

Установка `kDirtyTemplate` переключает на *грязный* шаблон (`{.dirty.}`), который **не** применяет автоматическое гигиеническое переименование. Символы в теле шаблона разрешаются в области видимости *вызывающего кода*, а не в собственной области шаблона. Это иногда необходимо как обходной путь для краевых случаев ошибок компилятора (например, issue #8220, связанный с `strformat`), но в общем случае следует избегать, поскольку это вновь вводит те самые гигиенические риски, от которых `genAst` был призван защитить.

Используйте `kDirtyTemplate` только тогда, когда у вас есть конкретная, задокументированная причина для грязной семантики.

**`kNoNewLit`**

По умолчанию, когда вы захватываете переменную `a` (или `x = выражение`), `genAst` автоматически оборачивает захваченное значение в `newLit(…)` (через внутренний помощник `newLitMaybe`, который пропускает `newLit` для типов, процедур, итераторов, функций и значений `NimNode`). Это преобразует рантаймовое значение в узел-литерал времени компиляции — именно то, что почти всегда нужно при генерации кода, встраивающего значение.

Установка `kNoNewLit` отключает это оборачивание. Захваченное выражение передаётся в шаблон как есть, без какой-либо конвертации через `newLit`. Используйте это, когда захваченное значение уже является `NimNode`, или когда нужно передать ссылку на символ, а не литерал.

## `genAstOpt`

```nim
macro genAstOpt*(options: static set[GenAstOpt], args: varargs[untyped]): untyped
```

### Описание

Основной макрос модуля. Принимает множество опций времени компиляции, список спецификаций захвата и финальный блок кода Nim, и возвращает `NimNode`, представляющий этот код с подставленными захваченными значениями.

На практике большинство кода использует удобную обёртку `genAst` вместо прямого вызова `genAstOpt`.

### Синтаксис списка захвата

Каждая запись в списке захвата (все аргументы, кроме финального блока) может быть одной из двух форм:

| Форма | Смысл |
|---|---|
| `a` | Захватить локальную переменную макроса `a` под тем же именем `a` в генерируемом коде |
| `a = выражение` | Вычислить `выражение` в теле макроса и сделать его доступным как `a` в генерируемом коде |

Если `kNoNewLit` не установлен, каждое захваченное значение проходит через `newLit` (через `newLitMaybe`), чтобы преобразовать его в AST-узел-литерал. Для типов, значений proc/func/iterator и значений `NimNode` это пропускается автоматически.

### Что доступно без захвата

- **`{.inject.}` процедуры и шаблоны**, определённые локально в теле макроса, видимы внутри блока `genAst` без захвата, поскольку они естественным образом внедряются в область видимости вызывающего кода механизмом шаблонов Nim.
- **`{.gensym.}` переменные** (все привязки `var`, `let` и `const` внутри тела макроса неявно получают `gensym`) должны быть захвачены явно, чтобы быть пригодными к использованию внутри блока.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `options` | `static set[GenAstOpt]` | Множество флагов опций; `{}` для значений по умолчанию |
| `args` | `varargs[untyped]` | Записи списка захвата, за которыми следует тело блока как последний аргумент |

### Возвращаемое значение

`NimNode` — AST тела блока с подставленными захваченными значениями.

### Примеры

**Минимальный пример — без захватов, чистая генерация кода:**

```nim
import std/[macros, genasts]

macro addOne(x: typed): untyped =
  genAst(x):
    x + 1

assert addOne(41) == 42
```

**Захват вычисленного в макросе значения:**

```nim
import std/[macros, genasts]

macro doubled(x: typed): untyped =
  # Вычисляем что-то внутри макроса и хотим запечь это в вывод.
  let factor = 2
  genAst(x, factor):
    x * factor   # `factor` здесь равен 2, запечён как литерал

assert doubled(21) == 42
```

**Переименование при захвате (форма `a = выражение`):**

```nim
import std/[macros, genasts]

macro check2(cond: bool): untyped =
  # cond — всё выражение целиком (например, `a * 2 == a + 3`)
  # Захватываем его под собственным именем, плюс отдельно извлекаем LHS и RHS.
  assert cond.kind == nnkInfix
  result = genAst(cond, s = repr(cond), lhs = cond[1], rhs = cond[2]):
    if not cond:
      raiseAssert "'$#' провалился: lhs='$#', rhs='$#'" % [s, $lhs, $rhs]

let a = 3
check2 a * 2 == a + 3   # проходит
# check2 a * 2 < a + 1  # выдало бы: 'a * 2 < a + 1' провалился: lhs='6', rhs='4'
```

**Передача `static`-параметра макроса как статического значения:**

```nim
import std/[macros, genasts]

macro makeConst(val: static int): untyped =
  genAst(val):
    const generated = val   # val запечён как целочисленный литерал
    generated

assert makeConst(99) == 99
```

**Использование `kDirtyTemplate` как обходного пути (продвинутый уровень):**

```nim
import std/[macros, genasts]

macro withDirty(x: typed): untyped =
  genAstOpt({kDirtyTemplate}, x):
    # Грязный режим: имена разрешаются в области видимости вызывающего.
    # Используйте только при наличии конкретной гигиенической причины.
    x + 1

assert withDirty(10) == 11
```

**Использование `kNoNewLit`, когда захваченное значение уже является `NimNode`:**

```nim
import std/[macros, genasts]

macro wrapNode(body: untyped): untyped =
  # body уже является NimNode; не хотим оборачивать в newLit.
  genAstOpt({kNoNewLit}, body):
    body   # передаётся как есть, без newLit

wrapNode(echo "hello")
```

## `genAst`

```nim
template genAst*(args: varargs[untyped]): untyped
```

### Описание

Удобная обёртка вокруг `genAstOpt`, использующая пустое множество опций (`{}`). Именно за этой версией следует тянуться по умолчанию. Она использует гигиеничный (не грязный) шаблон и автоматически оборачивает захваченные значения в `newLit` через `newLitMaybe`.

Вызов `genAst(захваты..., тело)` полностью эквивалентен `genAstOpt({}, захваты..., тело)`.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `args` | `varargs[untyped]` | Записи списка захвата, за которыми следует тело блока |

### Возвращаемое значение

`NimNode` — AST тела блока с подставленными захваченными значениями.

### Примеры

**Практическая реализация `assert` для иллюстрации:**

```nim
import std/[macros, genasts]

macro myAssert(cond: bool): untyped =
  genAst(cond, msg = "Проверка не пройдена: " & repr(cond)):
    if not cond:
      raise newException(AssertionDefect, msg)

let x = 5
myAssert x > 3   # проходит
# myAssert x > 10  # возбуждает AssertionDefect: "Проверка не пройдена: x > 10"
```

**Генерация типизированной процедуры-обёртки:**

```nim
import std/[macros, genasts]

macro makeLogger(prefix: static string): untyped =
  genAst(prefix):
    proc log(msg: string) =
      echo prefix, ": ", msg

makeLogger("INFO")
log("Сервер запущен")   # выводит: INFO: Сервер запущен
```

**Захват нескольких значений с переименованием:**

```nim
import std/[macros, genasts]

macro between(val, lo, hi: typed): untyped =
  genAst(val, lo, hi, errMsg = "значение вне диапазона"):
    if val < lo or val > hi:
      raise newException(RangeDefect, errMsg)
    val

let result = between(5, 1, 10)   # 5 в диапазоне, возвращается
# between(15, 1, 10)             # возбуждает RangeDefect
```

## `genAst` vs `quote do:` — сравнение

| Аспект | `quote do:` | `genAst` |
|---|---|---|
| Захват переменных | Неявный через `` `имяПеременной` `` | Явный через список захвата |
| Забытый захват | Разрешается в области видимости вызывающего — может молча взять неправильный символ | Ошибка компиляции: символ не найден |
| Гигиена | Гигиеничный по умолчанию | Гигиеничный по умолчанию |
| Встраивание вычисленного значения | `` `val` `` где `val` — `NimNode` | `val` в списке захвата (автоматически `newLit`) |
| Встраивание литерального значения | Нужно вручную вызвать `newLit` и присвоить `NimNode` | Автоматически через `newLitMaybe` |
| Локальные inject-процедуры | Доступны без апострофа | Доступны без захвата |
| Ясность намерений | Низкая (любая локальная переменная доступна через апостроф) | Высокая (список захвата — явный контракт) |

### Когда что использовать

- Используйте **`genAst`**, когда пишете новые макросы, захваченные значения — это обычные Nim-значения (целые, строки, булевы и т.д.), которые должны быть встроены как литералы, или когда нужен самый безопасный и читаемый код макроса.
- Используйте **`quote do:`** для быстрых, простых AST-шаблонов, где вы работаете преимущественно с уже готовыми `NimNode`-значениями (автоматический `newLit` не нужен) и семантика захвата очевидна.
- Используйте **`genAstOpt` с `kNoNewLit`**, когда захватываемые значения сами являются `NimNode` и вы не хотите оборачивания в `newLit`, но всё равно хотите гигиеничную структуру шаблона от `genAst`.
