# Справочник модуля `with`

> Предоставляет макрос `with` для удобного цепочечного вызова функций на одном объекте.  

---

## Какую проблему решает `with`

Когда нужно выполнить несколько вызовов процедур или несколько присваиваний на одной и той же переменной, приходится писать её имя снова и снова:

```nim
# Без with — повторяющийся и шумный код
var s = "Hello"
s.add(", ")
s.add("world")
s.add("!")
```

При длинных именах переменных или глубоко вложенных полях это становится ещё утомительнее. Макрос `with` устраняет повторение: он делает одну переменную неявным первым аргументом для целого блока вызовов.

---

## Экспортируемые символы

### `with` — макрос цепочечных вызовов

```nim
macro with*(arg: typed; calls: varargs[untyped]): untyped
```

**Что делает:** Принимает целевое выражение `arg` и блок операторов `calls`, затем переписывает каждый оператор в блоке так, чтобы `arg` автоматически подставлялся в качестве первого аргумента (или получателя) каждого вызова. Результат — `StmtList`, простая последовательность операторов, точно такая, как если бы вы написали их вручную в развёрнутом виде.

Под капотом `with` делегирует фактическое преобразование функции `underscoredCalls` из приватного модуля `std/private/underscored_calls`. Тот ищет в каждом вызове заполнитель `_`; если `_` отсутствует — автоматически подставляет `arg` первым аргументом.

> **Предостережение:** `arg` вычисляется заново для каждого оператора в блоке. Если `arg` — это выражение с побочными эффектами (например, вызов функции), побочные эффекты произойдут несколько раз. Если это важно — сначала сохраните результат в обычную переменную.

## Паттерны использования

### Паттерн 1 — вызов изменяющих процедур на переменной

Самый распространённый случай. Каждый вызов в блоке получает переменную в качестве первого аргумента:

```nim
import std/with

var s = "yay"
with s:
  add "abc"   # раскрывается в: s.add("abc")
  add "efg"   # раскрывается в: s.add("efg")

doAssert s == "yayabcefg"
```

Это в точности эквивалентно:

```nim
s.add("abc")
s.add("efg")
```

---

### Паттерн 2 — составные операторы присваивания

Вызовы операторов вроде `+=` и `-=` тоже обрабатываются как вызовы и получают `arg` в начало:

```nim
import std/with

var a = 44
with a:
  += 4    # раскрывается в: a += 4
  -= 5    # раскрывается в: a -= 5

doAssert a == 43
```

---

### Паттерн 3 — присваивание полей объектных типов

Обычные присваивания полям тоже переписываются, что делает инициализацию объектов аккуратной:

```nim
import std/with

type Config = object
  host: string
  port: int
  debug: bool

var cfg = Config()
with cfg:
  host  = "localhost"   # раскрывается в: cfg.host = "localhost"
  port  = 8080          # раскрывается в: cfg.port = 8080
  debug = true          # раскрывается в: cfg.debug = true
```

---

### Паттерн 4 — вложенность для вложенных объектов

Блоки `with` могут быть вложены друг в друга — каждый вводит нового неявного получателя для своей внутренней области:

```nim
import std/with

var foo = (bar: 1, qux: (baz: 2))
with foo:
  bar = 2        # foo.bar = 2
  with qux:      # теперь неявный получатель — foo.qux
    baz = 3      # foo.qux.baz = 3

doAssert foo.bar == 2
doAssert foo.qux.baz == 3
```

---

### Паттерн 5 — произвольное сочетание вызовов и присваиваний

Один блок `with` может содержать любую комбинацию вызовов, операторов и присваиваний:

```nim
import std/with

type Builder = object
  parts: seq[string]
  separator: string

proc add(b: var Builder, part: string) = b.parts.add(part)
proc setSep(b: var Builder, sep: string) = b.separator = sep

var b = Builder()
with b:
  add "первый"
  add "второй"
  setSep ", "
  parts.add "третий"   # прямой доступ к полю тоже работает

echo b.parts      # → @["первый", "второй", "третий"]
echo b.separator  # → ", "
```

## Как работает раскрытие

`with` — это исключительно преобразование на этапе компиляции. Никакого дополнительного кода во время выполнения он не генерирует — только сами операторы. Конструкция:

```nim
with someVar:
  foo(x)
  bar
```

превращается в:

```nim
someVar.foo(x)
someVar.bar
```

Если явно использовать заполнитель `_`, он управляет тем, куда именно подставляется `arg`:

```nim
with someVar:
  foo(_, extraArg)   # _ заменяется на someVar
  # → someVar.foo(someVar, extraArg)  -- используйте осторожно
```

Явное использование `_` — продвинутый приём; для обычного цепочечного вызова он не нужен.

## Предостережение о побочных эффектах — подробнее

Поскольку каждый оператор в блоке независимо получает `arg`, и макрос не сохраняет `arg` во временную переменную самостоятельно, любое выражение, переданное в качестве `arg`, вычисляется заново при каждом операторе:

```nim
proc makeObj(): MyObj =
  echo "создан!"    # этот побочный эффект сработает один раз на каждый оператор
  result = MyObj()

with makeObj():      # НЕ делайте так с выражениями с побочными эффектами
  field1 = 1
  field2 = 2
# "создан!" будет напечатано ДВАЖДЫ
```

**Безопасный паттерн:** сначала присвоить, потом использовать `with`:

```nim
var obj = makeObj()   # побочный эффект срабатывает ровно один раз
with obj:
  field1 = 1
  field2 = 2
```

## Сравнение с похожими подходами

| Подход | Запись в Nim | Особенность |
|--------|-------------|-------------|
| Ручное чередование | `s.add("a"); s.add("b")` | Явный, многословный |
| Соглашение `result` | неявный `result` в процедурах | Работает только внутри процедур |
| Макрос `with` | `with s: add "a"; add "b"` | Лаконичный, работает везде, только компиляция |
| Fluent API (цепочки методов) | `s.add("a").add("b")` | Требует соглашения «вернуть себя» в каждой процедуре |

`with` — наиболее универсальный вариант, поскольку не требует никаких изменений в вызываемых процедурах: любая существующая процедура, принимающая целевой тип первым аргументом, работает сразу.

## Быстрая шпаргалка

| Возможность | Поддержка |
|-------------|-----------|
| Вызов изменяющих процедур | ✓ |
| Составные операторы (`+=`, `-=`, …) | ✓ |
| Присваивание полей | ✓ |
| Вложенность (`with` внутри `with`) | ✓ |
| Явный заполнитель `_` | ✓ |
| Только выражения без побочных эффектов | ⚠ требует внимания |
| Накладные расходы во время выполнения | нет (только компиляция) |
