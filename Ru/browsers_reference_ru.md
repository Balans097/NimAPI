# Справочник модуля `browsers`

> **Модуль:** `std/browsers`  
> **Стабильность:** API нестабилен  
> **Назначение:** Кросс-платформенное открытие URL-адресов и локальных файлов в браузере по умолчанию.

---

## Общее описание

Модуль `browsers` предоставляет минималистичный, кросс-платформенный интерфейс для запуска браузера, установленного в системе по умолчанию. Он скрывает за собой специфику каждой ОС: `ShellExecute` на Windows, `open` на macOS, `xdg-open` на Linux/BSD — и позволяет открыть URL одним вызовом.

Модуль намеренно небольшой: он экспортирует одну константу и две процедуры (одна из которых устарела).

---

## Экспортируемые символы

---

### `osOpenCmd`

```nim
const osOpenCmd*: string
```

#### Что это такое

Константа времени компиляции, содержащая имя системной команды, которая открывает файл или URL в ассоциированном приложении. Напрямую вам редко понадобится её использовать, однако она экспортируется, чтобы другие инструменты (скрипты сборки, обёртки над командной оболочкой и т. д.) могли ссылаться на корректное имя команды, не прописывая его вручную.

| Платформа | Значение |
|---|---|
| Windows | `"open"` |
| macOS / Mac OS X | `"open"` |
| Linux, BSD и прочие | `"xdg-open"` |

#### Почему это важно

Если вы пишете инструмент, которому нужно самостоятельно вызвать системный «открыватель» (например, чтобы передать дополнительные флаги), использование `osOpenCmd` делает код переносимым без разбросанных по всему файлу блоков `when defined(...)`.

#### Примеры

```nim
import std/browsers

# Просто посмотрим значение константы (оно фиксируется во время компиляции)
echo "Эта система открывает URL командой: ", osOpenCmd
# Windows/macOS → "open"
# Linux         → "xdg-open"
```

```nim
import std/browsers, std/osproc

# Строим собственную shell-команду, используя ту же утилиту, что и модуль
let cmd = osOpenCmd & " https://nim-lang.org"
discard execShellCmd(cmd)
```

### `openDefaultBrowser(url: string)`

```nim
proc openDefaultBrowser*(url: string)
```

#### Что делает

Открывает указанный URL (или путь к локальному файлу) в браузере по умолчанию. **Вызов не блокирует программу** — он передаёт URL операционной системе и немедленно возвращает управление; программа продолжает работу, пока браузер открывается в фоне.

#### Как работает на каждой платформе

- **Windows** — вызывает `ShellExecuteW`, стандартный WinAPI для открытия документа в ассоциированном приложении.
- **macOS** — выполняет `open <url>` через оболочку. Команда `open` встроена в macOS и уважает настройку браузера по умолчанию.
- **Linux / BSD / прочий Unix** — сначала пробует `xdg-open` (стандарт freedesktop.org). Если это не удаётся (код выхода ≠ 0), процедура перебирает список браузеров из переменной окружения `BROWSER` (значения разделены двоеточием) и запускает первый найденный через `startProcess` (неблокирующий вызов).

#### Важные особенности и ловушки

1. **Пустая строка запрещена.** Передача пустой строки вызывает `doAssert` и завершает программу. Если нужна пустая страница, используйте `"about:blank"` (но смотрите устаревшую перегрузку ниже).
2. **Исключение при ошибке не выбрасывается.** Если открыть браузер не удалось, процедура возвращает управление молча. Ни возвращаемого значения, ни флага ошибки нет. Это намеренное решение разработчиков модуля.
3. **Локальные пути поддерживаются.** Внутренний вспомогательный `prepare` добавляет `"file://"` и делает путь абсолютным, если строка не содержит `"://"`. Однако `openDefaultBrowser` передаёт URL напрямую в `openDefaultBrowserRaw`, минуя `prepare`. Чтобы гарантированно открыть локальный файл, добавьте `file://` самостоятельно.

#### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `url` | `string` | Непустой URL (`https://…`, `http://…`, `file://…`) или путь. Не должен быть пустым. |

#### Примеры

```nim
import std/browsers

# Открываем сайт — браузер появляется, программа сразу продолжает работу
openDefaultBrowser("https://nim-lang.org")
echo "Браузер запущен (или, по крайней мере, мы постарались)."
```

```nim
import std/browsers

# Открываем локальный HTML-отчёт
openDefaultBrowser("file:///home/user/report.html")
```

```nim
import std/browsers, std/os

# Формируем корректный file://-URL из относительного пути
let reportPath = "file://" & absolutePath("build/report.html")
openDefaultBrowser(reportPath)
```

```nim
import std/browsers

# ПЛОХО — это приведёт к ошибке doAssert и завершению программы:
# openDefaultBrowser("")

# ХОРОШО — всегда проверяем URL, если он приходит извне
let url = getEnv("TARGET_URL", "https://example.com")
if url.len > 0:
  openDefaultBrowser(url)
```

### `openDefaultBrowser()` *(устарела)*

```nim
proc openDefaultBrowser*() {.deprecated: "not implemented, please open with a specific url instead".}
```

> **Статус:** Устарела начиная с Nim 1.1. **Не используйте** в новом коде.

#### Что планировалось

Задумывалось открывать браузер по умолчанию на пустой странице в соответствии с [IETF RFC 6694 §3](https://tools.ietf.org/html/rfc6694#section-3), резервирующим `"about:blank"` для этой цели.

#### Что происходит на самом деле (и почему устарела)

Реализация просто вызывает `openDefaultBrowserRaw("about:blank")`, однако результат непредсказуемо отличается от платформы к платформе:

| Платформа | Реальное поведение |
|---|---|
| **Windows** | Появляется диалог с вопросом, какое приложение должно обрабатывать протокол `about:` — браузер на пустую вкладку **не открывается**. |
| **macOS** | Выполняется `open "about:blank"`, браузер корректно открывается на пустой странице. |
| **Linux/BSD** | `xdg-open` сопоставляет `about:blank` с MIME-типом `text/html`, а не `x-scheme-handler/http`, поэтому может открыться просмотрщик HTML, а не браузер по умолчанию. |

Поскольку добиться единообразного поведения на всех платформах нетривиально, а необходимость открывать именно пустую страницу считается редким краевым случаем, команда Nim пометила эту перегрузку устаревшей и рекомендует всегда передавать явный URL.

#### Миграция

```nim
# Было (устарело):
openDefaultBrowser()

# Стало — явно указывайте, что хотите открыть:
openDefaultBrowser("https://example.com")
# Или, если всё же нужна пустая страница (с учётом платформенных оговорок):
openDefaultBrowser("about:blank")
```

## Типичные сценарии использования

### Открыть URL с проверкой корректности

```nim
import std/browsers, std/strutils

proc safeLaunch(url: string) =
  if url.len == 0:
    echo "Предупреждение: попытка открыть пустой URL"
    return
  if not (url.startsWith("https://") or url.startsWith("http://") or
          url.startsWith("file://")):
    echo "Предупреждение: URL без известной схемы: ", url
  openDefaultBrowser(url)

safeLaunch("https://nim-lang.org")
```

### Открыть сгенерированный отчёт после сборки

```nim
import std/browsers, std/os

let outFile = getCurrentDir() / "build" / "index.html"
if fileExists(outFile):
  openDefaultBrowser("file://" & outFile)
else:
  echo "Отчёт не найден по пути: ", outFile
```

## Замечания об обработке ошибок

Поскольку `openDefaultBrowser` не выбрасывает исключений и не возвращает никакого значения, программно узнать, открылся ли браузер, невозможно. Если вам нужны гарантии:

- **Linux/macOS:** убедитесь, что бинарник `xdg-open` / `open` существует до вызова процедуры.
- **Linux:** проверьте, что переменная окружения `BROWSER` задана (на случай отсутствия `xdg-open`).
- В редких случаях можно обернуть вызов в таймаут и проверить побочный эффект — но для большинства задач это избыточно.

Для большинства GUI-приложений и инструментов разработчика молчаливая ошибка вполне приемлема.
