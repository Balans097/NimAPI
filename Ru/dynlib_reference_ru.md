# Справочник модуля `dynlib`

**Модуль:** `dynlib` (часть стандартной библиотеки Nim)  
**Назначение:** Загрузка разделяемых библиотек (`.so`, `.dll`, `.dylib`) во время выполнения программы, поиск символов по имени и их вызов из кода Nim.

---

## Обзор

В большинстве случаев Nim компонует библиотеки *на этапе компиляции* — вы перечисляете их в `.nimble`-файле или передаёте флаги `-l`, и загрузчик ОС связывает всё до старта `main`. Но иногда нужно решить *во время работы программы*, какую именно библиотеку открыть: системы плагинов, опциональные драйверы устройств, бэкенды с выбором языка, горячая перезагрузка и прочее. Именно для этого и существует `dynlib`.

Модуль — тонкая кросс-платформенная обёртка:

| Платформа | Базовый механизм |
|---|---|
| Linux / macOS / большинство POSIX | `dlopen` / `dlsym` / `dlclose` (из `<dlfcn.h>`) |
| Windows | `LoadLibraryA` / `GetProcAddress` / `FreeLibrary` |
| Nintendo Switch | Не поддерживается — бросает `OSError` |
| Genode (без POSIX) | Не поддерживается — бросает `OSError` |

Всё строится вокруг одного непрозрачного типа `LibHandle`, который под капотом является просто `pointer`. Алгоритм работы: открыть библиотеку → получить дескриптор → разрешить символы → закрыть по завершении.

### Типичный рабочий сценарий

```
loadLib / loadLibPattern
        │
        ▼
   LibHandle ──► symAddr / checkedSymAddr ──► cast ──► вызов
        │
        ▼
    unloadLib
```

---

## Экспортируемый API

| Символ | Вид | Краткое описание |
|---|---|---|
| `LibHandle` | псевдоним типа | Непрозрачный дескриптор загруженной библиотеки |
| `loadLib(path, globalSymbols)` | процедура | Загрузить библиотеку по пути к файлу |
| `loadLib()` | процедура | Получить дескриптор самого исполняемого файла |
| `unloadLib(lib)` | процедура | Выгрузить загруженную библиотеку |
| `symAddr(lib, name)` | процедура | Найти символ; возвращает `nil` при неудаче |
| `checkedSymAddr(lib, name)` | процедура | Найти символ; бросает исключение при неудаче |
| `raiseInvalidLibrary(name)` | процедура | Бросить `LibraryError` для отсутствующего символа |
| `libCandidates(s, dest)` | процедура | Раскрыть шаблон имени в список кандидатов |
| `loadLibPattern(pattern, globalSymbols)` | процедура | Загрузить первую библиотеку, подходящую под шаблон |

## `LibHandle`

```nim
type LibHandle* = pointer
```

### Что это такое

`LibHandle` — непрозрачный токен, представляющий успешно загруженную разделяемую библиотеку. Конкретное значение — платформенный указатель (`HMODULE` на Windows, `void*` от `dlopen` на POSIX). Вы никогда не инспектируете его напрямую — только передаёте в `symAddr`, `checkedSymAddr` и `unloadLib`.

Значение `nil` означает «библиотека не загружена» — именно его возвращают функции загрузки при ошибке.

### Пример

```nim
import dynlib

let lib: LibHandle = loadLib("mylib.so")
if lib == nil:
  echo "Библиотека не найдена"
else:
  echo "Загружена успешно"
  unloadLib(lib)
```

## `loadLib(path, globalSymbols)` — загрузка по пути

```nim
proc loadLib*(path: string, globalSymbols = false): LibHandle {.gcsafe.}
```

### Что делает

Открывает разделяемую библиотеку по указанному пути файловой системы `path` и возвращает непрозрачный дескриптор. Возвращает `nil`, если файл не существует, не может быть открыт или имеет неразрешённые зависимости.

**`path`** следует тем же правилам, что и у загрузчика ОС:
- Абсолютный путь (`/usr/lib/libfoo.so`) используется как есть.
- Относительный путь (`./myplugin.so`) разрешается относительно текущей рабочей директории.
- Просто имя (`libfoo.so`) запускает поиск по системным путям ОС (`LD_LIBRARY_PATH`, `PATH` и т.д.).

**`globalSymbols`** (только POSIX, игнорируется на Windows):  
При `false` (по умолчанию) библиотека загружается с `RTLD_NOW` — все символы разрешаются немедленно, но остаются локальными для этого дескриптора (`RTLD_LOCAL`). При `true` — с `RTLD_NOW | RTLD_GLOBAL`: символы становятся видимы для последующих загружаемых библиотек. Это важно, когда плагин рассчитывает на доступность определённых символов глобально (например, он скомпонован с разделяемой средой выполнения, которую вы хотите переиспользовать).

### Когда использовать

Используйте эту процедуру, когда в момент вызова точно известно имя или путь нужной библиотеки. Для загрузки по шаблону или с учётом версии — см. `loadLibPattern`.

### Примеры

```nim
import dynlib

# Пример 1 — абсолютный путь с защитной проверкой
let lib = loadLib("/usr/lib/x86_64-linux-gnu/libz.so.1")
assert lib != nil, "zlib не найден по ожидаемому пути"
unloadLib(lib)

# Пример 2 — просто имя, ОС сама ищет
let lib2 = loadLib("libpng16.so.16")
if lib2 == nil:
  echo "libpng не найден в системе"

# Пример 3 — globalSymbols для плагина, которому нужна общая среда выполнения
let runtime = loadLib("myruntime.so", globalSymbols = true)
let plugin   = loadLib("plugin_a.so")   # теперь видит символы runtime
```

## `loadLib()` — дескриптор самого исполняемого файла

```nim
proc loadLib*(): LibHandle {.gcsafe.}
```

### Что делает

Возвращает дескриптор *текущего исполняемого файла*, а не внешней библиотеки. На POSIX вызывает `dlopen(nil, RTLD_NOW)`, на Windows — `LoadLibraryA(nil)`.

Это позволяет искать символы, скомпилированные непосредственно в сам исполняемый файл, — полезно для реализации таблиц диспетчеризации плагинов, интроспекции или вызова функций по строковым именам.

### Когда использовать

- Интроспекция экспортируемых символов собственного бинарника.
- Реализация моста для скриптовых языков, где скрипты вызывают функции по строковым именам.
- Тестирование `symAddr` / `checkedSymAddr` без внешней библиотеки.

### Пример

```nim
import dynlib

# Ищем функцию, скомпилированную в сам исполняемый файл
let self = loadLib()
assert self != nil

# Предположим, исполняемый файл экспортирует `myRegisteredCallback`
let cb = self.symAddr("myRegisteredCallback")
if cb != nil:
  echo "Нашли собственный callback по адресу: ", cast[int](cb)

unloadLib(self)
```

> **Замечание:** Чтобы символ был виден через дескриптор самого себя на Linux, исполняемый файл должен быть скомпилирован с `-rdynamic` (экспортирует все символы в таблицу динамических символов) или символы должны быть явно помечены для экспорта.

## `unloadLib`

```nim
proc unloadLib*(lib: LibHandle) {.gcsafe.}
```

### Что делает

Уменьшает счётчик ссылок ОС на загруженную библиотеку. Когда счётчик достигает нуля, библиотека убирается из адресного пространства процесса. На POSIX вызывает `dlclose`, на Windows — `FreeLibrary`.

### Важная семантика

- **Подсчёт ссылок.** `loadLib` на одном и том же пути увеличивает счётчик, `unloadLib` — уменьшает. Библиотека физически выгружается только при достижении нуля. Не считайте, что одиночный `unloadLib` немедленно освобождает память, если библиотека была загружена несколько раз.
- **Висячие указатели.** Любые указатели на функции или данные, полученные через `symAddr`, становятся недействительными после выгрузки библиотеки. Использование их — неопределённое поведение.
- **Порядок важен.** Если библиотека B зависит от библиотеки A, выгружайте B до A.

### Пример

```nim
import dynlib

type AddFn = proc(a, b: int): int {.cdecl.}

let lib = loadLib("libmath_ext.so")
assert lib != nil

let add = cast[AddFn](lib.symAddr("add_integers"))
echo add(3, 4)   # 7

unloadLib(lib)   # освобождаем ресурсы
# echo add(1, 2) # ← ОПАСНО: lib выгружена, `add` — висячий указатель
```

## `symAddr`

```nim
proc symAddr*(lib: LibHandle, name: cstring): pointer {.gcsafe.}
```

### Что делает

Ищет в таблице символов загруженной библиотеки символ с именем `name` и возвращает его адрес как сырой `pointer`. Возвращает `nil`, если символ отсутствует.

Возвращаемый указатель может представлять:
- **Функцию** — приводите к нужному типу `proc` через `cast` и вызывайте.
- **Глобальную переменную** — приводите к `ptr T` и разыменовывайте.

### Когда использовать

Используйте `symAddr`, когда отсутствие символа — допустимая, обрабатываемая ситуация (определение опциональных возможностей, проверка совместимости). Для обязательных символов предпочитайте `checkedSymAddr`.

### Пример

```nim
import dynlib

type VersionFn = proc(): cstring {.cdecl.}

let lib = loadLib("libsomething.so")
assert lib != nil

# Опциональная функция: используем только если есть
let getVer = cast[VersionFn](lib.symAddr("get_version"))
if getVer != nil:
  echo "Версия библиотеки: ", getVer()
else:
  echo "Функция версии недоступна в этой сборке"

unloadLib(lib)
```

## `checkedSymAddr`

```nim
proc checkedSymAddr*(lib: LibHandle, name: cstring): pointer
```

### Что делает

Работает точно так же, как `symAddr`, но вместо возврата `nil` при ошибке вызывает `raiseInvalidLibrary(name)`, что бросает исключение `LibraryError` с описательным сообщением.

Это правильный выбор для *обязательных* символов: если символа нет, программа не может продолжать работу осмысленно. Процедура избавляет от повторяющихся проверок `if result == nil: raise ...` после каждого обращения к символу.

### Когда использовать

Используйте `checkedSymAddr` для всех обязательных символов. `symAddr` — только когда вы явно хотите обработать отсутствие символа (опциональные функции, определение версии, адаптеры совместимости).

### Пример

```nim
import dynlib

type
  InitFn  = proc(): cint {.cdecl.}
  CleanFn = proc() {.cdecl.}

let lib = loadLib("libplugin.so")
assert lib != nil

# Эти символы обязаны быть — бросаем исключение если нет
let pluginInit  = cast[InitFn] (lib.checkedSymAddr("plugin_init"))
let pluginClean = cast[CleanFn](lib.checkedSymAddr("plugin_cleanup"))

discard pluginInit()
# ... используем плагин ...
pluginClean()

unloadLib(lib)
```

## `raiseInvalidLibrary`

```nim
proc raiseInvalidLibrary*(name: cstring) {.noinline, noreturn.}
```

### Что делает

Бросает исключение `LibraryError` с сообщением `"could not find symbol: <имя>"`. Помечена `noreturn` — выполнение никогда не продолжается за этим вызовом. Помечена `noinline`, чтобы ветка бросания исключения не раздувала горячий путь `checkedSymAddr`.

### Когда использовать

Напрямую вы будете вызывать эту процедуру редко. Главная её цель — быть единой точкой бросания для `checkedSymAddr`. Однако вы можете использовать её в собственных вспомогательных обёртках, загружающих несколько символов одновременно, чтобы получить единообразные сообщения об ошибках.

### Пример

```nim
import dynlib

proc requireSym(lib: LibHandle, name: string): pointer =
  let p = lib.symAddr(name)
  if p == nil:
    raiseInvalidLibrary(name)   # ← единообразная ошибка, не возвращается
  result = p

let lib = loadLib("libfoo.so")
assert lib != nil
let fn = requireSym(lib, "foo_compute")  # бросает LibraryError если нет
```

## `libCandidates`

```nim
proc libCandidates*(s: string, dest: var seq[string])
```

### Что делает

Раскрывает *шаблон* имени библиотеки (тот же мини-язык, что использует прагма `dynlib`) в список конкретных имён-кандидатов и добавляет их в `dest`.

Язык шаблонов имеет одну конструкцию: `(a|b|c)` — *группа альтернатив*. Всё вне скобок — буквальный префикс или суффикс. Процедура рекурсивно раскрывает все альтернативы, поэтому работают и вложенные группы.

| Шаблон | Производимые кандидаты |
|---|---|
| `libfoo.so` | `libfoo.so` |
| `libfoo(1\|2).so` | `libfoo1.so`, `libfoo2.so` |
| `lib(foo\|bar)(1\|2).so` | `libfoo1.so`, `libfoo2.so`, `libbar1.so`, `libbar2.so` |

### Когда использовать

Вызывайте эту процедуру, когда нужно перечислить кандидатов до попытки загрузить их — например, чтобы отобразить их в сообщении об ошибке, предварительно проверить наличие файлов на диске или реализовать собственную логику загрузки шире, чем предоставляет `loadLibPattern`.

### Пример

```nim
import dynlib

var candidates: seq[string]
libCandidates("libssl(1.0|1.1|3).so", candidates)
echo candidates
# @["libssl1.0.so", "libssl1.1.so", "libssl3.so"]

for name in candidates:
  echo "Попробуем: ", name
```

## `loadLibPattern`

```nim
proc loadLibPattern*(pattern: string, globalSymbols = false): LibHandle
```

### Что делает

Объединяет `libCandidates` и `loadLib` в одном удобном вызове. Раскрывает `pattern` в список кандидатов через `libCandidates` и пытается загрузить каждый по очереди, возвращая первый успешный дескриптор. Возвращает `nil`, если ни один кандидат не загрузился.

Именно это делает прагма `dynlib` внутри, когда вы пишете что-то вроде `{.dynlib: "libssl(1.0|1.1|3).so".}`.

**`globalSymbols`** имеет то же значение, что и в `loadLib`.

> **Предупреждение (из исходника):** Эта процедура использует GC (для построения последовательности кандидатов) и поэтому не может применяться для загрузки самого GC или вызываться до его инициализации.

### Когда использовать

Используйте `loadLibPattern`, когда библиотека может существовать под несколькими версионными именами и вы хотите перебрать их все без написания цикла вручную. Особенно полезно для системных библиотек, чей SONAME варьируется от дистрибутива к дистрибутиву Linux.

### Пример

```nim
import dynlib

# Попробовать libssl 3, затем 1.1, затем 1.0
let ssl = loadLibPattern("libssl(3|1.1|1.0).so")
if ssl == nil:
  quit("OpenSSL не найден в системе")

# Попробовать сначала Windows-имя, затем Linux-имя
when defined(windows):
  let zlib = loadLibPattern("(zlib1|zlib).dll")
else:
  let zlib = loadLibPattern("libz.so(|.1|.1.2)")

assert zlib != nil, "zlib не найден"
```

## Полный рабочий пример — система плагинов во время выполнения

```nim
# компиляция: nim c -r plugin_host.nim

import dynlib

# --- Объявления типов, соответствующих ABI плагина ---
type
  PluginInfo = object
    name: cstring
    version: cint
  GetInfoFn  = proc(): ptr PluginInfo {.cdecl.}
  ProcessFn  = proc(data: ptr UncheckedArray[byte], len: int): int {.cdecl.}

proc loadPlugin(path: string) =
  # 1. Открываем библиотеку
  let lib = loadLib(path)
  if lib == nil:
    echo "Не удалось загрузить плагин: ", path
    return

  # 2. Разрешаем обязательные символы
  let getInfo: GetInfoFn =
    cast[GetInfoFn](lib.checkedSymAddr("plugin_get_info"))
  let process: ProcessFn =
    cast[ProcessFn](lib.checkedSymAddr("plugin_process"))

  # 3. Разрешаем опциональный символ
  type FlushFn = proc() {.cdecl.}
  let flush = cast[FlushFn](lib.symAddr("plugin_flush"))

  # 4. Используем плагин
  let info = getInfo()
  echo "Загружен плагин: ", info.name, " v", info.version

  var buf = [byte 0x01, 0x02, 0x03]
  let result = process(cast[ptr UncheckedArray[byte]](addr buf[0]), buf.len)
  echo "Плагин обработал ", result, " байт"

  if flush != nil:
    flush()
    echo "Плагин сброшен"

  # 5. Освобождаем ресурсы
  unloadLib(lib)

loadPlugin("./plugins/compressor.so")
```

## Сводка по обработке ошибок

| Ситуация | Рекомендуемый подход |
|---|---|
| Файл библиотеки не найден | Проверяйте `nil` от `loadLib` / `loadLibPattern`; выводите понятное сообщение |
| Обязательный символ отсутствует | Используйте `checkedSymAddr`; обрабатывайте `LibraryError` на верхнем уровне |
| Опциональный символ отсутствует | Используйте `symAddr`; пропускайте функцию при `nil` |
| Использование указателей после `unloadLib` | **Никогда** — неопределённое поведение |

## Платформенные замечания

- **`globalSymbols`** молча игнорируется на Windows. Все Windows DLL по умолчанию используют единое плоское пространство имён.
- На **macOS** расширения библиотек обычно `.dylib`; на Linux — `.so` (часто с суффиксом версии, например `.so.6`).
- **Nintendo Switch** и **Genode** (без POSIX): все процедуры бросают `OSError`. Динамическая загрузка на этих платформах не поддерживается.
- Имена символов в C-библиотеках обычно немангленые (просто `add`), но в C++ имена могут быть искажены (`_ZN3Foo3addEii`). Используйте `extern "C"` на стороне C++, чтобы избежать этого.
