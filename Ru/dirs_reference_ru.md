# Справочник по модулю `dirs`

> **Модуль:** `std/dirs`  
> **Язык:** Nim  
> **Назначение:** Кросс-платформенные операции с директориями — создание, удаление, перемещение, копирование и обход файловой системы.

Все пути в этом модуле используют типизированную обёртку `Path` (из `std/paths`) вместо обычных строк. Это делает API надёжнее и выразительнее: система типов не позволяет случайно перепутать путь к файлу с произвольной строкой.

---

## Экспортируемый тип: `PathComponent`

Прежде чем разбирать функции, важно понять `PathComponent` — перечисление (enum), которое классифицирует тип найденного элемента файловой системы. Он реэкспортируется из стандартной библиотеки, поэтому никаких дополнительных импортов не требуется.

| Значение | Смысл |
|---|---|
| `pcFile` | Обычный файл |
| `pcLinkToFile` | Символическая ссылка, указывающая на файл |
| `pcDir` | Обычная директория |
| `pcLinkToDir` | Символическая ссылка, указывающая на директорию |

## Процедуры

### `dirExists`

```nim
proc dirExists*(dir: Path): bool
```

**Что делает:** Проверяет, существует ли директория по указанному пути. Возвращает `true`, если директория найдена, и `false` во всех остальных случаях — в том числе когда путь указывает на файл, а не на директорию. Символические ссылки прозрачно разыменовываются: если ссылка указывает на существующую директорию, результат будет `true`.

Это неразрушающая проверка только для чтения — её безопасно вызывать в любой момент.

**Когда использовать:** Перед попыткой открыть, просмотреть или создать что-то внутри директории; как защитное условие в скриптах, которые не должны продолжать работу при отсутствии нужной директории.

```nim
import std/dirs
import std/paths

let config = Path("/etc/myapp")

if dirExists(config):
  echo "Директория конфигурации найдена, продолжаем."
else:
  echo "Директория конфигурации отсутствует!"
```

### `createDir`

```nim
proc createDir*(dir: Path)
```

**Что делает:** Создаёт директорию, указанную в `dir`, включая все промежуточные родительские директории, которых ещё не существует. По сути это аналог команды `mkdir -p` в Unix.

Если директория уже существует, процедура ничего не делает и **не** выбрасывает ошибку. Такое «идемпотентное» поведение намеренно: в большинстве программ уже существующая директория — это не проблема.

Если создание завершается неудачей по любой другой причине (недостаточно прав, недопустимый путь и т.д.), выбрасывается `OSError`.

**Когда использовать:** Всякий раз, когда нужно убедиться в существовании директории перед записью в неё файлов. Поскольку вложенные уровни создаются автоматически, можно передавать глубоко вложенный путь, не беспокоясь о создании каждого уровня вручную.

```nim
import std/dirs
import std/paths

# Создаёт "output/reports/2024", даже если ни одного уровня ещё нет.
createDir(Path("output/reports/2024"))

# Повторный вызов совершенно безопасен — ошибки не будет.
createDir(Path("output/reports/2024"))
```

### `existsOrCreateDir`

```nim
proc existsOrCreateDir*(dir: Path): bool
```

**Что делает:** Объединяет проверку существования и создание директории в одном вызове. Процедура проверяет, существует ли уже `dir`:

- Если **существует** — возвращает `true` и больше ничего не делает.
- Если **не существует** — создаёт директорию и возвращает `false`.

**Важное ограничение:** В отличие от `createDir`, эта процедура **не** создаёт отсутствующие родительские директории. Если родительская директория не существует, будет выброшен `OSError`. Используйте эту функцию, когда вы знаете, что родитель уже есть, и нужно обеспечить только последний уровень.

**Когда использовать:** Когда важно знать, *была ли директория только что создана* — например, чтобы решить, нужно ли заполнить её начальными данными.

```nim
import std/dirs
import std/paths

let cache = Path("/tmp/myapp/cache")

let alreadyExisted = existsOrCreateDir(cache)
if alreadyExisted:
  echo "Кэш-директория уже была, используем имеющиеся данные."
else:
  echo "Кэш-директория только что создана, начинаем с чистого листа."
```

### `removeDir`

```nim
proc removeDir*(dir: Path, checkDir = false)
```

**Что делает:** Удаляет директорию `dir` вместе со **всем её содержимым** — поддиректориями, файлами, всем — рекурсивно. Это аналог `rm -rf` в Unix.

По умолчанию (`checkDir = false`) попытка удалить несуществующую директорию молча игнорируется. Это упрощает код очистки: можно безусловно удалять директорию, не проверяя её существование заранее.

Если установить `checkDir = true`, при отсутствии директории будет выброшен `OSError`. Используйте это, когда отсутствие директории означает логическую ошибку и не должно замалчиваться.

⚠️ **Операция необратима.** Удалённые файлы и директории восстановить невозможно.

**Когда использовать:** Очистка временных артефактов сборки, тестовых данных, кэш-директорий или любого дерева файлов, которое нужно уничтожить одной командой.

```nim
import std/dirs
import std/paths

let tmp = Path("build/tmp")

# Безопасно даже если "build/tmp" не существует.
removeDir(tmp)

# Выбросит OSError, если директории нет.
removeDir(tmp, checkDir = true)
```

### `moveDir`

```nim
proc moveDir*(source, dest: Path)
```

**Что делает:** Перемещает (переименовывает) директорию из `source` в `dest`. Всё дерево директории переносится на новое место. Если операционная система позволяет, это быстрое атомарное переименование; в противном случае содержимое может быть скопировано, а исходная директория удалена.

Символические ссылки внутри директории перемещаются как есть — перемещаются сами ссылки, а не файлы или директории, на которые они указывают.

При неудаче выбрасывается `OSError`.

**Когда использовать:** Переименование директории, архивирование старой версии папки, атомарная замена одной директории другой.

```nim
import std/dirs
import std/paths

# Переименовываем директорию (или перемещаем в другое место).
moveDir(Path("output/draft"), Path("output/final"))
```

### `walkDir` *(итератор)*

```nim
iterator walkDir*(dir: Path; relative = false, checkDir = false,
                  skipSpecial = false): tuple[kind: PathComponent, path: Path]
```

**Что делает:** Обходит **непосредственное** содержимое директории `dir` — только один уровень, без рекурсии. Для каждого элемента возвращает кортеж:

- `kind` — значение `PathComponent`, указывающее тип элемента (`pcFile`, `pcDir`, `pcLinkToFile`, `pcLinkToDir`).
- `path` — путь к этому элементу.

**Параметры:**

- `relative` — при значении `true` возвращаемые пути будут относительными (только имя элемента без родительских директорий). При `false` (по умолчанию) возвращается полный путь.
- `checkDir` — при `true` выбрасывает `OSError`, если `dir` не существует. При `false` (по умолчанию) итерация просто заканчивается ничего не вернув.
- `skipSpecial` — в Unix при `true` исключает специальные объекты файловой системы (именованные каналы, файлы устройств, сокеты); возвращаются только обычные файлы и директории.

**Когда использовать:** Получение списка прямых потомков директории, построение файлового браузера, обработка только верхнего уровня иерархии.

```nim
import std/dirs
import std/paths

for (kind, path) in walkDir(Path("/home/user/documents")):
  case kind
  of pcFile:    echo "  ФАЙЛ: ", path
  of pcDir:     echo "   ДИР: ", path
  of pcLinkToFile, pcLinkToDir:
                echo "ССЫЛКА: ", path
```

Использование `relative = true`:

```nim
for (kind, path) in walkDir(Path("/home/user/documents"), relative = true):
  echo path   # выводит просто "notes.txt", "projects" и т.д.
```

### `walkDirRec` *(итератор)*

```nim
iterator walkDirRec*(dir: Path,
                     yieldFilter = {pcFile},
                     followFilter = {pcDir},
                     relative = false, checkDir = false,
                     skipSpecial = false): Path
```

**Что делает:** Рекурсивно обходит всё дерево директорий, начиная с `dir`, возвращая `Path` для каждого подходящего элемента. В отличие от `walkDir`, автоматически спускается во вложенные директории.

Два фильтра управляют поведением:

- **`yieldFilter`** — множество значений `PathComponent`, указывающее, *какие элементы возвращать*. По умолчанию `{pcFile}` — возвращаются только обычные файлы. Добавьте `pcDir` для включения директорий, или `pcLinkToFile` / `pcLinkToDir` для символических ссылок.
- **`followFilter`** — множество значений `PathComponent`, указывающее, *в какие директории спускаться*. По умолчанию `{pcDir}` — обходятся только настоящие директории. Добавьте `pcLinkToDir` для следования по символическим ссылкам на директории (осторожно с циклическими ссылками!).

Параметры `relative`, `checkDir` и `skipSpecial` работают так же, как в `walkDir`.

⚠️ **Не изменяйте структуру директорий во время работы итератора.** Добавление или удаление файлов и директорий в процессе обхода приводит к неопределённому поведению.

**Когда использовать:** Рекурсивная обработка всех файлов в дереве проекта, построение индекса файлов по заданным критериям, реализация рекурсивного удаления или копирования вручную.

```nim
import std/dirs
import std/paths

# Вывести все обычные файлы в "src/".
for file in walkDirRec(Path("src")):
  echo file

# Также возвращать встреченные директории.
for entry in walkDirRec(Path("src"), yieldFilter = {pcFile, pcDir}):
  echo entry

# Следовать и по символическим ссылкам на директории.
for entry in walkDirRec(Path("src"),
                         followFilter = {pcDir, pcLinkToDir}):
  echo entry
```

### `setCurrentDir`

```nim
proc setCurrentDir*(newDir: Path)
```

**Что делает:** Изменяет текущую рабочую директорию процесса на `newDir`. Все последующие операции с относительными путями (открытие файлов, просмотр директорий и т.д.) будут разрешаться относительно этой новой директории.

Если `newDir` не существует или не может быть установлена по какой-либо причине, выбрасывается `OSError`.

**Когда использовать:** Когда программа должна переключить контекст работы — например, для выполнения операций относительно корня проекта или для перехода в директорию перед запуском дочернего процесса, который ищет файлы по относительным путям.

```nim
import std/dirs
import std/paths

setCurrentDir(Path("/var/myapp/data"))
# С этого момента относительные пути разрешаются от /var/myapp/data.
```

> **Подсказка:** Чтобы получить текущую рабочую директорию, используйте `getCurrentDir` из `std/paths`.

### `copyDir`

```nim
proc copyDir*(source, dest: Path; skipSpecial = false)
```

**Что делает:** Копирует всё дерево директорий из `source` в `dest`, включая все файлы и поддиректории. Поведение в отношении символических ссылок и прав доступа зависит от платформы:

- **Не-Windows:** Символические ссылки копируются как ссылки (воссоздаётся сама ссылка, а не файл, на который она указывает). Новые файлы и директории получают **права по умолчанию** для вновь создаваемых объектов в системе (с учётом `umask` пользователя). Если нужно сохранить исходные права, используйте `copyDirWithPermissions`.
- **Windows:** Символические ссылки пропускаются. Атрибуты файлов (включая права) копируются автоматически.

Если `skipSpecial = true`, специальные объекты Unix (именованные каналы, файлы устройств, сокеты) исключаются из копирования.

При неудаче выбрасывается `OSError`.

**Когда использовать:** Создание резервной копии директории, дублирование шаблонного проекта, подготовка файлов к развёртыванию, когда сохранение исходных прав не критично.

```nim
import std/dirs
import std/paths

# Копируем шаблон в новую директорию проекта.
copyDir(Path("templates/web"), Path("projects/my-new-site"))
```

### `copyDirWithPermissions`

```nim
proc copyDirWithPermissions*(source, dest: Path;
                              ignorePermissionErrors = true,
                              skipSpecial = false)
```

**Что делает:** Работает как `copyDir`, но дополнительно сохраняет права доступа (биты чтения/записи/выполнения) всех скопированных файлов и директорий. На Windows эквивалентна `copyDir`, поскольку та уже копирует атрибуты.

На не-Windows системах права применяются *после* копирования файла или директории. Это означает, что операция не атомарна: есть краткий момент, когда скопированный файл уже существует, но имеет неверные права. В высоконагруженных или многопоточных сценариях следует учитывать это состояние гонки.

**Параметры:**

- `ignorePermissionErrors` — при `true` (по умолчанию) ошибки чтения или установки прав молча игнорируются. При `false` будет выброшен `OSError` при любой ошибке прав — полезно, когда корректность прав критична.
- `skipSpecial` — то же, что в `copyDir`.

**Когда использовать:** Развёртывание конфигурационных файлов или скриптов, которые должны сохранять бит исполнения; архивирование директорий, где нужно точно воспроизвести структуру прав.

```nim
import std/dirs
import std/paths

# Копируем с сохранением прав; при любой ошибке прав — исключение.
copyDirWithPermissions(
  source = Path("/etc/myapp"),
  dest   = Path("/backup/myapp-config"),
  ignorePermissionErrors = false
)
```

## Краткое сравнение: `createDir` vs `existsOrCreateDir`

| | `createDir` | `existsOrCreateDir` |
|---|---|---|
| Создаёт родительские директории? | ✅ Да | ❌ Нет |
| Возвращает, существовала ли уже? | ❌ Нет | ✅ Да |
| Ошибка, если уже существует? | ❌ Нет | ❌ Нет |
| Лучше для | Обеспечения существования полного пути | Проверки + создания одного уровня |

## Краткое сравнение: `copyDir` vs `copyDirWithPermissions`

| | `copyDir` | `copyDirWithPermissions` |
|---|---|---|
| Копирует файлы и поддиректории | ✅ | ✅ |
| Сохраняет права доступа Unix | ❌ (использует умолчания) | ✅ |
| Работает на Windows | ✅ (копирует атрибуты) | ✅ (то же, что `copyDir` на Windows) |
| Дополнительный параметр | `skipSpecial` | `skipSpecial`, `ignorePermissionErrors` |
