# Модуль Cmdline — справочник

---

## Что делает этот модуль

`cmdline` предоставляет стандартные средства для чтения аргументов командной строки, переданных программе на Nim при запуске. Он отвечает на три вопроса, которые программа обычно задаёт:

- Сколько аргументов было передано? → `paramCount`
- Что представляет собой аргумент номер *N*? → `paramStr`
- Дай мне все аргументы списком → `commandLineParams`

Кроме того, модуль предоставляет `parseCmdLine` — низкоуровневый разборщик, разбивающий сырую строку командной строки на токены по платформозависимым правилам кавычек. Это полезно, когда командная строка хранится в строке, а не получена от ОС напрямую.

---

## Доступность и платформенные ограничения

В отличие от большинства модулей стандартной библиотеки, несколько процедур в `cmdline` **компилируются условно** и могут отсутствовать на некоторых целевых платформах. Это фундаментальное свойство модуля, а не граничный случай.

| Процедура | Когда может отсутствовать |
|-----------|--------------------------|
| `paramCount` | При сборке **динамической библиотеки** (`--app:lib`) на Posix. Портативного способа получить argv из разделяемого объекта не существует. Также отсутствует на Genode (вместо этого бросает `OSError`). |
| `paramStr` | Те же условия, что и для `paramCount`. |
| `commandLineParams` | Те же условия, что и для двух предыдущих; кроме того, при использовании в неподдерживаемом контексте библиотеки вызывает **ошибку компиляции**, а не ошибку времени выполнения. |

Правильный способ работы с этим в переносимом коде — проверять наличие через `declared()`:

```nim
when declared(commandLineParams):
  let args = commandLineParams()
else:
  # запасной вариант для контекста динамической библиотеки
  let args: seq[string] = @[]
```

## Нумерация аргументов

Соглашение об индексах следует модели **C argv**, но с одним отличием от C:

- Индекс `0`: сам исполняемый файл (его путь в том виде, в котором ОС его передала). **Не используйте `paramStr(0)` для получения пути к исполняемому файлу** — вместо этого используйте `getAppFilename()` из `std/os`, который надёжнее.
- Индексы `1` до `paramCount()`: реальные аргументы, набранные пользователем.
- `paramCount()` возвращает `0`, если аргументы не были переданы (а не `-1`, как `argc - 1` в C).

Таким образом, допустимый диапазон для `paramStr` при извлечении пользовательских аргументов: `1..paramCount()`. Передача любого целого числа за пределами диапазона `0..paramCount()` вызывает исключение `IndexDefect`.

---

## Особенность реализации на Windows

На Windows Nim **не опирается** на массив `argv` среды выполнения C. Вместо этого вызывается `GetCommandLineW()` и результат прогоняется через собственный парсер `parseCmdLine`. Это необходимо, поскольку Nim поддерживает GUI-приложения (точка входа `WinMain`), у которых нет доступа к предварительно разобранному argv. Полезным побочным эффектом является то, что поведение разбора аргументов на Windows всегда согласовано вне зависимости от того, каким компилятором C собран бинарник.

Разобранный результат кешируется в `ownArgv` на уровне потока после первого вызова `paramCount` или `paramStr`, поэтому последующие вызовы из того же потока бесплатны.

## Экспортируемые процедуры

---

### `parseCmdLine`

```nim
proc parseCmdLine*(c: string): seq[string]
```

#### Описание

Принимает сырую строку командной строки `c` и разбивает её на последовательность отдельных токенов-аргументов. Правила разбиения различаются между Windows и Posix и применяются во время компиляции в зависимости от целевой платформы.

Процедура **чистая** (помечена `noSideEffect`) — она не взаимодействует с ОС или каким-либо глобальным состоянием. Работает исключительно с входной строкой.

**Когда это полезно?** Когда командная строка находится в строке, которая не пришла напрямую от ОС — например, командная строка из конфигурационного файла, базы данных или полученная по сети. Для разбора реальных аргументов самой программы `commandLineParams` проще и прямолинейнее.

#### Правила разбора — Windows

Правила Windows реализуют [соглашения разбора среды выполнения Microsoft C](https://msdn.microsoft.com/en-us/library/17w5ykft.aspx):

- Аргументы разделяются пробелами или табуляциями.
- Символ вставки (`^`) **не** является экранирующим символом на этом уровне (он обрабатывается `cmd.exe` до того, как программа получает строку).
- Текст в двойных кавычках трактуется как единый аргумент вне зависимости от пробелов внутри.
- Обратный слэш непосредственно перед `"` экранирует кавычку: она воспринимается как литеральный символ `"`, а не разделитель.
- Обратные слэши являются литеральными везде, **кроме** позиции непосредственно перед `"`. При предшествии `"`:
  - Чётное количество обратных слэшей `\\...\\` даёт вдвое меньше литеральных слэшей, а `"` является разделителем.
  - Нечётное количество `\\...\` даёт `(N-1)/2` литеральных слэшей, а `"` становится литеральным символом кавычки.

#### Правила разбора — Posix

- Аргументы разделяются пробельными символами (пробел, табуляция, перенос строки).
- Текст в одинарных кавычках `'...'` является единым аргументом; внутри одинарных кавычек экранирование не выполняется.
- Текст в двойных кавычках `"..."` является единым аргументом; в данной реализации внутри двойных кавычек также не выполняется экранирование (в отличие от настоящего шелла, который обрабатывает `\"` внутри двойных кавычек).
- Экранирования обратным слэшем вне кавычек нет.

#### Параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `c` | `string` | Сырая строка командной строки для разбора. |

#### Возвращаемое значение

`seq[string]` — каждый элемент является одним разобранным токеном аргумента с разрешёнными кавычками и экранированием. Имя исполняемого файла включается в результат, если присутствует в `c` (как первый элемент).

#### Примеры

```nim
import cmdline

# Базовое разбиение
let args = parseCmdLine("hello world foo")
assert args == @["hello", "world", "foo"]

# Строка в кавычках с пробелами → один токен
let args2 = parseCmdLine("""say "hello world" now""")
assert args2 == @["say", "hello world", "now"]

# Пустая строка
let args3 = parseCmdLine("")
assert args3 == @[]

# Множественные пробелы схлопываются (они просто разделители)
let args4 = parseCmdLine("a   b   c")
assert args4 == @["a", "b", "c"]
```

```nim
# Windows: экранирование кавычек обратным слэшем
when defined(windows):
  # \" → литеральная "
  let a = parseCmdLine("""say \"quoted\" word""")
  assert a == @["say", "\"quoted\"", "word"]

  # \\" → один обратный слэш, затем конец строки в кавычках
  let b = parseCmdLine("""path\\" extra""")
  # Одна пара слэшей → один литеральный слэш; " закрывает несуществующую кавычку
  assert b[0] == "path\\"
```

```nim
# Posix: одинарные кавычки сохраняют пробелы дословно
when not defined(windows):
  let a = parseCmdLine("echo 'hello world'")
  assert a == @["echo", "hello world"]

  # Двойные кавычки тоже группируют
  let b = parseCmdLine("""echo "hello world" """)
  assert b == @["echo", "hello world"]
```

### `paramCount`

```nim
proc paramCount*(): int {.tags: [ReadIOEffect].}
```

#### Описание

Возвращает количество аргументов командной строки, переданных программе, **не считая** имени программы (индекс 0). Если программа была вызвана без аргументов, возвращает `0`.

Это эквивалент `argc - 1` в C.

После этого вызова можно безопасно итерировать `for i in 1..paramCount()` и вызывать `paramStr(i)` для каждого индекса.

**Доступность:** Не определена при сборке динамической библиотеки Posix (`--app:lib`). Используйте `declared(paramCount)` для проверки.

#### Возвращаемое значение

Неотрицательное целое число. `0` означает, что аргументы не были переданы.

#### Примеры

```nim
# Вызвано как: ./myapp foo bar baz
import cmdline

echo paramCount()   # → 3
```

```nim
# Вызвано как: ./myapp  (без аргументов)
import cmdline

echo paramCount()   # → 0
```

```nim
# Переносимая проверка
when declared(paramCount):
  echo "Получено аргументов: ", paramCount()
else:
  echo "Работаем как библиотека; argv недоступен"
```

### `paramStr`

```nim
proc paramStr*(i: int): string {.tags: [ReadIOEffect].}
```

#### Описание

Возвращает `i`-й аргумент командной строки в виде строки. Допустимый диапазон для пользовательских аргументов: `1..paramCount()`. Передача индекса за пределами `0..paramCount()` вызывает исключение `IndexDefect`.

Вызов `paramStr(0)` технически возможен и возвращает специфичную для ОС строку, обычно содержащую путь к исполняемому файлу в том виде, в котором он был передан в командной строке. Однако это ненадёжно — возвращаемое значение может быть пустой строкой, относительным или абсолютным путём в зависимости от того, как была запущена программа. Для получения пути к исполняемому файлу всегда используйте `getAppFilename()` из `std/os`.

**Доступность:** Не определена при сборке динамической библиотеки Posix (`--app:lib`). Используйте `declared(paramStr)` для проверки.

#### Параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `i` | `int` | Индекс аргумента. Допустимый диапазон пользовательских аргументов: `1..paramCount()`. |

#### Возвращаемое значение

Строка аргумента на позиции `i`.

#### Исключения

`IndexDefect` возникает, если `i < 0` или `i > paramCount()`.

#### Примеры

```nim
# Вызвано как: ./myapp Alice 42 true
import cmdline

echo paramStr(1)   # → "Alice"
echo paramStr(2)   # → "42"
echo paramStr(3)   # → "true"
```

```nim
import cmdline

# Перебор всех аргументов
for i in 1..paramCount():
  echo "Арг ", i, ": ", paramStr(i)
```

```nim
import cmdline

# Безопасный доступ с проверкой границ
proc getArg(i: int): string =
  if i >= 1 and i <= paramCount():
    paramStr(i)
  else:
    ""
```

### `commandLineParams`

```nim
proc commandLineParams*(): seq[string]
```

#### Описание

Возвращает все аргументы командной строки в виде последовательности строк, **исключая** имя исполняемого файла (индекс 0). Это наиболее удобный способ получить все аргументы сразу.

Внутренне просто вызывает `paramStr(i)` для `i` в диапазоне `1..paramCount()` и собирает результаты. Полностью эквивалентна ручному построению такой последовательности, но читается понятнее.

**Не используйте эту функцию для получения пути к исполняемому файлу.** Для этого вызывайте `getAppFilename()` из `std/os`.

**Доступность:** Не определена при сборке динамической библиотеки Posix (`--app:lib`). В этом контексте использование процедуры вызывает **ошибку компиляции** с сообщением `"commandLineParams() unsupported by dynamic libraries"`. Используйте `declared(commandLineParams)` для защиты от этого.

#### Возвращаемое значение

`seq[string]` — список аргументов, переданных пользователем, по порядку. Пустая последовательность, если аргументы не были переданы.

#### Примеры

```nim
# Вызвано как: ./myapp --verbose input.txt output.txt
import cmdline

let args = commandLineParams()
echo args          # → @["--verbose", "input.txt", "output.txt"]
echo args.len      # → 3
```

```nim
import cmdline

# Простая проверка наличия аргумента
let args = commandLineParams()
if "--help" in args:
  echo "Использование: myapp [опции] <ввод>"
  quit(0)
```

```nim
import cmdline

# Переносимое использование
when declared(commandLineParams):
  let args = commandLineParams()
  for arg in args:
    echo "Обрабатываем: ", arg
else:
  echo "Нет доступа к командной строке в режиме библиотеки"
```

```nim
import cmdline, strutils

# Разбор простого списка аргументов вида ключ=значение
let args = commandLineParams()
for arg in args:
  let parts = arg.split('=', maxsplit = 1)
  if parts.len == 2:
    echo "Ключ: ", parts[0], "  Значение: ", parts[1]
  else:
    echo "Флаг: ", arg
```

## Выбор подходящего инструмента

| Задача | Инструмент |
|--------|-----------|
| Все аргументы списком | `commandLineParams()` |
| Конкретный аргумент по индексу | `paramStr(i)` |
| Количество аргументов | `paramCount()` |
| Разобрать строку командной строки из конфига / сети / базы данных | `parseCmdLine(s)` |
| Разобрать флаги вроде `--verbose`, `-o file`, именованные опции | `std/parseopt` |
| Получить путь к самому исполняемому файлу | `std/os.getAppFilename()` |

---

## Взаимодействие с `std/parseopt`

`commandLineParams` и `parseCmdLine` дают сырые строки. Если программа принимает опции в стандартном Unix-стиле (`--flag`, `-f value`, `--key=value`), используйте `std/parseopt`, который строится поверх этих сырых средств доступа и берёт на себя разбор синтаксиса опций.

```nim
import std/parseopt

var p = initOptParser(commandLineParams())
for kind, key, val in p.getopt():
  case kind
  of cmdLongOption:  echo "Длинная опция: ", key, " = ", val
  of cmdShortOption: echo "Короткая опция: ", key
  of cmdArgument:    echo "Аргумент: ", key
  of cmdEnd:         break
```
