# Справочник модуля `wrapnils`

> Безопасная навигация через nil-указатели и поля вариантных объектов — без явных проверок на nil.  
> **Статус:** Экспериментальный модуль, нестабильный API.

---

## Какую проблему решает `wrapnils`

Глубокий доступ к вложенным ссылочным типам в Nim требует защитных проверок на nil на каждом уровне:

```nim
# Без wrapnils — многоярусная лестница проверок
var result: string
if obj != nil and obj.child != nil and obj.child.inner != nil:
  result = obj.child.inner.name
```

С вариантными объектами ситуация аналогична: обращение к полю, принадлежащему другой ветке дискриминанта, вызывает `FieldDefect` во время выполнения.

`wrapnils` предоставляет два макроса — `?.` и `??.` — которые прозрачно оборачивают любую цепочку обращений к полям и разыменований указателей, перехватывая оба вида опасностей на этапе компиляции без какого-либо шаблонного кода.

---

## Экспортируемые символы

---

### `?.` — безопасная навигация с возвратом значения по умолчанию

```nim
macro `?.`*(a: typed): auto
```

**Что делает:** Преобразует выражение `a` в версию, которая вычисляется безопасно даже когда промежуточные значения цепочки доступа оказываются `nil` или поле вариантного объекта имеет неверный дискриминант. При обнаружении любого такого условия макрос немедленно прекращает вычисление цепочки и возвращает `default(T)`, где `T` — статический тип выражения `a`.

**Ключевые свойства:**

- Тип возвращаемого значения в точности совпадает с типом исходного выражения `a`. Никакого типа-обёртки не вводится.
- `default(T)` означает: `""` для `string`, `0` для числовых типов, `nil` для ссылочных типов, `false` для `bool` и т. д.
- **Семантика lvalue сохраняется** — можно взять адрес результата через `.addr` и записывать значения через полученный указатель.
- Макрос — преобразование на этапе компиляции; сгенерированный код состоит из обычных операторов `if`/`break` без накладных расходов во время выполнения, кроме самих проверок на nil.

**Синтаксис:** Ставьте `?.` непосредственно перед выражением. Скобки позволяют управлять областью действия nil-защиты.

---

#### Базовая nil-безопасность

```nim
import std/wrapnils

type Node = ref object
  value: string
  next: Node

var head: Node   # nil

# Без wrapnils это вызовет панику:
# echo head.next.value  # NilAccessDefect!

echo ?.head.next.value   # → ""  (default(string))
```

---

#### Ограничение области скобками

По умолчанию `?.` защищает всю цепочку справа от себя. Используйте скобки, чтобы сузить защищённую область — всё снаружи скобок вычисляется обычным образом (и может вызвать исключение):

```nim
import std/wrapnils

type Foo = ref object
  x1: string
  x2: Foo
  x3: ref int

var f2 = Foo(x1: "a")
f2.x2 = f2   # цикл: f2.x2 ссылается на сам f2

# ?.() защищает только f2.x2.x2 — здесь цепочка гарантированно не nil
# Внешний .x3[] вычисляется обычным образом после разрешения защищённой части
assert ?.(f2.x2.x2).x3[] == 0

# Для сравнения: ?.  завершается на закрывающей скобке
# Результат (?.f2.x2.x2) имеет тип Foo (ссылка), которая может быть nil
assert (?.f2.x2.x2).x3 == nil   # внешний .x3 не защищён
```

#### Конструкторы внутри выражения

В качестве аргумента допускается любое корректное Nim-выражение:

```nim
import std/wrapnils

type Foo = ref object
  x1: string

assert ?.Foo(x1: "a").x1 == "a"   # конструктор — никогда не nil, работает отлично
```

---

#### Семантика lvalue — запись через безопасный результат

Поскольку `?.` сохраняет семантику lvalue, можно получить изменяемый указатель и писать через него:

```nim
import std/wrapnils

type B = object
  b0: int
  case cond: bool
  of false: discard
  of true:
    b1: float

var b = B(cond: true, b1: 4.5)

# Берём адрес b.b1 безопасно; p будет nil, если поле недоступно
if (let p = ?.b.b1.addr; p != nil):
  p[] = 4.7

assert b.b1 == 4.7
```

#### Безопасность полей вариантного объекта

Обращение к полю вариантного объекта, чей дискриминант указывает на другую ветку, вызывает `FieldDefect` в обычном Nim. `?.` перехватывает это при первом таком обращении и возвращает `default(T)`:

```nim
import std/wrapnils

type B = object
  b0: int
  case cond: bool
  of false: discard
  of true:
    b1: float

var b = B(cond: false, b0: 3)

# Обычный доступ вызывает исключение:
# discard b.b1  # FieldDefect!

assert ?.b.b1 == 0.0   # безопасно — возвращает default(float)

b = B(cond: true, b1: 4.5)
assert ?.b.b1 == 4.5   # теперь поле доступно — возвращается настоящее значение
```

---

#### Неоднозначность значения по умолчанию

`?.` всегда возвращает `default(T)` при неудаче. Это ограничение: если `default(T)` является допустимым значением в вашей предметной области (например, `0` для счётчика или `""` для строки, которая может быть пустой по смыслу), по возвращённому значению невозможно определить, успешно ли вычислилась цепочка. Используйте `??.`, когда это различие важно.

```nim
import std/wrapnils

type Foo = ref object
  x1: ref int

var f1 = Foo(x1: int.new)   # x1 указывает на int со значением 0

assert ?.f1.x1[] == 0   # успех — значение равно 0
# но также:
var f2: Foo
assert ?.f2.x1[] == 0   # неудача (f2 — nil), default тоже 0
# Два случая неотличимы при использовании ?.
```

### `??.` — безопасная навигация с возвратом `Option`

```nim
macro `??.`*(a: typed): Option
```

**Что делает:** Идентичное преобразование что и `?.`, но оборачивает результат в `std/options.Option[T]` вместо возврата голого значения. Возвращает `some(value)` при успешном вычислении цепочки и `none(T)` при обнаружении nil или неверного дискриминанта.

**Когда использовать `??.` вместо `?.`:**

- Когда `default(T)` является допустимым значением в предметной области и нужно отличить «успешно получили ноль» от «цепочка прервалась».
- Когда вызывающий код должен явно обработать отсутствие значения (показать ошибку, использовать запасной вариант).

```nim
import std/wrapnils, std/options

type Foo = ref object
  x1: ref int
  x2: int

var f1 = Foo(x1: int.new, x2: 2)   # x1 → 0, x2 = 2

# x1[] равно 0 — допустимо, но с ?. неотличимо от неудачи
assert (??.f1.x1[]).get == 0    # значение равно 0
assert (??.f1.x1[]).isSome      # но цепочка была корректной!

assert (??.f1.x2).get == 2

var f2: Foo   # nil
assert not (??.f2.x1[]).isSome  # f2 был nil → none
```

**Сравнение:**

```nim
assert ?.f2.x1[] == 0              # ?.  — возвращает 0, неоднозначно
assert not (??.f2.x1[]).isSome     # ??. — явно говорит «значения нет»
```

### `fakeDot` — доступ к полю через значение типа `Option`

```nim
template fakeDot*(a: Option, b): untyped
```

**Что делает:** Позволяет обратиться к полю `b` значения, обёрнутого в `Option`, с сохранением цепочки `none` — аналог опциональной навигации в других языках. Если `a` — `none`, результат — `none` соответствующего типа. Если `a` — `some`, но внутреннее значение является nil-ссылкой, результат тоже `none`. В противном случае возвращается `some(a.get.b)`.

**Почему "fakeDot":** В идеале это должно было бы записываться как `a.b` прямо для `Option`; поскольку Nim не поддерживает это прозрачно, используется явная запись `fakeDot(a, b)`.

```nim
import std/wrapnils, std/options

type Inner = ref object
  value: int

type Outer = ref object
  inner: Inner

var o = Outer(inner: Inner(value: 42))
let optOuter = option(o)

let result = fakeDot(optOuter, inner)   # Option[Inner]
assert result.isSome
assert fakeDot(result, value).get == 42
```

### `[]` (оператор индексирования для `Option`) — две перегрузки

#### `[](a: Option[T], i: I): auto`

```nim
func `[]`*[T, I](a: Option[T], i: I): auto {.inline.}
```

**Что делает:** Применяет оператор индексирования `[i]` к значению внутри `a`. Возвращает `some(a.get[i])`, если `a` — `some`; возвращает `none`, если `a` — `none`. Важно: если `a` — `some`, но контейнер пуст, нормально поднимается `IndexDefect` — этот оператор не подавляет ошибки выхода за границы, только распространяет отсутствие значения.

```nim
import std/wrapnils, std/options

let s = option(@[10, 20, 30])
assert s[1].get == 20

let empty: Option[seq[int]] = none(seq[int])
assert not empty[0].isSome   # none распространяется дальше
```

---

#### `[](a: Option[U]): auto`

```nim
func `[]`*[U](a: Option[U]): auto {.inline.}
```

**Что делает:** Разыменовывает указатель или ссылку внутри `a` (эквивалент `a.get[]`). Возвращает `some(a.get[])`, если `a` — `some` и внутренний указатель не nil; возвращает `none` в противном случае. Позволяет разыменовывать опциональные указатели без дополнительных проверок.

```nim
import std/wrapnils, std/options

var x = new int
x[] = 99
let optPtr = option(x)

assert optPtr[].get == 99   # разыменование через Option

let nilPtr: ref int = nil
let optNil = option(nilPtr)
assert not optNil[].isSome  # внутренний указатель nil → none
```

## Объединение `??.` и вспомогательных `[]`/`fakeDot`

Эти помощники нужны для того, чтобы строить длинные цепочки на значениях типа `Option` после того, как `??.` ввело обёртку `Option`:

```nim
import std/wrapnils, std/options

type Node = ref object
  data: seq[int]
  next: Node

var head = Node(data: @[1, 2, 3], next: Node(data: @[4, 5, 6]))

# Безопасно получить второй элемент head.next.data:
let result = (??.head.next.data)[1]
assert result.get == 5

var nilHead: Node
assert not ((??.nilHead.next.data)[1]).isSome
```

## `?.` против `??.` — руководство по выбору

| Ситуация | Используйте |
|----------|-------------|
| `default(T)` никогда не является допустимым значением (например, nil-указатель, всегда положительный счётчик) | `?.` |
| `default(T)` может быть допустимым значением; нужно отличить успех от неудачи | `??.` |
| Нужно продолжить цепочку доступа к полям на результате | `??.` + `fakeDot` + `[]` |
| Нужно просто получить значение или безопасный запасной вариант без накладных расходов `Option` | `?.` |

## Быстрая шпаргалка

| Символ | Тип | Возвращает | При nil/FieldDefect | Примечания |
|--------|-----|-----------|---------------------|-----------|
| `?.выражение` | макрос | `T` (тот же тип) | `default(T)` | Неоднозначно, если `default(T)` допустимо |
| `??.выражение` | макрос | `Option[T]` | `none(T)` | Однозначно; требует `std/options` |
| `fakeDot(opt, поле)` | шаблон | `Option[ТипПоля]` | `none(...)` | Доступ к полю через `Option` |
| `opt[i]` | func | `Option[ТипЭлемента]` | `none(...)` | Индексирование контейнера в `Option` |
| `opt[]` | func | `Option[ТипРазыменования]` | `none(...)` | Разыменование указателя в `Option` |
