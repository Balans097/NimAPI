# std/intsets — Справочное руководство

## Общее описание

`std/intsets` — это **тонкая обёртка-специализация** вокруг обобщённого модуля `std/packedsets`. Он существует исключительно для обратной совместимости и удобства именования: фиксирует параметр типа на `int`, экспортирует весь API `packedsets` в то же пространство имён и добавляет два сокращённых конструктора, специфичных для `int`.

В современном коде Nim вы можете использовать `std/packedsets` напрямую с любым типом `Ordinal`. Модуль `intsets` существует для того, чтобы код, написанный до обобщения `packedsets`, продолжал компилироваться без изменений.

### Что такое `IntSet` на самом деле

```nim
type IntSet* = PackedSet[int]
```

`IntSet` — это не новый тип, а **псевдоним типа**. Значение `IntSet` побитово идентично `PackedSet[int]`. Каждая процедура, принимающая `PackedSet[int]`, также принимает `IntSet`, и наоборот. Псевдоним лишь предоставляет более короткое и выразительное имя для распространённого случая.

### Структура данных: разреженное битовое множество

`PackedSet` (и, следовательно, `IntSet`) реализован как **разреженное битовое множество** — дерево Патрисии (префиксное дерево) из 64-битных битовых листьев. Это даёт фундаментально иные характеристики производительности по сравнению с хеш-множеством (`HashSet[int]`):

| Свойство | `IntSet` / `PackedSet[int]` | `HashSet[int]` |
|---|---|---|
| Память для плотных диапазонов | Очень мала (1 бит на элемент) | Высокая (один слот на элемент) |
| Память для разреженных значений | Умеренная (узлы дерева) | Пропорционально числу элементов |
| `incl` / `excl` | O(log N) в худшем случае | O(1) амортизированно |
| `contains` | O(log N) в худшем случае | O(1) амортизированно |
| Алгебра множеств (`union` и др.) | O(N) побитово по общим листьям | O(N) поэлементно |
| Порядок итерации | Всегда **по возрастанию** | Произвольный |
| Ограничение типа ключа | **Только Ordinal** | Любой хешируемый тип |

Выбирайте `IntSet` / `PackedSet[int]` когда:
- значения берутся из плотного диапазона (идентификаторы переменных компилятора, индексы инструкций, небольшие перечисления)
- нужна упорядоченная итерация без явного шага сортировки
- операции алгебры множеств (`union`, `difference`, `intersection`) выполняются часто
- важна память при больших плотных множествах

Выбирайте `HashSet[int]` когда:
- значения произвольные, разреженные или отрицательные целые в широком диапазоне
- O(1) среднего поиска важнее памяти или порядка итерации
- нужно хранить `float` или нечисловые типы (здесь неактуально, но полезно знать)

## API, специфичный для `intsets`

Эти две процедуры существуют только в `intsets` и не являются частью `packedsets`.

### `initIntSet`

```nim
proc initIntSet*(): IntSet {.inline.}
```

**Назначение:** создаёт и возвращает пустой `IntSet`. Это конструктор для псевдонима, специфичного для `int` — он полностью эквивалентен `initPackedSet[int]()`, просто имеет более короткое и явное имя.

Начиная с Nim v0.20, переменные `PackedSet`, объявленные как `var`, инициализируются нулями по умолчанию, поэтому явное создание необязательно. Однако вызов `initIntSet()` — хорошая практика, когда намерение должно быть понятно читателю, или при инициализации поля в объектном литерале.

```nim
import std/intsets

var s = initIntSet()
s.incl(1)
s.incl(2)
s.incl(3)
assert s.len == 3
assert 2 in s
```

---

### `toIntSet`

```nim
proc toIntSet*(x: openArray[int]): IntSet {.since: (1, 3), inline.}
```

**Назначение:** создаёт новый `IntSet`, предварительно заполненный каждым элементом из массива или последовательности `x`. Дублирующиеся значения тихо игнорируются — множество содержит каждое различимое значение ровно один раз. Доступно начиная с Nim 1.3.

Это наиболее удобный способ построить `IntSet` из известной коллекции значений в одном выражении.

```nim
import std/intsets

let evens = toIntSet([2, 4, 6, 8, 8, 8])   # дубликаты отброшены
assert evens.len == 4
assert 6 in evens
assert 7 notin evens

# Работает и с seq
let primes = toIntSet(@[2, 3, 5, 7, 11])
assert primes.len == 5
```

## Полный экспортируемый API (через `packedsets`)

Оператор `export packedsets` в `intsets` делает каждый публичный символ из `packedsets` доступным при `import std/intsets`. Ниже приведён полный API с подстановкой `IntSet` вместо `PackedSet[int]`.

---

### Создание и инициализация

#### `initPackedSet` / `initIntSet`

```nim
proc initPackedSet[A](): PackedSet[A]
proc initIntSet(): IntSet          # псевдоним для intsets
```

Возвращает пустое множество. Эквивалентны друг другу для элементного типа `int`.

```nim
var s = initIntSet()
assert s.len == 0
```

#### `toPackedSet` / `toIntSet`

```nim
proc toPackedSet[A](x: openArray[A]): PackedSet[A]
proc toIntSet(x: openArray[int]): IntSet    # псевдоним для intsets
```

Массово строит множество из массива или последовательности. Дубликаты тихо отбрасываются.

```nim
let s = toIntSet([10, 20, 30, 20])   # {10, 20, 30}
assert s.len == 3
```

---

### Добавление и удаление элементов

#### `incl` — включить один элемент

```nim
proc incl[A](s: var PackedSet[A]; key: A)
```

Добавляет `key` в `s`. Если `key` уже присутствует, вызов является no-op — `incl` идемпотентна.

```nim
var s = initIntSet()
s.incl(5)
s.incl(5)   # no-op
assert s.len == 1
```

#### `incl` — включить все элементы другого множества

```nim
proc incl[A](s: var PackedSet[A]; other: PackedSet[A])
```

Добавляет каждый элемент из `other` в `s` на месте. Это изменяющий аналог `union`. Эффективнее, чем вызов `incl` в цикле для больших множеств, так как работает с целыми битовыми листьями.

```nim
var a = toIntSet([1, 2, 3])
let b = toIntSet([3, 4, 5])
a.incl(b)   # a теперь {1, 2, 3, 4, 5}
assert a.len == 5
```

#### `excl` — удалить один элемент

```nim
proc excl[A](s: var PackedSet[A]; key: A)
```

Удаляет `key` из `s`. Если `key` нет в `s`, вызов является no-op — ошибка не возникает.

```nim
var s = toIntSet([1, 2, 3])
s.excl(2)
s.excl(99)   # no-op
assert s.len == 2
assert 2 notin s
```

#### `excl` — удалить все элементы другого множества

```nim
proc excl[A](s: var PackedSet[A]; other: PackedSet[A])
```

Удаляет каждый элемент из `other` из `s` на месте. Изменяющий аналог `difference`.

```nim
var a = toIntSet([1, 2, 3, 4])
let b = toIntSet([2, 4])
a.excl(b)   # a теперь {1, 3}
assert a.len == 2
```

#### `containsOrIncl` — добавить и сообщить о предшествующем присутствии

```nim
proc containsOrIncl[A](s: var PackedSet[A]; key: A): bool
```

Добавляет `key` в `s` и возвращает `true`, если `key` **уже присутствовал** до вызова, или `false`, если `key` был добавлен только что. Объединяет проверку членства и вставку в одну операцию — удобно для циклов дедупликации, где каждое значение нужно обработать ровно один раз.

```nim
var seen = initIntSet()
for n in [1, 2, 1, 3, 2]:
  if not seen.containsOrIncl(n):
    echo "впервые встречаем: ", n
# выводит: 1, 2, 3
```

#### `missingOrExcl` — удалить и сообщить о предшествующем отсутствии

```nim
proc missingOrExcl[A](s: var PackedSet[A]; key: A): bool
```

Удаляет `key` из `s` и возвращает `true`, если `key` **уже отсутствовал** до вызова, или `false`, если `key` присутствовал и был удалён. Логическое зеркало `containsOrIncl`.

```nim
var s = toIntSet([5, 10])
assert s.missingOrExcl(5) == false    # 5 присутствовал, теперь удалён
assert s.missingOrExcl(5) == true     # 5 теперь отсутствует
assert s.missingOrExcl(99) == true    # 99 никогда не было
```

### Запрос членства и размера

#### `contains` / `in` / `notin`

```nim
proc contains[A](s: PackedSet[A]; key: A): bool
```

Возвращает `true`, если `key` является членом `s`. Операторы `in` и `notin` вызывают `contains` под капотом.

```nim
let s = toIntSet([1, 2, 3])
assert s.contains(2)
assert 2 in s
assert 4 notin s
```

#### `len`

```nim
proc len[A](s: PackedSet[A]): int {.inline.}
```

Возвращает количество различных элементов в множестве. O(1) — счётчик поддерживается инкрементально.

```nim
let s = toIntSet([1, 1, 2, 3])
assert s.len == 3
```

#### `card` — псевдоним мощности

```nim
proc card[A](s: PackedSet[A]): int {.inline.}
```

Псевдоним для `len`. Название отражает математический термин *мощность* (кардинальное число) множества — количество его элементов. Оба имени одинаково допустимы; `len` более идиоматично в коде Nim.

```nim
let s = toIntSet([10, 20, 30])
assert s.card == 3
assert s.card == s.len
```

#### `isNil` — проверка пустоты

```nim
proc isNil[A](x: PackedSet[A]): bool {.inline.}
```

Возвращает `true`, если множество не содержит элементов. Несмотря на имя `isNil` (унаследованное от эпохи, когда `IntSet` имел внутренний указатель, который мог быть nil), это чистая проверка пустоты — эквивалентна `s.len == 0`.

```nim
var s = initIntSet()
assert s.isNil          # пусто
s.incl(42)
assert not s.isNil      # не пусто
s.excl(42)
assert s.isNil          # снова пусто
```

### Алгебра множеств — создание новых множеств

Все операторы ниже возвращают **новые множества** и не изменяют аргументы.

#### `union` / `+` — объединение

```nim
proc union[A](s1, s2: PackedSet[A]): PackedSet[A]
proc `+`[A](s1, s2: PackedSet[A]): PackedSet[A]   # псевдоним
```

Возвращает новое множество, содержащее каждый элемент, который есть в `s1`, в `s2` или в обоих.

```nim
let a = toIntSet([1, 2, 3])
let b = toIntSet([3, 4, 5])
assert union(a, b) == toIntSet([1, 2, 3, 4, 5])
assert (a + b).len == 5
```

#### `intersection` / `*` — пересечение

```nim
proc intersection[A](s1, s2: PackedSet[A]): PackedSet[A]
proc `*`[A](s1, s2: PackedSet[A]): PackedSet[A]   # псевдоним
```

Возвращает новое множество, содержащее только элементы, присутствующие **в обоих** `s1` и `s2`.

```nim
let a = toIntSet([1, 2, 3])
let b = toIntSet([2, 3, 4])
assert intersection(a, b) == toIntSet([2, 3])
assert (a * b) == toIntSet([2, 3])
```

#### `difference` / `-` — разность

```nim
proc difference[A](s1, s2: PackedSet[A]): PackedSet[A]
proc `-`[A](s1, s2: PackedSet[A]): PackedSet[A]   # псевдоним
```

Возвращает новое множество, содержащее элементы, которые есть в `s1`, но **не** в `s2`. Порядок операндов имеет значение.

```nim
let a = toIntSet([1, 2, 3])
let b = toIntSet([2, 3, 4])
assert difference(a, b) == toIntSet([1])
assert (a - b) == toIntSet([1])
assert (b - a) == toIntSet([4])   # другой результат
```

#### `symmetricDifference` — симметрическая разность

```nim
proc symmetricDifference[A](s1, s2: PackedSet[A]): PackedSet[A]
```

Возвращает новое множество, содержащее элементы, которые есть в `s1` или в `s2`, но **не в обоих** (операция XOR двух множеств). Операторного сокращения нет.

```nim
let a = toIntSet([1, 2, 3])
let b = toIntSet([3, 4, 5])
assert symmetricDifference(a, b) == toIntSet([1, 2, 4, 5])
```

---

### Отношения между множествами

#### `disjoint` — проверка непересечения

```nim
proc disjoint[A](s1, s2: PackedSet[A]): bool
```

Возвращает `true`, если `s1` и `s2` не имеют общих элементов. Эквивалентно `intersection(s1, s2).len == 0`, но без выделения временного множества.

```nim
let a = toIntSet([1, 2])
let b = toIntSet([3, 4])
let c = toIntSet([2, 5])
assert disjoint(a, b) == true
assert disjoint(a, c) == false   # оба содержат 2
```

#### `<=` — подмножество

```nim
proc `<=`[A](s1, s2: PackedSet[A]): bool
```

Возвращает `true`, если каждый элемент `s1` также есть в `s2`. `s1` и `s2` могут быть равны.

```nim
let a = toIntSet([1, 2])
let b = toIntSet([1, 2, 3])
assert a <= b    # a является подмножеством b
assert b <= b    # множество является подмножеством самого себя
assert not (b <= a)
```

#### `<` — собственное подмножество

```nim
proc `<`[A](s1, s2: PackedSet[A]): bool
```

Возвращает `true`, если `s1` является подмножеством `s2` **и** `s1 != s2` (то есть `s2` имеет строго больше элементов).

```nim
let a = toIntSet([1, 2])
let b = toIntSet([1, 2, 3])
assert a < b          # собственное подмножество
assert not (b < b)    # равные множества не являются собственными подмножествами друг друга
```

#### `==` — равенство

```nim
proc `==`[A](s1, s2: PackedSet[A]): bool
```

Возвращает `true`, если оба множества содержат в точности одни и те же элементы. Порядок не имеет значения (множества не упорядочены).

```nim
assert toIntSet([1, 2, 3]) == toIntSet([3, 1, 2])
assert toIntSet([1, 2, 2]) == toIntSet([1, 2])   # дубликаты игнорируются
```

### Массовые операции

#### `clear` — очистить

```nim
proc clear[A](result: var PackedSet[A])
```

Удаляет все элементы из множества, возвращая его в пустое состояние. Внутренние узлы дерева освобождаются, и множество готово к повторному использованию.

```nim
var s = toIntSet([1, 2, 3])
s.clear()
assert s.len == 0
assert s.isNil
```

#### `assign` *(устарело)*

```nim
proc assign[A](dest: var PackedSet[A]; src: PackedSet[A]) {.deprecated.}
```

Копирует `src` в `dest`. **Устарело** — используйте вместо этого встроенный оператор присваивания `=`, который автоматически выполняет то же самое глубокое копирование.

```nim
var a = initIntSet()
var b = toIntSet([1, 2, 3])
a = b   # предпочтительнее a.assign(b)
assert a == b
```

#### `=copy`

```nim
proc `=copy`[A](dest: var PackedSet[A]; src: PackedSet[A])
```

Хук копирования, вызываемый компилятором. Вызывается автоматически оператором `=`. Вы никогда не должны вызывать его напрямую — документируется здесь только для полноты. Выполняет корректное глубокое копирование внутреннего дерева, чтобы после присваивания `dest` и `src` были полностью независимы.

### Итерация

#### `items`

```nim
iterator items[A](s: PackedSet[A]): A
```

Выдаёт каждый элемент `s` в **порядке возрастания**. Это гарантировано — базовая структура дерева по своей природе производит отсортированный вывод. Изменение множества во время итерации недопустимо и вызовет аварийное завершение.

```nim
let s = toIntSet([30, 10, 20, 10])
for x in s:
  echo x    # выводит 10, затем 20, затем 30 (по возрастанию)
```

Синтаксис `for x in s` вызывает `items` неявно.

---

### Отображение

#### `$`

```nim
proc `$`[A](s: PackedSet[A]): string
```

Возвращает человекочитаемое строковое представление множества, например `{1, 2, 3}`. Элементы всегда перечисляются в порядке возрастания. Вызывается автоматически при `echo`.

```nim
let s = toIntSet([3, 1, 2])
assert $s == "{1, 2, 3}"
```

## Полный пример использования

```nim
import std/intsets

# Построение множества "посещённых" идентификаторов узлов при обходе графа
var visited = initIntSet()

proc dfs(graph: seq[seq[int]], node: int) =
  if visited.containsOrIncl(node):
    return   # уже посещали
  echo "посещаем узел ", node
  for neighbour in graph[node]:
    dfs(graph, neighbour)

let graph = @[
  @[1, 2],   # узел 0 → 1, 2
  @[3],      # узел 1 → 3
  @[3],      # узел 2 → 3
  @[]        # узел 3 (лист)
]

dfs(graph, 0)
# посещаем узел 0
# посещаем узел 1
# посещаем узел 3
# посещаем узел 2

echo "посещённые: ", visited   # {0, 1, 2, 3}
echo "число непосещённых: ", 4 - visited.len

# Алгебра множеств: какие узлы НЕ были посещены?
let allNodes = toIntSet([0, 1, 2, 3, 4, 5])
let unvisited = allNodes - visited
echo "недостижимые: ", unvisited   # {4, 5}
```
