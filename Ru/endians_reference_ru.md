# Справочник модуля `endians`

> **Стандартная библиотека Nim — вспомогательные функции порядка байт**
> Статус API: **нестабильный (Unstable)**.

---

## Общее описание

Каждое многобайтовое целое число, хранящееся в памяти, имеет **порядок байт** — последовательность, в которой его составные байты расположены по адресам. Существуют два соглашения:

- **Big-endian (BE, «от старшего»):** наиболее значимый байт хранится по наименьшему адресу. Используется большинством сетевых протоколов, а также архитектурами SPARC, PowerPC (исторически).
- **Little-endian (LE, «от младшего»):** наименее значимый байт хранится по наименьшему адресу. Используется в x86/x86-64, ARM (в режиме по умолчанию), RISC-V.

При чтении бинарных файлов, приёме сетевых пакетов или обмене данными с оборудованием часто требуется преобразование между этими порядками. Модуль `endians` предоставляет девять низкоуровневых процедур, покрывающих 16-, 32- и 64-битные значения в трёх семействах:

| Семейство | Что делает |
|-----------|-----------|
| `swapEndianN` | Безусловно разворачивает все N байт. |
| `littleEndianN` | Гарантирует, что результат в little-endian (нет операций на LE-машинах). |
| `bigEndianN` | Гарантирует, что результат в big-endian (нет операций на BE-машинах). |

Все процедуры работают с **сырой памятью** через аргументы типа `pointer` — они намеренно не зависят от типа данных и одинаково хорошо работают с целыми числами, числами с плавающей точкой или любым другим типом подходящего размера.

### Оптимизация на этапе компиляции

На GCC, Clang, ICC и MSVC операции обмена реализуются через встроенные функции компилятора (`__builtin_bswap16/32/64`, `_bswap`, `_byteswap_*`), которые обычно компилируются в **одну инструкцию CPU** (например, `BSWAP` на x86). На остальных компиляторах автоматически используется портируемая побайтовая реализация.

### Подключение

```nim
import std/endians
```

## Концептуальная основа: что значит «поменять порядок байт»?

Возьмём 32-битное шестнадцатеричное значение `0x01020304`. Его байты в памяти:

```
Адрес:       0x00  0x01  0x02  0x03
Big-endian:  0x01  0x02  0x03  0x04   ← старший байт первым
Little-end:  0x04  0x03  0x02  0x01   ← младший байт первым
```

Смена порядка байт означает разворот последовательности байт на месте: big-endian превращается в little-endian и наоборот.

## Справочник процедур

Все процедуры имеют одинаковую сигнатуру:

```nim
proc nameN*(outp, inp: pointer)
```

- `inp` — указатель на **исходный** буфер (чтение).
- `outp` — указатель на **целевой** буфер (запись).
- Оба указателя должны ссылаться на допустимую память размером **не менее N/8 байт** (N = 16, 32 или 64).
- `inp` и `outp` **могут совпадать** (алиасинг) — реализация использует промежуточную копию там, где это необходимо.
- Ни одна процедура не выделяет память в куче.

---

### `swapEndian16`

```nim
proc swapEndian16*(outp, inp: pointer)
```

#### Что делает

Читает 2 байта из `inp` и записывает их в `outp` в **обратном порядке** — байт 0 входа становится байтом 1 выхода, и наоборот. Операция безусловная: разворот производится всегда, вне зависимости от порядка байт текущего CPU.

Используйте эту процедуру, когда точно знаете, что исходный и целевой порядок байт различаются.

#### Расположение в памяти

```
inp:  [A] [B]
outp: [B] [A]
```

#### Пример

```nim
import std/endians

var src = [0x01'u8, 0x02]   # байты: 0x01, 0x02
var dst: array[2, uint8]

swapEndian16(addr dst, addr src)
assert dst == [0x02'u8, 0x01]  # байты развёрнуты

# Практика: чтение big-endian int16 на little-endian машине
var beBytes = [0x00'u8, 0xFF]  # big-endian представление числа 255
var value: int16
swapEndian16(addr value, addr beBytes[0])
assert value == 255
```

---

### `swapEndian32`

```nim
proc swapEndian32*(outp, inp: pointer)
```

#### Что делает

Читает 4 байта из `inp` и записывает их в `outp` с полным **разворотом всех четырёх байт**. Как и `swapEndian16`, операция безусловная.

#### Расположение в памяти

```
inp:  [A] [B] [C] [D]
outp: [D] [C] [B] [A]
```

#### Пример

```nim
import std/endians

var src = [0x01'u8, 0x02, 0x03, 0x04]
var dst: array[4, uint8]

swapEndian32(addr dst, addr src)
assert dst == [0x04'u8, 0x03, 0x02, 0x01]

# Практика: преобразование float32, полученного по сети в big-endian формате
var netFloat: array[4, uint8] = [0x42'u8, 0xF6, 0xE9, 0x79]  # 123.456 в BE
var hostFloat: float32
swapEndian32(addr hostFloat, addr netFloat[0])
# hostFloat теперь содержит 123.456 (на little-endian машине)
```

---

### `swapEndian64`

```nim
proc swapEndian64*(outp, inp: pointer)
```

#### Что делает

Читает 8 байт из `inp` и записывает их в `outp` с **разворотом всех восьми байт**. Самая «широкая» из трёх операций обмена — покрывает `int64`, `uint64`, `float64` и любой другой 8-байтный тип.

#### Расположение в памяти

```
inp:  [A] [B] [C] [D] [E] [F] [G] [H]
outp: [H] [G] [F] [E] [D] [C] [B] [A]
```

#### Пример

```nim
import std/endians

var a = [1'u8, 2, 3, 4, 5, 6, 7, 8]
var b: array[8, uint8]

swapEndian64(addr b, addr a)
assert b == [8'u8, 7, 6, 5, 4, 3, 2, 1]

# Разворот на месте (outp == inp безопасен):
swapEndian64(addr a, addr a)
assert a == [8'u8, 7, 6, 5, 4, 3, 2, 1]
```

### `littleEndian16`

```nim
proc littleEndian16*(outp, inp: pointer)
```

#### Что делает

Копирует 2 байта из `inp` в `outp`, **гарантируя, что результат хранится в little-endian порядке**.

- На **little-endian** машине (x86, ARM по умолчанию): простой `copyMem` — никаких операций.
- На **big-endian** машине: вызывает `swapEndian16` для разворота байт.

Это идиоматическая процедура для записи значений в форматы, требующие little-endian кодирования (большинство форматов Windows, заголовки WAV/BMP, USB-дескрипторы).

#### Пример

```nim
import std/endians

var value: uint16 = 0x0102   # нативное целое
var buf: array[2, uint8]

littleEndian16(addr buf, addr value)
# На LE-машине: buf == [0x02, 0x01]  (разворот не нужен, прямое копирование)
# На BE-машине: buf == [0x02, 0x01]  (разворот выполнен)
# В любом случае: buf содержит LE-представление значения
```

---

### `littleEndian32`

```nim
proc littleEndian32*(outp, inp: pointer)
```

#### Что делает

Копирует 4 байта из `inp` в `outp`, обеспечивая **little-endian порядок** на выходе — разворот выполняется только при необходимости (на big-endian машинах).

Типичные сценарии: запись 32-битных целых в форматы с фиксированным LE-кодированием (ZIP, ELF, большинство форматов изображений).

#### Пример

```nim
import std/endians

var value: uint32 = 0xDEADBEEF
var buf: array[4, uint8]

littleEndian32(addr buf, addr value)
# buf содержит 0xDEADBEEF в little-endian: [0xEF, 0xBE, 0xAD, 0xDE]
# — на любой машине
```

---

### `littleEndian64`

```nim
proc littleEndian64*(outp, inp: pointer)
```

#### Что делает

Копирует 8 байт из `inp` в `outp`, обеспечивая **little-endian порядок**. 64-битный аналог `littleEndian32` — полезен для 64-битных целых и `float64`.

#### Пример

```nim
import std/endians

var ts: int64 = 1_700_000_000_000'i64   # временна́я метка в миллисекундах
var buf: array[8, uint8]

littleEndian64(addr buf, addr ts)
# buf содержит ts в LE-порядке — готово для записи в бинарный LE-файл
```

---

### `bigEndian16`

```nim
proc bigEndian16*(outp, inp: pointer)
```

#### Что делает

Копирует 2 байта из `inp` в `outp`, гарантируя **big-endian порядок** на выходе — порядок байт, который используется в TCP/IP (сетевой порядок байт), большинстве аудиоформатов (AIFF, MIDI) и архитектурах типа SPARC.

- На **big-endian** машине: простой `copyMem`, без разворота.
- На **little-endian** машине: вызывает `swapEndian16`.

#### Пример

```nim
import std/endians

# Подготовка 16-битного номера порта для заголовка сырого сетевого пакета
var port: uint16 = 8080
var netBuf: array[2, uint8]

bigEndian16(addr netBuf, addr port)
# netBuf == [0x1F, 0x90]  (8080 = 0x1F90 в big-endian)
```

---

### `bigEndian32`

```nim
proc bigEndian32*(outp, inp: pointer)
```

#### Что делает

Копирует 4 байта из `inp` в `outp` в **big-endian порядке**. Незаменима при формировании или разборе сетевых пакетов, сериализации данных для MIDI, записи длин чанков PNG и в любых форматах с обязательным сетевым порядком байт.

#### Пример

```nim
import std/endians

# PNG хранит длины чанков как 32-битные целые в big-endian
var length: uint32 = 1024
var pngBuf: array[4, uint8]

bigEndian32(addr pngBuf, addr length)
# pngBuf == [0x00, 0x00, 0x04, 0x00] — корректная PNG-кодировка числа 1024
```

---

### `bigEndian64`

```nim
proc bigEndian64*(outp, inp: pointer)
```

#### Что делает

Копирует 8 байт из `inp` в `outp` в **big-endian порядке**. 64-битный аналог `bigEndian32` — используется для 64-битных временны́х меток, смещений в файлах форматов EXIF или MKV, и 64-битных полей сетевых протоколов.

#### Пример

```nim
import std/endians

var fileSize: uint64 = 9_000_000_000'u64
var buf: array[8, uint8]

bigEndian64(addr buf, addr fileSize)
# buf содержит fileSize в BE-порядке — готово для поля заголовка протокола
```

| Процедура | Байт | На LE-машине | На BE-машине |
|-----------|------|--------------|--------------|
| `swapEndian16` | 2 | всегда разворачивает | всегда разворачивает |
| `swapEndian32` | 4 | всегда разворачивает | всегда разворачивает |
| `swapEndian64` | 8 | всегда разворачивает | всегда разворачивает |
| `littleEndian16` | 2 | `copyMem` (нет операций) | разворачивает |
| `littleEndian32` | 4 | `copyMem` (нет операций) | разворачивает |
| `littleEndian64` | 8 | `copyMem` (нет операций) | разворачивает |
| `bigEndian16` | 2 | разворачивает | `copyMem` (нет операций) |
| `bigEndian32` | 4 | разворачивает | `copyMem` (нет операций) |
| `bigEndian64` | 8 | разворачивает | `copyMem` (нет операций) |

## Типичные приёмы и подводные камни

### Чтение структуры из бинарного файла

```nim
import std/endians

type Header = object
  magic: uint32
  length: uint32

proc readHeader(data: openArray[byte]): Header =
  var raw: Header
  copyMem(addr raw, unsafeAddr data[0], sizeof(Header))
  # Файл в big-endian (например, сетевой протокол):
  bigEndian32(addr result.magic, addr raw.magic)
  bigEndian32(addr result.length, addr raw.length)
```

### Подводный камень: слишком маленький буфер

Каждая процедура предполагает, что буферы содержат **не менее** стольких байт, сколько указано в суффиксе (2, 4 или 8). Передача более короткого буфера приводит к **неопределённому поведению** — проверки границ не выполняются.

```nim
var tiny: array[2, uint8]
# НЕВЕРНО — буфер только 2 байта, а swapEndian32 требует 4:
# swapEndian32(addr result, addr tiny[0])  # UB!
```

### Подводный камень: `swapEndian` вместо `bigEndian`/`littleEndian`

`swapEndianN` правилен **только** тогда, когда вы уверены, что исходный и целевой порядок байт различаются. Если использовать `swapEndianN` безусловно и запустить программу на big-endian машине, преобразование окажется неверным. Предпочитайте `bigEndianN`/`littleEndianN` — они переносимы по определению.

## Шпаргалка

```
Задача                                           Процедура
─────────────────────────────────────────────────────────────────
Развернуть байты всегда                          swapEndianN
Записать значение в LE-формат (WAV, ELF...)      littleEndianN
Записать значение в BE-формат (PNG, TCP/IP...)   bigEndianN
Прочитать значение из LE-формата → нативный      littleEndianN
Прочитать значение из BE-формата → нативный      bigEndianN
```
