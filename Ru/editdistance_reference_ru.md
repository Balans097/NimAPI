# editdistance — справочник по модулю

## Обзор

Модуль `editdistance` предоставляет две процедуры для вычисления **редакционного расстояния** (оно же **расстояние Левенштейна**) между двумя строками. Редакционное расстояние — это минимальное количество однознаковых операций — **вставок**, **удалений** и **замен** — необходимых для преобразования одной строки в другую.

Модуль предлагает два варианта одного и того же алгоритма:

| Процедура | Модель строки | Когда применять |
|---|---|---|
| `editDistance` | Юникодные руны (code points) | Любой язык: кириллица, CJK, эмодзи и т.д. |
| `editDistanceAscii` | Сырые байты (ASCII) | Только ASCII-ввод; быстрее за счёт простой итерации |

Обе процедуры **чистые** (помечены `noSideEffect`) и используют экономичную по памяти линейную реализацию алгоритма Левенштейна. Перед основными вычислениями алгоритм отрезает одинаковые **префикс** и **суффикс** — это ключевая оптимизация для строк с большим числом совпадающих символов в начале и в конце.

## `editDistance`

```nim
proc editDistance*(a, b: string): int
```

### Описание

Вычисляет расстояние Левенштейна между строками `a` и `b`, рассматривая их как **последовательности юникодных рун** (кодовых точек), а не сырых байтов. Это правильный выбор для любого текста, который может содержать многобайтовые символы — буквы с диакритикой, кириллицу, арабский, китайский, японский, корейский, эмодзи и так далее.

Результат — количество **операций на уровне рун** (не байтов), необходимых для преобразования `a` в `b`. Для текста, состоящего только из ASCII-символов, результат совпадает с результатом `editDistanceAscii`, но для многобайтового текста функции расходятся: `editDistanceAscii` считает байты, а `editDistance` — символы (руны).

Процедура **симметрична**: `editDistance(a, b) == editDistance(b, a)`. Внутри функции более короткая строка всегда помещается на позицию `a`, чтобы минимизировать количество выделений памяти.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `a` | `string` | Первая строка (кодировка UTF-8) |
| `b` | `string` | Вторая строка (кодировка UTF-8) |

### Возвращаемое значение

Неотрицательное целое `int`, равное минимальному количеству операций на уровне рун, разделяющих `a` и `b`. Возвращает `0`, если строки идентичны.

### Детали алгоритма

- **Отрезает общий префикс** (одинаковые ведущие руны) до основного цикла.
- **Отрезает общий суффикс** (одинаковые завершающие руны) после прохода по префиксу.
- Запускает стандартный DP Левенштейна с одной одномерной строкой (линейная память, O(min(|a|,|b|)) пространства).
- Обрабатывает **особые случаи** напрямую: одна строка пуста → вернуть длину другой; одна строка — одна руна → пройти по длинной строке и решить за O(n).

### Примеры

**Одна замена — один символ изменился:**

```nim
import editdistance

echo editDistance("Kitten", "Bitten")   # → 1  ("K" → "B")
echo editDistance("кот", "кит")         # → 1  ("о" → "и")
```

**Несколько операций:**

```nim
echo editDistance("суббота", "среда")   # → 5
echo editDistance("saturday", "sunday") # → 3
# sat-ur-day → s-un-day:
#   удалить "a", заменить "t"→"n", удалить "r"
```

**Одинаковые строки — всегда 0:**

```nim
echo editDistance("привет", "привет")   # → 0
```

**Пустая строка — расстояние равно длине второй строки:**

```nim
echo editDistance("", "nim")            # → 3
echo editDistance("nim", "")            # → 3  (симметрично)
```

**Юникод / многобайтовые символы — считаются руны, а не байты:**

```nim
# "кот" → "код": одна замена ("т" → "д")
echo editDistance("кот", "код")         # → 1

# Каждая кириллическая буква занимает 2 байта в UTF-8,
# но расстояние — это всё равно 1 руна
echo editDistance("café", "cave")       # → 2  ("f"→"v", "é"→"e")
```

**Эмодзи — каждый эмодзи считается одной руной:**

```nim
echo editDistance("😀😂", "😀😎")    # → 1
```

## `editDistanceAscii`

```nim
proc editDistanceAscii*(a, b: string): int
```

### Описание

Вычисляет расстояние Левенштейна между строками `a` и `b`, рассматривая их как **последовательности байтов**. Каждый байт — это один «символ» для данной функции. Подход корректен и эффективен для чистого ASCII-текста и заметно быстрее `editDistance`, поскольку полностью обходит стороной декодирование Unicode.

**Не используйте эту функцию с не-ASCII-вводом.** Многобайтовый UTF-8-символ (например, кириллическая буква, занимающая 2 байта) будет воспринят как два отдельных «символа», что даёт расстояния, не соответствующие числу видимых символов.

Процедура симметрична и внутри также всегда помещает более короткую строку первой.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `a` | `string` | Первая строка (ASCII) |
| `b` | `string` | Вторая строка (ASCII) |

### Возвращаемое значение

Неотрицательное целое `int`, равное минимальному количеству байтовых операций редактирования, разделяющих `a` и `b`. Возвращает `0`, если строки идентичны.

### Детали алгоритма

По структуре идентична `editDistance`, но вместо итерации по рунам (`fastRuneAt`, `runeAt`) использует прямую индексацию по байтам. Внутренний цикл становится значительно дешевле для ASCII-нагрузки. Оптимизация отрезания префикса/суффикса также присутствует.

### Примеры

**Одна замена — только ASCII:**

```nim
import editdistance

echo editDistanceAscii("Kitten", "Bitten")   # → 1
echo editDistanceAscii("hello", "hallo")     # → 1  ("e" → "a")
```

**Типичный сценарий — проверка орфографии / нечёткий поиск по ASCII-токенам:**

```nim
let dictionary = ["apple", "apply", "aptly", "appeal"]
let query = "appel"

for word in dictionary:
  echo word, ": ", editDistanceAscii(query, word)
# apple:  1
# apply:  2
# aptly:  3
# appeal: 2
```

**Почему НЕ стоит использовать с Unicode:**

```nim
# "кот" занимает 6 байтов в UTF-8 (2 байта на кириллическую букву)
# editDistanceAscii считает их 6 «символами», а не 3 рунами!
echo editDistanceAscii("кот", "код")   # → 2  (НЕВЕРНО: ожидалось 1)
echo editDistance("кот", "код")        # → 1  (верно)
```

## Как выбрать между двумя процедурами

```
Входные данные гарантированно содержат только ASCII?
    │
    ├─ Да  → используй editDistanceAscii   (быстрее, проще)
    │
    └─ Нет → используй editDistance        (корректно для любого Unicode-текста)
```

Практическое правило: если вы работаете с идентификаторами, именами файлов, токенами командной строки или английским текстом, `editDistanceAscii` подходит. Для пользовательского текста на любом языке всегда предпочтительнее `editDistance`.

## Типичные применения

- **Проверка орфографии** — предложить слово из словаря с наименьшим редакционным расстоянием до ошибочно написанного запроса.
- **Нечёткий поиск строк** — ранжировать кандидатов по количеству требуемых правок.
- **Сравнение версий / обнаружение изменений** — измерить, насколько похожи две версии строки.
- **Биоинформатика** — сравнивать последовательности ДНК или белков (ASCII-алфавиты хорошо ложатся на `editDistanceAscii`).
- **Постобработка OCR** — исправлять ошибки распознавания подбором ближайшего известного слова.

## Вычислительная сложность

| | Время | Память |
|---|---|---|
| Обе процедуры | O(|a| × |b|) в худшем случае | O(min(|a|,|b|)) |

Линейная память достигается благодаря однострочному DP: в каждый момент в памяти хранится только одна строка классической матрицы, а не вся таблица целиком.
