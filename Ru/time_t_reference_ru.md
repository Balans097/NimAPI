# Справочник модуля `time_t`

> **Назначение модуля:** `time_t` предоставляет единственный экспортируемый тип — `Time` — представляющий собой портируемую, платформенно-корректную обёртку над типом `time_t` из стандартной библиотеки C. Его единственная задача — позволить Nim-коду объявлять значения типа `time_t` безопасно, без ручного импорта заголовочного файла C и без необходимости разбираться, какой именно целочисленный тип — 32-битный или 64-битный — используется на текущей платформе.

---

## Что такое `time_t`?

`time_t` — это тип C, определённый в `<time.h>`, который хранит значение календарного времени. Практически повсеместно реализован как количество секунд, прошедших с начала эпохи Unix (1 января 1970 года, 00:00:00 UTC). Это основная «валюта» времени в большинстве системных API уровня C: `time()`, `difftime()`, `mktime()`, `localtime()` и так далее.

Точный размер целочисленного представления `time_t` **не задан стандартом C** — он определяется реализацией. На практике:

| Платформа / Компилятор | Размер |
|---|---|
| POSIX, 64-битные системы | Как правило `int64` (знаковый 64-бит) |
| Windows (современный, 64-бит) | `int64` (начиная с MSVC 2005) |
| Windows (32-бит, GCC) | `clong` (32-бит) |

32-битное представление `time_t` переполняется **19 января 2038 года** (проблема 2038 года). 64-битный `time_t` безопасен приблизительно до 292 миллиардов лет нашей эры.

## Справочник по типам

### `Time`

```nim
type Time* = ...  # платформенно-зависимый базовый тип
```

Тип Nim с ключевым словом `distinct`, оборачивающий `time_t` из `<time.h>`. Признак `distinct` означает, что значение `Time` нельзя случайно использовать как обычное целое число и наоборот — компилятор enforces различие на этапе компиляции. Явное приведение типа необходимо, если нужно получить сырое числовое значение.

`Time` — это тип, который используется при вызове C-функций, принимающих или возвращающих `time_t`. Он **не совпадает** с `times.Time` из `std/times` — высокоуровневым представлением времени в Nim с наносекундной точностью.

#### Платформенно-зависимые определения

Модуль выбирает правильное базовое представление на этапе компиляции через каскад условий `when`:

**POSIX (Linux, macOS, BSD, …)**

```nim
import std/posix
export posix.Time
```

На POSIX `Time` просто реэкспортируется из `std/posix`, который сам импортирует настоящий C-тип `time_t`. Это значит, что `time_t.Time` и `posix.Time` — **один и тот же тип**: никакое преобразование не нужно при работе с POSIX API.

**Windows (64-бит, или любой компилятор кроме 32-битного GCC)**

```nim
type Time* {.importc: "time_t", header: "<time.h>".} = distinct int64
```

На современном Windows, где `time_t` занимает 64 бита, `Time` отображается на `distinct int64`. Прагмы `{.importc.}` и `{.header.}` указывают компилятору Nim использовать реальный C-тип напрямую, а не создавать псевдоним на уровне Nim, — это гарантирует бинарную совместимость с системными библиотеками Windows.

**Windows (32-бит, только GCC)**

```nim
type Time* {.importc: "time_t", header: "<time.h>".} = distinct clong
```

На 32-битном Windows с GCC `time_t` является `clong` (знаковое 32-битное целое). Эта ветвь существует для точного соответствия ABI инструментальной цепочки GCC MinGW.

**Сборки документации (`nimdoc`)**

```nim
type Time* = distinct int64
```

При генерации API-документации Nim использует синтетическое определение `distinct int64`, чтобы тип отображался в документации без необходимости иметь окружение POSIX или Windows. Эта ветвь никогда не компилируется в реальные бинарники.

## Использование

### Импорт

```nim
import std/time_t
```

### Взаимодействие с C-функциями

Основное применение — оборачивание C-функций, работающих с `time_t`:

```nim
import std/time_t

# Объявляем функцию time() из стандартной библиотеки C
proc c_time(tloc: ptr Time): Time {.importc: "time", header: "<time.h>".}

# Получаем текущее время как сырое значение time_t
let now: Time = c_time(nil)
echo now.int64   # выводит секунды с начала эпохи Unix
```

Обёртка для `difftime`, вычисляющей разницу между двумя значениями `time_t`:

```nim
import std/time_t

proc c_difftime(time1, time0: Time): cdouble {.importc: "difftime", header: "<time.h>".}

let t0: Time = Time(1_000_000_000)  # 9 сентября 2001
let t1: Time = Time(1_700_000_000)  # 14 ноября 2023

let diff = c_difftime(t1, t0)
echo diff / 86400.0, " дней между двумя временными метками"
```

### Связь с `std/times`

`time_t.Time` — это **низкоуровневый тип для C-интеропа**. `std/times` предоставляет высокоуровневый `times.Time` с наносекундной точностью, поддержкой часовых поясов, форматированием и арифметикой. Для перехода между ними значение `time_t.Time` преобразуется в целое число, которое затем интерпретируется через `std/times`:

```nim
import std/time_t, std/times

proc c_time(tloc: ptr time_t.Time): time_t.Time {.importc: "time", header: "<time.h>".}

let raw: time_t.Time = c_time(nil)

# Преобразование сырого C time_t → std/times.Time
let nimTime: times.Time = fromUnix(raw.int64)
echo nimTime.format("yyyy-MM-dd HH:mm:ss")
```

### Явное приведение типов

Поскольку `Time` является типом `distinct`, арифметические операции и работа с целыми числами требуют явного приведения:

```nim
import std/time_t

let t = Time(1_700_000_000'i64)

# Извлечь сырое целочисленное значение
let seconds: int64 = t.int64

# Создать Time из известного смещения от эпохи
let oneHourLater = Time(seconds + 3600)
```

## Важные замечания

- Модуль не содержит **ни одной процедуры** — только определение типа `Time`. Все операции со временем (арифметика, форматирование, преобразование) предоставляет `std/times` либо C-функции, импортируемые через FFI.
- Никогда не смешивайте `time_t.Time` и `times.Time` без явного преобразования — это разные типы с разной точностью и семантикой. Компилятор Nim отклонит неявное использование одного вместо другого.
- На POSIX импорт `std/time_t` и импорт `std/posix` дают один и тот же тип `Time` — никакого двойного определения или конфликта не возникает.
- Прагмы `{.importc.}` и `{.header.}` в Windows-определениях означают, что в скомпилированном коде используется реальный C-тип, что обеспечивает корректное выравнивание ABI с системными библиотеками независимо от внутреннего представления целых чисел в Nim.
