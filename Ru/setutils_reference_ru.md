# std/setutils — Справочник модуля

> **Входит в стандартную библиотеку Nim** (`std/setutils`).

## Обзор

В Nim есть встроенный тип `set`, представляющий математическое множество небольших порядковых значений (символов, байтов, булевых значений, малых целых чисел, перечислений). Он чрезвычайно быстр — реализован в виде компактного битсета — но встроенные операторы покрывают лишь базовое: объединение (`+`), пересечение (`*`), разность (`-`), проверку принадлежности (`in`/`notin`), включение (`incl`), исключение (`excl`) и мощность (`card`).

Модуль `setutils` добавляет операции, которых заметно не хватает в этом базовом наборе:

- Преобразование любого итерируемого значения в множество (`toSet`)
- Построение полного множества всех значений некоторого типа (`fullSet`)
- Вычисление дополнения множества (`complement`)
- Условное включение/исключение через синтаксис индексного присваивания (`[]=`)
- Симметрическую разность — элементы, входящие ровно в одно из двух множеств — и как функцию (`symmetricDifference`), и как оператор (`-+-`)
- Симметрическую разность на месте (`toggle`)

Все операции работают с тем же встроенным типом `set[T]`; никаких обёрток или импорта альтернативных библиотек множеств не требуется.

---

## Допустимые типы элементов

Встроенный `set` в Nim может хранить только элементы типов с небольшим конечным порядковым диапазоном: `char`, `byte`, `bool`, `int8`, `uint8`, `int16`, `uint16` и любые `enum`. Это ограничение самого встроенного типа `set`, а не данного модуля.

## Справочник по API

### `toSet`

```nim
template toSet*(iter: untyped): untyped
```

**Преобразует любое итерируемое значение в встроенное множество, содержащее в точности те элементы, которые оно порождает.**

Это идиоматический способ построить множество из последовательности, массива, строки, диапазона или любого другого значения, по которому можно итерироваться. Тип элементов результирующего множества выводится автоматически из типа элементов итерируемого с помощью `elementType`. Повторяющиеся значения в источнике молча схлопываются — множества по определению содержат каждое значение не более одного раза.

Поскольку `toSet` — шаблон, он встраивается в точку вызова и не несёт никаких накладных расходов сверх самого цикла.

**Примеры:**

```nim
import std/setutils

# Из строки — каждый символ становится элементом множества:
assert "helloWorld".toSet == {'W', 'd', 'e', 'h', 'l', 'o', 'r'}
# Обратите внимание: дублирующиеся 'l' и 'o' появляются в результате лишь по одному разу.

# Из массива uint16:
assert toSet([10u16, 20, 30]) == {10u16, 20, 30}

# Из seq int16:
assert toSet(@[1321i16, 321, 90]) == {90i16, 321, 1321}

# Из диапазона:
assert toSet(0u8..10) == {0u8..10}

# Из массива из одного элемента:
assert toSet([false]) == {false}
```

**Типичное использование — сбор уникальных символов из строки:**

```nim
import std/setutils

proc uniqueLetters(s: string): set[char] =
  s.toSet

let letters = uniqueLetters("abracadabra")
echo letters   # {'a', 'b', 'c', 'd', 'r'}
```

### `fullSet`

```nim
func fullSet*[T](U: typedesc[T]): set[T] {.inline.}
```

**Возвращает множество, содержащее все возможные значения типа `T`.**

Полезно, когда нужно начать с «универсума» всех значений и затем вычитать из него — например, при вычислении дополнений или при проверке того, является ли набор флагов исчерпывающим.

Для обычных порядковых типов (`bool`, подтипы целых чисел, `range`) полное множество строится как `{T.low..T.high}`. Для перечислений с «дырами» (где значения перечисления — не подряд идущие целые числа) `fullSet` использует макрос для перечисления фактически объявленных значений.

**Примеры:**

```nim
import std/setutils

# Все булевы значения:
assert bool.fullSet == {true, false}

# Диапазонный тип — только значения в диапазоне:
type A = range[1..3]
assert A.fullSet == {1.A, 2, 3}

# int8 охватывает -128..127, то есть ровно 256 значений:
assert int8.fullSet.len == 256

# Перечисление с «дырой» — только объявленные члены:
type Colors = enum
  red, green = 3, blue
assert Colors.fullSet == {red, green, blue}
```

**Типичное использование — начать со всего и удалить ненужное:**

```nim
import std/setutils

type Permission = enum
  read, write, execute

# Выдать все права, затем отозвать одно:
var granted = Permission.fullSet
granted.excl(execute)
assert granted == {read, write}
```

### `complement`

```nim
func complement*[T](s: set[T]): set[T] {.inline.}
```

**Возвращает дополнение множества `s` — все значения типа `T`, которые *не* входят в `s`.**

Математически: если `U` — универсальное множество всех значений типа `T`, то `complement(s) == U - s`. Внутренне это просто `fullSet(T) - s`.

Операция отвечает на вопрос: «какие значения этого типа *отсутствуют* в данном множестве?» Особенно удобна при работе с флаговыми перечислениями или символьными классами — например, «все символы, которые *не* являются цифрами».

**Примеры:**

```nim
import std/setutils

type Colors = enum
  red, green = 3, blue

# Всё, кроме red и blue:
assert complement({red, blue}) == {green}

# Дополнение полного множества пусто:
assert complement({red, green, blue}).card == 0

# Все символы, не являющиеся ASCII-цифрами:
assert complement({'0'..'9'}) == {0.char..255.char} - {'0'..'9'}

# Диапазонные типы — дополнение остаётся в пределах диапазона:
assert complement({range[0..10](0), 1, 2, 3}) == {range[0..10](4), 5, 6, 7, 8, 9, 10}
```

**Типичное использование — инвертированный фильтр символов:**

```nim
import std/setutils

const nonDigits = complement({'0'..'9'})

proc stripDigits(s: string): string =
  for ch in s:
    if ch in nonDigits:
      result.add(ch)

echo stripDigits("a1b2c3")  # "abc"
```

### `[]=`

```nim
func `[]=`*[T](t: var set[T], key: T, val: bool) {.inline.}
```

**Условно включает или исключает один элемент из множества, в зависимости от булева значения.**

Этот оператор — краткая форма распространённого паттерна `if val: t.incl(key) else: t.excl(key)`. Он позволяет управлять принадлежностью элемента с помощью привычного синтаксиса индексного присваивания, рассматривая множество как отображение элементов в булевы значения. Особенно чисто выглядит, когда принадлежность элемента определяется условиями времени выполнения или конфигурационными флагами.

**Примеры:**

```nim
import std/setutils

type A = enum
  a0, a1, a2, a3

var s = {a0, a3}

# false → исключить:
s[a0] = false
s[a1] = false   # a1 не был в s; исключение отсутствующего элемента — нейтральная операция
assert s == {a3}

# true → включить:
s[a2] = true
s[a3] = true    # a3 уже был в s; включение уже присутствующего — нейтральная операция
assert s == {a2, a3}
```

**Типичное использование — построение множества прав из отдельных флагов:**

```nim
import std/setutils

type Flag = enum
  flagA, flagB, flagC

var flags: set[Flag]
let config = (a: true, b: false, c: true)

flags[flagA] = config.a
flags[flagB] = config.b
flags[flagC] = config.c

assert flags == {flagA, flagC}
```

### `symmetricDifference`

```nim
func symmetricDifference*[T](x, y: set[T]): set[T]
```

**Возвращает симметрическую разность двух множеств — множество элементов, которые входят ровно в одно из `x` или `y`, но не в оба сразу.**

Математически: `symmetricDifference(x, y) == (x - y) + (y - x)` — элементы, уникальные для `x`, плюс элементы, уникальные для `y`. Элементы, присутствующие в обоих множествах, взаимно уничтожаются.

На компиляторах, определяющих `nimHasXorSet` (Nim 2.x с встроенной поддержкой XOR), это одна инструкция `magic`, работающая непосредственно с битсетами, — столь же быстрая, как любая другая операция над множествами. На более старых компиляторах используется откат к `x + y - (x * y)`, что эквивалентно, но включает три операции.

**Примеры:**

```nim
import std/setutils

# Элементы, входящие в {1,2,3} или {2,3,4}, но не в оба:
assert symmetricDifference({1, 2, 3}, {2, 3, 4}) == {1, 4}
# 2 и 3 входят в оба → исключаются; 1 только в x, 4 только в y → включаются.

# Симметрическая разность с пустым множеством — само множество:
assert symmetricDifference({1, 2}, {}) == {1, 2}

# Симметрическая разность одинаковых множеств — пустое:
assert symmetricDifference({1, 2}, {1, 2}) == {}
```

**Типичное использование — обнаружение изменений между двумя состояниями:**

```nim
import std/setutils

type Option = enum
  optA, optB, optC, optD

let before = {optA, optB, optC}
let after  = {optB, optC, optD}

let changed = symmetricDifference(before, after)
# changed == {optA, optD}
# optA был удалён; optD был добавлен.
```

### `-+-`

```nim
proc `-+-`*[T](x, y: set[T]): set[T] {.inline.}
```

**Операторный псевдоним для `symmetricDifference(x, y)`.**

Обе формы дают идентичные результаты; выбирайте ту, которая лучше читается в контексте. Форма с оператором удобна в цепочечных выражениях, а именованная функция часто нагляднее в документации или когда само название уже всё объясняет.

**Примеры:**

```nim
import std/setutils

assert {1, 2, 3} -+- {2, 3, 4} == {1, 4}

# Цепочка: элементы, уникальные ровно для одного из трёх множеств (слева направо):
let r = {1, 2} -+- {2, 3} -+- {3, 4}
assert r == {1, 4}
```

### `toggle`

```nim
proc toggle*[T](x: var set[T], y: set[T]) {.inline.}
```

**Переключает принадлежность каждого элемента `y` в `x`: элементы, уже входящие в `x`, удаляются; элементы, отсутствующие в `x`, добавляются.**

Это мутирующий вариант на месте, эквивалентный `symmetricDifference`. Он изменяет `x` напрямую, не создавая нового множества, что избавляет от лишнего присваивания при обновлении переменной. Вызов `x.toggle(y)` полностью эквивалентен `x = symmetricDifference(x, y)`.

Название «toggle» («переключение») выбрано намеренно: каждый элемент из `y` переключает своё присутствие в `x`, подобно тому как тумблер переключается между «вкл» и «выкл».

**Примеры:**

```nim
import std/setutils

var x = {1, 2, 3}

# Переключить {2, 3, 4}: 2 и 3 уходят (были в x), 4 входит (не было в x):
x.toggle({2, 3, 4})
assert x == {1, 4}

# Переключить то же множество снова — всё возвращается обратно:
x.toggle({2, 3, 4})
assert x == {1, 2, 3}
```

**Типичное использование — переключение состояний выбора:**

```nim
import std/setutils

type Item = enum
  itemA, itemB, itemC, itemD

var selected: set[Item] = {itemA, itemB}

proc toggleSelection(toToggle: set[Item]) =
  selected.toggle(toToggle)

toggleSelection({itemB, itemC})
assert selected == {itemA, itemC}   # itemB снят, itemC выбран

toggleSelection({itemC, itemD})
assert selected == {itemA, itemD}   # itemC снят, itemD выбран
```

## Сводная таблица

| Символ | Сигнатура | Назначение |
|--------|-----------|------------|
| `toSet(iter)` | `untyped → set[T]` | Построить множество из любого итерируемого |
| `fullSet(T)` | `typedesc[T] → set[T]` | Множество всех значений типа |
| `complement(s)` | `set[T] → set[T]` | Все значения `T`, не входящие в `s` |
| `t[key] = val` | `(var set[T], T, bool) → void` | Условно включить или исключить один элемент |
| `symmetricDifference(x, y)` | `(set[T], set[T]) → set[T]` | Элементы ровно одного из двух множеств |
| `x -+- y` | `(set[T], set[T]) → set[T]` | Операторный псевдоним для `symmetricDifference` |
| `toggle(x, y)` | `(var set[T], set[T]) → void` | Симметрическая разность на месте |

---

## Связь между операциями симметрической разности

Три символа симметрической разности отражают одну и ту же концепцию на разных уровнях удобства:

```
symmetricDifference(x, y)   ← именованная функция; максимально явно
x -+- y                     ← инфиксный оператор; удобен в выражениях
x.toggle(y)                 ← мутация на месте; удобна при обновлении переменной
```

Все три дают одинаковый логический результат; выбор — сугубо стилистический.
