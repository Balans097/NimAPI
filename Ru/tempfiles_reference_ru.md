# Справочник модуля `tempfiles`

> **Назначение модуля:** `tempfiles` предоставляет безопасное, кроссплатформенное создание временных файлов и директорий. Модуль внутренне решает классическую проблему гонок при создании уникальных имён: генерирует случайное имя, атомарно открывает или создаёт объект и автоматически повторяет попытку при коллизии. Вызывающий код получает готовый дескриптор и путь, не беспокоясь об уникальности.
>
> ⚠️ **Экспериментальный API.** Интерфейс может измениться в будущих версиях Nim.

---

## Почему временные файлы — нетривиальная задача

Наивный подход — сначала придумать имя, потом открыть файл — уязвим к **гонке TOCTOU (time-of-check/time-of-use)**: в промежутке между проверкой доступности имени и фактическим созданием файла другой процесс может занять то же имя. `tempfiles` устраняет эту уязвимость, используя атомарные примитивы ОС (`O_EXCL` на POSIX, `CREATE_NEW` на Windows), которые немедленно завершаются неудачей, если путь уже занят, а затем автоматически повторяют попытку с новым случайным именем.

---

## Внутренние константы

Константы не экспортируются, но определяют поведение модуля:

| Константа | Значение | Назначение |
|---|---|---|
| `maxRetry` | `10000` | Максимальное число попыток генерации имени перед выбросом `OSError` |
| `letters` | `a-z A-Z 0-9` (62 символа) | Алфавит для случайных сегментов имени |
| `nimTempPathLength` | `8` (по умолчанию) | Длина случайного сегмента; переопределяется через `-d:nimTempPathLength=N` |

Случайный сегмент формируется из 62 символов на 8 позициях, что даёт 62⁸ ≈ 218 триллионов вариантов — коллизии на практике исключены.

### Настройка длины случайного сегмента

Длину можно изменить на этапе компиляции:

```sh
nim c -d:nimTempPathLength=16 myprogram.nim
```

Тогда каждое сгенерированное имя будет содержать 16-символьный случайный сегмент вместо стандартных 8.

## Справочник по процедурам

---

### `genTempPath`

```nim
proc genTempPath*(prefix, suffix: string, dir = ""): string
```

#### Что делает

Генерирует и возвращает **строку-кандидат** пути для временного файла или директории. Путь строится по формуле:

```
dir / (prefix & <случайный_сегмент> & suffix)
```

где `<случайный_сегмент>` — строка из `nimTempPathLength` случайно выбранных алфавитно-цифровых символов.

Процедура **только создаёт строку** — она не обращается к файловой системе, не проверяет существование пути и ничего не создаёт. Это строительный блок, используемый внутри `createTempFile` и `createTempDir`, но он также экспортируется для интеграции в собственную логику создания объектов.

#### Параметр `dir`

Если `dir` — пустая строка (по умолчанию), используется системная временная директория — тот же путь, что возвращает `getTempDir()` из `std/appdirs` (например `/tmp` на Linux/macOS, `%TEMP%` на Windows). Директория **должна уже существовать** — `genTempPath` её не создаёт.

#### Описание параметров

| Параметр | Тип | Описание |
|---|---|---|
| `prefix` | `string` | Текст перед случайным сегментом. Может быть пустым. |
| `suffix` | `string` | Текст после случайного сегмента. Может быть пустым. |
| `dir` | `string` | Директория для пути. Пустая строка = системная временная директория. |

**Возвращает:** `string` — путь-кандидат, не гарантированно свободный.

#### Примеры

```nim
import tempfiles

# Базовое использование — генерирует что-то вроде /tmp/tmp_aBcDeFgH_.dat
let candidate = genTempPath("tmp_", "_.dat")
echo candidate

# Конкретная директория
let inLocal = genTempPath("session_", ".lock", dir = "/run/myapp")
# → /run/myapp/session_XyZ12345.lock

# Без префикса и суффикса — просто случайное имя в системной temp-директории
let bare = genTempPath("", "")
echo bare  # например /tmp/kQmVwPnR

# Использование в собственном цикле создания со специальными флагами открытия
import std/os
var path: string
var f: File
for attempt in 0 ..< 100:
  path = genTempPath("work_", ".bin")
  if open(f, path, fmReadWrite):
    break
```

#### Когда использовать `genTempPath` напрямую

В большинстве случаев предпочтительнее `createTempFile` или `createTempDir` — они самостоятельно управляют повторными попытками и атомарным созданием. `genTempPath` полезен, когда нужно получить временный *путь* без немедленного создания файла — например, чтобы передать его как путь выходного файла внешнему инструменту.

---

### `createTempFile`

```nim
proc createTempFile*(prefix, suffix: string, dir = ""):
    tuple[cfile: File, path: string]
```

#### Что делает

Создаёт новый временный файл с уникальным именем и возвращает кортеж из открытого файлового дескриптора (`cfile`) и пути к файлу (`path`). Файл открывается в режиме чтения-записи (`w+`) и **создаётся исключительно атомарно** — гарантируется, что до возврата вызова файла не существовало. При коллизиях имён процедура автоматически повторяет попытку до `maxRetry` (10 000) раз.

Файл сразу готов к чтению и записи через возвращённый дескриптор `File`.

#### Ответственность вызывающего кода

Две вещи явно остаются на усмотрение вызывающего:

1. **Закрыть дескриптор** — вызвать `close(result.cfile)` по завершении работы.
2. **Удалить файл** — вызвать `removeFile(result.path)`, когда файл больше не нужен.

Модуль не регистрирует никаких финализаторов или atexit-хуков. Если программа аварийно завершится или об удалении забудут — файл останется на диске.

#### Условия ошибки

`OSError` выбрасывается, если:
- Директория `dir` не существует.
- После `maxRetry` попыток свободное имя не найдено (на практике невероятно).
- У процесса нет прав на запись в `dir`.

#### Описание параметров

| Параметр | Тип | Описание |
|---|---|---|
| `prefix` | `string` | Текст перед случайным сегментом в имени файла. Может быть пустым. |
| `suffix` | `string` | Текст после случайного сегмента (например, расширение). Может быть пустым. |
| `dir` | `string` | Директория для файла. Пустая строка = системная временная директория. |

**Возвращает:** `tuple[cfile: File, path: string]`
- `cfile` — открытый дескриптор `File` в режиме чтения-записи, позиция — начало файла
- `path` — абсолютный путь к созданному файлу

#### Примеры

```nim
import tempfiles, std/os

# Базовое использование
let (cfile, path) = createTempFile("tmpprefix_", "_end.tmp")
# path — что-то вроде: /tmp/tmpprefix_FDCIRZA0_end.tmp

cfile.write("hello, world")
cfile.setFilePos(0)
echo readAll(cfile)   # → hello, world

close(cfile)
echo readFile(path)   # по-прежнему доступен через путь после закрытия

removeFile(path)      # очистка
```

Использование конкретной директории:

```nim
import tempfiles, std/os

# Временный файл рядом с выходными файлами сборки
let (f, p) = createTempFile("build_", ".o", dir = "build/obj")
defer:
  close(f)
  removeFile(p)

# записать промежуточный результат компиляции в f ...
```

Гарантированная очистка через `defer`:

```nim
import tempfiles, std/os

proc processData(input: string) =
  let (tmp, tmpPath) = createTempFile("proc_", ".tmp")
  defer:
    close(tmp)
    removeFile(tmpPath)

  tmp.write(input)
  tmp.setFilePos(0)
  # передать tmpPath внешнему инструменту или в дальнейшую обработку ...
```

Несуществующая директория вызывает ошибку немедленно:

```nim
import tempfiles
doAssertRaises(OSError):
  discard createTempFile("", "", "nonexistent_dir")
```

#### Анатомия сгенерированного пути

```
/tmp / tmpprefix_ FDCIRZA0 _end.tmp
 ↑dir    ↑prefix  ↑случайн.  ↑suffix
```

Случайный сегмент по умолчанию — 8 алфавитно-цифровых символов (переопределяется при компиляции).

### `createTempDir`

```nim
proc createTempDir*(prefix, suffix: string, dir = ""): string
```

#### Что делает

Создаёт новую временную **директорию** с уникальным именем и возвращает её путь. Как и `createTempFile`, генерирует случайное имя через `genTempPath` и использует атомарный вызов ОС (`existsOrCreateDir`), гарантирующий, что директории не существовало до этого момента. При коллизиях автоматически повторяет попытку.

В отличие от `createTempFile`, файлового дескриптора нет — возвращается просто строка с путём к только что созданной пустой директории.

#### Ответственность вызывающего кода

Вызывающий код обязан **удалить директорию** (и её содержимое, если оно было добавлено) по завершении работы. Используйте `removeDir` из `std/os`.

#### Условия ошибки

`OSError` выбрасывается, если:
- Родительская директория `dir` не существует.
- После `maxRetry` попыток свободное имя не найдено.
- У процесса нет прав на запись в `dir`.

#### Описание параметров

| Параметр | Тип | Описание |
|---|---|---|
| `prefix` | `string` | Текст перед случайным сегментом в имени директории. Может быть пустым. |
| `suffix` | `string` | Текст после случайного сегмента. Может быть пустым. |
| `dir` | `string` | Родительская директория. Пустая строка = системная временная директория. |

**Возвращает:** `string` — абсолютный путь к только что созданной временной директории.

#### Примеры

```nim
import tempfiles, std/os

let tmpDir = createTempDir("tmpprefix_", "_end")
# tmpDir — что-то вроде: /tmp/tmpprefix_YEl9VuVj_end

assert dirExists(tmpDir)

# Используем директорию ...
writeFile(tmpDir / "output.txt", "results")

removeDir(tmpDir)   # удаляет директорию со всем содержимым
```

Гарантированная очистка через `defer`:

```nim
import tempfiles, std/os

proc runSandboxed(cmd: string) =
  let sandbox = createTempDir("sandbox_", "")
  defer: removeDir(sandbox)

  # запустить cmd с sandbox в качестве рабочей директории
  # все выходные файлы изолированы; очищаются автоматически
```

Использование конкретной родительской директории:

```nim
import tempfiles, std/os

# Временная директория рядом с кэшем проекта
let workDir = createTempDir("job_", "_work", dir = "cache")
defer: removeDir(workDir)
```

Несуществующий родитель вызывает ошибку немедленно:

```nim
import tempfiles
doAssertRaises(OSError):
  discard createTempDir("", "", "nonexistent")
```

## Внутренняя механика

### Генерация случайного имени

Состояние генератора случайных чисел хранится в потоковой переменной (`nimTempPathState`) и лениво инициализируется при первом обращении в каждом потоке. Это означает:

- **Потокобезопасность:** у каждого потока независимое состояние ГСЧ — параллельные вызовы из разных потоков не мешают друг другу.
- **Отсутствие глобальной блокировки:** генерация имён lock-free.
- **Ленивая инициализация:** ГСЧ засевается от источника энтропии ОС при первом использовании, а не при запуске программы.

### Атомарное создание (`safeOpen`)

Приватная процедура `safeOpen` оборачивает платформенно-специфичный атомарный вызов открытия:

| Платформа | Механизм | Гарантия |
|---|---|---|
| POSIX (Linux, macOS, …) | `open(O_RDWR \| O_CREAT \| O_EXCL)` | Атомарно завершается неудачей, если файл существует |
| Windows | `CreateFileW(CREATE_NEW)` | Атомарно завершается неудачей, если файл существует |

Если файл уже существует, `safeOpen` возвращает `nil` (без исключения), и вызывающий код повторяет попытку с новым случайным именем. Любая другая ошибка ОС пробрасывается как `OSError`.

| `createTempDir` | Директорию | Строку пути | `removeDir(path)` |

## Типичные паттерны использования

### Паттерн 1 — Временный файл как безопасный промежуточный результат

```nim
import tempfiles, std/os

proc atomicWrite(dest, content: string) =
  # Сначала пишем во временный файл, затем переименовываем
  # → атомарная замена с точки зрения читателей
  let (tmp, tmpPath) = createTempFile("atomic_", ".tmp",
                                       dir = parentDir(dest))
  defer: close(tmp)
  tmp.write(content)
  moveFile(tmpPath, dest)   # атомарна на большинстве ФС при одном разделе
```

### Паттерн 2 — Изолированная рабочая директория для подпроцесса

```nim
import tempfiles, std/os, std/osproc

proc runIsolated(exe: string, args: seq[string]): int =
  let workDir = createTempDir("run_", "")
  defer: removeDir(workDir)
  let p = startProcess(exe, workingDir = workDir, args = args)
  result = waitForExit(p)
```

### Паттерн 3 — Временный путь для передачи внешнему инструменту

```nim
import tempfiles, std/os, std/osproc

# Некоторые инструменты сами решают, создавать ли файл.
# Даём им гарантированно уникальный путь без создания файла.
let path = genTempPath("ffmpeg_out_", ".mp4")
discard execCmd("ffmpeg -i input.mkv " & path)
defer: removeFile(path)
```

## Важные замечания

- Все три процедуры принимают пустой `dir`, который незаметно подменяется на `getTempDir()`. В production-коде лучше указывать это явно для ясности.
- `createTempFile` открывает файл в режиме `w+` (чтение-запись). Позиция в файле начинается с 0. После записи перед чтением обратно вызовите `setFilePos(0)`.
- Случайный сегмент использует только алфавитно-цифровые символы (`a-z`, `A-Z`, `0-9`), поэтому сгенерированные имена безопасны на всех основных файловых системах и в командных оболочках.
- Модуль помечен **экспериментальным** — в production-использовании зафиксируйте версию Nim.
