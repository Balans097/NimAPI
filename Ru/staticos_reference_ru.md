# Справочник модуля `staticos`

> **Назначение модуля:** `staticos` предоставляет утилиты для работы с путями файловой системы, которые работают **исключительно во время компиляции** (`{.compileTime.}`). Модуль воспроизводит подмножество стандартного модуля `os`, но выполняется внутри виртуальной машины (VM) компилятора Nim — что делает его безопасным и полностью работоспособным при **кросс-компиляции** на платформы, где `os` недоступен или ненадёжен.

---

## Зачем нужен `staticos`?

При кросс-компиляции Nim-программы для встроенных систем, WebAssembly или экзотических ОС стандартный модуль `os` может быть недоступен во время выполнения. Но иногда необходимо проверить файлы или директории **прямо во время работы компилятора** — например, чтобы встроить все файлы из папки в бинарник или убедиться, что нужный ресурс существует ещё до завершения сборки.

`staticos` решает эту задачу: все его процедуры выполняются внутри VM компилятора. Ваш итоговый бинарник никогда не вызывает эти функции — вся работа выполняется до его создания.

## Типы

### `PathComponent`

```nim
type PathComponent* = enum
  pcFile,        # путь указывает на обычный файл
  pcLinkToFile,  # путь указывает на символическую ссылку на файл
  pcDir,         # путь указывает на директорию
  pcLinkToDir    # путь указывает на символическую ссылку на директорию
```

Перечисление, классифицирующее тип записи в файловой системе. Используется в поле `kind` результатов, возвращаемых `staticWalkDir`.

| Значение | Описание |
|---|---|
| `pcFile` | Обычный файл |
| `pcLinkToFile` | Символическая ссылка, цель которой — файл |
| `pcDir` | Директория |
| `pcLinkToDir` | Символическая ссылка, цель которой — директория |

## Процедуры

---

### `staticFileExists`

```nim
proc staticFileExists*(filename: string): bool {.compileTime.}
```

#### Что делает

Возвращает `true`, если `filename` существует на **хостовой** файловой системе (той, на которой запущен компилятор) и является **обычным файлом** или **символической ссылкой на файл**.

Возвращает `false` для:
- Директорий
- Файлов устройств (`/dev/sda` и т.п.)
- Именованных каналов (FIFO)
- Сокетов
- Всего, что не существует

#### Важное отличие

Процедура проверяет файловую систему **хост-машины** на этапе компиляции, а не целевой машины во время выполнения. Если вы кросс-компилируете на Linux под Windows, `staticFileExists` видит файловую систему Linux.

#### Пример

```nim
# На этапе компиляции проверяем наличие обязательного конфигурационного файла.
# Если он отсутствует — сборка завершается с понятной ошибкой.

const configPath = "assets/config.json"

static:
  if not staticFileExists(configPath):
    error "Обязательный файл '" & configPath & "' не найден. Сборка невозможна."

# Безопасно встраиваем — мы уже знаем, что файл существует.
const configData = staticRead(configPath)
```

Другое применение — условная компиляция в зависимости от наличия файла:

```nim
static:
  when staticFileExists("src/platform_override.nim"):
    echo "Замечание сборщика: используется платформенное переопределение"
```

### `staticDirExists`

```nim
proc staticDirExists*(dir: string): bool {.compileTime.}
```

#### Что делает

Возвращает `true`, если путь `dir` существует и является **директорией**. Следует по символическим ссылкам — если `dir` является симлинком, ведущим к директории, результат будет `true`.

Возвращает `false`, если:
- Путь не существует
- Путь существует, но является файлом, а не директорией

#### Связь с `staticFileExists`

Эти две процедуры взаимно дополняют друг друга и для обычных записей файловой системы являются взаимоисключающими: для заданного пути максимум одна из них может вернуть `true` (если путь не существует, обе вернут `false`).

#### Пример

```nim
# Проверяем наличие директории с ресурсами перед тем, как её обходить.

const assetsDir = "assets/images"

static:
  if not staticDirExists(assetsDir):
    error "Директория ресурсов '" & assetsDir & "' не найдена. Выполните 'make assets'."

# Теперь можно безопасно обходить директорию и встраивать изображения.
```

Также удобно для выбора между запасными путями:

```nim
const dataDir =
  when staticDirExists("data/override"):
    "data/override"
  else:
    "data/default"
```

### `staticWalkDir`

```nim
proc staticWalkDir*(dir: string; relative = false): seq[
    tuple[kind: PathComponent, path: string]] {.compileTime.}
```

#### Что делает

Перечисляет содержимое директории `dir` **на один уровень вглубь** (нерекурсивно) и возвращает его в виде `seq` кортежей `(kind, path)`. Каждый кортеж содержит:

- **`kind`** — значение `PathComponent`, описывающее тип записи
- **`path`** — путь к записи (абсолютный или относительный, в зависимости от параметра `relative`)

#### Параметр `relative`

| Значение `relative` | Возвращаемый путь |
|---|---|
| `false` (по умолчанию) | Полный путь: `"assets/images/logo.png"` |
| `true` | Путь относительно `dir`: `"logo.png"` |

#### Нерекурсивное поведение

`staticWalkDir` **не заходит** в поддиректории. Он перечисляет только то, что находится непосредственно внутри `dir`. Для рекурсивного обхода дерева нужно вызвать `staticWalkDir` повторно для каждой встреченной записи типа `pcDir` (см. пример ниже).

#### Пример — встроить все шейдеры из директории

```nim
import staticos

const shadersDir = "src/shaders"

# На этапе компиляции собираем все .glsl-файлы и встраиваем их исходный код.
const shaderSources = block:
  var result: seq[tuple[name, src: string]]
  for (kind, path) in staticWalkDir(shadersDir, relative = false):
    if kind == pcFile and path.endsWith(".glsl"):
      result.add((name: path, src: staticRead(path)))
  result

# shaderSources — константа времени компиляции. Никакого ввода-вывода в рантайме.
```

#### Пример — рекурсивный обход директории

```nim
proc collectFiles(dir: string): seq[string] {.compileTime.} =
  for (kind, path) in staticWalkDir(dir):
    case kind
    of pcFile, pcLinkToFile:
      result.add(path)
    of pcDir, pcLinkToDir:
      result.add(collectFiles(path))  # рекурсия

const allFiles = collectFiles("assets")
```

#### Пример — вывод типов записей

```nim
static:
  for (kind, path) in staticWalkDir("resources", relative = true):
    case kind
    of pcFile:       echo "  файл: ", path
    of pcLinkToFile: echo "  ссылка→файл: ", path
    of pcDir:        echo "  директория: ", path
    of pcLinkToDir:  echo "  ссылка→директория: ", path
```

## Компиляция vs Рантайм: сравнение

| Характеристика | `staticos` | Модуль `os` |
|---|---|---|
| Выполняется | Во время компиляции | Во время выполнения |
| Видит | Файловую систему хоста | Файловую систему цели |
| Безопасен при кросс-компиляции | ✅ Да | ⚠️ Может не работать |
| Результат встраивается в бинарник | Как `const` | Нет |
| Рекурсивный обход | Вручную (повторный вызов) | Доступен `walkDirRec` |

## Типичные паттерны использования

### Паттерн 1 — Быстрая проверка сборки (fail-fast)

```nim
static:
  doAssert staticFileExists("LICENSE"), "Файл LICENSE должен присутствовать в корне проекта"
  doAssert staticDirExists("assets"), "Директория assets/ обязательна"
```

Если условие не выполняется, компилятор немедленно выдаёт ошибку с понятным сообщением — никакого падения в рантайме.

### Паттерн 2 — Автоматическое встраивание всей директории

```nim
const files = block:
  var t: seq[tuple[name, content: string]]
  for (kind, path) in staticWalkDir("web/static"):
    if kind == pcFile:
      t.add((path, staticRead(path)))
  t
```

Весь каталог статических файлов превращается в константу времени компиляции — идеально для встроенных HTTP-серверов или ресурсов игры.

### Паттерн 3 — Условная компиляция по наличию файла

```nim
when staticFileExists("src/debug_helpers.nim"):
  include "src/debug_helpers.nim"
```

Позволяет опционально добавлять отладочный код без флагов командной строки.

## Технические замечания

- Все процедуры помечены **`{.compileTime.}`** — вызов вне `static:`-блока, определения `const` или другого контекста времени компиляции является ошибкой компиляции.
- Пути интерпретируются относительно **текущей рабочей директории компилятора**, которая обычно совпадает с корнем проекта (там, где запускается `nim c`).
- Реальные реализации этих процедур предоставляются слоем VM ops компилятора Nim (а не Nim-кодом), что отражено в теле каждой процедуры: `raiseAssert "implemented in the vmops"`.
