# SharedTable — справочное руководство

> **⚠ Устарело:** Модуль помечен `{.deprecated.}` в стандартной библиотеке Nim. Он сохранён для поддержки унаследованного кода. Для новых проектов рассмотрите `std/tables` в сочетании с явными примитивами синхронизации или специализированную конкурентную хеш-таблицу.

---

## Общее описание

`sharedtables` предоставляет **универсальную хеш-таблицу, живущую в разделяемой памяти** (shared memory) и пригодную для использования из нескольких потоков. Вся таблица защищена **единственным мьютексом**: только один поток в каждый момент времени может читать или записывать данные.

### Ключевые правила использования

- **Используйте только типы, не управляемые GC**, в качестве ключей (`A`) и значений (`B`). Таблица живёт в разделяемой памяти (`allocShared0`), о которой сборщик мусора Nim ничего не знает. Хранение управляемых GC типов (строки, `seq`, `ref`-объекты и т. д.) приведёт к повреждению памяти или аварийному завершению программы.
- **Всегда вызывайте `init` первым.** Таблица — это обычный `object` без автоматической инициализации. Без вызова `init` мьютекс и указатель на данные не инициализированы.
- **Всегда вызывайте `deinitSharedTable` по завершении работы.** Без этого вызова вы теряете выделенную память и ресурс мьютекса операционной системы.
- **Высокая конкуренция за блокировку — сигнал к переосмыслению алгоритма.** Документация самого модуля говорит: если конкуренция настолько высока, что вам нужна lock-free таблица, вы делаете что-то не так. Сначала пересмотрите дизайн.

### Сигнатура типа

```nim
type SharedTable*[A, B] = object
```

`A` — тип ключа, `B` — тип значения. Оба должны быть копируемы и иметь процедуру `hash`.

## Экспортируемый API

### `init`

```nim
proc init*[A, B](t: var SharedTable[A, B], initialSize = 32)
```

**Назначение:** Выделяет внутренний массив данных в разделяемой памяти и инициализирует мьютекс. Это конструктор — перед его вызовом никакие другие операции с таблицей недопустимы.

`initialSize` — *подсказка* о начальном числе слотов. Библиотека округляет значение вверх до ближайшей степени двойки, обеспечивающей приемлемый коэффициент заполнения. Если примерный объём данных известен заранее, задание `initialSize` позволяет избежать лишних перераспределений.

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init(64)          # подсказка: около 64 элементов до первого rehash
# ... работа с таблицей ...
deinitSharedTable(t)
```

---

### `deinitSharedTable`

```nim
proc deinitSharedTable*[A, B](t: var SharedTable[A, B])
```

**Назначение:** Освобождает выделенную разделяемую память и уничтожает мьютекс. Это деструктор — вызовите его ровно один раз, когда таблица больше не нужна. После вызова объект таблицы недействителен.

**Пример**

```nim
var t: SharedTable[int32, float32]
t.init()
# ... использование t ...
deinitSharedTable(t)   # обязательная очистка
```

---

### `[]=` (вставка или обновление)

```nim
proc `[]=`*[A, B](t: var SharedTable[A, B], key: A, val: B)
```

**Назначение:** Вставляет новую пару ключ-значение или **заменяет** существующее значение, если ключ уже есть в таблице. Это стандартная операция "upsert" — после выполнения в таблице всегда будет ровно одна запись для данного ключа.

Мьютекс удерживается на протяжении всей операции, поэтому обновление атомарно с точки зрения других потоков.

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init()
t[1'i32] = 100'i32     # вставка
t[1'i32] = 200'i32     # замена — теперь t[1] == 200
deinitSharedTable(t)
```

---

### `add`

```nim
proc add*[A, B](t: var SharedTable[A, B], key: A, val: B)
```

**Назначение:** Вставляет новую пару **без проверки** существующей записи с тем же ключом. Если ключ уже есть, таблица окажется содержащей **две записи с одинаковым ключом** — поведение, аналогичное мультимапу. Стандартный поиск (`mget`, `withValue`) найдёт только одну из них (ту, которую разрешит алгоритм открытой адресации), а вторая останется недостижимой.

Используйте `add` только если вы уверены, что ключ ещё не присутствует в таблице, или если намеренно создаёте мультимап-подобную структуру с собственной стратегией обхода. В подавляющем большинстве случаев правильный выбор — `[]=`.

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init()
t.add(1'i32, 10'i32)
t.add(1'i32, 20'i32)   # ⚠ дублирующийся ключ — вторая запись недостижима через mget
deinitSharedTable(t)
```

---

### `del`

```nim
proc del*[A, B](t: var SharedTable[A, B], key: A)
```

**Назначение:** Удаляет запись с заданным ключом. Если ключа нет в таблице, вызов тихо игнорируется — исключение не возникает.

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init()
t[5'i32] = 99'i32
t.del(5'i32)             # запись удалена
t.del(99'i32)            # ключ не найден — ошибки нет
deinitSharedTable(t)
```

---

### `len`

```nim
proc len*[A, B](t: var SharedTable[A, B]): int
```

**Назначение:** Возвращает текущее количество пар ключ-значение в таблице. Мьютекс захватывается для чтения внутреннего счётчика, поэтому результат согласован на момент вызова. В многопоточной системе к моменту использования возвращённого числа оно может уже устареть — воспринимайте результат как снимок (snapshot).

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init()
t[1'i32] = 1'i32
t[2'i32] = 2'i32
assert t.len() == 2
deinitSharedTable(t)
```

---

### `mget`

```nim
proc mget*[A, B](t: var SharedTable[A, B], key: A): var B
```

**Назначение:** Возвращает **изменяемую ссылку** на значение, хранящееся под ключом `key`. Если ключ отсутствует, выбрасывается исключение `KeyError`.

Поскольку `mget` возвращает ссылку (`var B`) во внутреннее хранилище таблицы, запись через эту ссылку после освобождения мьютекса **не является потокобезопасной** и может привести к гонке данных. Для безопасного изменения значения на месте под удерживаемым мьютексом используйте `withValue` или `withKey`.

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init()
t[7'i32] = 42'i32
let v = t.mget(7'i32)   # v == 42
# t.mget(99'i32)         # вызвало бы KeyError
deinitSharedTable(t)
```

### `mgetOrPut`

```nim
proc mgetOrPut*[A, B](t: var SharedTable[A, B], key: A, val: B): var B
```

**Назначение:** Ищет `key`. Если нашёл — возвращает изменяемую ссылку на существующее значение. Если не нашёл — вставляет `(key, val)` и возвращает ссылку на только что вставленное значение.

Вся операция (поиск + возможная вставка) выполняется под мьютексом. **Однако** возвращаемая ссылка указывает во внутреннее хранилище и становится небезопасной, как только другой поток изменит или перераспределит таблицу. Сам модуль явно помечает это как *inherently unsafe в контексте многопоточности*. Используйте только когда конкурентных изменений нет (например, на фазе однопоточной инициализации) или сразу скопируйте возвращённое значение.

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init()
# Ключ 3 отсутствует → вставляет (3, 0), возвращает ссылку на 0
var val = t.mgetOrPut(3'i32, 0'i32)
assert val == 0
# Ключ 3 теперь есть → возвращает ссылку на существующее значение
var val2 = t.mgetOrPut(3'i32, 99'i32)
assert val2 == 0   # исходное значение; 99 не использовалось
deinitSharedTable(t)
```

### `hasKeyOrPut`

```nim
proc hasKeyOrPut*[A, B](t: var SharedTable[A, B], key: A, val: B): bool
```

**Назначение:** Возвращает `true`, если `key` уже есть в таблице (без её изменения). Возвращает `false` и вставляет `(key, val)`, если ключ отсутствует. Проверка и возможная вставка — единая атомарная операция под мьютексом.

Это удобный примитив "вставить если отсутствует" — полезен для инициализации значений по умолчанию при первом обращении без отдельной проверки существования и последующей записи.

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init()
assert t.hasKeyOrPut(1'i32, 10'i32) == false  # нет → вставляет 10
assert t.hasKeyOrPut(1'i32, 20'i32) == true   # есть → без изменений
assert t.mget(1'i32) == 10'i32                # значение по-прежнему 10
deinitSharedTable(t)
```

### `withValue` (двухветочная форма — только если ключ найден)

```nim
template withValue*[A, B](t: var SharedTable[A, B], key: A,
                           value, body: untyped)
```

**Назначение:** Ищет `key` под мьютексом. Если нашёл — внедряет указатель (`value`) на хранимое значение и выполняет `body`, пока мьютекс удерживается. Если ключ отсутствует — `body` пропускается. Мьютекс освобождается по выходу из `body` (через `try/finally`).

Это **наиболее безопасный способ читать или изменять значение на месте**: мьютекс гарантирует эксклюзивный доступ на всё время выполнения `body`. Внедряемая переменная `value` имеет тип `ptr B` — разыменовывайте через `value[]`.

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init()
t[5'i32] = 100'i32

t.withValue(5'i32, v):
  echo v[]        # выводит 100
  v[] = 200       # безопасное изменение на месте под удерживаемым мьютексом

t.withValue(99'i32, v):
  echo "не вызовется — ключ отсутствует"

deinitSharedTable(t)
```

### `withValue` (трёхветочная форма — ключ найден / не найден)

```nim
template withValue*[A, B](t: var SharedTable[A, B], key: A,
                           value, body1, body2: untyped)
```

**Назначение:** То же, что двухветочная форма, но принимает второй блок (`body2`), который выполняется когда ключ **отсутствует**. Позволяет обработать оба случая — «нашёл» и «не нашёл» — атомарно, в рамках одного захвата мьютекса.

Для передачи второй ветки используется синтаксис `do:`.

**Пример**

```nim
var t: SharedTable[int32, int32]
t.init()
t[1'i32] = 10'i32

# Ключ есть: инкрементируем значение
t.withValue(1'i32, v):
  inc v[]
do:
  t[1'i32] = 1'i32   # сюда не попадём, но шаблон понятен

# Ключ отсутствует: вставляем значение по умолчанию
t.withValue(2'i32, v):
  inc v[]
do:
  t[2'i32] = 0'i32   # инициализируем; следующий вызов войдёт в первую ветку

assert t.mget(1'i32) == 11'i32
deinitSharedTable(t)
```

### `withKey`

```nim
proc withKey*[A, B](t: var SharedTable[A, B], key: A,
                    mapper: proc(key: A, val: var B, pairExists: var bool))
```

**Назначение:** Наиболее мощный и гибкий примитив мутации. Вызывает `mapper` **под мьютексом**, передавая текущий ключ, текущее (или нулевое по умолчанию) значение и булев флаг наличия записи. Mapper может:

- **Читать** `val` и `pairExists`, не изменяя их.
- **Изменять** `val` для обновления существующего значения или задания значения новой записи (при этом установите `pairExists = true`, чтобы подтвердить вставку).
- **Удалять** запись, установив `pairExists = false`.

Поскольку весь цикл «прочитать-изменить-записать» выполняется под мьютексом, гонки данных исключены — воспроизвести это безопасно через комбинацию `mget` + `[]=` невозможно.

Mapper должен быть **коротким и неблокирующим**: он удерживает глобальный мьютекс таблицы всё время своего выполнения, блокируя остальные потоки.

**Пример — атомарный декремент с автоудалением**

```nim
var t: SharedTable[int32, int32]
t.init()
t[1'i32] = 3'i32

# Декрементируем; удаляем ключ, когда счётчик обнуляется
t.withKey(1'i32) do (k: int32, v: var int32, exists: var bool):
  if exists:
    dec v
    if v <= 0:
      exists = false   # сигнал на удаление

assert t.len() == 1    # счётчик был 3 → стал 2, ключ ещё есть

deinitSharedTable(t)
```

**Пример — вставить только если отсутствует**

```nim
t.withKey(42'i32) do (k: int32, v: var int32, exists: var bool):
  if not exists:
    v = 100'i32
    exists = true      # подтверждаем вставку
```

## Сводка по потокобезопасности

| Операция | Атомарна? | Возвращает живую ссылку? |
|---|---|---|
| `[]=` | ✅ да | — |
| `del` | ✅ да | — |
| `add` | ✅ да | — |
| `len` | ✅ да (снимок) | — |
| `hasKeyOrPut` | ✅ да | — |
| `withValue` (обе формы) | ✅ да | `ptr B` действителен только внутри тела |
| `withKey` | ✅ да | — |
| `mget` | ⚠ мьютекс освобождён до возврата | `var B` — небезопасна после вызова |
| `mgetOrPut` | ⚠ мьютекс освобождён до возврата | `var B` — небезопасна после вызова |

## Полный рабочий пример

```nim
import std/sharedtables, std/os

var hits: SharedTable[int32, int32]
hits.init(16)

proc worker(id: int32) {.thread.} =
  for i in 0'i32 ..< 1000'i32:
    hits.withKey(id) do (k: int32, v: var int32, exists: var bool):
      inc v
      if not exists: exists = true

var threads: array[4, Thread[int32]]
for i in 0 ..< 4:
  createThread(threads[i], worker, int32(i))
joinThreads(threads)

for i in 0'i32 ..< 4'i32:
  echo "поток ", i, " → ", hits.mget(i), " итераций"

deinitSharedTable(hits)
```
