# Справочник модуля `widestrs`

> Поддержка «широких строк» C/C++ в Nim (последовательности символов в кодировке UTF-16).  
> Модуль входит в стандартную библиотеку Nim и служит мостом между нативными UTF-8 строками Nim и строками UTF-16, которые требуются Windows API и другим интерфейсам на C/C++.

---

## Зачем вообще нужны широкие строки?

Большинство современного мира хранит текст в **UTF-8** — кодировке переменной ширины, где ASCII-символы занимают 1 байт, а всё остальное — от 2 до 4 байт. Нативный тип `string` в Nim — тоже UTF-8.

Windows же изначально строилась вокруг **UTF-16**, где каждый символ занимает 2 байта (или 4 байта для символов за пределами Базовой многоязычной плоскости — через механизм *суррогатных пар*). «Широкие» (`W`-варианты) функции Windows API — `CreateFileW`, `MessageBoxW` и т. п. — ожидают строки в UTF-16, завершённые нулевым 16-битным словом.

Данный модуль предоставляет типы и процедуры преобразования, необходимые для работы с этим интерфейсом.

---

## Используемые концепции Unicode

| Термин | Значение |
|--------|---------|
| **Кодовая точка** | Скалярное значение Unicode, например U+0041 = `A`, U+1F600 = 😀 |
| **BMP** | Базовая многоязычная плоскость — кодовые точки U+0000…U+FFFF, умещаются в одном UTF-16-элементе |
| **Суррогатная пара** | Два 16-битных элемента (старший + младший суррогат), кодирующие символ выше U+FFFF |
| **Символ замены** | U+FFFD `<EFBFBD>` — подставляется вместо некорректной последовательности |

## Экспортируемые типы

### `Utf16Char`

```nim
type Utf16Char* = distinct int16
```

**Что это:** Один UTF-16-элемент — 16-битное целое число, объявленное как отдельный (`distinct`) тип, чтобы компилятор не давал случайно перепутать его с обычным `int16`. Каждый `Utf16Char` либо непосредственно представляет символ BMP, либо является одной половиной суррогатной пары.

---

### `WideCString`

```nim
# nimv2 (сборщик мусора ARC/ORC):
type WideCString* = ptr UncheckedArray[Utf16Char]

# Старый сборщик мусора:
type WideCString* = ref UncheckedArray[Utf16Char]
```

**Что это:** Сырой указатель (или управляемая ссылка — в зависимости от режима сборщика мусора) на завершённый нулём массив значений `Utf16Char`. Именно этот тип передаётся в функции C/C++, ожидающие `wchar_t*` или `LPWSTR`.

Поскольку в режиме `nimv2` `WideCString` является сырым указателем, он не несёт информации о владельце и не управляется автоматически. Для работы на стороне Nim предпочтительнее использовать `WideCStringObj`.

---

### `WideCStringObj`

```nim
# Только для nimv2:
type WideCStringObj* = object
  bytes: int
  data: WideCString
```

**Что это:** Владеющая обёртка над `WideCString`. Она управляет памятью буфера UTF-16: выделяет при создании и освобождает в деструкторе. Копирование намеренно запрещено (`=copy` помечен `{.error.}`); объект можно только перемещать.

В сборке со старым сборщиком мусора `WideCStringObj` — просто псевдоним для `WideCString` (тип-ссылка), и время жизни управляется сборщиком.

**Ключевые свойства:**
- Поддерживает `[]` и `[]=` для обращения к отдельным `Utf16Char` по индексу.
- Неявно преобразуется в `WideCString` через конвертер `toWideCString`, поэтому его можно передавать туда, где ожидается сырой указатель.
- Потокобезопасное выделение памяти: при компиляции с `--threads:on` использует `allocShared0`/`deallocShared`.

## Экспортируемые процедуры

---

### `len` — длина широкой строки

```nim
proc len*(w: WideCString): int
proc len*(w: WideCStringObj): int  # только nimv2, вызывает предыдущую
```

**Что делает:** Считает количество `Utf16Char`-элементов в широкой строке, **не включая** завершающий нуль. Работает линейным сканированием до нулевого `int16` — поля с хранимой длиной нет.

**Важная оговорка:** Считаются UTF-16 *элементы* (code units), а не кодовые точки Unicode. Один эмодзи или редкий символ CJK, закодированный суррогатной парой, даёт **2** элемента, хотя представляет **1** символ.

**Замечание о производительности:** Каждый вызов проходит всю строку. Если длина нужна несколько раз — сохраните её в переменную.

**Пример:**

```nim
# "Hi" в UTF-16 → 2 элемента + нулевой терминатор
let w = newWideCString("Hi")
echo w.len   # → 2

# Эмодзи, закодированный суррогатной парой → 2 элемента для 1 символа
let emoji = newWideCString("😀")
echo emoji.len   # → 2
```

### `newWideCString` — создание широкой строки

Четыре перегрузки; выбирайте ту, которая соответствует вашим исходным данным.

---

#### `newWideCString(size: int): WideCStringObj`

```nim
proc newWideCString*(size: int): WideCStringObj
```

**Что делает:** Выделяет пустой (обнулённый) буфер, достаточный для хранения `size` UTF-16-элементов плюс нулевой терминатор. Буфер заполнен нулями, поэтому результат сразу является корректной (пустой) широкой строкой.

**Область применения:** Предварительное выделение буфера, который будет заполнен вручную — например, при получении данных из вызова Windows API, записывающего результат в предоставленный буфер.

**Пример:**

```nim
# Зарезервировать место для 256 UTF-16-элементов
var buf = newWideCString(256)
# Передать buf в Windows API, который заполнит буфер, затем преобразовать обратно:
# let name = $buf
```

---

#### `newWideCString(source: cstring, L: int): WideCStringObj`

```nim
proc newWideCString*(source: cstring, L: int): WideCStringObj
```

**Что делает:** Преобразует UTF-8 строку `cstring` известной байтовой длины `L` в свежевыделенный UTF-16 `WideCStringObj`. Каждая UTF-8 кодовая точка декодируется и перекодируется в UTF-16:

- Символы BMP (U+0000…U+FFFF, кроме суррогатов) → один `Utf16Char`
- Символы выше U+FFFF → суррогатная пара (два `Utf16Char`)
- Некорректные суррогаты или вышедшие за диапазон кодовые точки → символ замены U+FFFD

**Предупреждение из исходного кода:** `source` должна быть выделена минимум на `L` байт. Проверки выхода за границы, кроме параметра длины, не производится.

**Пример:**

```nim
let s: cstring = "Привет"
let w = newWideCString(s, s.len)
echo w.len   # → 6  (6 кириллических символов, каждый — один UTF-16-элемент)
```

---

#### `newWideCString(s: cstring): WideCStringObj`

```nim
proc newWideCString*(s: cstring): WideCStringObj
```

**Что делает:** Удобная перегрузка, автоматически определяющая длину через `s.len`. Возвращает нулевую широкую строку (`nullWide`), если `s` равна nil.

**Пример:**

```nim
let w = newWideCString(cstring "Hello, мир")
echo w.len   # → 10
```

---

#### `newWideCString(s: string): WideCStringObj`

```nim
proc newWideCString*(s: string): WideCStringObj
```

**Что делает:** Наиболее удобная перегрузка — принимает нативный Nim-тип `string` напрямую. Внутри приводит к `cstring` и вызывает двухаргументную перегрузку с `s.len`.

**Именно эту перегрузку вы будете использовать чаще всего.**

**Пример:**

```nim
import widestrs

let greeting = "Héllo, wörld! 🌍"
let wide = newWideCString(greeting)

# wide — UTF-16 представление, готовое для передачи в Windows API
echo wide.len   # → 16 (15 BMP-символов + 1 суррогатная пара для 🌍 = 16 элементов)

# Преобразование обратно в Nim-строку
let back = $wide
echo back       # → "Héllo, wörld! 🌍"
assert back == greeting
```

### `$` — преобразование широкой строки в Nim-строку (UTF-8)

Четыре перегрузки; все декодируют UTF-16 → UTF-8.

---

#### `$(w: WideCString, estimate: int, replacement: int): string`

```nim
proc `$`*(w: WideCString, estimate: int, replacement: int = 0xFFFD): string
```

**Что делает:** Основная процедура преобразования. Читает `w` до нулевого терминатора, декодирует каждый UTF-16-элемент (корректно обрабатывая суррогатные пары) и возвращает результат как UTF-8 Nim-строку.

**Параметры:**

- `w` — широкая строка для преобразования.
- `estimate` — подсказка для начальной ёмкости строки. Хорошая оценка позволяет избежать перераспределений памяти. Реально выделяется `estimate + estimate / 4`.
- `replacement` — кодовая точка, подставляемая при обнаружении некорректной UTF-16-последовательности (по умолчанию U+FFFD `<EFBFBD>`).

**Обработка суррогатов:**
- Корректная суррогатная пара (старший, за ним младший) → декодируется в соответствующую кодовую точку Unicode.
- Одиночный суррогат (старший без следующего за ним младшего, или неожиданно встретившийся младший) → заменяется на `replacement`.

**Пример:**

```nim
let w = newWideCString("Hello 🌍")
# Используем достаточный estimate, чтобы избежать перераспределений
let s = `$`(w.data, 20)
echo s   # → "Hello 🌍"
```

---

#### `$(s: WideCString): string`

```nim
proc `$`*(s: WideCString): string
```

**Что делает:** Сокращённая форма, вызывает предыдущую процедуру с `estimate = 80`. Подходит для большинства строк. Если строка заведомо длинная — используйте перегрузку с явным `estimate` для лучшей производительности.

**Пример:**

```nim
let w = newWideCString("Nim — это круто!")
echo $w   # → "Nim — это круто!"
```

---

#### `$(s: WideCStringObj, estimate: int, replacement: int): string` *(только nimv2)*

```nim
proc `$`*(s: WideCStringObj, estimate: int, replacement: int = 0xFFFD): string
```

**Что делает:** Делегирует вызов перегрузке для `WideCString`, используя внутренний указатель объекта. Семантика идентична версии для сырого указателя.

---

#### `$(s: WideCStringObj): string` *(только nimv2)*

```nim
proc `$`*(s: WideCStringObj): string
```

**Что делает:** Краткая форма `$` для `WideCStringObj` с estimate по умолчанию равным 80. Самый удобный способ преобразовать широкую строку обратно в Nim-строку.

**Пример:**

```nim
import widestrs

let original = "Привет, мир! 🌏"
let wide = newWideCString(original)
let back = $wide
assert back == original
echo back   # → "Привет, мир! 🌏"
```

---

### `toWideCString` — неявный конвертер *(только nimv2)*

```nim
converter toWideCString*(x: WideCStringObj): WideCString {.inline.}
```

**Что делает:** Позволяет использовать `WideCStringObj` там, где ожидается `WideCString` (сырой указатель), без явного приведения типов. Именно благодаря этому передача `WideCStringObj` в функции FFI выглядит прозрачно.

**Это конвертер, а не обычная процедура** — вы никогда не вызываете его напрямую; компилятор вставляет его автоматически.

**Пример:**

```nim
proc someWindowsApi(s: WideCString) = discard  # заглушка FFI

let w = newWideCString("test")
someWindowsApi(w)   # WideCStringObj неявно преобразуется в WideCString
```

## Полный цикл преобразований

```nim
import widestrs

# --- UTF-8 → UTF-16 ---
let utf8 = "Héllo, 世界! 🎉"
let wide = newWideCString(utf8)

echo "UTF-16-элементов: ", wide.len
# BMP-символы: H é l l o ,   世 界 ! = 10
# Суррогатная пара для 🎉              = 2
# Итого: 12

# --- Обращение к отдельным элементам ---
echo cast[int16](wide[0])   # → 72  (= 'H')

# --- UTF-16 → UTF-8 ---
let roundTrip = $wide
assert roundTrip == utf8
echo roundTrip   # → "Héllo, 世界! 🎉"
```

## Типичный паттерн при работе с Windows API

Главное практическое назначение модуля — вызов функций Windows API:

```nim
import widestrs

# Допустим, мы оборачиваем CreateFileW из Windows API:
proc CreateFileW(
  lpFileName: WideCString,
  dwDesiredAccess: uint32,
  # ... остальные параметры
): int {.importc, dynlib: "kernel32".}

proc openFile(path: string): int =
  let widePath = newWideCString(path)   # UTF-8 → UTF-16
  result = CreateFileW(widePath,        # WideCStringObj → WideCString автоматически
                       0x80000000'u32)
```

## Быстрая шпаргалка

| Символ | Тип | Назначение |
|--------|-----|-----------|
| `Utf16Char` | тип | Один UTF-16-элемент (отдельный `int16`) |
| `WideCString` | тип | Сырой указатель на завершённый нулём массив UTF-16 |
| `WideCStringObj` | тип | Владеющая, перемещаемая обёртка над `WideCString` |
| `len(w)` | процедура | Число UTF-16-элементов (линейный поиск нуля) |
| `newWideCString(size)` | процедура | Выделить пустой буфер на `size` элементов |
| `newWideCString(cstring, int)` | процедура | UTF-8 cstring + длина → UTF-16 |
| `newWideCString(cstring)` | процедура | UTF-8 cstring (автодлина) → UTF-16 |
| `newWideCString(string)` | процедура | Nim-строка → UTF-16 (самый частый вариант) |
| `$w` | процедура | Широкая строка UTF-16 → Nim UTF-8 строка |
| `toWideCString` | конвертер | `WideCStringObj` → `WideCString` (неявный) |
