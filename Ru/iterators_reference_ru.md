# Справочник модуля iterators

> **Модуль:** `iterators`  
> **Часть:** Стандартная библиотека Nim  
> **Назначение:** Предоставляет стандартный протокол итерации для всех встроенных типов коллекций Nim. Каждый цикл `for` по строке, последовательности, массиву, множеству, кортежу или объекту опирается на итератор, определённый здесь. Модуль также устанавливает соглашения об именовании — `items`, `mitems`, `pairs`, `mpairs`, `fields`, `fieldPairs` — которые язык использует как точки расширения для пользовательских типов.

---

## Основные концепции

### Соглашения об именовании

Модуль следует четырём последовательным соглашениям:

| Имя | Что даёт | Мутация? |
|---|---|---|
| `items` | значение каждого элемента | нет |
| `mitems` | каждый элемент по ссылке `var` | **да** |
| `pairs` | кортежи `(индекс, элемент)` | нет |
| `mpairs` | кортежи `(индекс, var элемент)` | **да** |

`fields` и `fieldPairs` — структурные эквиваленты для кортежей и объектов, работающие во время компиляции.

### Тип возврата `lent`

Большинство итераторов `items` возвращают `lent T` (заимствование только для чтения), а не копию `T`. Это означает, что переменная цикла является псевдонимом в коллекцию — копирование не происходит, что особенно ценно для элементов большого размера. Псевдоним нельзя переприсваивать; используйте `mitems`, когда нужно изменять элементы на месте.

### Безопасность мутаций для `seq` и `string`

Итераторы по `seq[T]` и `string` запоминают длину перед началом цикла и проверяют её на каждой итерации. Это перехватывает случайные вызовы `add` или `delete` внутри тела цикла, которые сделали бы итерацию некорректной.

### `unCheckedInc`

Переменная индекса инкрементируется без проверки переполнения (`{.overflowChecks: off.}`). Это сделано намеренно: для коллекции длиной `high(int)` последний инкремент вызвал бы переполнение, но условие цикла проверяется *до* yield, поэтому переполнение не наблюдается. Это позволяет избежать лишних накладных расходов в плотных циклах.

## items — итерация по значениям

Все итераторы `items` поддерживают неявный синтаксис `for x in collection`. Ключевое слово `in` раскрывается в вызов `items`.

---

### `items` для `openArray[T]` (не char)

```nim
iterator items*[T: not char](a: openArray[T]): lent T {.inline.}
```

**Что это такое:** Самый универсальный итератор items — работает с любой непрерывной последовательностью: массивами, последовательностями и срезами, переданными как `openArray`. Возвращает каждый элемент как `lent T` (псевдоним только для чтения), избегая лишних копий. Ограничение `not char` существует потому, что для `char`-элементов есть отдельная перегрузка (см. ниже).

```nim
let nums = [10, 20, 30]
for n in nums:     # раскрывается в: for n in items(nums)
  echo n           # выводит 10, 20, 30

proc sumAll(a: openArray[int]): int =
  for x in a: result += x

echo sumAll([1, 2, 3])    # 6
echo sumAll(@[4, 5, 6])   # 15 — тот же итератор, другой контейнер
```

---

### `items` для `openArray[char]`

```nim
iterator items*[T: char](a: openArray[T]): T {.inline.}
```

**Что это такое:** Идентично версии для не-char, но возвращает `char` по значению, а не по ссылке `lent`. Причина — ошибка на уровне VM: взятие адреса элемента `openArray[char]`, полученного из строки, вызывает сбой в интерпретаторе времени компиляции. Поскольку возврат `char` по ссылке не даёт никакого выигрыша в производительности (один байт меньше указателя), возврат по значению одновременно корректен и эффективен.

```nim
let s = "hello"
for c in openArray[char](s):
  echo c   # h, e, l, l, o
```

---

### `items` для `array[IX, T]`

```nim
iterator items*[IX, T](a: array[IX, T]): T {.inline.}
```

**Что это такое:** Итерирует по массиву фиксированного размера. Тип индекса `IX` может быть любым порядковым типом — диапазоном целых чисел, перечислением или `char`. Для пустого массива цикл молча не производит итераций (проверяется во время компиляции через `when`), избегая неловкого краевого случая `low(IX) > high(IX)`.

Структура цикла `while true` с прерыванием по `i >= high(IX)` используется вместо простого `while i <= high(IX)`, чтобы избежать переполнения, когда `high(IX)` равен `high(int)`.

```nim
let arr = [1, 2, 3]
for x in arr: echo x   # 1, 2, 3

# Массив с индексом-перечислением:
type Dir = enum North, South, East, West
let labels: array[Dir, string] = ["N", "S", "E", "W"]
for s in labels: echo s
```

---

### `items` для `set[T]`

```nim
iterator items*[T](a: set[T]): T {.inline.}
```

**Что это такое:** Итерирует только по тем элементам, которые *фактически присутствуют* в множестве — не по всем возможным значениям, которые множество *может* содержать. Внутри сканирует полный диапазон `T` и возвращает только те значения, членство которых подтверждено. Порядок итерации всегда по возрастанию порядкового значения.

```nim
let s = {3, 1, 4, 1, 5}   # дубликаты схлопнуты
for x in s: echo x         # выводит 1, 3, 4, 5 (по порядку)

type Color = enum red, green, blue
let colors = {red, blue}
for c in colors: echo c    # red, blue
```

---

### `items` для `cstring`

```nim
iterator items*(a: cstring): char {.inline.}
```

**Что это такое:** Итерирует по символам C-строки до нулевого терминатора `'\0'`. На нативных бэкендах итерация останавливается на первом `'\0'`, поэтому встроенные нули в середине `cstring` никогда не достигаются. На бэкендах JS и NimVM длина вычисляется заранее.

> **Внимание:** Передача `nil` в качестве аргумента вызывает ошибку сегментации (SIGSEGV). Реализация намеренно не добавляет проверку на nil, чтобы не платить за неё на каждой итерации.

```nim
from std/sequtils import toSeq
assert toSeq("abc\0def".cstring) == @['a', 'b', 'c']
# '\0' завершает итерацию; "def" никогда не достигается
```

---

### `items` для `seq[T]`

```nim
iterator items*[T](a: seq[T]): lent T {.inline.}
```

**Что это такое:** Возвращает каждый элемент последовательности по ссылке `lent`. Запоминает длину перед циклом и проверяет её стабильность на каждом шаге — это утверждение перехватывает ошибки, когда тело цикла изменяет последовательность (добавляет или удаляет элементы), что сделало бы цикл некорректным.

```nim
let fruits = @["apple", "banana", "cherry"]
for f in fruits:
  echo f   # apple, banana, cherry

# Небезопасный паттерн, перехваченный утверждением:
var xs = @[1, 2, 3]
# for x in xs: xs.add(x)  # → ошибка утверждения: длина изменилась
```

---

### `items` для `string`

```nim
iterator items*(a: string): char {.inline.}
```

**Что это такое:** Возвращает каждый байт (символ) строки. Как и вариант для `seq`, запоминает длину и проверяет её стабильность. Обратите внимание: для многобайтовых UTF-8 последовательностей каждый `char` — это один *байт*, а не одна кодовая точка Unicode. Для итерации с учётом Unicode используйте `std/unicode.runes`.

```nim
for c in "hello":
  echo c   # h, e, l, l, o

var count = 0
for c in "Nim 🚀":
  if c != ' ': inc count
echo count   # считает байты, а не глифы
```

---

### `items` для дескриптора типа перечисления

```nim
iterator items*[T: enum and Ordinal](E: typedesc[T]): T
```

**Что это такое:** Итерирует по всем значениям типа перечисления. В отличие от `items` для множества, здесь используется сам *тип* (переданный как `typedesc`), а не экземпляр. Итерация охватывает весь непрерывный диапазон от `low(E)` до `high(E)`.

> **Важно:** Работает корректно только для перечислений *без дыр* (где все порядковые значения между `low` и `high` объявлены). Для перечислений с пропусками используйте `enumutils.items`.

```nim
type Goo = enum g0 = 2, g1, g2

from std/sequtils import toSeq
assert Goo.toSeq == [g0, g1, g2]

for v in Goo:
  echo v   # g0, g1, g2
```

---

### `items` для `Slice[T]`

```nim
iterator items*[T: Ordinal](s: Slice[T]): T
```

**Что это такое:** Итерирует по всем порядковым значениям внутри среза `s`, от `s.a` до `s.b` включительно. Это делает срезы напрямую пригодными для циклов `for` без необходимости явно прописывать синтаксис диапазона.

```nim
for i in (3 .. 7):   # items вызывается на Slice[int]
  echo i   # 3, 4, 5, 6, 7

for c in ('a' .. 'e'):
  echo c   # a, b, c, d, e
```

## mitems — изменяемая итерация

`mitems` возвращает каждый элемент как `var T` — изменяемую ссылку непосредственно в коллекцию. Изменение переменной цикла изменяет оригинальный элемент.

---

### `mitems` для `openArray[T]`

```nim
iterator mitems*[T](a: var openArray[T]): var T {.inline.}
```

**Что это такое:** Изменяемая итерация по элементам любой непрерывной коллекции. Контейнер должен быть объявлен как `var`. Это идиоматичный способ трансформировать каждый элемент массива или последовательности на месте без арифметики индексов.

```nim
var nums = [1, 2, 3, 4, 5]
for n in mitems(nums):
  n *= 2
echo nums   # [2, 4, 6, 8, 10]

var words = @["hello", "world"]
for w in mitems(words):
  w = w.toUpperAscii()
echo words   # @["HELLO", "WORLD"]
```

---

### `mitems` для `array[IX, T]`

```nim
iterator mitems*[IX, T](a: var array[IX, T]): var T {.inline.}
```

**Что это такое:** Изменяемая итерация по массиву фиксированного размера. Та же безопасная относительно переполнения структура цикла, что и в версии `items[IX,T]` только для чтения.

```nim
var data: array[3, float] = [1.0, 2.0, 3.0]
for x in mitems(data):
  x = x * x
echo data   # [1.0, 4.0, 9.0]
```

---

### `mitems` для `cstring`

```nim
iterator mitems*(a: var cstring): var char {.inline.}
```

**Что это такое:** Изменяемая посимвольная итерация по C-строке. Позволяет изменять отдельные символы на месте. На нативных бэкендах останавливается на первом `'\0'`. Рекомендуемый паттерн — использовать `prepareMutation` перед мутацией `cstring`, изначально являвшейся Nim-строкой, чтобы убедиться, что базовый буфер не является разделяемым.

```nim
from std/sugar import collect
var a = "abc\0def"
prepareMutation(a)
var b = a.cstring
let s = collect:
  for bi in mitems(b):
    if bi == 'b': bi = 'B'
    bi
assert s == @['a', 'B', 'c']
assert b == "aBc"
assert a == "aBc\0def"  # мутация распространилась на оригинальную строку
```

---

### `mitems` для `seq[T]`

```nim
iterator mitems*[T](a: var seq[T]): var T {.inline.}
```

**Что это такое:** Изменяемая итерация по элементам последовательности с тем же утверждением стабильности длины, что и в версии `items` только для чтения. Не добавляйте и не удаляйте элементы во время итерации — утверждение сработает.

```nim
var scores = @[85, 92, 78, 95]
for s in mitems(scores):
  if s < 80: s = 80   # установить нижнюю границу всех оценок на 80
echo scores   # @[85, 92, 80, 95]
```

---

### `mitems` для `string`

```nim
iterator mitems*(a: var string): var char {.inline.}
```

**Что это такое:** Изменяемая побайтовая итерация по строке. Позволяет изменять отдельные символы на месте. Стабильность длины проверяется.

```nim
var s = "hello world"
for c in mitems(s):
  if c == ' ': c = '_'
echo s   # "hello_world"
```

## pairs — индексированная итерация

`pairs` возвращает кортежи `(индекс, значение)`, давая одновременно позицию и элемент.

---

### `pairs` для `openArray[T]`

```nim
iterator pairs*[T](a: openArray[T]): tuple[key: int, val: T] {.inline.}
```

**Что это такое:** Возвращает `(индекс, элемент)` для каждого элемента коллекции. Индекс всегда является обычным `int`, независимо от типа индекса базовой коллекции.

```nim
let fruits = ["apple", "banana", "cherry"]
for i, f in fruits:
  echo i, ": ", f
# 0: apple
# 1: banana
# 2: cherry
```

---

### `pairs` для `array[IX, T]`

```nim
iterator pairs*[IX, T](a: array[IX, T]): tuple[key: IX, val: T] {.inline.}
```

**Что это такое:** Как `pairs` для `openArray`, но тип ключа — это реальный тип индекса массива `IX`, который может быть перечислением, символом или пользовательским целочисленным диапазоном. Это важно, когда массив индексируется чем-то, кроме обычного целого числа.

```nim
type Dir = enum North, South, East, West
let costs: array[Dir, int] = [10, 20, 15, 25]
for dir, cost in costs:
  echo dir, " стоит ", cost
# North стоит 10
# South стоит 20
# ...
```

---

### `pairs` для `seq[T]`

```nim
iterator pairs*[T](a: seq[T]): tuple[key: int, val: T] {.inline.}
```

**Что это такое:** Индексированная итерация по последовательности. Длина запоминается до начала цикла и проверяется на стабильность на каждом шаге.

```nim
let items = @["x", "y", "z"]
for idx, val in items:
  echo idx, " → ", val
```

---

### `pairs` для `string`

```nim
iterator pairs*(a: string): tuple[key: int, val: char] {.inline.}
```

**Что это такое:** Возвращает пары `(байтовый_индекс, char)` для каждого байта строки. Полезно, когда нужны одновременно позиция и символ — например, при построении строки замены с указанием мест изменений.

```nim
for i, c in "nim":
  echo i, ": ", c
# 0: n
# 1: i
# 2: m
```

---

### `pairs` для `cstring`

```nim
iterator pairs*(a: cstring): tuple[key: int, val: char] {.inline.}
```

**Что это такое:** Индексированная итерация по C-строке. На нативных бэкендах останавливается на `'\0'`. На JS использует вычисленную длину строки. Поведение со встроенными нулями аналогично `items` для `cstring`.

## mpairs — изменяемая индексированная итерация

`mpairs` сочетает доступ по индексу с изменяемостью: возвращает кортежи `(индекс, var элемент)`. Поле `val` возвращаемого кортежа является изменяемой ссылкой в коллекцию.

---

### `mpairs` для `openArray[T]`

```nim
iterator mpairs*[T](a: var openArray[T]): tuple[key: int, val: var T] {.inline.}
```

**Что это такое:** Индексированная изменяемая итерация по любой непрерывной коллекции. `key` даёт позицию; `val` — прямой доступ на запись к элементу. Полезно, когда логика мутации зависит от обоих: и от индекса, и от значения.

```nim
var data = @[10, 20, 30, 40]
for i, v in mpairs(data):
  if i mod 2 == 0: v = 0   # обнулить элементы с чётными индексами
echo data   # @[0, 20, 0, 40]
```

---

### `mpairs` для `array[IX, T]`

```nim
iterator mpairs*[IX, T](a: var array[IX, T]): tuple[key: IX, val: var T] {.inline.}
```

**Что это такое:** Индексированная изменяемая итерация по массиву фиксированного размера. Ключ сохраняет оригинальный тип индекса `IX`.

```nim
var arr: array[3, int] = [5, 10, 15]
for i, v in mpairs(arr):
  v += i
echo arr   # [5, 11, 17]
```

---

### `mpairs` для `seq[T]`

```nim
iterator mpairs*[T](a: var seq[T]): tuple[key: int, val: var T] {.inline.}
```

**Что это такое:** Индексированная изменяемая итерация с утверждением стабильности длины. Не изменяйте размер последовательности внутри цикла.

```nim
var words = @["apple", "banana", "cherry"]
for i, w in mpairs(words):
  w = $i & "_" & w
echo words   # @["0_apple", "1_banana", "2_cherry"]
```

---

### `mpairs` для `string`

```nim
iterator mpairs*(a: var string): tuple[key: int, val: var char] {.inline.}
```

**Что это такое:** Индексированная изменяемая побайтовая итерация по строке с утверждением стабильности длины.

```nim
var s = "abcde"
for i, c in mpairs(s):
  if i mod 2 == 0: c = '-'
echo s   # "-b-d-"
```

---

### `mpairs` для `cstring`

```nim
iterator mpairs*(a: var cstring): tuple[key: int, val: var char] {.inline.}
```

**Что это такое:** Индексированная изменяемая посимвольная итерация по C-строке. На нативных бэкендах останавливается на `'\0'`.

## fields — итерация по полям кортежа/объекта

### `fields` для одного кортежа или объекта

```nim
iterator fields*[T: tuple|object](x: T): RootObj {.magic: "Fields", noSideEffect.}
```

**Что это такое:** Итерирует по каждому полю кортежа или объекта, возвращая значение каждого поля. Переменная цикла имеет тип каждого поля по очереди — это *раскрутка во время компиляции*, а не цикл времени выполнения. Компилятор трансформирует оператор `for` в отдельный код для каждого поля, поэтому тело цикла может использовать фактический тип каждого поля.

> **Внимание:** Поскольку цикл раскручивается во время компиляции, нельзя использовать `if` для ветвления по типу — необходимо использовать `when` с оператором `is`. Также существует известная ошибка, влияющая на привязку символов в теле цикла; для сложных случаев предпочтите `fieldPairs`.

```nim
var t = (1, "foo")
for v in fields(t):
  v = default(typeof(v))
assert t == (0, "")

# Сброс всех полей объекта:
type Config = object
  debug: bool
  maxRetries: int
  host: string

var cfg = Config(debug: true, maxRetries: 3, host: "localhost")
for v in fields(cfg):
  v = default(typeof(v))
assert cfg == Config()
```

### `fields` для двух кортежей или объектов

```nim
iterator fields*[S: tuple|object, T: tuple|object](x: S, y: T): tuple[key: string, val: RootObj] {.magic: "Fields", noSideEffect.}
```

**Что это такое:** Одновременная итерация по соответствующим полям двух кортежей или объектов. На каждом шаге возвращается пара значений — одно из `x` и одно из `y` — позволяя выполнять попарные операции над структурно совместимыми типами.

```nim
var t1 = (1, "foo")
var t2 = default(typeof(t1))
for v1, v2 in fields(t1, t2):
  v2 = v1          # скопировать каждое поле из t1 в t2
assert t1 == t2
```

## fieldPairs — итерация по именованным полям

### `fieldPairs` для одного кортежа или объекта

```nim
iterator fieldPairs*[T: tuple|object](x: T): tuple[key: string, val: RootObj] {.magic: "FieldPairs", noSideEffect.}
```

**Что это такое:** Как `fields`, но дополнительно возвращает *имя* поля в виде строки вместе с его значением. `key` — имя поля как оно записано в исходном коде; `val` — значение поля. Как и `fields`, это раскрутка во время компиляции — используйте `when` вместо `if` для ветвления по типу.

Это стандартный инструмент для реализации обобщённой сериализации, красивой печати и интроспекции кортежей и объектов.

```nim
type Custom = object
  foo: string
  bar: bool

proc `$`(x: Custom): string =
  result = "Custom:"
  for name, value in x.fieldPairs:
    when value is bool:
      result.add("\n\t" & name & " is " & $value)
    else:
      result.add("\n\t" & name & " '" & value & "'")

echo Custom(foo: "hello", bar: true)
# Custom:
#   foo 'hello'
#   bar is true
```

### `fieldPairs` для двух кортежей или объектов

```nim
iterator fieldPairs*[S: tuple|object, T: tuple|object](x: S, y: T): tuple[key: string, a, b: RootObj] {.magic: "FieldPairs", noSideEffect.}
```

**Что это такое:** Попарная именованная итерация по полям двух структурно совместимых типов. На каждом шаге возвращается `(имя, значение_из_x, значение_из_y)`. Полезно для сравнения, слияния или избирательного копирования полей между двумя экземплярами.

```nim
type Foo = object
  x1: int
  x2: string

var a1 = Foo(x1: 12, x2: "abc")
var a2: Foo
for name, v1, v2 in fieldPairs(a1, a2):
  when name == "x2":
    v2 = v1   # скопировать только поле x2
assert a2 == Foo(x1: 0, x2: "abc")
```

## Краткий справочник

| Итератор | Типы коллекций | Возвращает | Мутация? |
|---|---|---|---|
| `items` | openArray, array, set, seq, string, cstring, typedesc, Slice | значение элемента | нет |
| `mitems` | openArray, array, seq, string, cstring | `var` элемент | **да** |
| `pairs` | openArray, array, seq, string, cstring | `(индекс, значение)` | нет |
| `mpairs` | openArray, array, seq, string, cstring | `(индекс, var значение)` | **да** |
| `fields` | tuple, object (×1 или ×2) | значение(я) поля | зависит |
| `fieldPairs` | tuple, object (×1 или ×2) | `(имя, значение)` или `(имя, v1, v2)` | зависит |
