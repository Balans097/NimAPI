# Справочник модуля `formatfloat`

> **Модуль:** `std/formatfloat` (стандартная библиотека Nim)  
> **Назначение:** Преобразование чисел с плавающей точкой (`float`, `float32`, `BiggestFloat`) в строковое представление.

Этот модуль предоставляет набор инструментов — от низкоуровневых процедур записи в буфер до удобных вспомогательных функций — все они решают одну задачу: как правильно и эффективно превратить число с плавающей точкой в человекочитаемый текст.

Модуль предлагает две базовые стратегии:

- **Roundtrip** (по умолчанию) — использует алгоритмы Dragonbox / Schubfach для получения *кратчайшей* десятичной строки, которая при обратном разборе даёт в точности тот же IEEE 754 бит-паттерн. Это современный, предпочтительный подход.
- **Sprintf** (устаревший, флаг `nimLegacySprintf`) — делегирует форматирование функции `snprintf` из стандартной библиотеки C с форматом `%.16g`, затем исправляет локаль-зависимые особенности (запятые вместо точек) и платформо-специфичные написания бесконечности и NaN.

## Экспортируемые функции

---

### `writeFloatToBufferRoundtrip` *(два варианта)*

```nim
proc writeFloatToBufferRoundtrip*(buf: var array[65, char]; value: BiggestFloat): int
proc writeFloatToBufferRoundtrip*(buf: var array[65, char]; value: float32): int
```

**Что делает**

Записывает число с плавающей точкой в фиксированный буфер `array[65, char]`, используя алгоритм Dragonbox (для `float64`/`BiggestFloat`) или Schubfach (для `float32`). В конец буфера всегда добавляется нуль-терминатор `'\0'`. Возвращает количество записанных символов *без учёта* нуль-терминатора.

Гарантия «roundtrip» означает: если распарсить полученную строку обратно любым стандартным парсером, вы получите *в точности тот же* float. Никакая точность не теряется, а вывод при этом настолько краток, насколько возможно.

**Когда использовать**

Используйте эту функцию, когда нужна низкоуровневая сериализация float без выделения памяти в куче — например, внутри собственного сериализатора, логгера или в любом месте, где heap-аллокации нежелательны.

**Параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `buf` | `var array[65, char]` | Выходной буфер. 65 символов всегда достаточно для любого float. Передаётся по ссылке и изменяется на месте. |
| `value` | `BiggestFloat` или `float32` | Число для преобразования. |

**Возвращает:** `int` — количество записанных символов (без нуль-терминатора).

**Пример**

```nim
import std/formatfloat

var buf: array[65, char]

# Вариант для float64
let n1 = writeFloatToBufferRoundtrip(buf, 3.14159265358979)
echo cast[cstring](buf[0].addr)  # "3.14159265358979"
echo n1                          # 16

# Вариант для float32
let n2 = writeFloatToBufferRoundtrip(buf, 1.5'f32)
echo cast[cstring](buf[0].addr)  # "1.5"
echo n2                          # 3
```

Свойство roundtrip в действии:

```nim
import std/strutils

var buf: array[65, char]
let original = 0.1 + 0.2              # 0.30000000000000004 по IEEE 754
discard writeFloatToBufferRoundtrip(buf, original)
let s = $cast[cstring](buf[0].addr)   # "0.30000000000000004"
assert parseFloat(s) == original      # true — точный roundtrip
```

### `writeFloatToBufferSprintf`

```nim
proc writeFloatToBufferSprintf*(buf: var array[65, char]; value: BiggestFloat): int
```

**Что делает**

Записывает `BiggestFloat` в буфер с помощью функции `snprintf` из стандартной библиотеки C, используя формат `"%.16g"` (до 16 значащих десятичных цифр). После форматирования выполняет несколько шагов нормализации:

1. Заменяет запятые (`','`), которые могут появиться из-за настроек локали, на точку (`'.'`) — результат всегда не зависит от локали.
2. Проверяет, есть ли в результате десятичная точка или маркер экспоненты; если нет — добавляет `.0`, чтобы строка всегда *выглядела* как вещественное число, а не целое.
3. Нормализует платформо-специфичные представления бесконечности и NaN: Windows может выдавать `1.#INF`, `-1.#IND`, `nan(ind)` и т. п. — всё это приводится к `inf`, `-inf` или `nan`.

**Когда использовать**

Как правило, эта функция вызывается не напрямую. Она является бэкендом, который используется при установке флага компиляции `nimLegacySprintf`. Существует для совместимости со старым кодом и окружениями, где алгоритмы Dragonbox/Schubfach недоступны.

**Параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `buf` | `var array[65, char]` | Выходной буфер, изменяется на месте. |
| `value` | `BiggestFloat` | Значение float64 для форматирования. |

**Возвращает:** `int` — количество записанных символов (без нуль-терминатора).

**Пример**

```nim
# Обычно функция используется косвенно через writeFloatToBuffer
# при компиляции с -d:nimLegacySprintf.
# Прямое использование:
var buf: array[65, char]
let n = writeFloatToBufferSprintf(buf, 42.0)
echo cast[cstring](buf[0].addr)  # "42.0"
echo n                           # 4

let n2 = writeFloatToBufferSprintf(buf, 1.23e10)
echo cast[cstring](buf[0].addr)  # "1.23e+10"  (зависит от платформы)

# Обработка специальных значений:
let n3 = writeFloatToBufferSprintf(buf, 1.0 / 0.0)
echo cast[cstring](buf[0].addr)  # "inf"

let n4 = writeFloatToBufferSprintf(buf, 0.0 / 0.0)
echo cast[cstring](buf[0].addr)  # "nan"
```

### `writeFloatToBuffer`

```nim
proc writeFloatToBuffer*(buf: var array[65, char]; value: BiggestFloat | float32): int {.inline.}
```

**Что делает**

Унифицированная точка входа для записи float в буфер с выбором бэкенда на этапе компиляции. В зависимости от того, собирается ли программа с флагом `-d:nimLegacySprintf`, вызывает либо `writeFloatToBufferSprintf` (устаревший вариант), либо `writeFloatToBufferRoundtrip` (по умолчанию). Принимает как `BiggestFloat`, так и `float32`.

Думайте об этой функции как о *главном* низкоуровневом инструменте записи float в буфер — она скрывает от вас выбор бэкенда. Используйте её вместо прямого вызова roundtrip- или sprintf-вариантов, если только вам специально не нужен один из них.

**Параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `buf` | `var array[65, char]` | Выходной буфер, изменяется на месте. |
| `value` | `BiggestFloat` или `float32` | Число для форматирования. |

**Возвращает:** `int` — количество записанных символов (без нуль-терминатора).

**Пример**

```nim
import std/formatfloat

var buf: array[65, char]

let n = writeFloatToBuffer(buf, 2.718281828)
echo cast[cstring](buf[0].addr)  # "2.718281828"

# Работает и с float32:
let n2 = writeFloatToBuffer(buf, 0.5'f32)
echo cast[cstring](buf[0].addr)  # "0.5"
```

### `addFloatRoundtrip`

```nim
proc addFloatRoundtrip*(result: var string; x: float | float32)
```

**Что делает**

Добавляет roundtrip-представление числа `x` к существующей строке. Это строковый аналог `writeFloatToBufferRoundtrip`. Внутри выделяется стековый буфер, число преобразуется, а символы копируются в строку — строка увеличивается ровно на нужное количество, без лишнего места.

Недоступна в контексте `nimvm` (виртуальная машина компиляции); вызов там вызовет ошибку assertion.

**Когда использовать**

Используйте, когда строка формируется постепенно и нужно вставить float с максимальной точностью и кратчайшим представлением. Это чуть более явный вариант, чем `addFloat`, — вы явно знаете, что получаете roundtrip-бэкенд независимо от флагов компиляции.

**Параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `result` | `var string` | Строка, к которой добавляется результат. |
| `x` | `float` или `float32` | Число для преобразования и добавления. |

**Пример**

```nim
import std/formatfloat

var s = "значение="
s.addFloatRoundtrip(0.1)
echo s  # "значение=0.1"

var s2 = "π≈"
s2.addFloatRoundtrip(3.141592653589793)
echo s2  # "π≈3.141592653589793"

# Построение CSV-строки из нескольких чисел:
var csv = ""
for v in [1.0, 2.5, 0.1 + 0.2]:
  if csv.len > 0: csv.add(',')
  csv.addFloatRoundtrip(v)
echo csv  # "1.0,2.5,0.30000000000000004"
```

### `addFloatSprintf`

```nim
proc addFloatSprintf*(result: var string; x: float)
```

**Что делает**

Добавляет к строке представление числа `x`, полученное через `snprintf`. Это строковый аналог `writeFloatToBufferSprintf`. Как и буферный вариант, нормализует локаль-зависимые особенности и платформо-специфичные написания специальных значений.

Недоступна в контексте `nimvm`.

**Когда использовать**

Прежде всего это функция совместимости с устаревшим кодом. Вызывайте её напрямую, если вам нужно именно поведение C-форматирования `%.16g` вне зависимости от флагов компиляции — например, чтобы вывод совпадал с более старой версией программы. В новом коде предпочтительнее `addFloat` или `addFloatRoundtrip`.

**Параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `result` | `var string` | Строка, к которой добавляется результат. |
| `x` | `float` | Число для преобразования и добавления. |

**Пример**

```nim
import std/formatfloat

var s = "результат: "
s.addFloatSprintf(12345.6789)
echo s  # "результат: 12345.6789"

var s2 = ""
s2.addFloatSprintf(1.0 / 0.0)
echo s2  # "inf"

var s3 = ""
s3.addFloatSprintf(0.0 / 0.0)
echo s3  # "nan"
```

### `addFloat`

```nim
proc addFloat*(result: var string; x: float | float32) {.inline.}
```

**Что делает**

Главная, высокоуровневая функция для добавления числа с плавающей точкой к строке. Именно к ней следует обращаться в повседневном коде. Диспетчеризация бэкенда происходит автоматически:

- **JS-цель:** использует нативную JS-функцию (`nimFloatToString`), которая гарантирует, что целые числа не будут напечатаны без десятичной точки, корректно обрабатывает `-0.0`, `NaN` и `Infinity`.
- **Нативная цель с `-d:nimLegacySprintf`:** вызывает `addFloatSprintf`.
- **Нативная цель (по умолчанию):** вызывает `addFloatRoundtrip`.

Функция всегда гарантирует, что вывод *выглядит как вещественное число* — она никогда не выдаст `"2"` для `2.0`, всегда будет `"2.0"`.

**Параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `result` | `var string` | Строка, к которой добавляется результат. |
| `x` | `float` или `float32` | Число для преобразования и добавления. |

**Пример**

```nim
import std/formatfloat

# Базовое использование:
var s = "foo:"
s.addFloat(45.67)
assert s == "foo:45.67"

# Целочисленные вещественные числа всегда получают ".0":
var s2 = ""
s2.addFloat(100.0)
assert s2 == "100.0"

# Специальные значения:
var s3 = ""
s3.addFloat(1.0 / 0.0)
assert s3 == "inf"

var s4 = ""
s4.addFloat(-1.0 / 0.0)
assert s4 == "-inf"

var s5 = ""
s5.addFloat(0.0 / 0.0)
assert s5 == "nan"

# float32:
var s6 = ""
s6.addFloat(3.14'f32)
echo s6  # "3.14"
```

### `$` *(для float и float32)*

```nim
func `$`*(x: float | float32): string
```

> **Доступность:** Только при компиляции с флагом `-d:nimPreviewSlimSystem`.

**Что делает**

Оператор `$` (универсальный оператор «преобразовать в строку» в Nim) для типов `float` и `float32`. Возвращает новую строку с текстовым представлением числа. Реализован как тонкая обёртка над `addFloat` — создаёт пустую строку и добавляет к ней число.

Когда `nimPreviewSlimSystem` не определён, этот оператор предоставляется модулем `system`. Версия здесь — современная, облегчённая замена.

**Параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `x` | `float` или `float32` | Число для преобразования. |

**Возвращает:** Новую `string` с текстовым представлением числа.

**Пример**

```nim
# Компиляция: nim c -d:nimPreviewSlimSystem yourfile.nim
import std/formatfloat

let s = $(3.14159)
echo s          # "3.14159"

let s2 = $(2.0)
echo s2         # "2.0"

# Часто используется при конкатенации строк:
let msg = "Вычисленное значение: " & $(1.0 / 3.0)
echo msg        # "Вычисленное значение: 0.3333333333333333"
```

## Архитектурные замечания

**Почему буфер 65 символов?** Формат IEEE 754 двойной точности даёт не более 17 значащих десятичных цифр. С учётом знака, десятичной точки, буквы `e`, знака показателя и до 3 цифр показателя степени максимальная длина представления не превышает 64 символов. Буфер имеет размер 65, чтобы всегда оставалось место для нуль-терминатора.

**Roundtrip vs. sprintf — что «точнее»?** Ни один из вариантов не теряет информацию — оба представляют float точно. Разница — в форме записи: `sprintf` с `%.16g` может выводить больше цифр, чем необходимо (например, `0.10000000000000001` вместо `0.1`), тогда как roundtrip-алгоритм всегда находит *кратчайшую* строку, которая отображается в тот же бит-паттерн.

**Сводная таблица специальных значений:**

| Значение | Вывод |
|----------|-------|
| `1.0 / 0.0` | `"inf"` |
| `-1.0 / 0.0` | `"-inf"` |
| `0.0 / 0.0` | `"nan"` |
| `-0.0` | `"-0.0"` |
| `2.0` | `"2.0"` (никогда `"2"`) |

## Быстрая шпаргалка

| Задача | Функция |
|--------|---------|
| Добавить float к строке (повседневный код) | `addFloat` |
| Добавить с гарантией roundtrip | `addFloatRoundtrip` |
| Добавить через C sprintf (легаси) | `addFloatSprintf` |
| Получить новую строку из float (`$`) | `$` (при `nimPreviewSlimSystem`) |
| Записать в буфер без аллокации (любой бэкенд) | `writeFloatToBuffer` |
| Записать в буфер, явно roundtrip | `writeFloatToBufferRoundtrip` |
| Записать в буфер, явно sprintf | `writeFloatToBufferSprintf` |
