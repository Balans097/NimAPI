# Справочник модуля `base64`

**Модуль:** `base64` (часть стандартной библиотеки Nim)  
**Статус:** ⚠️ Нестабильный API — интерфейсы могут измениться в следующих версиях Nim.  
**Назначение:** Кодирование двоичных данных в строку Base64 и обратное декодирование с поддержкой стандартного, URL-безопасного и MIME (email) вариантов.

---

## Обзор

Base64 — схема преобразования двоичных данных в текст. Каждые три входных байта (24 бита) разбиваются на четыре 6-битных фрагмента, каждый из которых отображается в один из 64 печатных ASCII-символов. Результат примерно на треть длиннее оригинала, но состоит исключительно из безопасных печатных символов, которые могут передаваться через текстовые каналы: электронную почту, JSON, URL и HTML-атрибуты.

### Два алфавита

Модуль поддерживает два набора символов, выбираемых флагом `safe`:

| Позиция | Стандартный (`safe=false`) | URL-безопасный (`safe=true`) |
|---|---|---|
| 62 | `+` | `-` |
| 63 | `/` | `_` |
| Дополнение | `=` | `=` |

URL-безопасный алфавит (RFC 4648 §5) избегает символов `+` и `/`, которые имеют особое значение в URL и путях файловой системы. В остальном оба алфавита идентичны.

### Дополнение (padding)

Если длина входных данных не кратна 3, в конец дописываются один или два символа `=`, чтобы длина выходной строки стала кратной 4:

| Длина входа mod 3 | Символов дополнения `=` |
|---|---|
| 0 | нет |
| 1 | `==` |
| 2 | `=` |

## Экспортируемый API

| Символ | Вид | Описание |
|---|---|---|
| `encode[T](s, safe)` | proc | Кодировать `openArray[byte\|char]` в строку Base64 |
| `encodeMime(s, lineLen, newLine, safe)` | proc | Кодировать строку с переносами строк для MIME/email |
| `initDecodeTable()` | proc | Построить и вернуть таблицу декодирования из 256 элементов |
| `decode(s)` | proc | Декодировать строку Base64 обратно в исходные байты |

> Вторая перегрузка `encode`, принимающая `SomeInteger and not byte`, **устарела** и в этом справочнике не рассматривается.

---

## `encode`

```nim
proc encode*[T: byte|char](s: openArray[T], safe = false): string
```

### Что делает

Преобразует любую последовательность значений `byte` или `char` в строку, закодированную в Base64. Входные данные могут быть строкой `string`, `seq`, массивом-литералом или любым другим `openArray`.

Параметр `safe` выбирает алфавит:
- `false` (по умолчанию) — стандартный алфавит RFC 4648 §4 с символами `+` и `/`.
- `true` — URL/файловой системы-безопасный алфавит RFC 4648 §5 с `-` и `_` вместо `+` и `/`.

Возвращаемая строка всегда чисто ASCII. Её длина приблизительно равна `⌈len/3⌉ × 4`.

### Когда использовать

Используйте `encode` для любых данных, которым нужно пережить текстовый транспорт: встраивание изображений в HTML/CSS (URI `data:`), отправка двоичных нагрузок в JSON, хранение двоичных блобов в базах данных, ожидающих строки, генерация заголовков HTTP Basic Auth и т.д. Используйте `safe = true`, когда закодированная строка будет находиться в параметре URL-запроса или имени файла.

### Примеры

```nim
import base64

# Обычная строка
assert encode("Hello World") == "SGVsbG8gV29ybGQ="

# Длина входа кратна 3 — дополнение не нужно
assert encode("nim") == "bmlt"

# Длина входа mod 3 == 1 — два символа дополнения '=='
assert encode("a") == "YQ=="

# Длина входа mod 3 == 2 — один символ дополнения '='
assert encode("ab") == "YWI="

# Массив байт (например, сырые двоичные данные)
assert encode([1'u8, 2, 3, 4, 5]) == "AQIDBAU="

# Массив символов
assert encode(['h', 'e', 'y']) == "aGV5"

# URL-безопасный алфавит: '/' становится '_', '+' становится '-'
assert encode("c\xf7>", safe = false) == "Y/c+"
assert encode("c\xf7>", safe = true)  == "Y_c-"

# Практика: встроить небольшой двоичный файл как фрагмент data URI
let png = [0x89'u8, 0x50, 0x4E, 0x47]  # магические байты PNG
echo "data:image/png;base64," & encode(png)
```

## `encodeMime`

```nim
proc encodeMime*(s: string, lineLen = 75.Positive, newLine = "\r\n",
                 safe = false): string
```

### Что делает

Кодирует строку в Base64, а затем разбивает результат на строки фиксированной длины, разделённые настраиваемой последовательностью переноса. Это соответствует формату MIME для электронной почты, описанному в RFC 2045, который требует, чтобы содержимое Base64 было разбито на строки не длиннее 76 символов.

Внутри процедура сначала вызывает `encode`, получая полную строку Base64, затем нарезает её на сегменты по `lineLen` символов и объединяет их через `newLine`.

**Особые случаи:**
- Если `s` пуста — немедленно возвращается пустая строка.
- Если закодированный результат уже `≤ lineLen` символов — возвращается как есть, без переносов.
- Если `newLine` — пустая строка `""` — результат возвращается без разбивки (эквивалентно вызову `encode`).

**`lineLen`** — максимальное количество символов Base64 в одной строке. Должен быть типа `Positive` (≥ 1). По умолчанию 75, что вместе с двухсимвольным переносом `\r\n` даёт строки по 77 байт — чуть выше лимита MIME в 76 символов для самого содержимого Base64.

**`newLine`** — строка-разделитель строк. По умолчанию `"\r\n"` (CRLF), как требуют стандарты электронной почты. Используйте `"\n"` для Unix-стиля или в контекстах, где строгое соответствие MIME не нужно.

**`safe`** — то же значение, что и в `encode`.

### Когда использовать

Используйте `encodeMime` при составлении тела письма или вложения MIME. Многие почтовые серверы отклоняют сообщения со строками Base64 длиннее 76 символов. Для всех остальных случаев (API, data URI, токены) предпочитайте обычный `encode`.

### Примеры

```nim
import base64

# Стандартное MIME-кодирование — CRLF, 75 символов на строку
let mime = encodeMime("Hello, this is a test of MIME base64 encoding for email!")
# Каждая строка содержит не более 75 символов Base64, затем \r\n

# Пользовательская длина строки и Unix-перенос — удобно для читаемого вывода
assert encodeMime("Hello World", 4, "\n") == "SGVs\nbG8g\nV29y\nbGQ="

# Пустой ввод возвращает пустую строку
assert encodeMime("") == ""

# Результат короче lineLen — перенос не нужен
assert encodeMime("Hi", 100) == "SGk="

# Полностью отключить разбивку, передав пустую строку переноса
let noWrap = encodeMime("Hello World", 4, "")
assert noWrap == encode("Hello World")  # идентично обычному encode

# URL-безопасный вариант для MIME
let safeMime = encodeMime("c\xf7>", 4, "\n", safe = true)
echo safeMime  # Y_c- (достаточно коротко для одной строки)
```

## `initDecodeTable`

```nim
proc initDecodeTable*(): array[256, char]
```

### Что делает

Строит и возвращает таблицу поиска из 256 элементов, отображающую каждое возможное байтовое значение ASCII в его 6-битное значение Base64. Эта таблица используется внутри `decode` для преобразования каждого символа Base64 в числовой эквивалент за время O(1).

Таблица вычисляется **во время компиляции** (результат хранится в константе `decodeTable`) и одновременно обрабатывает оба алфавита:

| Диапазон символов | Отображаемое значение |
|---|---|
| `A`–`Z` | 0–25 |
| `a`–`z` | 26–51 |
| `0`–`9` | 52–61 |
| `+` или `-` | 62 |
| `/` или `_` | 63 |
| Любой другой байт | 255 (= `invalidChar`, маркер недопустимого символа) |

Обратите внимание: и `+`, и `-` отображаются в 62, и `/`, и `_` — в 63. Это означает, что `decode` принимает входные данные, закодированные любым из двух алфавитов, не требуя никакого флага.

### Когда использовать

Вы редко будете вызывать эту процедуру напрямую — она существует, чтобы константа `decodeTable` могла инициализироваться во время компиляции. Однако она экспортирована, поэтому вы можете вызвать её, если нужна собственная копия таблицы: например, для построения пользовательского декодера с иной обработкой ошибок или для проверки, какие символы считаются допустимыми.

### Пример

```nim
import base64

let table = initDecodeTable()

# Проверяем, что 'A' отображается в 0, а 'a' — в 26
echo int(table[ord('A')])   # 0
echo int(table[ord('Z')])   # 25
echo int(table[ord('a')])   # 26
echo int(table[ord('z')])   # 51
echo int(table[ord('0')])   # 52
echo int(table[ord('9')])   # 61
echo int(table[ord('+')])   # 62
echo int(table[ord('-')])   # 62  (то же, что '+')
echo int(table[ord('/')])   # 63
echo int(table[ord('_')])   # 63  (то же, что '/')
echo uint8(table[ord('!')]) # 255 — маркер недопустимого символа
```

## `decode`

```nim
proc decode*(s: string): string
```

### Что делает

Преобразует строку, закодированную в Base64, обратно в её исходное двоичное представление. Возвращает декодированные байты как строку Nim (`string` в Nim — байтовый буфер и может содержать произвольные двоичные данные).

#### Обработка пробельных символов

Декодер молча пропускает ведущие пробелы в начале строки, а также встречающиеся внутри данных символы `\n`, `\r` и пробел. Это означает, что он может напрямую принимать многострочный вывод `encodeMime`, не требуя предварительного удаления переносов.

Завершающие символы `=`, `\n`, `\r` и пробел удаляются перед началом обработки.

#### Независимость от алфавита

Благодаря дизайну таблицы декодирования (см. `initDecodeTable`), `decode` принимает входные данные, закодированные как стандартным алфавитом (`+`, `/`), так и URL-безопасным (`-`, `_`), или даже смесью обоих — без каких-либо флагов или параметров.

#### Обработка ошибок

Если встречается любой символ, который не является допустимым символом Base64, пробельным символом или `=`, выбрасывается исключение `ValueError` с подробным сообщением, включающим нарушающий символ, его порядковый номер и позицию во входной строке:

```
ValueError: Invalid base64 format character `!` (ord 33) at location 4.
```

### Когда использовать

Используйте `decode` всякий раз, когда получаете данные, закодированные в Base64, и вам нужны исходные байты: чтение вложений MIME в электронной почте, обработка ответов API, встраивающих двоичные данные как Base64, разбор URI `data:` и т.д.

### Примеры

```nim
import base64

# Базовый цикл туда-обратно
assert decode(encode("Hello World")) == "Hello World"

# Варианты дополнения
assert decode("YQ==") == "a"    # 1 исходный байт
assert decode("YWI=") == "ab"   # 2 исходных байта
assert decode("YWJj") == "abc"  # 3 исходных байта, дополнение не нужно

# Ведущие и встроенные пробельные символы молча игнорируются
assert decode("  SGVsbG8gV29ybGQ=") == "Hello World"
assert decode("SGVs\r\nbG8g\r\nV29y\r\nbGQ=") == "Hello World"  # строки MIME

# URL-безопасно закодированный ввод декодируется без флагов
assert decode("Y_c-") == "c\xf7>"
assert decode("Y/c+") == "c\xf7>"   # стандартный алфавит — тот же результат

# Декодирование сырых двоичных данных
let original = [0x00'u8, 0xFF, 0x10, 0xAB]
let encoded  = encode(original)
let decoded  = decode(encoded)
assert decoded.len == 4
assert byte(decoded[0]) == 0x00
assert byte(decoded[1]) == 0xFF

# Ошибка на недопустимом символе
import std/unittest
expect ValueError:
  discard decode("SGVs!!GQ=")
```

## Полный пример «туда и обратно»

```nim
import base64, strutils

# --- Двоичные данные (имитация небольшого файла) ---
let binaryData = "PK\x03\x04\x14\x00\x00\x00"  # магические байты ZIP

# Кодируем для встраивания в поле JSON
let encoded = encode(binaryData)
echo "Закодировано: ", encoded
echo "Длина: ", encoded.len, " символов для ", binaryData.len, " байт"

# Декодируем обратно для проверки целостности
let decoded = decode(encoded)
assert decoded == binaryData
echo "Цикл туда-обратно OK"

# --- Вложение электронной почты ---
let attachment = "Уважаемый пользователь,\n\nОтчёт во вложении.\n\nС уважением"
let mime = encodeMime(attachment, lineLen = 76, newLine = "\r\n")
echo "\nMIME-закодированное вложение (первые 80 символов):"
echo mime[0 .. min(79, mime.high)]

# Декодируем вывод MIME напрямую (переносы строк удаляются автоматически)
let recovered = decode(mime)
assert recovered == attachment
echo "MIME-цикл туда-обратно OK"

# --- URL-безопасный токен (например, токен сброса пароля) ---
let tokenBytes = "\xde\xad\xbe\xef\xca\xfe\xba\xbe"
let token = encode(tokenBytes, safe = true)
echo "\nURL-безопасный токен: ", token  # без символов +//
assert decode(token) == tokenBytes      # decode обрабатывает оба алфавита
```

## Типичные ошибки

**Двоичные данные в декодированных строках.** Строка `string` в Nim — байтовый буфер, необязательно валидный UTF-8. Передача декодированной двоичной строки в процедуры, ожидающие текст UTF-8, может дать искажённый вывод или исключение. Преобразуйте в `seq[byte]`, если нужно работать с результатом как с сырыми байтами.

**Длина строки `encodeMime` и лимит MIME.** RFC 2045 требует строк не длиннее 76 символов *включая* CRLF. При `lineLen = 75` по умолчанию и `newLine = "\r\n"` (2 символа) каждая передаваемая строка составляет 77 байт, что укладывается в лимит в 76 символов для самого содержимого Base64. Если ваша почтовая библиотека считает CRLF в счёт 76 символов, используйте `lineLen = 74`.

**URL-безопасное кодирование и дополнение `=`.** Некоторые URL-контексты также требуют удаления завершающего `=`, поскольку `=` используется как разделитель строки запроса. `encode` и `encodeMime` не удаляют дополнение — это нужно делать вручную при необходимости.

**Устаревшая перегрузка для целых чисел.** Вызов `encode` с массивом типа `int`, `int32` или других целочисленных типов (не `byte` и не `char`) вызывает предупреждение об устаревании. Сначала преобразуйте в `seq[byte]` или `array[N, byte]`.

**Это нестабильный API.** Заголовок модуля предупреждает, что интерфейсы могут измениться. Зафиксируйте версию Nim, если полагаетесь на точное поведение.
