# arithmetics.nim — полный справочник

> **Модуль**: `system/arithmetics` — стандартная библиотека Nim  
> **Назначение**: Определяет все встроенные арифметические операторы и несколько вспомогательных функций для порядковых типов для каждого числового типа Nim. Всё здесь реализовано как *магия* компилятора — Nim напрямую транслирует эти операции в инструкции процессора или операторы C, не вызывая никакого реального тела функции.  
> **Доступность**: автоматически в каждой Nim-программе через неявный импорт `system`.

---

## Как читать этот файл

Большинство записей несут прагму `{.magic: "XYZ", noSideEffect.}`. Это означает:

- Тело процедуры (если есть) предназначено только для документации и выполняемых примеров — компилятор заменяет каждый вызов соответствующей низкоуровневой инструкцией.
- `noSideEffect` гарантирует компилятору отсутствие наблюдаемых побочных эффектов, кроме возвращаемого значения, что позволяет использовать операторы в `func`, константных выражениях и строгом режиме.

Операторы определены для каждого конкретного числового типа (`int`, `int8`, `int16`, `int32`, `int64`, `uint`, `uint8`, …, `float32`, `float64`). Этот справочник группирует перегрузки и объясняет семантику один раз, отмечая специфику для конкретных типов там, где она существует.

## 1. Вспомогательные функции для порядковых типов

Эти четыре процедуры работают с любым **порядковым типом** — целыми числами, символами, булевыми значениями и перечислениями. «Порядковый» означает, что тип имеет чётко определённую, конечную, упорядоченную последовательность значений.

---

### `succ[T: Ordinal, V: SomeInteger](x: T, y: V = 1): T`

Возвращает `y`-й **последователь** `x` — значение, которое находится на `y` шагов *после* `x` в порядковой последовательности. Шаг по умолчанию равен 1. Вызывает `OverflowDefect` во время выполнения (или ошибку времени компиляции для констант), если результат выходит за диапазон типа.

**Когда использовать**: везде, где нужно продвинуть порядковое значение на произвольный шаг с явной обработкой переполнения. Для простого `+1` у обычного целого числа идиоматичнее `inc`.

```nim
assert succ(5) == 6          # один шаг вперёд
assert succ(5, 3) == 8       # три шага вперёд
assert succ('a') == 'b'      # работает и с char
# succ(true) — OverflowDefect: у bool нет последователя после true
```

---

### `pred[T: Ordinal, V: SomeInteger](x: T, y: V = 1): T`

Возвращает `y`-й **предшественник** `x` — значение на `y` шагов *до* `x`. Вызывает `OverflowDefect` при выходе за нижнюю границу диапазона.

```nim
assert pred(5) == 4          # один шаг назад
assert pred(5, 3) == 2       # три шага назад
assert pred('z') == 'y'
```

---

### `inc[T: Ordinal, V: SomeInteger](x: var T, y: V = 1)`

**Изменяет** `x` на месте, продвигая его на `y` шагов вперёд. Эквивалентно `x = succ(x, y)`. При переполнении вызывает `OverflowDefect`.

```nim
var i = 2
inc(i)        # i == 3
inc(i, 3)     # i == 6
```

`inc` и `dec` — идиоматический способ Nim продвигать счётчики циклов и порядковые переменные состояния. Они точнее передают намерение, чем `i += 1`, особенно для не-числовых порядковых типов.

---

### `dec[T: Ordinal, V: SomeInteger](x: var T, y: V = 1)`

**Изменяет** `x` на месте, отступая на `y` шагов назад. Эквивалентно `x = pred(x, y)`. При выходе за нижнюю границу вызывает `OverflowDefect`.

```nim
var i = 2
dec(i)        # i == 1
dec(i, 3)     # i == -2
```

## 2. Арифметика знаковых целых

Все операторы в этом разделе работают с `int`, `int8`, `int16`, `int32` и `int64`. При включённой проверке переполнения (по умолчанию в отладочных сборках) выход за диапазон типа вызывает `OverflowDefect`.

---

### Унарный `+x`

```nim
proc `+`*(x: int): int
```

Возвращает `x` без изменений. Унарный плюс — операция без эффекта, предоставляемая для симметрии с унарным минусом и для синтаксических контекстов, требующих выражения.

```nim
let x = +42    # x == 42
```

---

### Унарный `-x`

```nim
proc `-`*(x: int): int
```

Возвращает **арифметическое отрицание** `x`. Для знаковых целых: `-x == 0 - x`. Обратите внимание: отрицание `low(int)` (самого отрицательного значения) переполняется, так как его положительный аналог не помещается в тип.

```nim
assert -7 == -7
assert -(-7) == 7
# -low(int8) вызовет OverflowDefect в режиме с проверками
```

---

### `x + y` (бинарный)

```nim
proc `+`*(x, y: int): int
```

Стандартное **сложение**. Результат имеет тот же тип, что и операнды. Переполнение проверяется в отладочных сборках.

```nim
assert 3 + 4 == 7
# 127'i8 + 1'i8 вызовет OverflowDefect в режиме с проверками
```

---

### `x - y` (бинарный)

```nim
proc `-`*(x, y: int): int
```

Стандартное **вычитание**. Переполнение проверяется.

```nim
assert 10 - 3 == 7
# -128'i8 - 1'i8 вызовет OverflowDefect в режиме с проверками
```

---

### `x * y`

```nim
proc `*`*(x, y: int): int
```

Стандартное **умножение**. Переполнение проверяется.

```nim
assert 6 * 7 == 42
```

## 3. Целочисленное деление и остаток

Эти операторы реализуют семантику **усечения к нулю** — такую же, как `/` и `%` в C. Знак результата определяется знаками операндов по конкретным правилам; см. таблицы ниже.

---

### `x div y`

```nim
proc `div`*(x, y: int): int
```

Вычисляет **целое частное**, усекая к нулю. Приближённо равно `trunc(x / y)` при вещественном делении.

Деление на ноль вызывает `DivByZeroDefect`.

| x | y | x div y | Пояснение |
|---|---|---------|-----------|
| 7 | 3 | 2 | 7 / 3 = 2,33… → усечение до 2 |
| -7 | 3 | -2 | -7 / 3 = -2,33… → усечение до -2 |
| 7 | -3 | -2 | 7 / -3 = -2,33… → усечение до -2 |
| -7 | -3 | 2 | -7 / -3 = 2,33… → усечение до 2 |

```nim
assert (7 div 3) == 2
assert (-7 div 3) == -2
assert (7 div -3) == -2
assert (-7 div -3) == 2
```

---

### `x mod y`

```nim
proc `mod`*(x, y: int): int
```

Вычисляет **остаток** после `div`. Определяется как `x - (x div y) * y`, поэтому знак результата всегда совпадает со знаком `x` (делимого), независимо от знака `y`.

Деление на ноль вызывает `DivByZeroDefect`.

| x | y | x mod y | Проверка: x - (x div y)*y |
|---|---|---------|--------------------------|
| 7 | 5 | 2 | 7 - 1*5 = 2 |
| -7 | 5 | -2 | -7 div 5 = -1; -7 - (-1)*5 = -2 |
| 7 | -5 | 2 | 7 div -5 = -1; 7 - (-1)*(-5) = 2 |
| -7 | -5 | -2 | -7 div -5 = 1; -7 - 1*(-5) = -2 |

```nim
assert (7 mod 5) == 2
assert (-7 mod 5) == -2
assert (7 mod -5) == 2
assert (-7 mod -5) == -2
```

> **Важно**: это *не* математическое деление по модулю (которое всегда неотрицательно). Для неотрицательного результата используйте `floorMod` из `std/math`.

## 4. Побитовые операции (знаковые)

Эти операторы работают с двоичным представлением значения. Для знаковых целых используется представление в дополнительном коде (two's complement).

---

### `not x`

```nim
proc `not`*(x: int): int
```

**Побитовое дополнение** — инвертирует каждый бит. Эквивалентно `-x - 1` в дополнительном коде.

```nim
assert not 0'u8 == 255         # все биты установлены
assert not 0'i8 == -1          # дополнительный код: все биты = -1
assert not 1000'u16 == 64535
assert not 1000'i16 == -1001   # -(1000+1)
```

---

### `x and y`

```nim
proc `and`*(x, y: int): int
```

**Побитовое И** — результирующий бит равен 1 только если оба соответствующих входных бита равны 1. Часто используется для **маскирования** битов.

```nim
assert (0b0011 and 0b0101) == 0b0001   # только бит 0 установлен в обоих
assert (0xFF and 0x0F) == 0x0F         # оставить только младший нибл
```

---

### `x or y`

```nim
proc `or`*(x, y: int): int
```

**Побитовое ИЛИ** — результирующий бит равен 1, если *хотя бы один* соответствующий входной бит равен 1. Часто используется для **установки** конкретных битов.

```nim
assert (0b0011 or 0b0101) == 0b0111
# (flags or 0b0100) — установить бит 2 в flags
```

---

### `x xor y`

```nim
proc `xor`*(x, y: int): int
```

**Побитовое исключающее ИЛИ** — результирующий бит равен 1, если соответствующие входные биты *различаются*. Часто используется для **переключения** битов или проверки различий.

```nim
assert (0b0011 xor 0b0101) == 0b0110
assert (x xor x) == 0    # XOR с самим собой всегда даёт ноль
```

## 5. Битовые сдвиги (знаковые)

### `x shr y`

```nim
proc `shr`*(x: int, y: SomeInteger): int
```

**Арифметический сдвиг вправо** (с расширением знакового бита). Сдвигает биты `x` вправо на `y` позиций. Освобождающиеся позиции слева заполняются **знаковым битом** — поэтому отрицательные числа остаются отрицательными, а положительные — положительными.

`y` берётся по модулю `sizeof(x) * 8` (например, для `int32` сдвиг на 35 эквивалентен сдвигу на 3).

> **Важно**: приоритет операторов в Nim отличается от C. В Nim `a + b shr c` разбирается как `a + (b shr c)`, потому что `shr` имеет *меньший* приоритет, чем `+`.

```nim
assert 16 shr 2 == 4           # 0b10000 → 0b00100
assert -16 shr 2 == -4         # знаковый бит сохраняется
assert 0b1000_0000'i8 shr 4 == 0b1111_1000'i8  # расширение единицами
assert -1 shr 5 == -1          # сдвиг -1 вправо всегда даёт -1
```

Если определён `nimOldShiftRight`, `shr` для знаковых целых выполняет **логический** (заполняющий нулями) сдвиг и помечен как устаревший.

Для беззнаковых целых `shr` всегда выполняет **логический** (заполняющий нулями) сдвиг.

---

### `ashr(x, y)` — явный арифметический сдвиг вправо

```nim
proc ashr*(x: int, y: SomeInteger): int
```

По поведению идентичен `shr` для знаковых целых (арифметический, с расширением знакового бита). Предоставляется как **именованная функция** (не оператор) для случаев, когда синтаксис оператора неоднозначен или когда нужно явно выразить намерение в коде.

```nim
assert ashr(0b0001_0000'i8, 2) == 0b0000_0100'i8
assert ashr(0b1000_0000'i8, 1) == 0b1100_0000'i8  # знаковый бит копируется
assert ashr(0b1000_0000'i8, 8) == 0b1000_0000'i8  # сдвиг ≥ ширине: все знаковые биты
```

---

### `x shl y`

```nim
proc `shl`*(x: int, y: SomeInteger): int
```

**Сдвиг влево**. Сдвигает биты влево на `y` позиций; освобождающиеся позиции справа заполняются нулями. Эквивалентен умножению на 2^y (без проверки переполнения). `y` берётся по модулю `sizeof(x) * 8`.

```nim
assert 1'i32 shl 4 == 16          # 0b1 → 0b10000
assert 1'i64 shl 4 == 16
assert 0b0000_0001'i8 shl 7 == 0b1000_0000'i8   # устанавливает знаковый бит
```

## 6. Арифметика беззнаковых целых

Все операторы в этом разделе работают с `uint`, `uint8`, `uint16`, `uint32`, `uint64`. Беззнаковая арифметика **никогда не вызывает ошибок переполнения** — она оборачивается по модулю 2^N, где N — ширина типа в битах.

---

### `x + y`, `x - y`, `x * y` (беззнаковые)

```nim
proc `+`*(x, y: uint): uint
proc `-`*(x, y: uint): uint
proc `*`*(x, y: uint): uint
```

Стандартные беззнаковые сложение, вычитание и умножение. Всё тихо оборачивается при переполнении/недотечении.

```nim
assert 255'u8 + 1'u8 == 0'u8    # оборачивание
assert 0'u8 - 1'u8 == 255'u8    # недотечение оборачивается к максимуму
assert 200'u8 * 2'u8 == 144'u8  # 400 mod 256 = 144
```

---

### `x div y` (беззнаковое)

```nim
proc `div`*(x, y: uint): uint
```

Беззнаковое целочисленное деление — всегда усекает к нулю (что для неотрицательных чисел совпадает с усечением к минус бесконечности). Нет сложностей со знаком. При делении на ноль — `DivByZeroDefect`.

```nim
assert 7'u div 3'u == 2'u
```

---

### `x mod y` (беззнаковое)

```nim
proc `mod`*(x, y: uint): uint
```

Беззнаковый остаток. Результат всегда в `[0, y)`. При нулевом делителе — `DivByZeroDefect`.

```nim
assert 7'u mod 5'u == 2'u
```

## 7. Побитовые операции и сдвиги (беззнаковые)

### `not x` (беззнаковое)

```nim
proc `not`*(x: uint): uint
```

Побитовое дополнение. Для беззнаковых типов результат — просто `(2^N - 1) xor x`, что всегда неотрицательно и помещается в тип.

```nim
assert not 0'u8 == 255'u8
assert not 255'u8 == 0'u8
```

---

### `x and y`, `x or y`, `x xor y` (беззнаковые)

Та же побитовая семантика, что и для знаковых типов. Работают с сырым двоичным представлением.

```nim
assert (0xFF'u8 and 0x0F'u8) == 0x0F'u8
assert (0xF0'u8 or 0x0F'u8) == 0xFF'u8
assert (0xFF'u8 xor 0xFF'u8) == 0x00'u8
```

---

### `x shr y` (беззнаковый)

```nim
proc `shr`*(x: uint, y: SomeInteger): uint
```

**Логический сдвиг вправо** (с заполнением нулями). Освобождающиеся позиции всегда заполняются нулями, независимо от значения старшего бита. Это естественное поведение для беззнаковых типов.

```nim
assert 0xFF'u8 shr 4 == 0x0F'u8   # старший нибл исчезает, заполняется нулями
```

---

### `x shl y` (беззнаковый)

```nim
proc `shl`*(x: uint, y: SomeInteger): uint
```

**Сдвиг влево** с заполнением нулями справа. Эквивалентен умножению на 2^y с оборачиванием при переполнении.

```nim
assert 1'u8 shl 7 == 128'u8
assert 3'u8 shl 6 == 192'u8
```

## 8. Операторы составного присваивания

Эти операторы изменяют левый операнд на месте. Они — тонкие обёртки: `x += y` раскрывается в тот же код, что и `x = x + y`.

---

### `x += y` — сложение с присваиванием

```nim
proc `+=`*[T: SomeInteger](x: var T, y: T)
```

Увеличивает `x` на `y`. Работает со всеми целыми типами (знаковыми и беззнаковыми). Для типов с плавающей точкой есть отдельная перегрузка (см. §9).

```nim
var n = 10
n += 5    # n == 15
```

---

### `x -= y` — вычитание с присваиванием

```nim
proc `-=`*[T: SomeInteger](x: var T, y: T)
```

Уменьшает `x` на `y`.

```nim
var n = 10
n -= 3    # n == 7
```

---

### `x *= y` — умножение с присваиванием

```nim
proc `*=`*[T: SomeInteger](x: var T, y: T)
```

Умножает `x` на `y` на месте. Реализовано как `x = x * y`.

```nim
var n = 6
n *= 7    # n == 42
```

## 9. Арифметика с плавающей точкой

Все операторы ниже работают как с `float` (= `float64`, 64-битная двойная точность IEEE 754), так и с `float32` (32-битная одинарная точность). Применяются правила IEEE 754: операции с `NaN` дают `NaN`; переполнение даёт `Inf` или `-Inf`; деление на ноль тоже даёт `Inf`/`-Inf`, а не исключение.

---

### Унарный `+x`, `-x` (float)

```nim
proc `+`*(x: float): float
proc `-`*(x: float): float
```

Унарный плюс (без эффекта) и отрицание. Отрицание инвертирует знаковый бит — корректно обрабатывает `+0.0`, `-0.0`, `Inf`, `-Inf` и `NaN` по IEEE 754.

```nim
assert -3.14 == -3.14
assert -(-3.14) == 3.14
assert -(Inf) == -Inf
```

---

### `x + y`, `x - y`, `x * y` (float)

Стандартные сложение, вычитание и умножение по IEEE 754. Результат — округлённое вещественное число.

```nim
assert 0.1 + 0.2 != 0.3      # классическое округление с плавающей точкой
assert 1.0 + Inf == Inf
assert Inf - Inf != Inf - Inf # NaN != NaN
```

---

### `x / y` (float)

```nim
proc `/`*(x, y: float): float
```

Деление с плавающей точкой по IEEE 754. Деление на ноль даёт `Inf`, `-Inf` или `NaN` (при 0.0/0.0) — без исключений.

```nim
assert 1.0 / 2.0 == 0.5
assert 1.0 / 0.0 == Inf
assert -1.0 / 0.0 == -Inf
assert 0.0 / 0.0 != 0.0 / 0.0   # NaN != NaN
```

> Для целочисленного (усечённого) деления используйте `div`; `/` определён только для типов с плавающей точкой.

---

### Составное присваивание для float: `+=`, `-=`, `*=`

```nim
proc `+=`*[T: float|float32|float64](x: var T, y: T)
proc `-=`*[T: float|float32|float64](x: var T, y: T)
proc `*=`*[T: float|float32|float64](x: var T, y: T)
```

Изменяют `x` на месте. Встраиваемые обёртки над базовыми операторами.

```nim
var f = 1.0
f += 0.5    # f == 1.5
f *= 2.0    # f == 3.0
f -= 0.1    # f ≈ 2.9
```

---

### `/=` — деление с присваиванием (float)

```nim
proc `/=`*(x: var float64, y: float64)
proc `/=`*[T: float|float32](x: var T, y: T)
```

Делит `x` на `y` на месте.

```nim
var f = 10.0
f /= 4.0    # f == 2.5
```

## 10. Оборачивающие операторы арифметики (`%`)

Эти операторы интерпретируют свои операнды как **беззнаковые**, независимо от объявленного типа, и всегда оборачиваются при переполнении/недотечении без каких-либо исключений. Доступны для всех знаковых целых типов (`int`, `int8`, `int16`, `int32`, `int64`).

Суффикс `%` — визуальное напоминание о том, что операция *модульная* (никакого `OverflowDefect` никогда).

---

### `x +% y` — оборачивающее сложение

```nim
proc `+%`*(x, y: int): int
```

Складывает `x` и `y` как беззнаковые числа, затем переинтерпретирует битовый шаблон как знаковый тип. Эквивалентно `cast[int](cast[uint](x) + cast[uint](y))`.

```nim
assert high(int8) +% 1'i8 == low(int8)   # 127 + 1 = -128 (оборачивание)
assert -1'i8 +% 1'i8 == 0'i8
```

**Область применения**: хэш-функции, аккумуляторы контрольных сумм и любая арифметика, где намеренно нужно оборачивание в дополнительном коде без исключения.

---

### `x -% y` — оборачивающее вычитание

```nim
proc `-%`*(x, y: int): int
```

Вычитает без проверки переполнения, интерпретируя операнды как беззнаковые.

```nim
assert low(int8) -% 1'i8 == high(int8)   # -128 - 1 = 127 (оборачивание)
assert 0'i8 -% 1'i8 == -1'i8             # совпадает с обычным вычитанием
```

---

### `x *% y` — оборачивающее умножение

```nim
proc `*%`*(x, y: int): int
```

Умножает без проверки переполнения, интерпретируя операнды как беззнаковые перед умножением.

```nim
assert 100'i8 *% 100'i8 == 16'i8   # 10000 mod 256, переинтерпретированное как int8
```

---

### `x /% y` — оборачивающее деление

```nim
proc `/%`*(x, y: int): int
```

Делит, интерпретируя оба операнда как беззнаковые. Для неотрицательных значений совпадает с `div`. Ключевое различие проявляется для отрицательных значений (которые как беззнаковые — большие числа):

```nim
assert (-1'i8 /% 2'i8) == 127'i8
# Потому что cast[uint8](-1) = 255, и 255 div 2 = 127
```

---

### `x %% y` — оборачивающий остаток

```nim
proc `%%`*(x, y: int): int
```

Вычисляет остаток, интерпретируя оба операнда как беззнаковые. Результат всегда неотрицателен при интерпретации как знакового типа (в диапазоне беззнаковых значений типа).

```nim
assert (-7'i32 %% 5'i32) == 3'i32
# cast[uint32](-7) = 4294967289, 4294967289 mod 5 = 3
# (переинтерпретированное как int32: 3)
```

> **Важно**: в отличие от `mod`, знак результата `%%` определяется *беззнаковой интерпретацией*, а не знаком `x`.

## 11. Краткая таблица приоритетов операторов

Приоритеты операторов из этого модуля в Nim, от высшего к низшему:

| Уровень | Операторы | Примечания |
|---------|-----------|------------|
| 9 (высший арифметический) | унарный `+`, унарный `-`, `not` | Справа налево |
| 8 | `*`, `/`, `div`, `mod`, `shl`, `shr`, `%%`, `*%`, `/%` | Слева направо |
| 7 | `+`, `-`, `+%`, `-%`, `or`, `xor` | Слева направо |
| 6 | `and` | Слева направо |

> **Важно**: `shl` и `shr` имеют *меньший* приоритет, чем `+`, в отличие от C. Выражение `a + b shl 2` в Nim означает `a + (b shl 2)`, а не `(a + b) shl 2`.
