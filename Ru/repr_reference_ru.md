# Справочник модуля `repr`

> **Модуль:** `repr.nim` — стандартная библиотека Nim  
> **Назначение:** Универсальный инструмент отладки. Предоставляет строковое представление произвольных значений Nim, включая сложные вложенные структуры, указатели и низкоуровневые типы. Вывод близок к синтаксису Nim и понятен человеку, что делает модуль незаменимым при разработке и отладке.

---

## Общая картина

Модуль `repr` — это движок, стоящий за встроенной процедурой `repr()` в Nim. Он умеет обходить информацию о типах во время исполнения (RTTI) для любого значения и формировать описательную, читаемую строку — по аналогии с `repr()` в Python или форматом `{:?}` в Rust, но для Nim.

Все функции модуля вызываются прежде всего **компилятором** (помечены `compilerproc` или `compilerRtl`): вы пишете `repr(someValue)`, а компилятор сам выбирает нужную функцию на основе типа значения.

---

## Экспортируемые функции

---

### `reprInt`

```nim
proc reprInt(x: int64): string
```

**Что делает:**  
Преобразует 64-битное знаковое целое число в его десятичное строковое представление. Это самый базовый кирпичик — он используется напрямую и вызывается изнутри другими repr-функциями, когда нужно отобразить числовой компонент (например, коды управляющих символов в строках).

**Почему важно:**  
Все целочисленные типы (`int`, `int8`, `int16`, `int32`, `int64`) в конечном счёте приходят к этой функции (после расширения до `int64`). По сути, это обёртка над оператором `$` для `int64`.

**Пример:**
```nim
echo reprInt(42)      # "42"
echo reprInt(-1000)   # "-1000"
echo reprInt(0)       # "0"
```

---

### `reprFloat`

```nim
proc reprFloat(x: float): string
```

**Что делает:**  
Преобразует число с плавающей точкой в строку. Как и `reprInt`, оборачивает оператор `$`, который для `float` даёт десятичное представление с достаточной точностью для точного восстановления значения.

**Почему важно:**  
Числа с плавающей точкой имеют тонкости представления. Оператор `$` для `float` в Nim гарантирует, что `parseFloat(reprFloat(x)) == x` — значение сохраняется без потерь. Это важно при отладке вычислений над вещественными числами.

**Пример:**
```nim
echo reprFloat(3.14)      # "3.14"
echo reprFloat(1.0e10)    # "10000000000.0"
echo reprFloat(1.0/3.0)   # "0.3333333333333333"
```

### `reprPointer`

```nim
proc reprPointer(x: pointer): string
```

**Что делает:**  
Форматирует адрес сырого указателя в виде шестнадцатеричной строки с помощью спецификатора формата `%p` языка C (через `snprintf`). На 64-битной системе результат выглядит как `0x7ffd3a1c2b40`.

**Почему важно:**  
При отладке ссылочных типов, последовательностей или сырых указателей знание реального адреса памяти позволяет выявить алиасинг (две переменные, указывающие на один объект) или отследить идентичность объекта между вызовами `repr`. Эта функция также используется внутренне для отображения адресов последовательностей и ref-значений.

**Пример:**
```nim
var x = 42
echo reprPointer(addr x)  # например: "0x7ffee3b2a004"

# Два ref, указывающие на один объект, покажут одинаковый адрес
var a: ref int = new int
var b = a
echo reprPointer(cast[pointer](a))  # тот же адрес, что у b
echo reprPointer(cast[pointer](b))  # тот же адрес, что у a
```

### `reprStr`

```nim
proc reprStr(s: string): string
```

**Что делает:**  
Формирует отладочное представление строки Nim. Результат всегда заключён в двойные кавычки, специальные символы экранируются:
- `"` → `\"`
- `\` → `\\`
- Символ новой строки (`\10`) → `\10"`, затем настоящий перевод строки и `"` (разбивает строку для удобства чтения)
- Другие управляющие символы или байты > 127 → `\NNN` (десятичный escape)

Кроме того, для ненулевых строк с длиной > 0 перед кавычками добавляется адрес памяти базового C-строкового буфера — это помогает обнаружить алиасинг строк.

**Почему важно:**  
В отличие от оператора `$`, функция `reprStr` делает невидимые символы видимыми. Это критично при отладке строк, содержащих встроенные нулевые байты, управляющие последовательности или бинарные данные.

**Пример:**
```nim
echo reprStr("hello")         # 0x... "hello"
echo reprStr("say \"hi\"")    # 0x... "say \"hi\""
echo reprStr("line1\nline2")  # 0x... "line1\10"
                               #       "line2"
echo reprStr("")              # ""
```

### `reprBool`

```nim
proc reprBool(x: bool): string
```

**Что делает:**  
Возвращает `"true"` или `"false"` для булевых значений. Просто и прямолинейно.

**Почему важно:**  
Когда булевы значения встречаются внутри сложных структур (кортежей, объектов), `repr` использует эту функцию для их отображения. Явные строки `"true"`/`"false"` (а не `"1"`/`"0"`) делают вывод читаемым.

**Пример:**
```nim
echo reprBool(true)   # "true"
echo reprBool(false)  # "false"

# Внутри кортежа:
type Flags = tuple[active: bool, count: int]
var f: Flags = (true, 5)
echo repr(f)  # [active = true, count = 5]
```

### `reprChar`

```nim
proc reprChar(x: char): string
```

**Что делает:**  
Преобразует одиночный символ в repr-строку, всегда обёрнутую в одиночные кавычки. Специальные символы экранируются:
- `"` → `\"` внутри кавычек
- `\` → `\\`
- Управляющие символы (0–31) и байты 127–255 → `\NNN` (десятичный escape)

**Почему важно:**  
Без экранирования вывод нулевого байта или символа табуляции был бы невидимым или вводящим в заблуждение. `reprChar` делает фактическое значение байта однозначным.

**Пример:**
```nim
echo reprChar('A')     # "'A'"
echo reprChar('\n')    # "'\10'"
echo reprChar('\\')    # "'\\'"
echo reprChar('\0')    # "'\0'"
echo reprChar('"')     # "'\"'"
```

### `reprEnum`

```nim
proc reprEnum(e: int, typ: PNimType): string
```

**Что делает:**  
Принимает целочисленное значение перечисления и его RTTI (`PNimType`), возвращает имя соответствующего варианта перечисления в виде строки.

Используются две стратегии поиска:
1. **Прямая индексация** — для перечислений без «дыр» (последовательные значения): использует ординал как индекс массива, поиск за O(1).
2. **Линейный поиск** — для перечислений с «дырами» (непоследовательные значения): сканирует все варианты в поиске совпадения.

Если целое число не соответствует ни одному варианту, возвращается `"<значение> (invalid data!)"`.

**Почему важно:**  
Перечисления хранятся как целые числа во время исполнения. Без RTTI вы бы видели просто `2` вместо `"Green"`. Эта функция восполняет этот пробел, делая значения перечислений читаемыми в отладочном выводе.

**Пример:**
```nim
type Color = enum
  Red, Green, Blue

var c = Green
echo repr(c)  # "Green"

# Перечисление с «дырами»:
type Status = enum
  Ok = 0, Warning = 10, Error = 100

var s = Warning
echo repr(s)  # "Warning"

# Пример некорректных данных (гипотетически):
# reprEnum(99, statusType)  →  "99 (invalid data!)"
```

### `reprSet`

```nim
proc reprSet(p: pointer, typ: PNimType): string
```

**Что делает:**  
Формирует repr-строку для множества (`set`) Nim. Вывод форматируется как `{elem1, elem2, ...}` (или `{}` для пустого множества). Функция считывает базовый битовый массив (множества в Nim хранятся как битовые поля) и восстанавливает присутствующие элементы, отображая каждый через `addSetElem`.

Формат отображения элемента зависит от типа элемента:
- Элементы-перечисления → имена вариантов
- Булевы элементы → `true`/`false`
- Символьные элементы → символьные литералы
- Целочисленные элементы → десятичные числа

**Почему важно:**  
Множества Nim — это компактные битовые карты во время исполнения. Инспекция сырых байтов ничего не скажет. `reprSet` декодирует битовую карту обратно в набор значений.

**Пример:**
```nim
type Fruit = enum
  Apple, Banana, Cherry

var basket: set[Fruit] = {Apple, Cherry}
echo repr(basket)  # "{Apple, Cherry}"

var bits: set[char] = {'a', 'b', 'z'}
echo repr(bits)    # "{'a', 'b', 'z'}"

var empty: set[int8] = {}
echo repr(empty)   # "{}"
```

### `reprOpenArray`

```nim
proc reprOpenArray(p: pointer, length: int, elemtyp: PNimType): string
```

**Что делает:**  
Формирует repr-строку для открытого массива (`openArray[T]` в Nim — параметрический тип, принимающий как обычные массивы, так и последовательности). Вывод форматируется как `[elem0, elem1, ...]`, каждый элемент отображается через универсальный диспетчер `reprAux`.

`openArray` во время исполнения представляет собой просто указатель + длину, поэтому функция получает их раздельно и обходит элементы вручную.

**Почему важно:**  
`openArray` — распространённый паттерн в Nim для написания гибких процедур. Возможность вызвать `repr` для них при отладке необходима, хотя у них нет самостоятельного типа во время исполнения.

**Пример:**
```nim
proc process(data: openArray[int]) =
  echo repr(data)  # внутри использует reprOpenArray

process([1, 2, 3])    # "[1, 2, 3]"
process(@[10, 20])    # "[10, 20]"
```

### `reprAny`

```nim
proc reprAny(p: pointer, typ: PNimType): string
```

**Что делает:**  
Это **главный диспетчер** — функция, которую вызывает компилятор, когда вы пишете `repr(someValue)`. Она принимает сырой указатель на любое значение и его RTTI, инициализирует состояние обхода (`ReprClosure`) и вызывает внутренний рекурсивный диспетчер `reprAux`.

Контекст `ReprClosure` отслеживает:
- **`recdepth`**: ограничение глубины рекурсии (по умолчанию: без ограничений, `-1`). При достижении лимита вместо содержимого выводится `"..."`, что предотвращает бесконечные циклы на циклических ссылках.
- **`indent`**: текущий уровень отступа для многострочного вывода.
- **`marked`**: множество уже посещённых GC-ячеек. Предотвращает бесконечные циклы на циклических графах `ref`/`ptr`: каждая ячейка показывается полностью только один раз, при повторном посещении выводится лишь её адрес.

**Почему важно:**  
Через `reprAny` проходит всё. Это единственная точка входа, благодаря которой `repr()` работает для любого типа в Nim. Понимание этой функции объясняет, почему циклические структуры не вызывают бесконечный цикл и почему глубоко вложенные структуры обрабатываются корректно.

**Пример:**
```nim
# reprAny напрямую не вызывается — используйте repr():
type Node = ref object
  val: int
  next: Node

var n1 = Node(val: 1)
var n2 = Node(val: 2)
n1.next = n2
n2.next = n1  # цикл!

echo repr(n1)
# ref 0x... --> [val = 1,
# next = ref 0x... --> [val = 2,
# next = ref 0x... ]]   ← цикл обнаружен, адрес показан, но содержимое не раскрывается повторно
```

## Внутренние вспомогательные функции (не публичные, но важные для понимания)

Они не экспортируются для прямого использования, но определяют наблюдаемое поведение:

| Функция | Роль |
|---|---|
| `reprStrAux` | Основная логика экранирования строк; используется в `reprStr` и ветке `tyCstring` |
| `reprArray` | Отображает массивы фиксированного размера как `[e0, e1, ...]` |
| `reprSequence` | Отображает последовательности как `0x... @[e0, e1, ...]` (адрес + `@[...]`) |
| `reprRecord` | Отображает кортежи и объекты как `[field = val, ...]` |
| `reprRef` | Отображает `ref`/`ptr` с адресом и разыменованным содержимым, защита от циклов |
| `reprAux` | Центральный рекурсивный диспетчер по всем видам типов |
| `addSetElem` | Отображает один элемент множества в зависимости от его типа |
| `initReprClosure` | Инициализирует состояние обхода (захватывает блокировку кучи в многопоточном режиме) |
| `deinitReprClosure` | Завершает состояние обхода (освобождает блокировку кучи) |

---

## Ключевые особенности проектирования

**Защита от циклов:** `reprAny` / `reprRef` используют `CellSet` (множество указателей GC-ячеек) для отслеживания посещённых объектов. `ref` или `ptr`, который уже был посещён, отображается только по адресу, без раскрытия содержимого. Это предотвращает бесконечную рекурсию на циклических графах.

**Потокобезопасность:** Когда определены `hasThreadSupport` и `hasSharedHeap`, функция `initReprClosure` захватывает блокировку кучи на время операции repr. Это необходимо, поскольку `CellSet` выделяет память внутри себя.

**Интеграция с компилятором:** Функции, помеченные `compilerproc` или `compilerRtl`, вызываются непосредственно из сгенерированного кода на C. Компилятор знает их имена и вставляет вызовы — вы не диспетчеризируете их на уровне Nim.

**Устаревший перевод строки:** Директива `nimLegacyReprWithNewline` заставляет `reprAny` добавлять `\n` в конец вывода, воспроизводя старое поведение Nim (до PR #16034).
