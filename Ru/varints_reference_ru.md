# Справочник модуля `varints`

> Кодирование целых чисел переменной длины, вдохновлённое SQLite.  
> **Статус API:** нестабильный.

---

## Что такое varint?

*Varint* (variable-length integer, целое число переменной длины) — компактный двоичный формат, в котором маленькие числа занимают меньше байт, чем большие. Вместо того чтобы всегда отводить 8 байт под `uint64`, значение вроде `42` хранится всего в **1 байте**, при этом весь 64-битный диапазон по-прежнему укладывается максимум в **9 байт**.

Это особенно ценно в файловых форматах, сетевых протоколах и базах данных, где большинство чисел невелики и важна экономия места.

Кодирование повторяет схему из **SQLite 4** и работает так:

| Диапазон значений   | Байт |
|---------------------|------|
| 0 – 240             | 1    |
| 241 – 2 287         | 2    |
| 2 288 – 67 823      | 3    |
| 67 824 – 16 777 215 | 4    |
| …до 2⁶⁴−1           | до 9 |

## Экспортируемые символы

### `maxVarIntLen` — константа

```nim
const maxVarIntLen* = 9
```

**Что это:** Максимальное количество байт, которое может занять один varint.

Используйте эту константу для безопасного выделения буферов — ни один varint не превысит 9 байт, какое бы значение ни кодировалось.

**Пример:**

```nim
var buf: array[maxVarIntLen, byte]   # всегда достаточно места
let written = writeVu64(buf, someValue)
```

---

### `writeVu64` — кодирование беззнакового 64-битного числа

```nim
proc writeVu64*(z: var openArray[byte], x: uint64): int
```

**Что делает:** Записывает беззнаковое целое `x` в байтовый буфер `z` в формате varint. Возвращает количество фактически записанных байт (от 1 до 9).

**Параметры:**

- `z` — изменяемый байтовый буфер. Для любого входного значения должен быть длиной не менее `maxVarIntLen` (9) байт.
- `x` — значение для кодирования.

**Возвращаемое значение:** Количество записанных байт. Передайте его в `readVu64`, чтобы декодер знал, где заканчивается текущий varint и начинается следующий.

**Ключевая идея:** Маленькие числа дёшевы. Значение `0` занимает 1 байт; значение `2^64 − 1` занимает 9 байт. Всё, что между ними, масштабируется плавно.

**Пример:**

```nim
import varints

var buf: array[maxVarIntLen, byte]

# Маленькое число — занимает всего 1 байт
var n = writeVu64(buf, 100u64)
echo n          # → 1
echo buf[0]     # → 100

# Большое число — требует больше байт
n = writeVu64(buf, 300_000u64)
echo n          # → 4
```

### `readVu64` — декодирование беззнакового 64-битного числа

```nim
proc readVu64*(z: openArray[byte]; pResult: var uint64): int
```

**Что делает:** Читает varint из байтового буфера `z`, записывает декодированное значение в `pResult` и возвращает количество потреблённых байт. Возвращает `0`, если буфер слишком короткий и содержит неполный varint.

**Параметры:**

- `z` — байтовый буфер с закодированными данными (только для чтения).
- `pResult` — выходная переменная, куда помещается декодированный `uint64`.

**Возвращаемое значение:** Количество потреблённых байт — то же самое, что вернул `writeVu64` при кодировании. Значение `0` сигнализирует, что буфер обрезан и данных не хватает.

**Ключевая идея:** Первый байт играет роль *дискриминатора* — по его значению декодер сразу определяет, сколько дополнительных байт нужно считать. Это делает декодирование однопроходным: никаких циклов, только ветвления.

**Пример:**

```nim
import varints

var buf: array[maxVarIntLen, byte]
var value: uint64

# Цикл: кодируем, затем декодируем
let written = writeVu64(buf, 12345u64)
let read    = readVu64(buf, value)

assert written == read     # одинаковое число байт
assert value == 12345u64
echo value   # → 12345

# Обнаружение усечённого буфера
var shortBuf: array[1, byte]
shortBuf[0] = 252u8           # дискриминатор, требующий 6 байт
let result = readVu64(shortBuf, value)
echo result  # → 0  (буфер слишком короткий, value не изменилось)
```

### `encodeZigzag` — отображение знаковых чисел в беззнаковые

```nim
proc encodeZigzag*(x: int64): uint64 {.inline.}
```

**Что делает:** Преобразует *знаковый* `int64` в *беззнаковый* `uint64` по схеме «зигзаг-кодирования», после чего значение можно эффективно сжать с помощью `writeVu64`.

**Зачем это нужно:** Varint-кодирование экономично только для *малых* беззнаковых значений. Отрицательное число вроде `-1` имеет битовое представление `0xFFFFFFFFFFFFFFFF` и потребовало бы все 9 байт. Зигзаг-кодирование решает эту проблему, чередуя положительные и отрицательные числа:

```
0  → 0
-1 → 1
1  → 2
-2 → 3
2  → 4
...
```

Таким образом, числа с малым *абсолютным значением* (как положительные, так и отрицательные) дают маленькие беззнаковые результаты и хорошо сжимаются.

**Формула:** `(x << 1) XOR (x >> 63)`

**Пример:**

```nim
import varints

echo encodeZigzag(0'i64)    # → 0
echo encodeZigzag(-1'i64)   # → 1
echo encodeZigzag(1'i64)    # → 2
echo encodeZigzag(-2'i64)   # → 3
echo encodeZigzag(100'i64)  # → 200
echo encodeZigzag(-100'i64) # → 199
```

**Типичный паттерн — кодирование знакового числа как компактного varint:**

```nim
var buf: array[maxVarIntLen, byte]
let signed: int64 = -42
let n = writeVu64(buf, encodeZigzag(signed))
echo n  # → 1  (всего 1 байт для числа с малым модулем!)
```

### `decodeZigzag` — восстановление знакового числа из зигзаг-формы

```nim
proc decodeZigzag*(x: uint64): int64 {.inline.}
```

**Что делает:** Точная обратная операция к `encodeZigzag`. Принимает зигзаг-кодированный `uint64` и возвращает исходный `int64`.

**Формула:** `(x >> 1) XOR -(x AND 1)` (в исходном коде реализована через битовое приведение типов).

**Пример:**

```nim
import varints

echo decodeZigzag(0u64)   # → 0
echo decodeZigzag(1u64)   # → -1
echo decodeZigzag(2u64)   # → 1
echo decodeZigzag(3u64)   # → -2
echo decodeZigzag(199u64) # → -100
echo decodeZigzag(200u64) # → 100
```

**Полный цикл для знаковых чисел:**

```nim
import varints

var buf: array[maxVarIntLen, byte]
var raw: uint64

let original: int64 = -9999

# Кодирование
let written = writeVu64(buf, encodeZigzag(original))

# Декодирование
let consumed = readVu64(buf, raw)
let recovered = decodeZigzag(raw)

assert recovered == original
echo recovered  # → -9999
echo written    # → 2  (малый модуль = короткий varint)
```

## Объединяем всё — пример потоковой обработки

```nim
import varints

# Представим, что нам нужно записать поток знаковых дельт между
# показаниями датчика. Дельты обычно небольшие, так что
# varint + zigzag даёт отличное сжатие.

let deltas: seq[int64] = @[0, 5, -3, 120, -1, 0, 1000, -800]

var stream: seq[byte] = @[]
var tmp: array[maxVarIntLen, byte]

for d in deltas:
  let n = writeVu64(tmp, encodeZigzag(d))
  stream.add(tmp[0 ..< n])

echo "Размер после кодирования: ", stream.len, " байт"
# Намного меньше, чем 8×8 = 64 байта при хранении в виде int64

# --- Декодирование ---
var pos = 0
var raw: uint64
while pos < stream.len:
  let n = readVu64(toOpenArray(stream, pos, stream.high), raw)
  if n == 0: break   # усечённые данные
  echo decodeZigzag(raw)
  pos += n
```

## Быстрая шпаргалка

| Символ           | Тип       | Назначение                                           |
|------------------|-----------|------------------------------------------------------|
| `maxVarIntLen`   | константа | Максимальное число байт varint (= 9)                 |
| `writeVu64`      | процедура | Кодировать `uint64` → байты varint                   |
| `readVu64`       | процедура | Декодировать байты varint → `uint64`                 |
| `encodeZigzag`   | процедура | Отобразить `int64` → `uint64` для сжатия со знаком   |
| `decodeZigzag`   | процедура | Отобразить `uint64` → `int64` (обратная операция)    |
