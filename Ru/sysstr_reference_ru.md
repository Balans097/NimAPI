# Справочник модуля sysstr

> **Модуль:** `sysstr`  
> **Часть:** Стандартная библиотека Nim  
> **Назначение:** Реализует низкоуровневые примитивы строк и последовательностей, на которые опирается кодогенератор Nim. Любая конкатенация строк, срез, изменение размера и `add` в пользовательском Nim-коде в конечном счёте компилируется в вызовы этого модуля. Понимание его устройства критично для рассуждений о производительности строк, расположении памяти и границе между Nim-строками и C-строками.

---

## Основные концепции

### Как строки Nim расположены в памяти

`NimString` — это указатель на блок памяти в куче со следующей структурой:

```
 ┌────────────────────────────────────────────────────┐
 │  Заголовок TGenericSeq                             │
 │  ├─ len      : int   (текущее число символов)      │
 │  ├─ reserved : int   (ёмкость + флаговые биты)     │
 │  └─ [elemSize: int   (только с gogc)]              │
 ├────────────────────────────────────────────────────┤
 │  data[0..len-1]  : массив char                     │
 │  data[len]       : '\0'  (терминатор C-строки)     │
 └────────────────────────────────────────────────────┘
```

- `len` считает *байты*, а не кодовые точки Unicode.
- `reserved` хранит текущую ёмкость; старшие биты могут нести специальные флаги (`seqShallowFlag`, `strlitFlag`).
- Завершающий `\0` означает, что любой `NimString` можно привести к `cstring` с нулевыми накладными расходами.

### Чем последовательности отличаются от строк

У `PGenericSeq` тот же заголовок `TGenericSeq`, но за ним следуют данные элементов с выравниванием. Размер и выравнивание элемента берутся из дескриптора типа времени выполнения (`PNimType`), а не из заголовка.

### Стратегия роста (`resize`)

По всему модулю единая функция роста управляет расширением ёмкости:

```
ёмкость ≤ 0        → 4
0 < ёмкость < 64K  → ёмкость × 2
ёмкость ≥ 64K      → ёмкость × 1,5
```

Это балансирует быстрое удвоение для маленьких буферов и снижение расхода памяти для больших.

## Выделение памяти для строк

### `rawNewString`

```nim
proc rawNewString(space: int): NimString {.compilerproc.}
```

**Что это такое:** Выделяет новый пустой `NimString` с зарезервированной ёмкостью не менее `space` байт. Строка полностью инициализирована: `len` равен `0`, первый байт — `'\0'`, так что она немедленно пригодна как C-строка. Минимальная фактическая ёмкость всегда равна 7 (чтобы избежать крохотных, быстро перераспределяемых буферов).

Это рабочая лошадка за `""` в сгенерированном C-коде и за `newStringOfCap`.

```nim
# Сгенерированный C-код для:  s = ""
# → rawNewString(0)
#
# Сгенерированный C-код для предзаказа конкатенации известной длины:
# → rawNewString(6 + name.len + 17)
```

> **Замечание о `NoInit`-варианте:** Внутри используется `rawNewStringNoInit`, когда вызывающий код гарантирует, что сам заполнит `len` и нулевой терминатор — небольшая экономия на обнулении. Публичный `rawNewString` всегда оставляет строку в безопасном, пригодном состоянии.

---

### `mnewString`

```nim
proc mnewString(len: int): NimString {.compilerproc.}
```

**Что это такое:** Выделяет `NimString` *ровно* из `len` байт, все установлены в `'\0'`. И `len`, и `reserved` устанавливаются в `len`, вся область данных обнуляется. Именно так реализуется `newString(n)` — вы получаете строку, предзаполненную нулевыми байтами, готовую к записи побайтово.

Отличие от `rawNewString`: `rawNewString` даёт *пустую* строку с зарезервированным местом; `mnewString` даёт строку, уже *имеющую* запрошенную длину, заполненную нулями.

```nim
# Nim:
let s = newString(10)   # len=10, все байты '\0'
# Внутри → mnewString(10)
```

## Конвертация строк

### `nimToCStringConv`

```nim
proc nimToCStringConv(s: NimString): cstring {.compilerproc, nonReloadable, inline.}
```

**Что это такое:** Мост с нулевой стоимостью из `NimString` в `cstring`. Поскольку строки Nim уже нуль-терминированы, это просто приведение указателя к `addr s.data` — никакого копирования, никакого выделения памяти. Если `s` равен `nil` или пустой, возвращает литерал пустой C-строки `""`, чтобы не передавать нулевой указатель в C-функции.

Вызывается каждый раз, когда Nim передаёт `string` в параметр `cstring` функции C.

```nim
proc puts(s: cstring) {.importc, header: "<stdio.h>".}

let msg = "Привет"
puts(msg)   # компилятор вставляет nimToCStringConv(msg) — бесплатная операция
```

---

### `toNimStr`

```nim
proc toNimStr(str: cstring, len: int): NimString {.compilerproc.}
```

**Что это такое:** Создаёт новый `NimString`, копируя ровно `len` байт из C-строки `str`. Результат правильно терминируется `'\0'`. Используется компилятором, когда C-строка известной длины должна стать Nim-строкой (например, строковые литералы, возвращаемые значения из FFI, где длина уже вычислена).

```nim
// Когда C-функция возвращает char* с известной длиной,
// компилятор оборачивает: toNimStr(cResult, cResultLen)
```

---

### `cstrToNimstr`

```nim
proc cstrToNimstr(str: cstring): NimString {.compilerRtl.}
```

**Что это такое:** Конвертирует нуль-терминированную C-строку *неизвестной* длины в `NimString`, сначала вызывая `str.len` (что сканирует до `'\0'`), затем делегируя в `toNimStr`. Возвращает `nil`, если `str` равен `nil`.

Это функция времени выполнения за `$someCstring`, когда длина должна быть определена во время выполнения.

```nim
proc getenv(key: cstring): cstring {.importc, header: "<stdlib.h>".}

let path = $getenv("PATH")   # компилятор вставляет cstrToNimstr(getenv("PATH"))
```

---

### `copyStrLast`

```nim
proc copyStrLast(s: NimString, start, last: int): NimString {.compilerproc.}
```

**Что это такое:** Извлекает подстроку `s[start..last]` в новый `NimString`. `start` ограничивается снизу `0`; `last` ограничивается сверху `s.len - 1`. Возвращает `nil`, если `s` равен `nil`. Эта процедура существует главным образом для **бутстрэппинга** — современные версии компилятора больше не генерируют её вызовы, но она должна оставаться, чтобы компилятор мог скомпилировать сам себя.

```nim
# Эквивалентно (более старый компилятор):  s[2..5]
# → copyStrLast(s, 2, 5)
```

---

### `copyStr`

```nim
proc copyStr(s: NimString, start: int): NimString {.compilerproc.}
```

**Что это такое:** Как `copyStrLast`, но копирует от `start` до конца строки. Делегирует в `copyStrLast(s, start, s.len - 1)`. Также существует исключительно для совместимости при **бутстрэппинге**.

```nim
# Эквивалентно (более старый компилятор):  s[3..^1]
# → copyStr(s, 3)
```

## Копирование строк

### `copyString`

```nim
proc copyString(src: NimString): NimString {.compilerRtl.}
```

**Что это такое:** Стандартное глубокое копирование `NimString`. Возвращает `nil`, если `src` равен `nil`. Если `src` несёт `seqShallowFlag` (помечает его как поверхностную копию), возвращается тот же указатель без копирования. В противном случае выделяется новый буфер и копируются все байты, включая нулевой терминатор.

Когда определён `nimShallowStrings` и `src` является строковым литералом (`strlitFlag`), копия делается поверхностной, чтобы избежать ненужного дублирования константных данных.

Вызывается компилятором там, где семантика значений Nim требует копии — например, при присваивании строки новой переменной в не-`move`-контексте.

```nim
var a = "привет"
var b = a          # компилятор вызывает copyString(a)
b[0] = 'П'        # a не изменяется
```

---

### `copyStringRC1`

```nim
proc copyStringRC1(src: NimString): NimString {.compilerRtl.}
```

**Что это такое:** Вариант копирования, оптимизированный для бэкендов GC с подсчётом ссылок (ARC/ORC). Под этими бэкендами, если доступен быстрый путь `newObjRC1`, аллокация выполняется со счётчиком ссылок, предустановленным в 1, что избегает лишнего атомарного инкремента. Под регионным или не-RC GC откатывается к `toOwnedCopy`, делая его идентичным `copyString`. Логика `seqShallowFlag` та же.

---

### `copyDeepString`

```nim
proc copyDeepString(src: NimString): NimString {.inline, raises: [].}
```

**Что это такое:** Безусловное глубокое копирование — всегда выделяет свежий буфер независимо от поверхностных флагов. Используется в контекстах, где требуется семантика `deepCopy` Nim (например, передача строк между потоками или в каналы).

```nim
# Отправка через канал всегда глубоко копирует:
chan.send(deepCopy(s))   # внутри использует copyDeepString
```

## Мутация и рост строк

### `resizeString`

```nim
proc resizeString(dest: NimString, addlen: int): NimString {.compilerRtl.}
```

**Что это такое:** Обеспечивает наличие в `dest` достаточной ёмкости ещё для `addlen` байт, **не обновляя `len`**. Если `dest` уже имеет достаточно места, возвращается без изменений. Если нет, выделяется новый буфер большего размера по стратегии `resize` (не менее `dest.len + addlen`, возможно больше), существующее содержимое копируется, и возвращается новый буфер.

Это *шаг предварительного выделения памяти*, который компилятор вставляет перед серией вызовов `appendString`. Двухфазный паттерн (один `resize`, затем несколько `append`) — ключ к эффективному `&=` и многочастной конкатенации через `&`.

```nim
# Nim:   s &= "Привет " & name & "!"
# Сгенерированный C (концептуально):
#   s = resizeString(s, 7 + name.len + 1)
#   appendString(s, "Привет ")
#   appendString(s, name)
#   appendString(s, "!")
```

> **Важно:** После `resizeString` возвращённый указатель может отличаться от входного (если произошло перераспределение). Всегда используйте возвращаемое значение.

---

### `appendString`

```nim
proc appendString(dest, src: NimString) {.compilerproc, inline.}
```

**Что это такое:** Копирует все байты `src` (включая нулевой терминатор) в конец `dest` и увеличивает `dest.len` на `src.len`. **Не проверяет ёмкость** — в сгенерированном коде ему всегда предшествует вызов `resizeString`. Является no-op, если `src` равен `nil`.

Это *максимально дешёвый append*: один `copyMem` и один инкремент целого числа.

---

### `appendChar`

```nim
proc appendChar(dest: NimString, c: char) {.compilerproc, inline.}
```

**Что это такое:** Добавляет один символ `c` к `dest`, обновляет нулевой терминатор и инкрементирует `len`. Как и `appendString`, не проверяет ёмкость. Вызывается в плотных циклах, где компилятор точно знает, что добавляется ровно один символ.

```nim
# Nim:  s.add('\n')
# → appendChar(s, '\n')  (после проверки ёмкости)
```

---

### `addChar`

```nim
proc addChar(s: NimString, c: char): NimString
```

**Что это такое:** *Безопасный* append одного символа, который самостоятельно проверяет ёмкость и при необходимости перераспределяет память. Если `s` равен `nil`, выделяет новую строку. Если `s` заполнен, увеличивает его по стратегии `resize`. Возвращает (возможно новый) указатель на строку. Используется в `ccgexprs.nim`, когда добавление одного символа нельзя статически предварительно размерить.

---

### `setLengthStr`

```nim
proc setLengthStr(s: NimString, newLen: int): NimString {.compilerRtl.}
```

**Что это такое:** Изменяет размер строки до ровно `newLen` символов. Чисто обрабатывает все три случая:

- **Рост:** выделяет буфер большего размера, копирует старое содержимое, обнуляет новый хвост.
- **Уменьшение:** оставляет тот же буфер, обновляет `len` и помещает `'\0'` в новый конец.
- **Без изменения:** всё равно обновляет нулевой терминатор в `s[newLen]` для безопасности.

Отрицательный `newLen` трактуется как 0. Если `s` равен `nil` и `newLen > 0`, возвращает обнулённую строку нужной длины через `mnewString`.

Это базовая реализация `setLen(s, n)` для строк.

```nim
var s = "Привет, Мир!"
s.setLen(6)    # → setLengthStr(s, 6) → s теперь "Привет"
s.setLen(10)   # → setLengthStr(s, 10) → "Привет\0\0\0\0"
```

## Операции с последовательностями

### `incrSeq`

```nim
proc incrSeq(seq: PGenericSeq, elemSize, elemAlign: int): PGenericSeq {.compilerproc.}
```

**Что это такое:** Увеличивает последовательность ровно на один элемент (инкрементирует `len` на 1) и возвращает (возможно перераспределённый) указатель на последовательность. Если текущая длина равна ёмкости, вызывается `growObj` со следующим шагом `resize`. Вызывающий код затем обязан записать фактический элемент в `seq[seq.len - 1]`.

Это прямая реализация `seq.add(elem)` в сгенерированном C-коде.

```nim
# Nim:
var s: seq[int]
s.add(42)
# Сгенерированный C (концептуально):
#   s = incrSeq(s, sizeof(int), alignof(int))
#   s[s.len - 1] = 42
```

---

### `incrSeqV3`

```nim
proc incrSeqV3(s: PGenericSeq, typ: PNimType): PGenericSeq {.compilerproc.}
```

**Что это такое:** Более новая версия `incrSeq`, принимающая полный дескриптор `PNimType` вместо сырых размера/выравнивания элемента — это позволяет корректно использовать `newSeq` для перераспределения (важно для корректности GC). Обрабатывает `nil`-ввод, выделяя свежую последовательность длиной 1 с `len`, установленным в 0. При росте «крадёт» содержимое из старой последовательности (`s.len = 0`), чтобы предотвратить двойной учёт GC.

Суффикс «V3» означает, что это текущий вариант по умолчанию, используемый компилятором для большинства операций `add` с последовательностями.

---

### `setLengthSeqV2`

```nim
proc setLengthSeqV2(s: PGenericSeq, typ: PNimType, newLen: int, isTrivial: bool): PGenericSeq {.compilerRtl.}
```

**Что это такое:** Изменяет размер обобщённой последовательности до `newLen` элементов, инициализируя новые элементы нулями. Обрабатывает все три случая:

- **Рост:** при необходимости расширяет ёмкость через `extendCapacityRaw`, затем обнуляет новые слоты.
- **Уменьшение:** вызывает `truncateRaw`, который под трассирующими бэкендами GC декрементирует счётчики ссылок для нетривиальных типов и обнуляет освобождённые слоты.
- **Без изменения:** обновляет `len` на месте.

Базовая реализация `setLen(mySeq, n)` для последовательностей.

Флаг `isTrivial` сигнализирует, что элементы не содержат управляемых ссылок — когда `true`, логика деструктора/обнуления при усечении пропускается, делая уменьшение быстрее.

```nim
var s = @[1, 2, 3, 4, 5]
s.setLen(2)    # → setLengthSeqV2(..., 2, true) → @[1, 2]
s.setLen(10)   # → setLengthSeqV2(..., 10, true) → @[1, 2, 0, 0, 0, 0, 0, 0, 0, 0]
```

---

### `setLengthSeqUninit`

```nim
proc setLengthSeqUninit(s: PGenericSeq; typ: PNimType; newLen: int; isTrivial: bool): PGenericSeq {.compilerRtl.}
```

**Что это такое:** Идентично `setLengthSeqV2`, за исключением того, что **новые элементы не обнуляются при росте**. Используется, когда компилятор может доказать, что каждый новый слот будет записан раньше, чем прочитан — скорость в обмен на безопасность. Берётся путь `doInit = false` в базовом шаблоне.

> **Не полагайтесь на содержимое новых элементов после `setLengthSeqUninit`.** Они содержат то, что оказалось в памяти аллокатора по этому адресу.

## Запрос ёмкости

### `capacity` (строка)

```nim
func capacity*(self: string): int {.inline.}
```

**Что это такое:** Возвращает количество байт, которые строка может вместить *без перераспределения* — то есть значение `reserved` в базовом `NimString`. Возвращает `0` для `nil`-строк. Это строковый аналог `seq.capacity`.

Знание ёмкости важно, когда вы хотите заранее выделить ровно нужное количество памяти и избежать перераспределений в середине цикла.

```nim
var s = newStringOfCap(42)
s.add "Nim"
assert s.capacity == 42   # всё ещё 42; перераспределения не было
assert s.len == 3
```

---

### `capacity` (seq)

```nim
func capacity*[T](self: seq[T]): int {.inline.}
```

**Что это такое:** Возвращает количество элементов, которые последовательность может вместить без перераспределения — поле `space` базового `PGenericSeq`. Возвращает `0` для `nil`-последовательностей.

```nim
var lst = newSeqOfCap[string](42)
lst.add "Nim"
assert lst.capacity == 42
assert lst.len == 1

# Предварительное выделение для цикла известной длины:
var results = newSeqOfCap[int](n)
for i in 0..<n:
  results.add computeSomething(i)
assert results.capacity >= n   # произошло не более одного выделения
```
