# Справочник модуля `decls`

> **Назначение модуля:** `decls` предоставляет синтаксический сахар для объявлений, которые неудобно выражать на чистом Nim. На данный момент модуль содержит один макрос — `byaddr` — который привносит в Nim l-value ссылки в стиле C++. В будущем здесь могут появиться другие эргономичные формы объявлений.

## Предыстория: что такое l-value ссылка?

В большинстве системных языков **ссылка** (или *псевдоним*, *алиас*) — это имя, которое указывает непосредственно на существующую ячейку памяти, а не хранит собственную копию данных. Изменение ссылки изменяет оригинал.

В C++ это записывается так:
```cpp
auto& a = someArray[0];
a += 100;  // изменяет someArray[0] на месте
```

В Nim нет встроенного синтаксиса объявления ссылки для локальных переменных. Обычные обходные пути — передача параметров `var`, использование явных указателей — либо невозможны в определённых областях видимости, либо синтаксически громоздки. `byaddr` заполняет этот пробел, автоматически генерируя необходимую работу с указателями и скрывая её за чистым синтаксисом объявления.

## `byaddr`

```nim
macro byaddr*(sect)
```

### Что делает

`byaddr` — это **макрос-прагма переменной**, который превращает привязку `let` или `var` в псевдоним — имя, которое прозрачно читает из памяти выражения в правой части и пишет в неё. После объявления каждое использование нового имени эквивалентно прямому обращению к исходному местоположению.

Синтаксис объявления зеркально повторяет обычное объявление переменной Nim, с `{.byaddr.}`, помещённым между именем и `=`:

```nim
var a {.byaddr.} = someExpression
```

Аннотация типа необязательна, но поддерживается:

```nim
var b {.byaddr.}: int = someExpression
```

### Ключевые свойства

| Свойство | Поведение |
|---|---|
| Изменение псевдонима | Изменяет исходную ячейку памяти |
| Тип псевдонима | Тип элемента выражения (не указатель) |
| Адрес псевдонима | Идентичен адресу оригинала |
| Исходное выражение | Вычисляется один раз, в момент объявления |
| Работает с | Любым l-value: элементы массива, элементы последовательности, поля объекта, разыменованные указатели |

### Синтаксис использования

```nim
import std/decls

# Форма 1: тип выводится автоматически
var a {.byaddr.} = expression

# Форма 2: явная аннотация типа
var b {.byaddr.}: SomeType = expression
```

> `byaddr` должен появляться как прагма на одном объявлении переменной — не на блоке `var` с несколькими привязками, и не на секции `const` или `type`.

### Примеры

#### Пример 1 — Псевдоним для элемента последовательности

Самая наглядная демонстрация: изменение `a` изменяет `s[0]`, потому что они разделяют один и тот же адрес.

```nim
import std/decls

var s = @[10, 11, 12]
var a {.byaddr.} = s[0]

a += 100

assert s == @[110, 11, 12]   # s[0] изменился через псевдоним
assert a is int               # псевдоним имеет тип элемента, а не тип указателя
```

#### Пример 2 — Два псевдонима, один адрес

И `a`, и `b` ссылаются на `s[0]`, поэтому их адреса идентичны. Это подтверждает, что копии не создаются.

```nim
import std/decls

var s = @[10, 11, 12]
var a {.byaddr.} = s[0]
var b {.byaddr.}: int = s[0]

assert a.addr == b.addr   # оба указывают на s[0]
```

#### Пример 3 — Псевдоним для поля объекта

`byaddr` не ограничен последовательностями. Он работает с любым l-value, включая поля структур.

```nim
import std/decls

type Point = object
  x, y: float

var p = Point(x: 1.0, y: 2.0)
var px {.byaddr.} = p.x

px = 99.0
assert p.x == 99.0   # поле было изменено через псевдоним
```

#### Пример 4 — Псевдоним для вложенной структуры

Полезно для избавления от повторяющейся глубокой индексации в плотном блоке кода.

```nim
import std/decls

var grid = @[@[1, 2, 3], @[4, 5, 6]]

var cell {.byaddr.} = grid[1][2]   # псевдоним для элемента со значением 6
cell = 999
assert grid[1][2] == 999
```

#### Пример 5 — Псевдоним с явной аннотацией типа

Необязательная аннотация типа полезна для удобочитаемости или когда нужно, чтобы компилятор проверил ожидаемый тип на месте объявления.

```nim
import std/decls

var values = [10.0, 20.0, 30.0]
var first {.byaddr.}: float = values[0]

first = 0.5
assert values[0] == 0.5
```

#### Пример 6 — Избавление от повторяющейся индексации в теле цикла

Распространённое реальное применение: создать короткий псевдоним для глубоко вложенного элемента в начале тела цикла, чтобы сохранить читаемость остальной части тела.

```nim
import std/decls

type Config = object
  thresholds: seq[int]

var cfg = Config(thresholds: @[10, 20, 30, 40])

for i in 0 ..< cfg.thresholds.len:
  var t {.byaddr.} = cfg.thresholds[i]
  t = t * 2   # изменяет cfg.thresholds[i] напрямую

assert cfg.thresholds == @[20, 40, 60, 80]
```

## Как `byaddr` работает внутри

Понимание генерируемого кода объясняет как мощь макроса, так и его ограничения.

Когда вы пишете:
```nim
var a {.byaddr.} = s[0]
```

Макрос раскрывает это приблизительно в:
```nim
let tmp: ptr typeof(s[0]) = addr(s[0])
template a: untyped = tmp[]
```

Происходят две вещи:

1. **Захватывается указатель.** `addr(s[0])` берёт адрес выражения и сохраняет его в неизменяемой привязке `let` с именем `tmp`. Выражение вычисляется здесь ровно один раз. Тип `tmp` — это `ptr T`, где `T` — тип элемента.

2. **Шаблон становится псевдонимом.** Определяется беспараметрический шаблон с именем `a`, который раскрывается в `tmp[]` (разыменование указателя) каждый раз, когда `a` появляется в коде. Поскольку это шаблон, а не переменная, он встраивается в каждом месте использования, делая чтение и запись прозрачно проходящими через указатель.

Когда вы позднее пишете `a += 100`, компилятор видит `tmp[] += 100` — прямую запись в исходную память. Когда вы пишете `a.addr`, компилятор видит `addr(tmp[])`, что идентично `addr(s[0])`.

Форма с явным типом `var b {.byaddr.}: int = s[0]` генерирует:
```nim
let tmp: ptr int = addr(s[0])
template b: untyped = tmp[]
```

Хитрость в том, что снаружи `a` выглядит и ведёт себя в точности как обычная переменная типа `int` — компилятор нигде не "протекает" тип указателя через интерфейс шаблона.

## Ограничения и стабильность

### ⚠️ Экспериментальные функции

`byaddr` построен на двух функциях Nim, которые сами помечены как экспериментальные:

- **Беспараметрические шаблоны, инстанцированные как символы** — шаблоны с нулём аргументов, которые выглядят и ведут себя как имена переменных.
- **Макросы-прагмы переменных** — возможность использовать макрос как прагму на объявлении переменной.

Из-за этого поведение `byaddr` **не гарантируется стабильным** между версиями Nim. На практике он работает надёжно, но при решении, использовать ли его в долгоживущем или широко распространяемом коде, следует принимать во внимание зависимость от экспериментальных внутренностей.

### Непреднамеренное переопределение

Текущая реализация допускает объявление одного и того же имени с `{.byaddr.}` более одного раза в одной области видимости. Это непреднамеренный побочный эффект реализации на основе шаблонов, а не намеренная функция. Не полагайтесь на это.

### r-value не поддерживаются

`byaddr` требует **l-value** в правой части — выражения, имеющего стабильный адрес в памяти. Передача временного значения или r-value (результата функции, возвращающей по значению, литерала, арифметического выражения) завершится ошибкой на вызове `addr()`:

```nim
# ❌ Не скомпилируется — целочисленный литерал не имеет адреса
var x {.byaddr.} = 42

# ❌ Не скомпилируется — результат функции является временным
var y {.byaddr.} = someProc()

# ✅ Работает — элемент массива является l-value
var arr = [1, 2, 3]
var z {.byaddr.} = arr[0]
```

### Не ссылка на уровне языка

Псевдоним, создаваемый `byaddr`, является шаблоном, а не полноценным ссылочным типом. Его нельзя хранить в структуре данных, возвращать из функции или передавать как параметр `var`. Его область видимости строго лексическая — он существует только в блоке, где объявлен.

### Только одно объявление

Макрос ожидает ровно одно объявление переменной на вызов прагмы. Блоки `var` с несколькими именами и единственным `{.byaddr.}` не поддерживаются.

## Практические паттерны

### Паттерн 1 — Чистая мутация на месте по сложному пути

```nim
import std/decls

type Matrix = seq[seq[float]]

proc normalize(m: var Matrix, row, col: int) =
  var cell {.byaddr.} = m[row][col]
  if cell < 0.0: cell = 0.0
  if cell > 1.0: cell = 1.0
  # Не нужно каждый раз писать m[row][col]
```

### Паттерн 2 — Обмен значений через псевдонимы

```nim
import std/decls

var data = @[5, 3, 8, 1]

var a {.byaddr.} = data[0]
var b {.byaddr.} = data[2]

if a > b:
  let tmp = a   # копирует значение int, а не псевдоним
  a = b
  b = tmp
```

### Паттерн 3 — Читаемый доступ к глубоко вложенной конфигурации

```nim
import std/decls

type
  Server = object
    host: string
    port: int
  AppConfig = object
    server: Server

var config = AppConfig(server: Server(host: "localhost", port: 8080))

block:
  var host {.byaddr.} = config.server.host
  var port {.byaddr.} = config.server.port

  host = "0.0.0.0"
  port = 443

assert config.server.host == "0.0.0.0"
assert config.server.port == 443
```
