# parsecsv — справочник по модулю

> **Стандартная библиотека Nim** | `import std/parsecsv`

Простой высокопроизводительный парсер формата CSV (значения, разделённые запятыми). Модуль читает CSV-данные строка за строкой, возвращая каждую строку в виде последовательности строк. Парсер работает потоково — файл не загружается в память целиком, что делает его пригодным для больших объёмов данных.

## Типы

### `CsvRow`

```nim
type CsvRow* = seq[string]
```

Псевдоним типа для `seq[string]`. Представляет одну разобранную строку CSV-файла — упорядоченный список строковых полей. Каждый элемент соответствует значению одной ячейки (одного столбца) в этой строке.

---

### `CsvParser`

```nim
type CsvParser* = object of BaseLexer
  row*: CsvRow
  headers*: seq[string]
```

Центральный объект модуля. Хранит внутреннее состояние парсера и предоставляет два публичных поля:

- **`row`** — последняя успешно прочитанная строка в виде `CsvRow` (`seq[string]`). Обновляется при каждом вызове `readRow`.
- **`headers`** — список имён столбцов, прочитанных с помощью `readHeaderRow`. Используется как таблица поиска при обращении к `rowEntry`.

Перед использованием объект `CsvParser` необходимо инициализировать одной из процедур `open`, а после завершения работы — освободить вызовом `close`.

---

### `CsvError`

```nim
type CsvError* = object of IOError
```

Исключение, которое выбрасывается при ошибке разбора CSV (например, неожиданный символ вместо разделителя или незакрытое кавычками поле). Наследует `IOError`. Сообщение об ошибке включает имя файла, номер строки и столбца (если они доступны).

## Процедуры

---

### `open` (из потока)

```nim
proc open*(self: var CsvParser, input: Stream, filename: string,
           separator = ',', quote = '"', escape = '\0',
           skipInitialSpace = false)
```

Инициализирует парсер для чтения из готового объекта `Stream`. Параметр `filename` используется исключительно для формирования читаемых сообщений об ошибках — он не влияет на открытие файла.

**Параметры:**

| Параметр | По умолчанию | Описание |
|---|---|---|
| `separator` | `','` | Символ-разделитель полей внутри строки. |
| `quote` | `'"'` | Символ, которым оборачиваются поля, содержащие спецсимволы (разделитель, перенос строки и т.д.). Значение `'\0'` отключает обработку кавычек. |
| `escape` | `'\0'` | Если задан ненулевой символ (например, `'\\'`), он используется как префикс экранирования: следующий за ним символ трактуется буквально. Если `'\0'` и кавычки включены, два подряд идущих символа кавычки означают одну кавычку в значении (формат RFC 4180). |
| `skipInitialSpace` | `false` | Если `true`, пробелы сразу после разделителя игнорируются. Удобно для CSV вида `a, b, c`. |

**Когда использовать:** Когда у вас уже есть объект `Stream` — например, `StringStream` для данных в памяти или поток из сетевого источника.

```nim
import std/[parsecsv, streams]

var strm = newStringStream("имя,возраст\nАлиса,30\nБоб,25")
var p: CsvParser
p.open(strm, "данные.csv")

while p.readRow():
  echo p.row[0], " — ", p.row[1], " лет"

p.close()
strm.close()
# Вывод:
# имя — возраст лет
# Алиса — 30 лет
# Боб — 25 лет
```

### `open` (из файла)

```nim
proc open*(self: var CsvParser, filename: string,
           separator = ',', quote = '"', escape = '\0',
           skipInitialSpace = false)
```

Удобная перегрузка, которая самостоятельно открывает файл по переданному пути и создаёт поток внутри. Все остальные параметры работают идентично потоковой версии `open`.

**Когда использовать:** Самый распространённый случай — есть путь к CSV-файлу, и нужно просто начать чтение.

```nim
import std/parsecsv

var p: CsvParser
p.open("сотрудники.csv", separator = ';')

while p.readRow():
  for field in p.row:
    stdout.write(field & "\t")
  echo ""

p.close()
```

**Важно:** Если файл не удаётся открыть, немедленно выбрасывается `CsvError`.

---

### `close`

```nim
proc close*(self: var CsvParser) {.inline.}
```

Закрывает парсер и освобождает связанный входной поток. Всегда вызывайте `close` по завершении работы с данными — это освобождает дескриптор файла и внутренний буфер.

```nim
var p: CsvParser
p.open("данные.csv")
# ... читаем данные ...
p.close()  # ← не забыть
```

**Примечание:** После `close` объект `CsvParser` не следует использовать без повторного вызова `open`.

### `readRow`

```nim
proc readRow*(self: var CsvParser, columns = 0): bool
```

Читает и разбирает следующую непустую строку из входного потока. Заполняет `self.row` полями этой строки и возвращает `true`. Возвращает `false`, когда данные закончились (достигнут конец файла). **Пустые строки в файле автоматически пропускаются.**

**Параметр `columns`:** Если больше 0, парсер проверяет, что каждая строка содержит ровно столько столбцов. При несовпадении выбрасывается `CsvError`. Полезно для строгой валидации формата.

```nim
import std/[parsecsv, streams]

# В середине CSV — пустая строка, она автоматически пропускается
var strm = newStringStream("1,2,3\n\n4,5,6")
var p: CsvParser
p.open(strm, "test.csv")

while p.readRow():
  echo p.row  # выведет @["1","2","3"], затем @["4","5","6"]

p.close()
strm.close()
```

**Строгий контроль числа столбцов:**

```nim
# Выбросит CsvError, если в какой-то строке не ровно 3 столбца
while p.readRow(columns = 3):
  обработать(p.row)
```

### `readHeaderRow`

```nim
proc readHeaderRow*(self: var CsvParser)
```

Читает первую строку файла и сохраняет её содержимое в `self.headers`. После вызова `self.row` тоже содержит значения заголовка. Последующие вызовы `readRow` будут переходить к строкам данных, тогда как `self.headers` остаётся неизменным — выступая постоянным реестром имён столбцов.

Эта процедура нужна для именованного доступа к столбцам через `rowEntry`. Без предварительного вызова `readHeaderRow` функция `rowEntry` работать корректно не будет.

```nim
import std/[parsecsv, streams]

var strm = newStringStream("город,страна,население\nБерлин,Германия,3600000\nТокио,Япония,13960000")
var p: CsvParser
p.open(strm, "города.csv")

p.readHeaderRow()
echo p.headers  # @["город", "страна", "население"]

while p.readRow():
  echo p.rowEntry("город"), " (", p.rowEntry("страна"), ")"

p.close()
strm.close()
# Вывод:
# Берлин (Германия)
# Токио (Япония)
```

### `rowEntry`

```nim
proc rowEntry*(self: var CsvParser, entry: string): var string
```

Возвращает ссылку на значение в текущей строке (`self.row`), соответствующее столбцу с именем `entry` из `self.headers`. Это основной способ обращаться к полям строки по имени, а не по числовому индексу.

**Требование:** До использования этой процедуры необходимо вызвать `readHeaderRow`. Иначе `self.headers` пуст, и любое обращение вызовет `KeyError`.

**Выбрасывает:** `KeyError`, если `entry` не совпадает ни с одним именем в `self.headers`.

Поскольку результат имеет тип `var string` (изменяемая ссылка), значение можно и присваивать — хотя это требуется редко.

```nim
import std/[parsecsv, streams]

let csv = "товар,цена,количество\nЯблоко,0.99,100\nБанан,0.45,200"
var strm = newStringStream(csv)
var p: CsvParser
p.open(strm, "магазин.csv")
p.readHeaderRow()

while p.readRow():
  let название  = p.rowEntry("товар")
  let цена      = p.rowEntry("цена")
  let количество = p.rowEntry("количество")
  echo название, ": $", цена, " × ", количество, " шт."

p.close()
strm.close()
# Вывод:
# Яблоко: $0.99 × 100 шт.
# Банан: $0.45 × 200 шт.
```

**Обращение к несуществующему столбцу:**

```nim
try:
  echo p.rowEntry("скидка")
except KeyError as e:
  echo "Столбец не найден: ", e.msg
```

### `processedRows`

```nim
proc processedRows*(self: var CsvParser): int {.inline.}
```

Возвращает общее количество вызовов `readRow` — вне зависимости от того, вернул ли каждый вызов `true` (строка прочитана) или `false` (конец файла). Счётчик начинается с 0 и увеличивается на 1 при каждом вызове `readRow`, включая вызовы после исчерпания данных.

**Варианты применения:**
- Отслеживание прогресса при чтении больших файлов.
- Отладка и логирование (например, «ошибка на строке N»).
- Проверка, что файл содержит ожидаемое число строк данных.

```nim
import std/[parsecsv, streams]

var strm = newStringStream("a,b\n1,2\n3,4")
var p: CsvParser
p.open(strm, "test.csv")

while p.readRow():
  echo "Строка #", p.processedRows(), ": ", p.row

# После цикла readRow вернул false — счётчик ещё раз увеличился
echo "Итого вызовов readRow: ", p.processedRows()

p.close()
strm.close()
# Вывод:
# Строка #1: @["a", "b"]
# Строка #2: @["1", "2"]
# Строка #3: @["3", "4"]
# Итого вызовов readRow: 3
```

> **Важно:** `processedRows` считает *вызовы `readRow`*, а не строки с данными в файле. Пустые строки, которые пропускаются при разборе, в счётчик не входят. Однако каждый вызов `readRow`, вернувший `false` (конец файла), счётчик увеличивает.

## Полный пример использования

Пример демонстрирует полный рабочий цикл: открытие файла, чтение заголовков, итерация по строкам данных с доступом по именам столбцов и обработка ошибок.

```nim
import std/parsecsv

let content = """id,имя,оценка
1,Алиса,95
2,Боб,87
3,Карол,91
"""
writeFile("результаты.csv", content)

var p: CsvParser
p.open("результаты.csv")
p.readHeaderRow()

echo "Столбцы: ", p.headers

while p.readRow():
  echo p.rowEntry("имя"), " получила оценку ", p.rowEntry("оценка")

echo "Обработано строк: ", p.processedRows()
p.close()

# Вывод:
# Столбцы: @["id", "имя", "оценка"]
# Алиса получила оценку 95
# Боб получил оценку 87
# Карол получила оценку 91
# Обработано строк: 5
```

## Краткая сводка

| Процедура | Назначение |
|---|---|
| `open(parser, stream, filename, ...)` | Инициализация из готового потока |
| `open(parser, filename, ...)` | Инициализация напрямую из файла |
| `close(parser)` | Освобождение парсера и потока |
| `readRow(parser, columns?)` | Читает следующую непустую строку → `bool` |
| `readHeaderRow(parser)` | Читает первую строку и сохраняет её как `headers` |
| `rowEntry(parser, name)` | Получить значение текущей строки по имени столбца |
| `processedRows(parser)` | Получить общее число вызовов `readRow` |
