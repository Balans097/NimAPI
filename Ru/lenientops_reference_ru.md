# lenientops — справочник по модулю

## Обзор

Модуль `lenientops` предоставляет перегруженные версии стандартных бинарных операторов `+`, `-`, `*`, `/`, `<` и `<=`, которые принимают **смешанные операнды: целые числа и числа с плавающей точкой**. Без этого модуля строгая система типов Nim требует явного приведения одного из операндов перед выполнением такой операции — например, `float(x) + y`. После импорта `lenientops` подобные выражения работают напрямую.

### Основное правило

При совместном использовании целого числа и числа с плавающей точкой **целое число преобразуется к типу числа с плавающей точкой**. Результат всегда является числом с плавающей точкой — а именно тем типом с плавающей точкой, который уже присутствует в выражении. Неявного повышения из `float32` в `float64` (или наоборот) не происходит: тип числа с плавающей точкой сохраняется в точности.

```
int  op  float32  →  float32
int  op  float64  →  float64
float32  op  int  →  float32
float64  op  int  →  float64
```

### Почему это отдельный модуль?

По умолчанию Nim не выполняет неявных числовых преобразований. Преобразование целого числа в число с плавающей точкой может **незаметно потерять информацию**: 64-битное целое может содержать значения, которые 64-битное число с плавающей точкой не способно представить точно (целые числа, превышающие 2⁵³, не все представимы в `float64`). Поскольку такой компромисс не всегда приемлем, операторы не входят в стандартную область видимости библиотеки — вы подключаете их намеренно, явно импортируя `lenientops`.

### А как же `==`?

Оператор равенства `==` **намеренно не предоставляется**. Сравнение числа с плавающей точкой и целого числа на равенство неоднозначно: следует ли привести целое к float (рискуя потерять точность) или округлить float до целого? Правильный ответ зависит от контекста, поэтому явный выбор остаётся за вами.

### А как же `>=` и `>`?

Эти операторы тоже **не предоставляются**. `system.nim` уже определяет шаблоны `>=` и `>` с сигнатурой `(x, y: untyped): untyped`, и добавление перегрузок здесь привело бы к ошибкам неоднозначного вызова. На практике `a > b` эквивалентно `b < a`, а `a >= b` — `b <= a`, поэтому сравнения можно переписать через доступные `<` и `<=`.

## Экспортируемые операторы

Все операторы обобщённые по `I: SomeInteger` (любой целочисленный тип: `int`, `int8`, `int16`, `int32`, `int64`, `uint`, `uint8`, …) и `F: SomeFloat` (`float32` или `float64`). Каждый оператор — это `func` (чистая функция без побочных эффектов) с пометкой `inline` (нулевые накладные расходы на вызов).

Каждый оператор представлен в **двух перегрузках**: одна для порядка `(целое, float)`, другая для `(float, целое)`, охватывая оба возможных порядка аргументов.

---

### `+` — Сложение

```nim
func `+`*[I: SomeInteger, F: SomeFloat](i: I, f: F): F
func `+`*[I: SomeInteger, F: SomeFloat](f: F, i: I): F
```

Складывает целое число и число с плавающей точкой. Целое число сначала преобразуется к типу `F`, затем выполняется обычное сложение с плавающей точкой. Тип результата — `F`.

**Пример:**

```nim
import lenientops

let a = 3 + 1.5       # int + float64  →  4.5 : float64
let b = 2.0'f32 + 10  # float32 + int  →  12.0 : float32
let c = 7 + 0.5'f32   # int + float32  →  7.5 : float32

echo a  # 4.5
echo b  # 12.0
echo c  # 7.5
```

---

### `-` — Вычитание

```nim
func `-`*[I: SomeInteger, F: SomeFloat](i: I, f: F): F
func `-`*[I: SomeInteger, F: SomeFloat](f: F, i: I): F
```

Вычитает число с плавающей точкой из целого или целое из числа с плавающей точкой. Целое число преобразуется к `F` до выполнения операции; порядок операндов соблюдается в обычном смысле.

**Пример:**

```nim
import lenientops

let a = 10 - 3.5      # int - float64  →  6.5 : float64
let b = 10.0 - 3      # float64 - int  →  7.0 : float64
let c = 1 - 1.5'f32   # int - float32  →  -0.5 : float32

echo a  # 6.5
echo b  # 7.0
echo c  # -0.5
```

---

### `*` — Умножение

```nim
func `*`*[I: SomeInteger, F: SomeFloat](i: I, f: F): F
func `*`*[I: SomeInteger, F: SomeFloat](f: F, i: I): F
```

Умножает целое число на число с плавающей точкой. Поскольку умножение коммутативно, обе перегрузки дают одинаковый числовой результат независимо от порядка аргументов, однако тип результата всегда `F`.

**Пример:**

```nim
import lenientops

let a = 4 * 2.5        # int * float64  →  10.0 : float64
let b = 0.1'f32 * 3    # float32 * int  →  0.3 : float32
let c = 100 * 0.01     # int * float64  →  1.0 : float64

echo a  # 10.0
echo b  # 0.3
echo c  # 1.0
```

---

### `/` — Деление

```nim
func `/`*[I: SomeInteger, F: SomeFloat](i: I, f: F): F
func `/`*[I: SomeInteger, F: SomeFloat](f: F, i: I): F
```

Делит целое число на число с плавающей точкой или число с плавающей точкой на целое. Целое преобразуется к `F` до выполнения операции, после чего выполняется стандартное деление с плавающей точкой. В отличие от целочисленного деления (`div`), результат не усекается — всегда возвращается вещественная дробь.

**Пример:**

```nim
import lenientops

let a = 7 / 2.0           # int / float64  →  3.5 : float64
let b = 1.0 / 4           # float64 / int  →  0.25 : float64
let c = 1'i32 / 3.0'f32   # int32 / float32  →  0.3333... : float32

echo a  # 3.5
echo b  # 0.25
echo c  # 0.33333334
```

---

### `<` — Строгое сравнение «меньше»

```nim
func `<`*[I: SomeInteger, F: SomeFloat](i: I, f: F): bool
func `<`*[I: SomeInteger, F: SomeFloat](f: F, i: I): bool
```

Возвращает `true`, если левый операнд строго меньше правого — после приведения целого к типу `F`. Поскольку преобразование больших целых чисел в float может потерять точность, результаты вблизи границ представимых float-значений следует интерпретировать с осторожностью.

**Пример:**

```nim
import lenientops

echo 3 < 3.5      # true  — int приводится к float64: 3.0 < 3.5
echo 4 < 3.5      # false — 4.0 < 3.5 ложно
echo 2.9 < 3      # true  — 2.9 < 3.0
echo 3.0 < 3      # false — 3.0 < 3.0 ложно (строгое неравенство)
```

---

### `<=` — Нестрогое сравнение «меньше или равно»

```nim
func `<=`*[I: SomeInteger, F: SomeFloat](i: I, f: F): bool
func `<=`*[I: SomeInteger, F: SomeFloat](f: F, i: I): bool
```

Возвращает `true`, если левый операнд меньше правого или равен ему — после приведения целого к типу `F`. Это нестрогий вариант оператора `<`.

**Пример:**

```nim
import lenientops

echo 3 <= 3.0     # true  — 3.0 <= 3.0
echo 3 <= 2.9     # false — 3.0 <= 2.9 ложно
echo 2.9 <= 3     # true  — 2.9 <= 3.0
echo 3.0 <= 3     # true  — 3.0 <= 3.0
```

**Замена для `>=` и `>`:** поскольку эти операторы не экспортируются, используйте следующие эквиваленты:

```nim
import lenientops

let x = 5
let y = 4.5

# Вместо x > y пишите:
echo y < x     # true

# Вместо x >= y пишите:
echo y <= x    # true
```

## Предупреждение о точности

Преобразование целого числа в число с плавающей точкой **не всегда выполняется без потерь**. У `float64` 52-битная мантисса: он точно представляет все целые числа вплоть до 2⁵³ (9 007 199 254 740 992), однако большие значения `int64` могут быть округлены. У `float32` 23-битная мантисса, и он точно представляет целые числа лишь до 2²⁴ (16 777 216).

```nim
import lenientops

let big = 9_007_199_254_740_993'i64  # 2^53 + 1
echo big + 0.0  # может вывести 9007199254740992.0 — последний бит теряется
```

Именно поэтому модуль называется «lenient» («снисходительный»), а его документация предупреждает: *«используйте с осторожностью»*.

## Сводная таблица

| Оператор | Сигнатура | Возвращает | Примечание |
|---|---|---|---|
| `+` | `(int, float)` или `(float, int)` | `F` | Целое приводится к `F` |
| `-` | `(int, float)` или `(float, int)` | `F` | Порядок операндов соблюдается |
| `*` | `(int, float)` или `(float, int)` | `F` | Коммутативен |
| `/` | `(int, float)` или `(float, int)` | `F` | Вещественное деление, без усечения |
| `<` | `(int, float)` или `(float, int)` | `bool` | Строгое неравенство |
| `<=` | `(int, float)` или `(float, int)` | `bool` | Нестрогое неравенство |
| `==` | — | — | **Не предоставляется** — семантика неоднозначна |
| `>` / `>=` | — | — | **Не предоставляется** — конфликт с шаблонами `system.nim` |
