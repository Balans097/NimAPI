# logic.nim — полный справочник модуля

> **Область применения:** Этот модуль расширяет встроенный булев инструментарий Nim операторами формальной логики, предназначенными для использования внутри **контрактных прагм** — `.ensures`, `.requires`, `.invariant`. Это *не* обычные функции с возвратом значения во время выполнения — это специальные *magic*-конструкции, понимаемые компилятором Nim на уровне формальной верификации и спецификации. Использование этих операторов вне контрактного контекста является либо нейтральным, либо неопределённым.

## `->` — Логическая импликация

```nim
proc `->`*(a, b: bool): bool {.magic: "Implies".}
```

### Что это означает

Оператор импликации читается как: **«если `a`, то `b`»**. Он возвращает `false` только тогда, когда `a` истинно, а `b` ложно — то есть импликация нарушается исключительно в случае, когда предпосылка выполнена, но вывод не следует из неё.

| `a`   | `b`   | `a -> b` |
|-------|-------|----------|
| false | false | **true** |
| false | true  | **true** |
| true  | false | **false** |
| true  | true  | **true** |

Интуитивно: *обещание нарушается только тогда, когда у тебя была возможность его выполнить, но ты этого не сделал.*

### Когда применять

Используйте `->` внутри постусловий (`.ensures`) или инвариантов, когда одно условие должно **условно гарантировать** другое. Это позволяет выразить *условную* корректность: «только когда верно X, должно быть верно Y».

### Пример

```nim
proc safeDivide(a, b: int): int {.ensures: (b != 0) -> (result * b == a).} =
  if b == 0: 0 else: a div b
```

Постусловие утверждает: *если делитель не равен нулю, то результат, умноженный на `b`, должен равняться `a`*. Когда `b` равно нулю, никакого обещания о результате мы не даём — предпосылка импликации ложна, и она выполняется автоматически.

## `<->` — Логическая эквивалентность

```nim
proc `<->`*(a, b: bool): bool {.magic: "Iff".}
```

### Что это означает

Биконди-циональный оператор читается как: **«`a` тогда и только тогда, когда `b`»**. Оба операнда должны иметь одинаковое значение истинности, чтобы выражение было `true`. Это симметричная, двунаправленная версия импликации: `(a -> b) and (b -> a)`.

| `a`   | `b`   | `a <-> b` |
|-------|-------|-----------|
| false | false | **true**  |
| false | true  | **false** |
| true  | false | **false** |
| true  | true  | **true**  |

### Когда применять

Используйте `<->`, когда два свойства *эквивалентны по определению* — всякий раз, когда выполняется одно, обязательно выполняется и другое, без исключений в любую сторону. Часто встречается в инвариантах, связывающих два флага или состояния воедино.

### Пример

```nim
type Stack[T] = object
  data: seq[T]
  isEmpty: bool

proc push[T](s: var Stack[T]; val: T)
    {.ensures: s.isEmpty <-> (s.data.len == 0).} =
  s.data.add(val)
  s.isEmpty = s.data.len == 0
```

Постусловие утверждает, что флаг `isEmpty` и фактическая пустота `data` *всегда синхронизированы* — ни одно из них не может быть истинным без другого.

## `forall` — Квантор всеобщности

```nim
proc forall*(args: varargs[untyped]): bool {.magic: "Forall".}
```

### Что это означает

`forall` выражает математический квантор **∀ (для всех)**. Он утверждает, что некоторое свойство выполняется для *каждого* элемента коллекции или диапазона. Воспринимайте это как спецификационный аналог `all()`, но работающий символически на уровне верификатора, а не итерирующий по данным во время выполнения.

Стандартный синтаксис вызова отражает математическую нотацию:

```nim
forall(x, коллекция, предикат(x))
```

- **`x`** — связанная переменная (имя выбирается автором)
- **`коллекция`** — область значений для квантификации
- **`предикат(x)`** — булево выражение, которое должно быть истинным для каждого `x`

### Когда применять

Используйте `forall` в прагмах `.ensures` или `.invariant`, когда нужно гарантировать что-либо о *всех* элементах структуры данных — например, каждый элемент положителен, сохраняется инвариант сортировки, нет нулевых указателей.

### Пример

```nim
proc sortAscending(arr: var seq[int])
    {.ensures: forall(i, 0..<arr.len-1, arr[i] <= arr[i+1]).} =
  arr.sort()
```

Постусловие читается так: *для всех допустимых индексов `i` элемент с индексом `i` меньше или равен элементу с индексом `i+1`* — это формальное и полное утверждение об инварианте сортировки по возрастанию.

## `exists` — Квантор существования

```nim
proc exists*(args: varargs[untyped]): bool {.magic: "Exists".}
```

### Что это означает

`exists` выражает математический квантор **∃ (существует)**. Он утверждает, что *хотя бы один* элемент коллекции удовлетворяет заданному свойству. Это двойственный по отношению к `forall` оператор: там, где `forall` требует универсальной истинности, `exists` требует лишь единственного «свидетеля».

```nim
exists(x, коллекция, предикат(x))
```

### Когда применять

Используйте `exists`, когда гарантия корректности функции носит *экзистенциальный* характер — функция должна произвести или оставить после себя хотя бы один элемент с определённым свойством, не уточняя, какой именно и сколько.

### Пример

```nim
proc findPositive(arr: seq[int]): int
    {.requires: exists(x, arr, x > 0),
      ensures:  result > 0.} =
  for v in arr:
    if v > 0: return v
  0
```

Предусловие использует `exists`, чтобы потребовать от вызывающего кода передавать массив, в котором есть хотя бы один положительный элемент. Постусловие затем гарантирует, что возвращённое значение положительно.

## `old` — Захват значения до вызова

```nim
proc old*[T](x: T): T {.magic: "Old".}
```

### Что это означает

`old(x)` захватывает **значение `x` в момент вызова процедуры** — до любых изменений, произведённых телом процедуры. Это незаменимо в постусловиях, где нужно соотнести *финальное* состояние с *начальным*. Без `old` постусловие не имеет возможности обратиться к тому, каким было изменяемое значение до вызова.

Параметр типа `[T]` означает, что `old` работает с любым типом: целые числа, строки, последовательности, объекты — что угодно.

### Когда применять

Используйте `old` исключительно внутри прагм `.ensures`, когда корректность процедуры с побочными эффектами выражается как *изменение* или *соотношение* состояний «до» и «после». Если ваше постусловие говорит только о результате или о финальном состоянии само по себе — `old` не нужен. Но как только вы говорите «список вырос на один элемент» или «счётчик увеличился», вам нужно зафиксировать состояние «до» с помощью `old`.

### Пример

```nim
proc appendItem[T](s: var seq[T]; item: T)
    {.ensures: s.len == old(s.len) + 1 and s[^1] == item.} =
  s.add(item)
```

Постусловие даёт два обещания:
1. Последовательность ровно на один элемент длиннее, чем была до вызова (`old(s.len) + 1`).
2. Последний элемент — это именно тот элемент, который был добавлен.

Без `old(s.len)` не было бы возможности выразить «на один длиннее, чем до вызова» — только «некоторая длина».

## Сводная таблица

| Оператор / Процедура | Формальный смысл | Типичная прагма |
|----------------------|------------------|-----------------|
| `a -> b`             | Если `a`, то `b` (импликация) | `.ensures`, `.invariant` |
| `a <-> b`            | `a` тогда и только тогда, когда `b` | `.ensures`, `.invariant` |
| `forall(x, S, P(x))` | ∀ x ∈ S: P(x) | `.ensures`, `.requires` |
| `exists(x, S, P(x))` | ∃ x ∈ S: P(x) | `.ensures`, `.requires` |
| `old(x)`             | Значение `x` до вызова | только `.ensures` |

---

> **Примечание:** Все пять конструкций являются компиляторной *магией* — у них нет обычной реализации для времени выполнения. Они существуют для того, чтобы инструментарий формальной спецификации и верификации Nim мог рассуждать о корректности программ. Рассматривайте этот модуль как **язык спецификаций, встроенный в Nim**, а не как библиотеку утилит времени выполнения.
