# Справочник модуля `appdirs`

> **Назначение модуля:** `appdirs` предоставляет кросс-платформенные вспомогательные функции для определения стандартных системных директорий, которые приложения используют для хранения данных, конфигурации, кэша и временных файлов. Вместо того чтобы жёстко прописывать пути вроде `~/.config` или `C:\Users\Alice\AppData`, вы спрашиваете у модуля — и он сам разбирается, где правильное место для текущей ОС и пользователя.

## `getHomeDir`

```nim
proc getHomeDir*(): Path
```

### Что делает

Возвращает **домашнюю директорию** текущего вошедшего в систему пользователя. Это личная корневая папка, в которой ОС хранит всё, что принадлежит одной учётной записи.

Функция намеренно является низкоуровневым строительным блоком. Для обработки путей, приходящих из пользовательских конфигурационных файлов (например, путей, начинающихся с `~`), существует более высокоуровневая `expandTilde`, которая оборачивает `getHomeDir`.

### Поведение на разных платформах

| ОС | Типичный результат |
|----|--------------------|
| Linux / BSD | `/home/alice/` |
| macOS | `/Users/alice/` |
| Windows | `C:\Users\Alice\` |

### Когда использовать

Используйте `getHomeDir`, когда вам нужно построить пользовательский путь, который не вписывается ни в одну из более специализированных категорий (данные, конфиг, кэш, временные файлы). Например, для создания dotfile прямо в домашней директории — хотя это считается устаревшей практикой; для конфигурации предпочтительнее `getConfigDir`.

### Пример

```nim
import std/appdirs
import std/paths

let home = getHomeDir()
echo home                          # /home/alice/

let dotfile = home / Path(".myapp")
echo dotfile                       # /home/alice/.myapp
```

## `getDataDir`

```nim
proc getDataDir*(): Path
```

### Что делает

Возвращает директорию, в которой приложения должны хранить **постоянные пользовательские данные** — то, что принадлежит пользователю и должно сохраняться между сессиями, но не является конфигурацией (например, локальная база данных, загруженные ресурсы, сохранения игры).

На Linux и BSD эта функция следует **спецификации XDG Base Directory**: сначала проверяется переменная окружения `XDG_DATA_HOME`. Если она не установлена, используется `~/.local/share`. Это означает, что опытные пользователи могут перенаправить хранилище данных, просто установив переменную окружения — без изменений в коде.

### Поведение на разных платформах

| ОС | Проверяемая переменная | Запасной вариант |
|----|------------------------|------------------|
| Linux / BSD | `XDG_DATA_HOME` | `~/.local/share/` |
| macOS | `XDG_DATA_HOME` | `~/Library/Application Support/` |
| Windows | — | `%APPDATA%\` |

### Когда использовать

Выбирайте `getDataDir` для всего, что пользователь не хотел бы потерять: локальная БД, импортированные файлы, сохранения игры, офлайн-контент (когда он является основным, а не временным). Не используйте для временных файлов или машинных кэшей, которые можно восстановить.

### Пример

```nim
import std/appdirs
import std/paths
import std/os

let dataRoot = getDataDir()
let appData  = dataRoot / Path("myapp")

createDir(appData)                          # создаём директорию, если её нет
let db = appData / Path("storage.db")
echo "База данных находится по пути: ", db
# Linux:   /home/alice/.local/share/myapp/storage.db
# macOS:   /Users/alice/Library/Application Support/myapp/storage.db
# Windows: C:\Users\Alice\AppData\Roaming\myapp\storage.db
```

## `getConfigDir`

```nim
proc getConfigDir*(): Path
```

### Что делает

Возвращает директорию, в которой приложения должны хранить **пользовательские конфигурационные файлы** — настройки, предпочтения, привязки клавиш и тому подобное. Путь привязан к пользователю: у каждой учётной записи на машине своя конфигурация.

На Linux и BSD функция соответствует спецификации XDG Base Directory, сначала проверяя `XDG_CONFIG_HOME` и по умолчанию используя `~/.config/`. На macOS также проверяется XDG-переменная, и тоже используется `~/.config/`, поскольку широко распространённого Apple-нативного эквивалента нет.

> **Завершающий разделитель:** Возвращаемый путь всегда заканчивается разделителем директорий (`/` на POSIX, `\` на Windows). Учитывайте это при ручном конкатенировании путей; оператор `/` из `std/paths` обрабатывает это корректно на любой платформе.

### Поведение на разных платформах

| ОС | Проверяемая переменная | Запасной вариант |
|----|------------------------|------------------|
| Linux / BSD | `XDG_CONFIG_HOME` | `~/.config/` |
| macOS | `XDG_CONFIG_HOME` | `~/.config/` |
| Windows | — | `%APPDATA%\` |

### Когда использовать

Храните здесь всё, что пользователь намеренно редактирует, чтобы изменить поведение вашего приложения: `config.toml`, `settings.json`, `keybindings.yaml` и т.д.

### Пример

```nim
import std/appdirs
import std/paths
import std/os

let cfgDir = getConfigDir() / Path("myapp")
createDir(cfgDir)

let settings = cfgDir / Path("settings.toml")
echo "Файл конфигурации: ", settings
# Linux:   /home/alice/.config/myapp/settings.toml
# Windows: C:\Users\Alice\AppData\Roaming\myapp\settings.toml
```

## `getCacheDir` (пользователь)

```nim
proc getCacheDir*(): Path
```

### Что делает

Возвращает директорию, в которой приложения должны хранить **кэш-данные** текущего пользователя. Кэш — это расходный материал: его можно удалить в любой момент, и приложение должно уметь заново сгенерировать или перезагрузить содержимое без потери данных.

Функция проверяет платформенные переменные окружения, уважая переопределения пользователя или системного администратора, а затем использует известные значения по умолчанию.

### Поведение на разных платформах

| ОС | Логика определения пути |
|----|------------------------|
| Linux / BSD | `XDG_CACHE_HOME` → `~/.cache/` |
| macOS | `XDG_CACHE_HOME` → `~/Library/Caches/` |
| Windows | `%LOCALAPPDATA%\` |

Обратите внимание: на Windows используется `LOCALAPPDATA` (а не `APPDATA`) — это намеренно, потому что кэши привязаны к конкретной машине и не должны синхронизироваться между устройствами в доменной среде.

### Когда использовать

Используйте для миниатюр изображений, скомпилированных шейдеров, кэшей сетевых ответов, предобработанных данных — всего, что ускоряет работу, но безопасно для удаления. Операционные системы и пользователи могут очищать директории кэша без предупреждения.

### Пример

```nim
import std/appdirs
import std/paths

let cacheRoot = getCacheDir()
echo cacheRoot
# Linux:   /home/alice/.cache/
# macOS:   /Users/alice/Library/Caches/
# Windows: C:\Users\Alice\AppData\Local\
```

## `getCacheDir` (приложение)

```nim
proc getCacheDir*(app: Path): Path
```

### Что делает

Удобная перегрузка, которая возвращает **поддиректорию кэша, привязанную к конкретному приложению**. Вы передаёте имя приложения (или любой идентификатор) как `app`, и функция автоматически строит полный путь.

На Windows добавляется дополнительный сегмент `"cache"` (потому что Windows не придерживается той же конвенции POSIX-систем с `~/.cache/<appname>`):

- **Windows:** `getCacheDir() / app / "cache"`
- **Все остальные платформы:** `getCacheDir() / app`

### Когда использовать

Это та перегрузка, которую вы будете использовать чаще всего. Вместо того чтобы вручную объединять `getCacheDir()` и имя приложения, этот единственный вызов делает всё за вас и прозрачно обрабатывает особенность Windows.

### Пример

```nim
import std/appdirs
import std/paths
import std/os

let myCache = getCacheDir(Path("myapp"))
createDir(myCache)

echo myCache
# Linux:   /home/alice/.cache/myapp
# macOS:   /Users/alice/Library/Caches/myapp
# Windows: C:\Users\Alice\AppData\Local\myapp\cache

let thumbnail = myCache / Path("thumb_42.png")
echo "Миниатюра сохраняется по пути: ", thumbnail
```

## `getTempDir`

```nim
proc getTempDir*(): Path
```

### Что делает

Возвращает **системную директорию для временных файлов** — место, куда приложения кладут файлы, нужные лишь в течение одного запуска или короткой операции. Временные файлы не гарантированно переживают перезагрузку и никогда не должны содержать важную для пользователя информацию.

Стратегия определения пути различается по платформам:

- **Windows:** вызывает нативную функцию WinAPI `GetTempPath`.
- **POSIX (Linux, macOS, BSD):** последовательно проверяет переменные окружения `TMPDIR`, `TEMP`, `TMP` и `TEMPDIR`.
- **Запасной вариант (все платформы):** возвращает `/tmp`, если ни один из вышеперечисленных вариантов не дал результата.

Результат можно переопределить **на этапе компиляции**, передав `-d:tempDir=mypath` компилятору Nim — удобно для sandbox-окружений сборки или детерминированного тестирования.

> **Важно:** `getTempDir` **не проверяет**, существует ли возвращаемый путь на диске. Всегда проверяйте или создавайте его перед записью файлов.

### Когда использовать

Используйте для промежуточных результатов вычислений, файлов-блокировок, распакованных архивов до перемещения в конечное место назначения — для любого файла, потеря которого при сбое приемлема. Никогда не храните здесь пользовательские данные.

### Пример

```nim
import std/appdirs
import std/paths
import std/os

let tmp = getTempDir()
echo tmp
# Linux:   /tmp/
# macOS:   /tmp/   (или значение $TMPDIR, например /var/folders/.../T/)
# Windows: C:\Users\Alice\AppData\Local\Temp\

# Создаём уникально именованный временный файл
let workFile = tmp / Path("myapp_work_" & $getCurrentProcessId() & ".dat")
echo "Рабочий файл: ", workFile
```

## Сводная таблица по платформам

Краткий справочник, показывающий, что возвращает каждая функция на трёх основных платформах при настройках по умолчанию (без переопределения XDG-переменных) для пользователя `alice`.

| Функция | Linux | macOS | Windows |
|---|---|---|---|
| `getHomeDir()` | `/home/alice/` | `/Users/alice/` | `C:\Users\Alice\` |
| `getDataDir()` | `/home/alice/.local/share/` | `/Users/alice/Library/Application Support/` | `C:\Users\Alice\AppData\Roaming\` |
| `getConfigDir()` | `/home/alice/.config/` | `/home/alice/.config/` | `C:\Users\Alice\AppData\Roaming\` |
| `getCacheDir()` | `/home/alice/.cache/` | `/Users/alice/Library/Caches/` | `C:\Users\Alice\AppData\Local\` |
| `getCacheDir(Path("app"))` | `/home/alice/.cache/app` | `/Users/alice/Library/Caches/app` | `C:\Users\Alice\AppData\Local\app\cache` |
| `getTempDir()` | `/tmp/` | `/tmp/` (или `$TMPDIR`) | `C:\Users\Alice\AppData\Local\Temp\` |
