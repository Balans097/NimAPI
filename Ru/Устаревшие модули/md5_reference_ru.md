# md5 — Справочник по модулю

> **⚠️ Предупреждение об устаревании:** Этот модуль признан устаревшим в актуальных версиях Nim.
> Рекомендуется перейти на пакет `checksums`:
> ```
> nimble install checksums
> ```
> После этого используйте `import checksums/md5`.

---

## Обзор

Модуль `md5` предоставляет инструменты для вычисления **MD5-контрольных сумм** — фиксированных 128-битных (16-байтовых) «отпечатков» произвольных данных. MD5 — широко известная хеш-функция, определённая в [RFC 1321](https://www.ietf.org/rfc/rfc1321.txt).

**Ключевые свойства:**
- Одинаковые входные данные всегда дают одинаковый 128-битный дайджест.
- Даже минимальное изменение входных данных приводит к полностью другому дайджесту.
- Работает **на этапе компиляции** и в целевых средах **JavaScript**.
- MD5 **не является криптографически стойким** алгоритмом для защиты от коллизий и не должен применяться в задачах безопасности (хеширование паролей, цифровые подписи). При этом он по-прежнему пригоден для проверки целостности данных, контрольных сумм и совместимости с устаревшими системами.

---

## Экспортируемые типы

### `MD5Digest`

```nim
type MD5Digest* = array[0..15, uint8]
```

Представляет MD5-контрольную сумму в виде «сырого» массива из 16 байт. Это основной тип результата, возвращаемый функцией `toMD5`. Его можно преобразовать в читаемую шестнадцатеричную строку с помощью оператора `$` или сравнить два дайджеста оператором `==`.

---

### `MD5Context`

```nim
type MD5Context* {.final.} = object
  ...
```

Непрозрачный объект, хранящий **промежуточное состояние** инкрементального хеширования. Используется совместно с `md5Init`, `md5Update` и `md5Final`, когда данные поступают частями (например, при чтении большого файла по блокам). Внутренние поля объекта не предназначены для прямого использования.

---

## Экспортируемые процедуры

---

### `toMD5`

```nim
proc toMD5*(s: string): MD5Digest
```

**Самый простой способ вычислить MD5-контрольную сумму.** Принимает строку целиком и возвращает `MD5Digest`.

Внутри процедура самостоятельно создаёт `MD5Context` и управляет им — вам не нужно вручную вызывать `md5Init`, `md5Update` и `md5Final`.

**Пример:**

```nim
import md5

let digest = toMD5("hello, world")
echo digest          # вывод сырого массива
echo $digest         # e4d7f1b4ed2e42d15898f4b27b019da4
```

**Когда использовать:** когда данные целиком находятся в памяти в виде строки и нужно получить дайджест одним вызовом.

---

### `$` (MD5Digest в строку)

```nim
proc `$`*(d: MD5Digest): string
```

Преобразует `MD5Digest` (16 сырых байт) в привычную **32-символьную строку в нижнем регистре шестнадцатеричного формата**. Это стандартное человекочитаемое представление MD5-хеша.

**Пример:**

```nim
import md5

let digest = toMD5("abc")
echo $digest   # 900150983cd24fb0d6963f7d28e17f72
```

**Когда использовать:** всякий раз, когда нужно вывести, залогировать, сравнить в виде текста или сохранить MD5-значение как строку.

---

### `getMD5`

```nim
proc getMD5*(s: string): string
```

**Удобный однострочный вариант**: вычисляет MD5 строки и сразу возвращает результат в виде шестнадцатеричной строки. Это в точности то же самое, что `$toMD5(s)`, но избавляет от явного преобразования.

**Пример:**

```nim
import md5

echo getMD5("abc")          # 900150983cd24fb0d6963f7d28e17f72
echo getMD5("abc") ==
     getMD5("abc")          # true — одинаковые входные данные
echo getMD5("abc") ==
     getMD5("ABC")          # false — MD5 чувствителен к регистру
```

**Когда использовать:** когда контрольная сумма нужна сразу в виде строки, а данные уже полностью находятся в памяти.

---

### `==` (сравнение MD5Digest)

```nim
proc `==`*(D1, D2: MD5Digest): bool
```

Сравнивает два значения `MD5Digest` **побайтово** и возвращает `true`, только если все 16 байт совпадают. Это стандартный способ проверить, что две порции данных имеют одинаковую контрольную сумму.

**Пример:**

```nim
import md5

let a = toMD5("hello")
let b = toMD5("hello")
let c = toMD5("world")

echo a == b   # true  — одинаковые входные данные
echo a == c   # false — разные входные данные
```

**Когда использовать:** когда нужно сравнить два `MD5Digest` без промежуточного преобразования в строки. Это эффективнее, чем конвертировать оба значения и сравнивать строки.

---

### `md5Init`

```nim
proc md5Init*(c: var MD5Context) {.raises: [], tags: [], gcsafe.}
```

**Инициализирует `MD5Context`**, сбрасывая его в начальное состояние, установленное стандартом MD5. Это всегда должен быть первый вызов в рамках потокового API. Процедура записывает в четыре внутренних 32-битных слова состояния магические константы из спецификации MD5, а также обнуляет счётчики и внутренний буфер.

**Пример:**

```nim
import md5

var ctx: MD5Context
md5Init(ctx)   # ctx готов принимать данные
```

**Когда использовать:** всегда, как обязательный первый шаг при использовании низкоуровневого API `md5Init` / `md5Update` / `md5Final`. При использовании `toMD5` или `getMD5` вызов происходит автоматически.

---

### `md5Update` (вариант с openArray)

```nim
proc md5Update*(c: var MD5Context, input: openArray[uint8]) {.raises: [], tags: [], gcsafe.}
```

**Передаёт очередной фрагмент байт** в текущее MD5-вычисление. Контекст накапливает данные внутри, обрабатывая полные 64-байтовые блоки по мере их заполнения. Можно вызывать любое количество раз с фрагментами произвольного размера.

**Пример:**

```nim
import md5

var ctx: MD5Context
var digest: MD5Digest

md5Init(ctx)
md5Update(ctx, [0x61'u8, 0x62'u8, 0x63'u8])  # байты строки "abc"
md5Final(ctx, digest)

echo $digest   # 900150983cd24fb0d6963f7d28e17f72
```

**Когда использовать:** когда данные уже представлены в виде байтового массива, или при построении потокового решения с чтением двоичных данных по частям.

---

### `md5Update` (вариант с cstring)

```nim
proc md5Update*(c: var MD5Context, input: cstring, len: int) {.raises: [], tags: [], gcsafe.}
```

Перегрузка `md5Update`, принимающая `cstring` (указатель на символы в стиле C) и явную длину в байтах. Внутри делегирует работу варианту с `openArray[uint8]`, нарезая cstring в нужный срез. Полезна при взаимодействии с C-библиотеками или работе с «сырыми» строковыми указателями.

**Пример:**

```nim
import md5

var ctx: MD5Context
var digest: MD5Digest
let raw: cstring = "hello"

md5Init(ctx)
md5Update(ctx, raw, 5)   # явно передаём длину
md5Final(ctx, digest)

echo $digest   # 5d41402abc4b2a76b9719d911017c592
```

**Когда использовать:** когда у вас есть `cstring` и известна её длина — как правило, при работе с C FFI или низкоуровневыми буферами.

---

### `md5Final`

```nim
proc md5Final*(c: var MD5Context, digest: var MD5Digest) {.raises: [], tags: [], gcsafe.}
```

**Завершает MD5-вычисление** и записывает 16-байтовый результат в `digest`. Процедура применяет дополнение по стандарту MD5 (байт `0x80`, затем нули, затем общая длина данных в битах как 64-битное целое в порядке little-endian), выполняет последние раунды сжатия и извлекает финальное состояние в выходной массив. После вызова `md5Final` контекст обнуляется и не должен использоваться без нового вызова `md5Init`.

**Пример:**

```nim
import md5

var ctx: MD5Context
var digest: MD5Digest

md5Init(ctx)
md5Update(ctx, [0x61'u8])  # "a"
md5Update(ctx, [0x62'u8])  # "b" — два отдельных фрагмента
md5Update(ctx, [0x63'u8])  # "c"
md5Final(ctx, digest)

# Тот же результат, что и toMD5("abc") — порядок кусков не важен
echo $digest   # 900150983cd24fb0d6963f7d28e17f72
```

**Когда использовать:** как обязательный завершающий шаг потокового API, после того как все данные переданы через `md5Update`.

## Паттерны использования

### Паттерн 1 — Единовременный (самый распространённый)

Используйте этот паттерн, когда данные полностью доступны в виде строки.

```nim
import md5

# Сразу возвращает шестнадцатеричную строку
let checksum = getMD5("The quick brown fox jumps over the lazy dog")
echo checksum   # 9e107d9d372bb6826bd81d3542a419d6

# Возвращает MD5Digest — удобно для бинарного сравнения
let digest = toMD5("The quick brown fox jumps over the lazy dog")
if digest == toMD5("The quick brown fox jumps over the lazy dog"):
  echo "Данные не повреждены"
```

---

### Паттерн 2 — Потоковый (данные поступают частями)

Используйте этот паттерн, когда данные поступают фрагментами — например, при чтении файла блоками.

```nim
import md5

proc hashChunks(chunks: seq[string]): string =
  var ctx: MD5Context
  var digest: MD5Digest
  md5Init(ctx)
  for chunk in chunks:
    md5Update(ctx, chunk.toOpenArrayByte(0, chunk.high))
  md5Final(ctx, digest)
  return $digest

let parts = @["hel", "lo, ", "world"]
echo hashChunks(parts)
# То же, что getMD5("hello, world") — e4d7f1b4ed2e42d15898f4b27b019da4
```

---

### Паттерн 3 — Контрольная сумма на этапе компиляции

Поскольку модуль работает на этапе компиляции, контрольные суммы можно вычислять в блоках `static` или в выражениях `const`.

```nim
import md5

const compiledHash = getMD5("моя константа времени компиляции")
echo compiledHash
```

|---|---|---|---|
