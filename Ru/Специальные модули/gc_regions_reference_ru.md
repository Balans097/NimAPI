# Справочник модуля gc_regions

> **Модуль:** `gc_regions`  
> **Часть:** Стандартная библиотека Nim  
> **Назначение:** Реализует «стековый GC» — регион-базированный аллокатор с bump-указателем, предназначенный для встраиваемых систем, задач реального времени или любых сценариев, где требуется детерминированное управление памятью с минимальными задержками. Вместо трассировки объектов и сборки мусора память организована в чанки и освобождается вся сразу при уничтожении *региона*.

---

## Основная концепция

Этот модуль заменяет стандартный сборщик мусора Nim на **аллокатор регионов** (известный также как *арена* или *obstack*). Ментальная модель — это стек памяти:

```
 ┌──────────────────────────────────────────┐
 │  Чанк 1  │  Чанк 2  │  ... │  Чанк N    │  ← связный список
 └──────────────────────────────────────────┘
                                     ▲
                               bump-указатель
                      (здесь находится следующий свободный байт)
```

- Аллокация выполняется за **O(1)**: просто сдвигаем bump-указатель.
- Освобождение отдельных объектов **опционально** и не производит слияния блоков.
- Освобождение всего региона занимает **O(чанков)**: обход списка и возврат страниц ОС.
- Регионы являются **потокозависимыми** по умолчанию, но могут передаваться между потоками.

## Типы

### `MemRegion`

```nim
type MemRegion* = object
```

**Что это такое:** Центральная структура данных модуля. `MemRegion` содержит целую арену памяти: связный список чанков, выделенных ОС, текущий bump-указатель и учёт размеров. Создаётся простым `var r: MemRegion` — нулевая инициализация уже означает пустой регион.

**Ключевые внутренние поля:**
- `bump` / `remaining` — куда идёт следующая аллокация и сколько байт осталось в текущем чанке.
- `head` / `tail` — первый и последний чанки связного списка.
- `nextChunkSize` — размер, который будет запрошен у ОС для следующего чанка (удваивается каждый раз, нижняя граница — `PageSize * nimMinHeapPages`).
- `totalSize` — сумма размеров всех чанков, которыми сейчас владеет регион.
- `lock` — присутствует только при определённом `hasThreadSupport`; защищает межпотоковый доступ.

```nim
var arena: MemRegion   # начать с пустого региона; нулевая инициализация — валидное состояние
```

---

### `StackPtr`

```nim
type StackPtr = object
  bump: pointer
  remaining: int
  current: Chunk
```

**Что это такое:** Лёгкий снимок состояния аллокации региона — по сути, *сохранённая позиция bump-указателя*. Получается через `obstackPtr()`, восстанавливается через `setObstackPtr()`. Именно так реализуются scoped-, стекоподобные времена жизни памяти внутри региона.

---

## API потокозависимого региона

Потокозависимая переменная `tlRegion` — это *неявный* регион, который среда выполнения Nim использует для всех обычных вызовов `new` и `newSeq` при активном регионном GC. Перечисленные ниже процедуры работают с `tlRegion` без явной передачи региона в качестве параметра.

---

### `obstackPtr` (потокозависимый)

```nim
proc obstackPtr*(): StackPtr
```

**Что это такое:** Делает снимок текущей позиции аллокации в потокозависимом регионе. Вызовите эту функцию перед блоком временной работы; после блока вызовите `setObstackPtr` для освобождения всего аллоцированного между ними — за O(1) на каждый освобождённый чанк.

Это регион-аналог «сохранить указатель стека».

```nim
let saved = obstackPtr()
# ... выделяем множество временных объектов ...
setObstackPtr(saved)   # все эти объекты исчезли; память возвращена в пул
```

---

### `setObstackPtr` (потокозависимый)

```nim
proc setObstackPtr*(sp: StackPtr)
```

**Что это такое:** Восстанавливает потокозависимый регион до позиции, захваченной в `sp`, освобождая всю память, выделенную после этой точки. Финализаторы объектов, у которых они есть, вызываются до того, как память возвращается в пул.

```nim
let mark = obstackPtr()
let tmp = newString(1024)   # выделено в tlRegion
# ... используем tmp ...
setObstackPtr(mark)         # tmp исчез; его память снова в пуле
```

> **Важно:** После вызова `setObstackPtr` любые указатели на объекты, выделенные после `sp`, становятся висячими. Не используйте их.

---

### `deallocAll` (потокозависимый)

```nim
proc deallocAll*()
```

**Что это такое:** Освобождает **все** чанки потокозависимого региона и сбрасывает его в пустое состояние. Финализаторы вызываются для всех объектов, у которых они есть. Сам регион обнуляется и готов к повторному использованию.

```nim
# После этого tlRegion пуст; вся память возвращена ОС
deallocAll()
```

---

### `withScratchRegion`

```nim
template withScratchRegion*(body: untyped)
```

**Что это такое:** Шаблон, который на время выполнения `body` подставляет **новый пустой регион** в качестве потокозависимого, а затем уничтожает всю память, выделенную во время `body`, и восстанавливает предыдущий регион. Представьте это как scoped-блокнот для черновой работы — идеально для временных вычислений, которые не должны засорять основной регион.

```nim
withScratchRegion:
  let temp = newSeq[int](10_000)
  doSomeWork(temp)
  # temp и всё остальное, выделенное здесь, освобождается автоматически
# исходный tlRegion полностью восстановлен; никаких утечек и лишних объектов
```

> **Это самый высокоуровневый и безопасный API модуля.** Предпочитайте его ручным парам `obstackPtr`/`setObstackPtr` везде, где область видимости временных аллокаций совпадает с блоком кода.

## API явного региона

Эти процедуры принимают явный параметр `MemRegion`, позволяя управлять несколькими независимыми аренами одновременно.

---

### `withRegion`

```nim
template withRegion*(r: var MemRegion; body: untyped)
```

**Что это такое:** Временно делает `r` активным потокозависимым регионом на время выполнения `body`. Все аллокации среды выполнения Nim (`new`, `newSeq`, строковые операции), происходящие внутри `body`, идут в `r`. При выходе — нормальном или через исключение — предыдущий потокозависимый регион восстанавливается, а `r` обновляется с учётом произошедших аллокаций.

```nim
var myArena: MemRegion
withRegion(myArena):
  let obj = new(MyType)   # выделено в myArena, а не в tlRegion
  doWork(obj)
# myArena теперь владеет obj; tlRegion не изменился
```

---

### `obstackPtr` (явный регион)

```nim
proc obstackPtr*(r: MemRegion): StackPtr
```

**Что это такое:** То же, что потокозависимая версия, но для явного региона `r`. Возвращает снимок `StackPtr` текущего состояния аллокации `r`.

```nim
var arena: MemRegion
let mark = arena.obstackPtr()
```

---

### `setObstackPtr` (явный регион)

```nim
proc setObstackPtr*(r: var MemRegion; sp: StackPtr)
```

**Что это такое:** Откатывает регион `r` до состояния аллокации, захваченного в `sp`, освобождая и финализируя всё, выделенное после этой точки.

```nim
var arena: MemRegion
let mark = arena.obstackPtr()
withRegion(arena):
  let tmp = new(HeavyObject)
arena.setObstackPtr(mark)  # HeavyObject освобождён, финализатор вызван если есть
```

---

### `deallocAll` (явный регион)

```nim
proc deallocAll*(r: var MemRegion)
```

**Что это такое:** Освобождает всю память в регионе `r`, вызывает все ожидающие финализаторы и сбрасывает `r` в пустое состояние. После этого вызова `r` ведёт себя как только что объявленный.

```nim
var scratch: MemRegion
withRegion(scratch):
  # ... много работы ...
scratch.deallocAll()   # очищаем всё
```

---

### `isOnHeap`

```nim
proc isOnHeap*(r: MemRegion; p: pointer): bool
```

**Что это такое:** Возвращает `true`, если указатель `p` указывает на память, находящуюся внутри региона `r`. Полезно для проверок безопасности и отладки — например, чтобы убедиться, что полученный указатель действительно принадлежит той арене, которую вы ожидаете.

Реализация проверяет хвостовой чанк первым (самый быстрый путь, поскольку именно он содержит активный bump-указатель), затем обходит остальной список чанков.

```nim
var arena: MemRegion
withRegion(arena):
  let obj = new(MyType)
  assert arena.isOnHeap(cast[pointer](obj))
```

---

### `getOccupiedMem` (явный регион)

```nim
proc getOccupiedMem*(r: MemRegion): int
```

**Что это такое:** Возвращает количество байт, **занятых** в регионе `r`. Вычисляется как `totalSize - remaining` — то есть всё пространство выделенных чанков минус неиспользованный хвост последнего чанка.

```nim
echo "Занято: ", myArena.getOccupiedMem(), " байт"
```

---

### `getFreeMem` (явный регион)

```nim
proc getFreeMem*(r: MemRegion): int
```

**Что это такое:** Возвращает количество байт, ещё **доступных** в текущем (хвостовом) чанке, прежде чем у ОС будет запрошен новый чанк. Это просто `r.remaining` — сюда **не входит** пространство, которое появится в новом чанке.

```nim
echo "Свободно в текущем чанке: ", myArena.getFreeMem(), " байт"
```

---

### `getTotalMem` (явный регион)

```nim
proc getTotalMem*(r: MemRegion): int
```

**Что это такое:** Возвращает суммарный объём байт во **всех** чанках, которыми сейчас владеет регион `r`, — занятых и свободных.

```nim
echo "Общий размер арены: ", myArena.getTotalMem(), " байт"
```

## Статистика памяти

Это потокозависимые аналоги функций статистики из предыдущего раздела. Они запрашивают `tlRegion`.

### `getOccupiedMem`

```nim
proc getOccupiedMem(): int
```

**Что это такое:** Байты, занятые в потокозависимом регионе.

```nim
echo "Занято в TL-регионе: ", getOccupiedMem()
```

---

### `getFreeMem`

```nim
proc getFreeMem(): int
```

**Что это такое:** Байты, оставшиеся в текущем чанке потокозависимого региона.

---

### `getTotalMem`

```nim
proc getTotalMem(): int
```

**Что это такое:** Суммарный объём байт в потокозависимом регионе (занятые + свободные в хвостовом чанке).

```nim
let ratio = getOccupiedMem().float / getTotalMem().float
echo "Утилизация: ", ratio * 100.0, "%"
```

## Заглушки совместимости с GC

Поскольку этот модуль заменяет стандартный GC, он должен предоставлять пустые реализации всех процедур управления GC, которые может вызвать другой код Nim. Все они безопасны для вызова, но ничего не делают.

| Процедура | Сигнатура | Описание |
|---|---|---|
| `GC_disable` | `proc GC_disable()` | Заглушка. GC нечего отключать. |
| `GC_enable` | `proc GC_enable()` | Заглушка. |
| `GC_fullCollect` | `proc GC_fullCollect()` | Заглушка. Цикла сборки не существует. |
| `GC_setStrategy` | `proc GC_setStrategy(strategy: GC_Strategy)` | Заглушка. |
| `GC_enableMarkAndSweep` | `proc GC_enableMarkAndSweep()` | Заглушка. |
| `GC_disableMarkAndSweep` | `proc GC_disableMarkAndSweep()` | Заглушка. |
| `GC_getStatistics` | `proc GC_getStatistics(): string` | Всегда возвращает `""`. |
| `nimGC_setStackBottom` | `proc nimGC_setStackBottom(p: pointer)` | Заглушка. |
| `nimGCref` | `proc nimGCref(x: pointer)` | Заглушка. Суррогат подсчёта ссылок. |
| `nimGCunref` | `proc nimGCunref(x: pointer)` | Заглушка. Суррогат подсчёта ссылок. |

```nim
# Вызывать безопасно; в режиме регионов ничего не делает
GC_fullCollect()
GC_disable()
```

## Внутренние процедуры / Compiler-RT

Эти процедуры вызываются кодом, сгенерированным компилятором Nim, или внутренними механизмами среды выполнения. В пользовательском коде вы почти никогда не будете вызывать их напрямую, но понимание их работы помогает при чтении диагностики среды выполнения или написании низкоуровневого кода.

---

### `newObj`

```nim
proc newObj(typ: PNimType, size: int): pointer {.compilerRtl.}
```

**Что это такое:** Компилятор вызывает эту процедуру для выделения нового объекта, не являющегося последовательностью или строкой. Выделяет `size` байт в `tlRegion`, обнуляет память и регистрирует финализатор, если он есть у типа. Нельзя использовать для последовательностей и строк.

---

### `newObjNoInit`

```nim
proc newObjNoInit(typ: PNimType, size: int): pointer {.compilerRtl.}
```

**Что это такое:** Как `newObj`, но **пропускает вызов `zeroMem`**. Немного быстрее, когда вызывающий код всё равно инициализирует каждое поле. Используется компилятором, когда он может доказать, что все поля будут записаны перед чтением.

---

### `newSeq`

```nim
proc newSeq(typ: PNimType, len: int): pointer {.compilerRtl.}
```

**Что это такое:** Выделяет новую последовательность из `len` элементов в `tlRegion`. Выделенный блок обнуляется. Оба поля `len` и `reserved` устанавливаются равными `len`. Каждая последовательность несёт `SeqHeader` с обратным указателем на свой регион, что позволяет ей расти через `growObj`.

---

### `newStr`

```nim
proc newStr(typ: PNimType, len: int; init: bool): pointer {.compilerRtl.}
```

**Что это такое:** Выделяет новый строковый буфер из `len` байт в `tlRegion`. Если `init = true`, буфер обнуляется. Длина строки устанавливается в `0`, зарезервированная ёмкость — в `len`. Как и последовательности, строки хранят обратный указатель на свой регион.

---

### `growObj`

```nim
proc growObj(old: pointer, newsize: int): pointer {.rtl.}
```

**Что это такое:** Вызывается компилятором, когда последовательности или строке нужно вырасти сверх текущей ёмкости. Читает `SeqHeader` старого объекта, чтобы найти его регион, выделяет больший блок в том же регионе, копирует старые данные, обнуляет новый хвост и освобождает старый блок (используя откат bump-указателя, если возможно).

Ключевая идея: поскольку `SeqHeader` записывает исходный регион, последовательность может расти, даже если `tlRegion` изменился с момента её создания.

---

### `unsureAsgnRef` / `asgnRef` / `asgnRefNoCycle`

```nim
proc unsureAsgnRef(dest: PPointer, src: pointer) {.compilerproc, inline.}
proc asgnRef(dest: PPointer, src: pointer) {.compilerproc, inline.}
proc asgnRefNoCycle(dest: PPointer, src: pointer) {.compilerproc, inline, deprecated.}
```

**Что это такое:** Вспомогательные функции присваивания, которые компилятор вызывает при сохранении ссылки в слот указателя. В трассирующем GC они обновляли бы барьеры записи или счётчики ссылок. В режиме регионов они представляют собой тривиальные присваивания `dest[] = src` — отслеживать нечего.

`asgnRefNoCycle` устарела и существует только для обратной совместимости со старым скомпилированным кодом.

---

### `allocImpl` / `alloc0Impl` / `reallocImpl` / `realloc0Impl` / `deallocImpl`

```nim
proc allocImpl(size: Natural): pointer
proc alloc0Impl(size: Natural): pointer
proc reallocImpl(p: pointer, newsize: Natural): pointer
proc realloc0Impl(p: pointer, oldsize, newsize: Natural): pointer
proc deallocImpl(p: pointer)
```

**Что это такое:** Низкоуровневые реализации, стоящие за `alloc`, `alloc0`, `realloc`, `realloc0` и `dealloc` в системном модуле. Они полностью обходят регион и напрямую вызывают `c_malloc` / `c_realloc` / `c_free`. При неудаче аллокации выбрасывают `OutOfMemError`.

> **Примечание:** Эти функции предназначены для *системных* аллокаций (например, каналов, мьютексов), которые должны существовать вне любого региона. Для объектов приложения предпочитайте `withRegion` + обычные `new` / `newSeq`.

---

### `allocSharedImpl` / `allocShared0Impl` / `reallocSharedImpl` / `reallocShared0Impl` / `deallocSharedImpl`

```nim
proc allocSharedImpl(size: Natural): pointer
proc allocShared0Impl(size: Natural): pointer
proc reallocSharedImpl(p: pointer, newsize: Natural): pointer
proc reallocShared0Impl(p: pointer, oldsize, newsize: Natural): pointer
proc deallocSharedImpl(p: pointer)
```

**Что это такое:** Реализации семейства `allocShared` / `deallocShared`. В данном аллокаторе они идентичны своим несовместным аналогам — разделяемой кучи не существует; вся «разделяемая» память тоже проходит через `c_malloc`. Потокобезопасность должна обеспечиваться вызывающей стороной.
