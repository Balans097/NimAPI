# `ansi_c.nim` — справочник модуля

> **Часть стандартной библиотеки времени выполнения Nim**  
> Компилируется с `{.push hints:off, stack_trace:off, profiler:off, raises:[].}`

---

## Общее описание

`ansi_c.nim` — это **связующий слой между рантаймом Nim и стандартной библиотекой C**. Модуль импортирует тщательно отобранный набор C-функций — операции с памятью, строковые утилиты, примитивы ввода-вывода, обработку сигналов и выделение кучевой памяти — в пространство имён Nim под единым префиксом `c_`, чтобы избежать любых пересечений с именами стандартной библиотеки самого Nim.

Этот модуль **не предназначен для использования на уровне прикладного кода**. Он является фундаментом рантайма: сборщик мусора, механизм исключений, система ввода-вывода и обработчик аварийных завершений — все они используют эти обёртки. Понимание этого модуля даёт чёткое представление о том, что происходит «на уровень ниже» обычного Nim-кода.

Каждая процедура либо помечена `{.inline.}`, либо напрямую отображается на C-функцию, поэтому **никаких накладных расходов** сверх самой C-функции нет. Никаких трассировок стека, никаких хуков профилировщика, никаких аннотаций исключений — этот модуль намеренно существует вне всех этих механизмов.

---

## Прагмы уровня модуля

```nim
{.push hints:off, stack_trace:off, profiler:off, raises:[].}
```

Все символы в этом файле компилируются с активными прагмами:

- **`hints:off`** — подавляет подсказки компилятора Nim, сохраняя сборку рантайма чистой.
- **`stack_trace:off`** — эти процедуры не отображаются в трассировках стека Nim. Намеренно: они работают на уровне C и только засоряли бы отладочный вывод.
- **`profiler:off`** — исключены из профилировщика Nim. Сырые C-вызовы не имеет смысла профилировать на уровне Nim.
- **`raises:[]`** — компилятору сообщается, что эти процедуры не генерируют никаких исключений Nim. Это принципиально важно: рантайм использует их именно в тех местах, где генерация исключений невозможна или опасна (например, внутри GC или обработчика краша).

## Определения типов

### `C_JmpBuf*`

```nim
type C_JmpBuf* = ...  # зависит от платформы
```

Nim-обёртка для C-типа `jmp_buf` — буфера, который используется парой `setjmp`/`longjmp` для сохранения и восстановления состояния регистров процессора. Конкретное определение выбирается на этапе компиляции:

| Условие | Определение |
|---|---|
| `-d:nimBuiltinSetjmp` | `array[5, pointer]` — встроенный в компилятор, без C-заголовка |
| Linux + amd64 | `object` с полем `abi: array[200 div sizeof(clong), clong]` — точно соответствует ABI Linux x86-64 |
| Всё остальное | Непрозрачный `object`, импортированный из `<setjmp.h>` |

`C_JmpBuf` используется исключительно в `c_setjmp` и `c_longjmp`. Прикладному коду никогда не нужно работать с ним напрямую.

---

### `CFilePtr*`

```nim
type
  CFile {.importc: "FILE", header: "<stdio.h>", incompleteStruct.} = object
  CFilePtr* = ptr CFile
```

Указатель на C-структуру `FILE` — фундаментальный дескриптор для всего ввода-вывода на уровне C. Собственный тип `File` в Nim оборачивает именно его. `CFilePtr` присутствует во всех I/O-процедурах этого модуля.

## Глобальные переменные

### `cstdin*`, `cstdout*`, `cstderr*`

```nim
var cstderr* {.importc: stderrName, header: "<stdio.h>".}: CFilePtr
var cstdout* {.importc: stdoutName, header: "<stdio.h>".}: CFilePtr
var cstdin*  {.importc: stdinName,  header: "<stdio.h>".}: CFilePtr
```

Прямые Nim-привязки к трём стандартным потокам C. Имя фактического C-символа выбирается на этапе компиляции: `__stderrp`/`__stdoutp`/`__stdinp` на macOS/FreeBSD/DragonFly, где `stdio.h` использует макросы; `stderr`/`stdout`/`stdin` везде остальном.

Они используются по всему рантайму для низкоуровневого вывода — например, в обработчиках краша и вспомогательных функциях `rawWrite` — именно потому, что обходят весь высокоуровневый буферинг и механизм исключений Nim.

## Константы сигналов

Эти целочисленные константы идентифицируют POSIX/C-сигналы. На известных платформах они определены как константы времени компиляции; на неизвестных — импортируются из C-заголовков.

| Константа | Типичное значение | Значение |
|---|---|---|
| `SIGABRT*` | 6 (POSIX), 22 (Windows) | Аварийное завершение программы (`abort()`) |
| `SIGFPE*`  | 8 | Ошибка с плавающей точкой (деление на ноль, переполнение) |
| `SIGILL*`  | 4 | Недопустимая инструкция |
| `SIGINT*`  | 2 | Прерывание с клавиатуры (Ctrl+C) |
| `SIGSEGV*` | 11 | Ошибка сегментации (обращение к недопустимой памяти) |
| `SIGTERM*` | 15 | Запрос на завершение |
| `SIGPIPE*` | 13 (POSIX) / 7 (Haiku) | Обрыв канала (запись в закрытый сокет/пайп) |
| `SIGBUS*`  | 10 (macOS) / 30 (Haiku) | Ошибка шины (невыровненный доступ к памяти) |
| `SIG_DFL*` | 0 / nil | Восстановить обработчик сигнала по умолчанию |

> **Примечание:** `SIGTERM` не экспортируется на Windows (нет `*`). `SIGPIPE` и `SIGBUS` — только POSIX, на Windows недоступны.

## Процедуры работы с памятью

Это прямые обёртки C-функций из `<string.h>`. Все работают с «сырыми» аргументами типа `pointer` — без контроля типов, без проверки границ.

---

### `c_memchr`

```nim
proc c_memchr*(s: pointer, c: cint, n: csize_t): pointer
```

Ищет в первых `n` байтах, начиная с адреса `s`, **первое вхождение** байта со значением `c`. Возвращает указатель на найденный байт или `nil`, если байт не найден.

Это C-аналог побайтового сканирования. Рантайм использует его при поиске конкретного байта в сыром блоке памяти.

```nim
var buf = [byte(0), 1, 2, 42, 3]
let p = c_memchr(addr buf[0], 42, csize_t(buf.len))
# p указывает на buf[3], где находится значение 42
```

---

### `c_memcmp`

```nim
proc c_memcmp*(a, b: pointer, size: csize_t): cint {.noSideEffect.}
```

Сравнивает `size` байт памяти по адресу `a` с памятью по адресу `b` **побайтово** (беззнаковое сравнение). Возвращает:
- `0`, если области идентичны,
- отрицательное значение, если `a` лексикографически меньше `b`,
- положительное значение, если `a` больше.

Помечена `noSideEffect`, поскольку только читает память и даёт детерминированный результат. Используется в рантайме для сравнения сырых блоков данных — например, в определённых проверках барьеров записи GC и сравнениях дескрипторов типов.

```nim
var x = [byte(1), 2, 3]
var y = [byte(1), 2, 3]
assert c_memcmp(addr x[0], addr y[0], 3) == 0  # идентичны

var z = [byte(1), 2, 4]
assert c_memcmp(addr x[0], addr z[0], 3) < 0   # x < z
```

---

### `c_memcpy`

```nim
proc c_memcpy*(a, b: pointer, size: csize_t): pointer {.discardable.}
```

Копирует ровно `size` байт из источника `b` в приёмник `a`. Области источника и приёмника **не должны пересекаться** — для перекрывающихся областей используйте `c_memmove`. Возвращает `a`.

Возвращаемое значение помечено `discardable`, поскольку вызывающий код почти всегда его игнорирует. На современных процессорах компилируется в SIMD-оптимизированное копирование памяти.

```nim
var src = [byte(10), 20, 30]
var dst: array[3, byte]
c_memcpy(addr dst[0], addr src[0], 3)
# dst теперь содержит [10, 20, 30]
```

---

### `c_memmove`

```nim
proc c_memmove*(a, b: pointer, size: csize_t): pointer {.discardable.}
```

Копирует `size` байт из `b` в `a`, **корректно обрабатывая перекрывающиеся области**. Это безопасная альтернатива `c_memcpy`, когда источник и приёмник могут ссылаться на одну и ту же память. Чуть дороже `c_memcpy`, потому что должна определить направление копирования.

```nim
var buf = [byte(1), 2, 3, 4, 5]
# Сдвигаем элементы вправо на одну позицию (области перекрываются)
c_memmove(addr buf[1], addr buf[0], 4)
# buf теперь [1, 1, 2, 3, 4]
```

---

### `c_memset`

```nim
proc c_memset*(p: pointer, value: cint, size: csize_t): pointer {.discardable.}
```

Заполняет первые `size` байт памяти, начиная с `p`, значением байта `value` (используются только 8 младших бит). Возвращает `p`. Активно используется рантаймом для обнуления свежевыделенных блоков памяти.

```nim
var buf: array[8, byte]
c_memset(addr buf[0], 0, 8)   # обнулить
# buf = [0, 0, 0, 0, 0, 0, 0, 0]

c_memset(addr buf[0], 0xFF, 4)  # заполнить первые 4 байта значением 0xFF
# buf = [255, 255, 255, 255, 0, 0, 0, 0]
```

## Строковые процедуры

### `c_strcmp`

```nim
proc c_strcmp*(a, b: cstring): cint {.noSideEffect.}
```

Сравнивает две нуль-терминированные C-строки **лексикографически** (по кодам символов). Возвращает `0` при равенстве, отрицательное — если `a < b`, положительное — если `a > b`. Обе строки должны быть корректными (ненулевыми и нуль-терминированными).

```nim
assert c_strcmp("apple", "apple") == 0
assert c_strcmp("apple", "banana") < 0
assert c_strcmp("zebra", "ant") > 0
```

---

### `c_strlen`

```nim
proc c_strlen*(a: cstring): csize_t {.noSideEffect.}
```

Возвращает количество байт в нуль-терминированной строке `a`, **не считая** завершающего символа `\0`. Это измерение длины строки на уровне C, которое рантайм Nim использует всякий раз, когда нужно узнать длину `cstring` перед её обработкой.

```nim
assert c_strlen("hello") == 5
assert c_strlen("") == 0
```

---

### `c_strstr`

```nim
proc c_strstr*(haystack, needle: cstring): cstring {.noSideEffect.}
```

Ищет **первое вхождение** подстроки `needle` в строке `haystack`. Возвращает указатель на начало найденной подстроки внутри `haystack` или `nil`, если подстрока не найдена.

```nim
let result = c_strstr("hello world", "world")
# result указывает на 'w' в "world" внутри "hello world"

let missing = c_strstr("hello world", "xyz")
# missing == nil
```

## Процедуры управления потоком выполнения

### `c_abort`

```nim
proc c_abort*() {.noSideEffect, noreturn.}
```

Вызывает C-функцию `abort()`, которая **немедленно аварийно завершает программу**. Она посылает `SIGABRT` и — в зависимости от платформы и конфигурации — создаёт дамп памяти. Помечена `noreturn`: компилятор знает, что выполнение никогда не продолжится после этого вызова.

Используется рантаймом Nim как последнее средство, когда программа достигает невосстановимого состояния — например, при сбое утверждения (assertion) внутри самого менеджера памяти или критической внутренней ошибке, при которой даже генерация исключения Nim небезопасна.

```nim
# Внутри рантайма:
if internalInvariantBroken:
  c_abort()  # программа завершается здесь, безусловно
```

---

### `c_setjmp`

```nim
proc c_setjmp*(jmpb: C_JmpBuf): cint
```

Сохраняет текущее состояние регистров процессора (указатель стека, счётчик команд, callee-saved регистры) в `jmpb` и возвращает `0`. Если впоследствии выполнение будет перенесено сюда через `c_longjmp`, функция вернёт ненулевое значение, переданное в `c_longjmp`.

Это одна из самых тонких функций во всём C: она **возвращается дважды** — один раз при обычном вызове (возвращает `0`) и второй раз при срабатывании `c_longjmp` (возвращает ненулевое значение). В Nim она используется механизмом исключений для реализации try/catch на уровне C без опоры на исключения C++.

Фактическая вызываемая C-функция зависит от флагов компиляции:

| Флаг | C-функция |
|---|---|
| `-d:nimSigSetjmp` | `sigsetjmp(jmpb, 0)` — также сохраняет маску сигналов |
| `-d:nimBuiltinSetjmp` | `__builtin_setjmp` — встроенная функция GCC/Clang |
| `-d:nimRawSetjmp` (Windows) | `setjmp` или `_setjmp(jmpb, NULL)` |
| `-d:nimRawSetjmp` (POSIX) | `_setjmp` — не сохраняет маску сигналов |
| По умолчанию | `setjmp` — стандартный ANSI C |

```nim
var buf: C_JmpBuf
let code = c_setjmp(buf)
if code == 0:
  echo "Первый проход — нормальное выполнение"
  # ... где-то глубже вызывается c_longjmp(buf, 1) ...
else:
  echo "Прыжок выполнен! Возвращённое значение: ", code
```

---

### `c_longjmp`

```nim
proc c_longjmp*(jmpb: C_JmpBuf, retval: cint)
```

Восстанавливает состояние процессора из `jmpb` (ранее сохранённого через `c_setjmp`) и заставляет `c_setjmp` вернуть `retval`. Если `retval` равно `0`, стандарт C трактует его как `1`, чтобы отличить возврат от прыжка от исходного вызова. **Из самого `c_longjmp` выполнение не возвращается.**

Это механизм нелокального выхода рантайма. Когда Nim генерирует исключение, а исключения C++ недоступны (например, при компиляции с `-d:nimRawSetjmp`), механизм исключений использует `c_longjmp` для раскрутки стека к ближайшему ограждению `c_setjmp`.

```nim
var buf: C_JmpBuf
if c_setjmp(buf) == 0:
  echo "Инициализация..."
  c_longjmp(buf, 42)   # прыжок обратно к setjmp
  echo "Никогда не выполнится"
else:
  echo "Возврат из longjmp"  # это выполнится
```

> **Предупреждение:** Все стековые переменные, изменявшиеся между `c_setjmp` и `c_longjmp`, должны быть объявлены `volatile` в C, чтобы иметь определённое поведение. В Nim это обеспечивается компилятором.

## Процедуры обработки сигналов

### `c_signal`

```nim
proc c_signal*(sign: cint, handler: CSighandlerT): CSighandlerT {.discardable.}
```

Устанавливает `handler` в качестве обработчика сигнала с номером `sign`. Возвращает предыдущий обработчик. Используйте `SIG_DFL` как обработчик для восстановления поведения ОС по умолчанию.

Рантайм Nim использует эту функцию для установки обработчиков краша (для `SIGSEGV`, `SIGFPE` и т.д.), чтобы ошибка сегментации или ошибка плавающей точки превращалась в исключение Nim или аккуратный отчёт о сбое, а не в молчаливое завершение ОС.

Тип обработчика `CSighandlerT` — это `proc(a: cint) {.noconv.}`: коллбэк в соглашении о вызовах C, принимающий номер сигнала.

```nim
proc myHandler(sig: cint) {.noconv.} =
  c_fprintf(cstderr, "Пойман сигнал %d\n", sig)
  c_abort()

c_signal(SIGSEGV, myHandler)  # установить обработчик краша
```

---

### `c_raise`

```nim
proc c_raise*(sign: cint): cint
```

Отправляет сигнал `sign` **текущему процессу**. Возвращает `0` при успехе, ненулевое значение при ошибке. Это программный эквивалент нажатия Ctrl+C (для `SIGINT`) или любого другого сигнала, посылаемого из программы самой себе.

```nim
# Имитировать прерывание изнутри программы:
discard c_raise(SIGINT)
```

## Процедуры ввода-вывода

### `c_fprintf`

```nim
proc c_fprintf*(f: CFilePtr, frmt: cstring): cint {.varargs, discardable.}
```

C-функция `fprintf`: форматирует строку по `frmt` (спецификаторы формата в стиле printf) и записывает её в файловый дескриптор `f`. Дополнительные аргументы следуют после `frmt` в стиле C varargs. Возвращает количество записанных символов (обычно игнорируется).

Используется по всему рантайму для диагностического вывода — особенно в обработчиках краша и внутренних утверждениях, где высокоуровневому вводу-выводу Nim доверять нельзя.

```nim
c_fprintf(cstderr, "Ошибка: значение равно %d\n", 42)
c_fprintf(cstdout, "Привет, %s!\n", cstring("world"))
```

---

### `c_printf`

```nim
proc c_printf*(frmt: cstring): cint {.varargs, discardable.}
```

Аналог `c_fprintf`, но всегда пишет в `stdout`. Тонкая обёртка вокруг C-функции `printf`. Используется в рантайме редко, преимущественно для отладочного вывода.

```nim
c_printf("Debug: x = %d, y = %f\n", 10, 3.14)
```

---

### `c_fputs`

```nim
proc c_fputs*(c: cstring, f: CFilePtr): cint {.discardable.}
```

Записывает строку `c` (без нуль-терминатора) в файл `f`. Никакого форматирования — только сырой строковый вывод. Возвращает неотрицательное значение при успехе, `EOF` при ошибке. Быстрее `c_fprintf`, когда форматирование не нужно.

```nim
c_fputs("hello\n", cstdout)
c_fputs("сообщение об ошибке", cstderr)
```

---

### `c_fputc`

```nim
proc c_fputc*(c: char, f: CFilePtr): cint {.discardable.}
```

Записывает **один символ** `c` в файл `f`. Возвращает записанный символ как беззнаковый `cint` или `EOF` при ошибке. Наиболее гранулярный примитив ввода-вывода в этом модуле.

```nim
c_fputc('\n', cstdout)   # записать перевод строки
c_fputc('X', cstderr)    # записать 'X' в stderr
```

---

### `c_sprintf`

```nim
proc c_sprintf*(buf, frmt: cstring): cint {.varargs, noSideEffect.}
```

Форматирует строку в буфер `buf` по формату `frmt`. **Без проверки границ** — буфер должен быть достаточно большим для результата. Помечена `noSideEffect`, потому что пишет только в явный выходной буфер.

> **Замечание по безопасности:** Комментарий в исходнике гласит: *«мы используем её только так, что это не может привести к проблемам безопасности»*. На практике переполнение буфера через `sprintf` — классическая уязвимость безопасности C. В рантайме она вызывается только с контролируемыми строками формата и буферами заранее известного размера.

```nim
var buf: array[64, char]
discard c_sprintf(cast[cstring](addr buf[0]), "%d + %d = %d", 1, 2, 3)
# buf содержит "1 + 2 = 3\0"
```

---

### `c_snprintf`

```nim
proc c_snprintf*(buf: cstring, n: csize_t, frmt: cstring): cint {.varargs, noSideEffect.}
```

Как `c_sprintf`, но **безопасная**: записывает не более `n - 1` символов в `buf` и всегда добавляет нуль-терминатор. Возвращает количество символов, *которые были бы записаны*, если бы `n` было достаточно большим — поэтому если возвращаемое значение ≥ `n`, вывод был усечён. Это предпочтительный примитив форматирования.

```nim
var buf: array[16, char]
let written = c_snprintf(cast[cstring](addr buf[0]), 16, "Value: %d", 12345)
# buf = "Value: 12345\0", written = 12
```

---

### `c_fwrite`

```nim
proc c_fwrite*(buf: pointer, size, n: csize_t, f: CFilePtr): csize_t
```

Записывает `n` элементов по `size` байт каждый из `buf` в файл `f`. Возвращает количество успешно записанных элементов (может быть меньше `n` при ошибке). Это рабочая лошадка для бинарного ввода-вывода без форматирования и без предположений о нуль-терминаторах.

```nim
var data = [byte(65), 66, 67]  # "ABC"
let written = c_fwrite(addr data[0], 1, 3, cstdout)
# выводит: ABC (записано 3 байта)
```

---

### `c_fflush`

```nim
proc c_fflush*(f: CFilePtr): cint
```

Сбрасывает буфер записи файла `f` в ОС. Возвращает `0` при успехе, `EOF` при ошибке. Передача `nil` сбрасывает **все открытые потоки**. Без сброса вывод, записанный через `c_fprintf` или `c_fwrite`, может оставаться в пользовательском буфере и так и не появиться на экране или в файле.

```nim
c_fprintf(cstdout, "Сейчас произойдёт краш...\n")
discard c_fflush(cstdout)  # убедиться, что сообщение видно до краша
c_abort()
```

## Процедуры выделения памяти

Это Nim-обёртки функций динамического выделения памяти из C. На Zephyr RTOS без `libc`-malloc они переназначаются на `k_malloc`/`k_calloc`/`k_free`/`k_realloc` из `<kernel.h>`. На всех остальных платформах вызывают стандартные функции `<stdlib.h>`.

---

### `c_malloc`

```nim
proc c_malloc*(size: csize_t): pointer
```

Выделяет `size` байт **неинициализированной** кучевой памяти. Возвращает указатель на блок или `nil` при неудаче. Содержимое возвращённой памяти **не гарантировано** — там могут быть произвольные данные от предыдущих аллокаций.

```nim
let p = c_malloc(csize_t(64))
if p != nil:
  c_memset(p, 0, csize_t(64))  # обнулить вручную
  # ... используем память ...
  c_free(p)
```

---

### `c_calloc`

```nim
proc c_calloc*(nmemb, size: csize_t): pointer
```

Выделяет `nmemb * size` байт кучевой памяти, **обнулённой**. Двухаргументная форма существует, чтобы C-рантайм мог обнаружить переполнение при умножении. Возвращает `nil` при неудаче.

Предпочтительнее `c_malloc` + `c_memset` при необходимости обнулённой памяти, так как `calloc` может использовать нулевые страницы ОС и на практике работать быстрее.

```nim
let p = c_calloc(csize_t(10), csize_t(sizeof(int)))
# p указывает на 10 заполненных нулями слотов размером int
defer: c_free(p)
```

---

### `c_free`

```nim
proc c_free*(p: pointer)
```

Освобождает память, ранее выделенную через `c_malloc`, `c_calloc` или `c_realloc`. Передача `nil` безопасна и ничего не делает. Передача любого другого недопустимого указателя (уже освобождённого или не принадлежащего аллокатору) — **неопределённое поведение**, которое, вероятно, повредит кучу.

```nim
let p = c_malloc(csize_t(32))
# ... используем p ...
c_free(p)
# После этого p — висящий указатель, никогда не используйте его снова
```

---

### `c_realloc`

```nim
proc c_realloc*(p: pointer, newsize: csize_t): pointer
```

Изменяет размер блока памяти по адресу `p` до `newsize` байт. Блок может быть перемещён по новому адресу; старый указатель `p` нельзя использовать после вызова. Возвращает новый указатель или `nil` при неудаче (в этом случае исходный блок **не освобождается**). Если `p` равно `nil`, ведёт себя как `c_malloc`.

> **Замечание для Zephyr:** На Zephyr без libc функция `c_realloc` реализована вручную: выделяет новый блок, копирует старые данные и освобождает старый указатель. Она не умеет эффективно уменьшать аллокации.

```nim
var p = c_malloc(csize_t(16))
p = c_realloc(p, csize_t(64))  # расширить блок
if p == nil:
  echo "Realloc не удался"
else:
  defer: c_free(p)
```

## Высокоуровневые Nim-вспомогательные функции (compilerproc)

Эти две процедуры — не сырые C-импорты; они являются Nim-обёртками, в которые **сам компилятор** вставляет вызовы. Они помечены `{.compilerproc.}`, что означает: генератор кода Nim знает их имена и вставляет вызовы к ним напрямую.

---

### `rawWriteString`

```nim
proc rawWriteString*(f: CFilePtr, s: cstring, length: int) {.compilerproc, nonReloadable, inline.}
```

Записывает ровно `length` байт строки `s` в файл `f`, затем сбрасывает буфер. Использует `c_fwrite` + `c_fflush`. Не генерирует исключений — намеренно: это должно работать даже в контекстах, где система исключений сломана или недоступна (например, во время фатального краша).

Это процедура, которую компилятор вызывает, когда ему нужно безусловно и немедленно вывести строку в C-файловый дескриптор.

```nim
rawWriteString(cstderr, "фатальная ошибка\n", 18)
# Записывает ровно 18 байт в stderr и сбрасывает буфер
```

---

### `rawWrite`

```nim
proc rawWrite*(f: CFilePtr, s: cstring) {.compilerproc, nonReloadable, inline.}
```

Аналогична `rawWriteString`, но определяет длину автоматически с помощью `c_strlen`. Записывает всю нуль-терминированную строку `s` в `f` и сбрасывает буфер. Также защищена от исключений по дизайну.

Разница от `rawWriteString` — исключительно в том, знает ли вызывающий код длину заранее. Если знает, `rawWriteString` экономит лишний вызов `strlen`.

```nim
rawWrite(cstdout, "Привет из рантайма\n")
```
