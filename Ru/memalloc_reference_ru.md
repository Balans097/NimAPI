# `memalloc.nim` — справочник модуля

> **Модуль:** `memalloc` — стандартная библиотека Nim, выделение памяти и низкоуровневые операции  
> **Доступность:** Большая часть модуля требует `hasAlloc` (не JS, не NimScript). Сырые операции с памятью (`zeroMem`, `copyMem` и др.) доступны на большинстве не-JS платформ. Подробности — в примечаниях к каждому символу.  
> **Безопасность:** Каждая функция этого модуля явно **небезопасна**. Они работают ниже системы типов Nim. Используйте их только при необходимости ручного управления памятью для FFI с C, высокопроизводительных структур данных или встраиваемого/системного программирования.

---

## Обзор

`memalloc.nim` предоставляет два дополняющих друг друга уровня управления памятью:

**Уровень 1 — Сырые операции с памятью** (без выделения, только побайтовые манипуляции):  
`zeroMem`, `copyMem`, `moveMem`, `equalMem`, `cmpMem`

**Уровень 2 — Аллокатор** (две отдельные кучи):
- **Потокозависимая куча** — память принадлежит потоку, который её выделил, и должна быть освобождена тем же потоком: `alloc`, `alloc0`, `realloc`, `realloc0`, `dealloc` и их типизированные обёртки `create`, `createU`, `resize`.
- **Разделяемая куча** — память может передаваться между потоками и освобождаться любым потоком: `allocShared`, `allocShared0`, `reallocShared`, `reallocShared0`, `deallocShared` и их типизированные обёртки `createShared`, `createSharedU`, `resizeShared`, `freeShared`.

Дополнительно модуль предоставляет статистику памяти GC (`getOccupiedMem`, `getFreeMem`, `getTotalMem`) и опциональные счётчики выделений (`AllocStats`, `getAllocStats`, `dumpAllocstats`).

### Ключевая ментальная модель

```
Потокозависимая куча    Разделяемая куча
────────────────────    ──────────────────────────────────
alloc / alloc0          allocShared / allocShared0
realloc / realloc0      reallocShared / reallocShared0
dealloc                 deallocShared
────────────────────    ──────────────────────────────────
createU / create        createSharedU / createShared
resize                  resizeShared
(dealloc)               freeShared
```

Суффикс `U` всегда означает **U**ninitialized — **н**еинициализированный (быстрее, опаснее). Отсутствие `U` означает инициализацию нулями (чуть медленнее, безопаснее).

## Группа 1: Сырые операции с памятью

Эти пять процедур работают с сырыми байтовыми диапазонами. Доступны на всех не-JS, не-NimScript платформах. Напрямую соответствуют `memset`/`memcpy`/`memmove`/`memcmp` из стандартной библиотеки C.

---

### `zeroMem`

```nim
proc zeroMem*(p: pointer, size: Natural) {.inline, noSideEffect, tags: [], raises: [], enforceNoRaises.}
```

**Что делает:**  
Заполняет ровно `size` байт, начиная с адреса `p`, значением `0`. Каждый бит в диапазоне становится нулём. Эквивалент C-вызова `memset(p, 0, size)`.

**Когда использовать:**  
- Безопасная очистка конфиденциальных данных (пароли, ключи шифрования) из памяти перед её освобождением.
- Инициализация сырого буфера в заведомо известное состояние перед использованием.
- Выполнение требований C API, ожидающих нулезаполненные структуры.

**Чем отличается от присваивания через `=`:**  
`zeroMem` полностью игнорирует систему типов Nim — записывает нули безусловно, без вызовов деструкторов, без взаимодействия с GC и без проверки границ (за исключением переданного `size`). Именно это делает её подходящей для очистки памяти, которой система типов Nim «не владеет» (например, буферы, выделенные через `alloc`).

```nim
# Безопасно затереть буфер пароля перед освобождением
var buf = alloc(64)
# ... используем buf ...
zeroMem(buf, 64)   # затираем до dealloc — секрет не остаётся в памяти
dealloc(buf)
```

```nim
# Обнулить C-структуру, полученную через FFI
type CConfig {.importc, header: "config.h".} = object
  flags: cint
  size:  cint
var cfg: CConfig
zeroMem(addr cfg, sizeof(CConfig))
```

> ⚠️ **Небезопасна.** Неверный `size` молча портит соседнюю память.

---

### `copyMem`

```nim
proc copyMem*(dest, source: pointer, size: Natural) {.inline, gcsafe, tags: [], raises: [], enforceNoRaises.}
```

**Что делает:**  
Копирует ровно `size` байт из `source` в `dest`. Эквивалент `memcpy`. **Области источника и назначения не должны перекрываться.** Если перекрытие возможно — используйте `moveMem`.

**Когда использовать:**  
- Копирование сырых буферов (аудиосэмплы, пиксели, сетевые пакеты) между заранее выделенными областями.
- Массовое копирование типа-значения в сырой буфер для сериализации.
- Взаимодействие с C-кодом, работающим с байтовыми массивами.

**Перекрывающиеся области:**  
Если `dest` и `source` перекрываются — результат неопределён: некоторые байты могут быть прочитаны уже после перезаписи. Это то же ограничение, что и у `memcpy` в C.

```nim
var src = [1'u8, 2, 3, 4, 5]
var dst: array[5, uint8]
copyMem(addr dst, addr src, 5)
# dst теперь [1, 2, 3, 4, 5]
```

```nim
# Скопировать структуру в байтовый буфер для передачи по сети
type Header = object
  magic: uint32
  length: uint32

var h = Header(magic: 0xDEADBEEF'u32, length: 42)
var wire: array[8, byte]
copyMem(addr wire, addr h, sizeof(Header))
```

> ⚠️ **Небезопасна.** Нет проверки границ. Перекрывающиеся области → неопределённое поведение.

---

### `moveMem`

```nim
proc moveMem*(dest, source: pointer, size: Natural) {.inline, gcsafe, tags: [], raises: [], enforceNoRaises.}
```

**Что делает:**  
Копирует ровно `size` байт из `source` в `dest`, корректно обрабатывая случай, когда области **перекрываются**. Эквивалент `memmove`. Чуть медленнее `copyMem` из-за обработки перекрытий, но безопасна при любом взаимном расположении областей.

**Правило выбора:**
- Области точно **не** перекрываются → `copyMem` (быстрее).
- Области **могут** перекрываться → `moveMem` (безопасно).

```nim
# Сдвинуть элементы влево на одну позицию в том же массиве (перекрытие)
var data = [10'u8, 20, 30, 40, 50]
moveMem(addr data[0], addr data[1], 4)
# data теперь [20, 30, 40, 50, 50]  (последний байт устаревший — это нормально)
```

> ⚠️ **Небезопасна.** Нет проверки границ. Неопределённое поведение при выходе `size` за пределы выделенной области.

---

### `equalMem`

```nim
proc equalMem*(a, b: pointer, size: Natural): bool {.inline, noSideEffect, tags: [], raises: [], enforceNoRaises.}
```

**Что делает:**  
Побайтово сравнивает ровно `size` байт начиная с `a` и `b`. Возвращает `true`, если все байты идентичны, иначе `false`. Эквивалент `memcmp(a, b, size) == 0`.

**Когда использовать:**  
- Сравнение сырых буферов без значимого типа Nim.
- Быстрая проверка равенства структур фиксированного размера, когда вы уверены, что байты выравнивания детерминированы.

**Оговорка про выравнивание:** Структуры могут содержать байты выравнивания (padding), вставленные компилятором со случайными значениями. `equalMem` сравнивает *все* байты включая padding — две логически равные структуры могут не пройти `equalMem`, если их padding отличается. Для структур предпочитайте поэлементное сравнение, если только не знаете наверняка, что padding отсутствует.

```nim
var a = [1'u8, 2, 3]
var b = [1'u8, 2, 3]
var c = [1'u8, 2, 4]
echo equalMem(addr a, addr b, 3)  # true
echo equalMem(addr a, addr c, 3)  # false
```

> ⚠️ **Небезопасна.** Нет проверки границ. Байты выравнивания в структурах могут давать неожиданные результаты.

---

### `cmpMem`

```nim
proc cmpMem*(a, b: pointer, size: Natural): int {.inline, noSideEffect, tags: [], raises: [], enforceNoRaises.}
```

**Что делает:**  
Лексикографически сравнивает `size` байт начиная с `a` и `b`, интерпретируя каждый байт как беззнаковое значение. Возвращает:
- **Отрицательное** целое — если первый отличающийся байт в `a` меньше, чем в `b`.
- **Положительное** целое — если первый отличающийся байт в `a` больше.
- **Ноль** — если все `size` байт идентичны.

Эквивалент `memcmp(a, b, size)`.

**Когда использовать:**  
- Реализация пользовательской сортировки или упорядочивания байтовых массивов.
- Сравнение ключей фиксированной длины (в хеш-таблице или при бинарном поиске).

```nim
var x = [1'u8, 2, 3]
var y = [1'u8, 2, 5]
let r = cmpMem(addr x, addr y, 3)
if r < 0:
  echo "x < y"   # эта ветка будет выполнена
```

> ⚠️ **Небезопасна.** Нет проверки границ.

## Группа 2: Выделение памяти в потокозависимой куче

Память в потокозависимой куче **должна быть освобождена тем же потоком, который её выделил**. Никогда не передавайте эту память другому потоку для освобождения там. Для межпоточных данных используйте разделяемую кучу.

Доступна при `hasAlloc and not defined(js)`.

---

### `alloc`

```nim
template alloc*(size: Natural): pointer
```

**Что делает:**  
Выделяет сырой **неинициализированный** блок как минимум `size` байт в потокозависимой куче. Возвращает `pointer`. Содержимое блока не определено — чтение до записи является неопределённым поведением.

**Жизненный цикл:** Должен быть освобождён через `dealloc(p)` или `realloc(p, 0)`. Если забыть — утечка памяти на всё время жизни процесса.

```nim
let buf = alloc(256)        # сырой, неинициализированный блок
zeroMem(buf, 256)           # явно обнуляем перед чтением
# ... используем buf ...
dealloc(buf)
```

> Используйте `alloc0` для автоматического обнуления. Используйте `createU` для типизированного указателя.

---

### `alloc0`

```nim
template alloc0*(size: Natural): pointer
```

**Что делает:**  
Как `alloc`, но **инициализирует весь блок нулями** перед возвратом. Каждый байт гарантированно равен `0`. Чуть медленнее `alloc` из-за дополнительного прохода записи.

```nim
let p = alloc0(sizeof(int) * 10)   # 10 int-ов, все нулевые
let arr = cast[ptr UncheckedArray[int]](p)
echo arr[0]   # безопасно читать: гарантированно 0
dealloc(p)
```

---

### `createU`

```nim
proc createU*(T: typedesc, size = 1.Positive): ptr T {.inline, gcsafe, raises: [].}
```

**Что делает:**  
Типизированная обёртка над `alloc`. Выделяет `T.sizeof * size` байт и возвращает `ptr T`. Блок **неинициализирован**. `size` — количество элементов типа `T`, а не байт — именно это главное преимущество перед сырым `alloc`.

**Когда предпочесть перед `alloc`:** Всегда, когда известен тип выделяемых данных — параметр типа отловит ошибки в расчёте размера на этапе компиляции.

```nim
type Vec3 = object
  x, y, z: float32

let v = createU(Vec3)   # один Vec3, неинициализированный
v.x = 1.0; v.y = 2.0; v.z = 3.0
dealloc(v)

# Массив из 100 структур Vec3:
let arr = createU(Vec3, 100)
# arr[0] .. arr[99] доступны, но неинициализированы
dealloc(arr)
```

---

### `create`

```nim
proc create*(T: typedesc, size = 1.Positive): ptr T {.inline, gcsafe, raises: [].}
```

**Что делает:**  
Типизированная обёртка над `alloc0`. Выделяет `T.sizeof * size` **нулезаполненных** байт и возвращает `ptr T`. Безопаснее `createU` — память гарантированно чистая.

```nim
type Node = object
  value: int
  next: ptr Node

let n = create(Node)     # нулезаполнен: value = 0, next = nil
echo n.next.isNil        # true — гарантируется нулевой инициализацией
dealloc(n)
```

---

### `realloc`

```nim
template realloc*(p: pointer, newSize: Natural): pointer
```

**Что делает:**  
Изменяет размер ранее выделенного блока. Поведение по случаям:
- `p` равно `nil` → работает как `alloc(newSize)`, выделяет новый блок.
- `newSize == 0` и `p != nil` → работает как `dealloc(p)`, освобождает блок.
- Иначе → увеличивает или уменьшает блок до как минимум `newSize` байт. Указатель `p` становится **недействительным** после вызова; используйте только возвращённый указатель.

**Новые байты при увеличении** — **неинициализированные**. Используйте `realloc0`, если нужны нулевые.

```nim
var p = alloc(64)
# Нужно больше места:
p = realloc(p, 128)   # старый p недействителен; используем возвращённый
# ... используем p со 128 байтами ...
dealloc(p)
```

---

### `realloc0`

```nim
template realloc0*(p: pointer, oldSize, newSize: Natural): pointer
```

**Что делает:**  
Как `realloc`, но **инициализирует нулями все новые байты** при увеличении. Требует явной передачи `oldSize`, чтобы рантайм знал, какие байты являются новыми. Медленнее `realloc` при увеличении, но предотвращает чтение неинициализированных данных из расширенной области.

```nim
var p = alloc0(32)
p = realloc0(p, 32, 64)  # байты 32..63 гарантированно нулевые
dealloc(p)
```

---

### `resize`

```nim
proc resize*[T](p: ptr T, newSize: Natural): ptr T {.inline, gcsafe, raises: [].}
```

**Что делает:**  
Типизированная обёртка над `realloc`. Изменяет размер блока до `T.sizeof * newSize` байт. `newSize` — количество элементов, не байт.

```nim
var arr = createU(int, 10)
arr = resize(arr, 20)    # теперь 20 int-ов; новые слоты неинициализированы
dealloc(arr)
```

---

### `dealloc`

```nim
proc dealloc*(p: pointer) {.noconv, compilerproc, rtl, gcsafe, raises: [], tags: [].}
```

**Что делает:**  
Освобождает блок, ранее полученный от `alloc`, `alloc0`, `realloc`, `create` или `createU`. После вызова `dealloc(p)` указатель `p` становится **висячим** — любое обращение к нему является неопределённым поведением.

**Правила:**
- Освобождать потокозависимую память может только **поток-аллокатор**.
- Двойное освобождение — неопределённое поведение (вероятно, краш или повреждение памяти).
- Передача `nil` может или не может вызвать краш в зависимости от нижележащего аллокатора — не полагайтесь на это.

```nim
let p = alloc(128)
# ... используем p ...
dealloc(p)
# p теперь висячий — не использовать
```

> ⚠️ **Крайне опасна.** Утечки, двойное освобождение и обращение к освобождённой памяти — всё это молчаливо на уровне Nim и неопределённо на уровне C.

## Группа 3: Выделение памяти в разделяемой куче

К разделяемой куче можно обращаться **из любого потока**. Выделенная здесь память может передаваться другим потокам и освобождаться там. Внутренне защищена мьютексом или системным `malloc` (с `-d:useMalloc`).

Доступна при `hasAlloc and not defined(js)`.

---

### `allocShared`

```nim
template allocShared*(size: Natural): pointer
```

**Что делает:**  
Выделяет как минимум `size` **неинициализированных** байт в разделяемой куче. Блок может использоваться и освобождаться любым потоком.

```nim
# Поток-производитель выделяет, поток-потребитель освобождает — валидно с allocShared
let buf = allocShared(1024)
sendToOtherThread(buf)
```

---

### `allocShared0`

```nim
template allocShared0*(size: Natural): pointer
```

**Что делает:**  
Как `allocShared`, но инициализирует блок нулями.

```nim
let zeroBuf = allocShared0(512)
// Все 512 байт равны 0
```

---

### `createSharedU`

```nim
proc createSharedU*(T: typedesc, size = 1.Positive): ptr T {.inline, tags: [], gcsafe, raises: [].}
```

**Что делает:**  
Типизированная обёртка над `allocShared`. Выделяет `T.sizeof * size` **неинициализированных** байт в разделяемой куче, возвращает `ptr T`.

```nim
type Msg = object
  kind: int
  data: array[64, byte]

let m = createSharedU(Msg)
m.kind = 42
sendMessage(m)   # m будет освобождён принимающим потоком
```

---

### `createShared`

```nim
proc createShared*(T: typedesc, size = 1.Positive): ptr T {.inline.}
```

**Что делает:**  
Типизированная обёртка над `allocShared0`. Выделяет нулезаполненные `T.sizeof * size` байт в разделяемой куче.

```nim
let m = createShared(Msg)   // m.kind == 0, m.data полностью нулевой
```

---

### `reallocShared`

```nim
template reallocShared*(p: pointer, newSize: Natural): pointer
```

**Что делает:**  
Изменяет размер блока в разделяемой куче. Та же семантика, что и у `realloc` (nil → выделить, 0 → освободить, иначе изменить размер). Новые байты неинициализированы.

---

### `reallocShared0`

```nim
template reallocShared0*(p: pointer, oldSize, newSize: Natural): pointer
```

**Что делает:**  
Как `reallocShared`, но инициализирует новые байты нулями. Требует явного `oldSize`.

---

### `resizeShared`

```nim
proc resizeShared*[T](p: ptr T, newSize: Natural): ptr T {.inline, raises: [].}
```

**Что делает:**  
Типизированная обёртка над `reallocShared`. Изменяет размер до `T.sizeof * newSize` байт, возвращает `ptr T`.

---

### `deallocShared`

```nim
proc deallocShared*(p: pointer) {.noconv, compilerproc, rtl, gcsafe, raises: [], tags: [].}
```

**Что делает:**  
Освобождает блок, полученный из `allocShared`, `allocShared0` или `reallocShared`. Как `dealloc`, но для разделяемой кучи. Может вызываться из любого потока.

```nim
// Принимающий поток освобождает память, выделенную производителем
proc receiver(buf: pointer) =
  processData(buf)
  deallocShared(buf)
```

> ⚠️ Все те же опасности, что и у `dealloc`: двойное освобождение, использование после освобождения, висячие указатели.

---

### `freeShared`

```nim
proc freeShared*[T](p: ptr T) {.inline, gcsafe, raises: [].}
```

**Что делает:**  
Типизированная удобная обёртка над `deallocShared`. Принимает `ptr T` вместо сырого `pointer`. Функционально идентична `deallocShared(p)`.

```nim
let m = createShared(Msg)
// ... используем m ...
freeShared(m)   // типизировано — чище в месте вызова, чем deallocShared(m)
```

---

## Группа 4: Статистика и диагностика выделений

Доступна при `hasAlloc and not defined(js)`. Трекинг счётчиков требует `-d:nimAllocStats`.

---

### `AllocStats`

```nim
type AllocStats* = object
  allocCount: int
  deallocCount: int
```

**Что это:**  
Простой объект, хранящий два счётчика: общее количество вызовов выделения и освобождения памяти с момента запуска программы (или с момента последнего снимка при использовании вычитания).

Поля напрямую не экспортированы (строчными буквами), но `getAllocStats` возвращает полное значение, а оператор `-` позволяет вычислять разность двух снимков.

---

### `getAllocStats`

```nim
proc getAllocStats*(): AllocStats
```

**Что делает:**  
Возвращает текущие глобальные счётчики выделений. При компиляции с `-d:nimAllocStats` счётчики атомарно инкрементируются при каждом вызове `alloc`/`dealloc`. Без `-d:nimAllocStats` всегда возвращает нулевую статистику (без накладных расходов).

```nim
let before = getAllocStats()
runSomeCode()
let after = getAllocStats()
let delta = after - before
# обратитесь к полям delta для получения значений счётчиков
```

---

### `dumpAllocstats`

```nim
template dumpAllocstats*(code: untyped)
```

**Что делает:**  
Удобный шаблон, оборачивающий блок кода: делает снимок статистики до и после, вычисляет разность и выводит её через `echo`. Полезен для быстрой проверки «сколько выделений делает эта функция?» в ходе разработки.

**Важно:** Без `-d:nimAllocStats` все счётчики будут нулевыми и вывод будет неинформативным.

```nim
dumpAllocstats:
  var s = newSeq[int](1000)
  for i in 0..<1000:
    s[i] = i * i
// Выводит дельту выделений памяти, вызванных блоком выше
```

## Группа 5: Запросы статистики памяти

Эти функции запрашивают у GC/аллокатора информацию об использовании памяти процессом. Доступны при `hasAlloc`.

---

### `getOccupiedMem`

```nim
proc getOccupiedMem*(): int {.rtl.}
```

**Что делает:**  
Возвращает количество байт, которые в данный момент **используются** — принадлежат процессу и содержат живые данные. Это сумма всех текущих выделенных, ещё не освобождённых блоков, отслеживаемых аллокатором Nim. На JS возвращает `-1`.

```nim
echo "Живые данные: ", getOccupiedMem(), " байт"
```

---

### `getFreeMem`

```nim
proc getFreeMem*(): int {.rtl.}
```

**Что делает:**  
Возвращает количество байт, которые процесс **владеет, но не использует** — память, запрошенная у ОС, но находящаяся в списке свободных аллокатора в ожидании повторного использования. На JS возвращает `-1`.

```nim
// Большой свободный пул означает, что аллокатор заранее запросил у ОС лишнюю память
echo "Свободный пул: ", getFreeMem(), " байт"
```

---

### `getTotalMem`

```nim
proc getTotalMem*(): int {.rtl.}
```

**Что делает:**  
Возвращает общее количество байт, которыми владеет процесс с точки зрения аллокатора: `getTotalMem() == getOccupiedMem() + getFreeMem()`. Это не то же самое, что RSS процесса по данным ОС, — учитывается только память, отслеживаемая аллокатором Nim. На JS возвращает `-1`.

```nim
echo "Всего куча: ", getTotalMem(), " байт"
```

---

### `getOccupiedSharedMem`

```nim
proc getOccupiedSharedMem*(): int {.rtl.}
```

**Что делает:**  
Как `getOccupiedMem`, но для **разделяемой кучи**. Доступна только при включённых потоках (`--threads:on`) и без системного malloc (`-d:useMalloc`).

---

### `getFreeSharedMem`

```nim
proc getFreeSharedMem*(): int {.rtl.}
```

**Что делает:**  
Как `getFreeMem`, но для разделяемой кучи. Те же ограничения по доступности.

---

### `getTotalSharedMem`

```nim
proc getTotalSharedMem*(): int {.rtl.}
```

**Что делает:**  
Как `getTotalMem`, но для разделяемой кучи. Те же ограничения по доступности.

---

## Выбор правильного аллокатора

| Потребность | Используйте |
|---|---|
| Типизировано, потокозависимо, неинициализировано | `createU(T)` |
| Типизировано, потокозависимо, нулезаполнено | `create(T)` |
| Типизировано, потокозависимо, изменить размер | `resize(p, n)` |
| Сырые байты, потокозависимо | `alloc` / `alloc0` / `realloc` / `realloc0` |
| Освободить потокозависимое | `dealloc(p)` |
| Типизировано, разделяемое (межпоточное), неинициализировано | `createSharedU(T)` |
| Типизировано, разделяемое, нулезаполнено | `createShared(T)` |
| Типизировано, разделяемое, изменить размер | `resizeShared(p, n)` |
| Сырые байты, разделяемое | `allocShared` / `allocShared0` / `reallocShared` |
| Освободить разделяемое | `deallocShared(p)` или `freeShared(p)` |

---

## Сводка по безопасности

| Риск | Как избежать |
|---|---|
| Утечка памяти | Каждому `alloc*` / `create*` должен соответствовать `dealloc` / `free` |
| Обращение после освобождения | Устанавливайте указатели в `nil` после освобождения; никогда не читайте освобождённый указатель |
| Двойное освобождение | Чёткая ответственность за владение; освобождает только один путь выполнения |
| Смешивание куч | `dealloc` ≠ `deallocShared`; никогда не смешивайте их |
| Перекрывающиеся области в `copyMem` | Используйте `moveMem`, когда области могут перекрываться |
| Байты выравнивания в `equalMem` | Используйте `equalMem` только для байтовых буферов без struct-padding |
