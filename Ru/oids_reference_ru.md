# oids — Справочник по модулю

> **Стандартная библиотека Nim** | `import std/oids`

## Что такое OID?

**OID (Object Identifier, идентификатор объекта)** — это 12-байтовый глобально уникальный идентификатор, разработанный для независимой генерации на множестве машин без какой-либо координации между ними, но при этом гарантирующий отсутствие коллизий. Реализация вдохновлена форматом MongoDB BSON ObjectId.

Каждый OID собирается из трёх независимых компонентов, хранящихся в фиксированном бинарном формате:

```
┌──────────────────┬─────────────────┬───────────────────┐
│   time (8 байт)  │  fuzz (4 байта) │  count (4 байта)  │
│  Unix-timestamp  │ случ. для проц. │  атомарный счётч. │
└──────────────────┴─────────────────┴───────────────────┘
                     Итого: 16 байт (128 бит)
                     Строковое представление: 24 hex-символа
```

- **`time`** — секунды от начала эпохи Unix в момент генерации. Кодирует *когда* был создан OID; идентификаторы, сгенерированные позже, лексикографически следуют за более ранними.
- **`fuzz`** — случайное 32-битное значение, вычисляемое однократно при первой загрузке модуля. Служит «отпечатком» процесса/машины, гарантируя уникальность OID, сгенерированных разными процессами (или на разных машинах в одну секунду).
- **`count`** — атомарно возрастающий 32-битный целочисленный счётчик. Гарантирует уникальность внутри одного процесса в пределах одной секунды, в том числе между потоками.

Сочетание всех трёх компонентов делает коллизию практически невозможной: два OID совпадут только если они сгенерированы в одном процессе, в одну секунду, и счётчик прошёл полный оборот до того же значения — событие с астрономически малой вероятностью.

**Формат передачи:** При сериализации через `$` OID становится строкой из 24 строчных шестнадцатеричных символов, например `"5fc7f546ddbbc84800006aaf"`.

**Инициализация:** Случайное начальное значение и `fuzz` вычисляются автоматически при первой загрузке модуля. Никакой процедуры настройки вызывать не нужно.

## Типы

### `Oid`

```nim
type Oid* = object
```

12-байтовый тип-значение (хранится встроенно, без выделения памяти в куче), содержащий три компонента OID. Поскольку `Oid` — это `object` (а не `ref`), присваивание копирует значение — алиасинга нет.

Поля `time`, `fuzz` и `count` приватны. Взаимодействовать с OID можно только через процедуры данного модуля. Каноническое внешнее представление — 24-символьная hex-строка, возвращаемая `$`.

```nim
var a = genOid()   # значение на стеке, без выделения памяти
var b = a          # b — независимая копия
```

## Генерация

### `genOid`

```nim
proc genOid*(): Oid
```

Создаёт и возвращает новый глобально уникальный OID. Это единственная процедура, необходимая в большинстве случаев.

Функция захватывает текущий Unix-timestamp, атомарно увеличивает счётчик уровня модуля и объединяет оба значения с фиксированным `fuzz`, вычисленным при загрузке модуля. Потокобезопасна: инкремент счётчика выполняется атомарной CPU-инструкцией, поэтому параллельные вызовы из нескольких потоков дают различные OID без использования блокировок.

```nim
let id = genOid()
echo $id           # например, "5fc7f546ddbbc84800006aaf"
echo ($id).len     # 24 — всегда ровно 24 символа

# Каждый вызов даёт уникальное значение
let id1 = genOid()
let id2 = genOid()
assert id1 != id2
```

```nim
# Генерация большого числа OID в цикле — безопасно и быстро
var ids: seq[Oid]
for i in 1..1000:
  ids.add(genOid())
```

## Сериализация и разбор

### `$` (в строку)

```nim
proc `$`*(oid: Oid): string
```

Преобразует OID в каноническую 24-символьную строку строчных шестнадцатеричных символов. Байты сериализуются в порядке big-endian, поэтому строка сохраняет порядок «сначала временна́я метка»: OID, сгенерированные раньше, лексикографически предшествуют более поздним.

Результат всегда имеет длину ровно 24 символа и содержит только символы `0–9` и `a–f`.

```nim
let id = genOid()
let s = $id
echo s          # например, "5fc7f546ddbbc84800006aaf"
echo s.len      # 24

# Удобно для хранения в БД, JSON, файлах журналов, HTTP-заголовках и т.д.
let json = """{"_id": """ & "\"" & $id & "\"" & "}"
```

### `parseOid`

```nim
proc parseOid*(str: cstring): Oid
```

Разбирает 24-символьную шестнадцатеричную строку обратно в `Oid`. Это операция, обратная `$`. Входная строка должна быть корректной 24-символьной hex-строкой — функция не проверяет входные данные и при некорректном вводе молча вернёт мусорный `Oid`.

Параметр имеет тип `cstring` (C-строка с нулевым терминатором). Nim-строки неявно конвертируются в `cstring`, поэтому Nim-строку можно передавать напрямую.

```nim
let original = genOid()
let s = $original              # "5fc7f546ddbbc84800006aaf"

let restored = parseOid(s)     # передаём Nim-строку — неявная конверсия в cstring
assert original == restored    # туда-обратно без потерь

# Разбор строки из жёсткого кода (например, из базы данных или конфиг-файла):
let known = parseOid("5fc7f546ddbbc84800006aaf")
echo $known  # 5fc7f546ddbbc84800006aaf
```

> **Важно:** `parseOid` не проверяет входные данные. Передача строки короче 24 символов или содержащей не-hex-символы приводит к неопределённому поведению. Всегда проверяйте длину и допустимость символов в строке до вызова `parseOid` с пользовательскими данными.

## Инспекция

### `generatedTime`

```nim
proc generatedTime*(oid: Oid): Time
```

Извлекает и возвращает Unix-временну́ю метку, встроенную в OID, в виде значения `Time` из `std/times`. Это говорит вам *когда* был создан OID с точностью до секунды.

Полезно для:
- Отображения читаемого времени создания без хранения отдельного поля временно́й метки в модели данных.
- Фильтрации или сортировки записей по времени создания, используя только OID.
- Отладки — быстрой проверки, когда был сгенерирован OID.

```nim
import std/[oids, times]

let id = genOid()
let t = id.generatedTime()
echo t.format("yyyy-MM-dd HH:mm:ss")  # например, "2024-01-15 14:32:07"

# Время с точностью до секунды
let before = getTime()
let id2 = genOid()
let after = getTime()
assert id2.generatedTime() >= before
assert id2.generatedTime() <= after
```

## Равенство и хеширование

### Оператор `==`

```nim
proc `==`*(oid1: Oid, oid2: Oid): bool
```

Возвращает `true` тогда и только тогда, когда все три компонента (`time`, `fuzz`, `count`) идентичны. Два OID равны тогда и только тогда, когда это один и тот же OID — «эквивалентных, но различных» представлений не существует, поскольку цикл `$` → `parseOid` не имеет потерь.

```nim
let a = genOid()
let b = a            # копия
let c = genOid()     # новый, другой OID

assert a == b        # одинаковые значения
assert a != c        # разные OID
assert not (a == c)
```

---

### `hash`

```nim
proc hash*(oid: Oid): Hash
```

Вычисляет хеш OID для использования в `Table`, `HashSet` и других хеш-контейнерах Nim. Хеш объединяет все три компонента, обеспечивая хорошее распределение.

Вызывать `hash` напрямую никогда не нужно — контейнер подхватывает его автоматически, когда `Oid` используется в качестве ключа или элемента множества.

```nim
import std/[oids, tables, sets]

# OID в роли ключей таблицы
var registry = initTable[Oid, string]()
let id1 = genOid()
let id2 = genOid()
registry[id1] = "первая запись"
registry[id2] = "вторая запись"
echo registry[id1]  # первая запись

# OID в хеш-множестве
var seen = initHashSet[Oid]()
seen.incl(id1)
assert id1 in seen
assert id2 notin seen
```

## Низкоуровневый вспомогательный инструмент

### `hexbyte`

```nim
proc hexbyte*(hex: char): int
```

Преобразует один шестнадцатеричный символ (`'0'`–`'9'`, `'a'`–`'f'`, `'A'`–`'F'`) в его целочисленное значение (0–15). Используется внутри `parseOid` для побайтового декодирования hex-строки.

Функция экспортирована, но в прикладном коде нужна редко. Доступна, если вы реализуете собственную логику сериализации или разбора OID.

```nim
assert hexbyte('0') == 0
assert hexbyte('9') == 9
assert hexbyte('a') == 10
assert hexbyte('f') == 15
assert hexbyte('A') == 10
assert hexbyte('F') == 15
```

## Полный пример

Следующий пример демонстрирует типичный жизненный цикл OID: генерацию, хранение в виде строки, восстановление через разбор, извлечение временно́й метки создания и использование в роли ключа таблицы.

```nim
import std/[oids, times, tables, sequtils]

# --- Генерация ---
let id = genOid()
echo "Новый OID: ", $id                                        # 24-символьный hex
echo "Создан:    ", id.generatedTime().format("yyyy-MM-dd")    # дата создания
echo "Длина:     ", ($id).len                                  # 24

# --- Сериализация и восстановление ---
let serialised = $id                   # сохранить в БД, JSON, журнал и т.д.
let restored   = parseOid(serialised)  # восстановить позже
assert id == restored
echo "Туда-обратно без потерь: ", id == restored  # true

# --- Использование как ключ таблицы ---
var store = initTable[Oid, string]()
for name in ["Алиса", "Боб", "Карол"]:
  store[genOid()] = name

for oid, name in store:
  echo $oid, "  ->  ", name

# --- Сортировка по времени создания ---
# Поскольку $oid big-endian и временна́я метка идёт первой,
# лексикографический порядок совпадает с хронологическим
# для OID, сгенерированных в одном процессе.
var ids: seq[Oid]
for _ in 1..5:
  ids.add(genOid())

let timestamps = ids.mapIt(it.generatedTime())
# Все временны́е метки равны или возрастают (точность — секунда)
for i in 1 ..< timestamps.len:
  assert timestamps[i] >= timestamps[i-1]
```

## Краткая сводка

| Символ | Сигнатура | Описание |
|---|---|---|
| `Oid` | `object` | 12-байтовый глобально уникальный идентификатор (тип-значение) |
| `genOid` | `()` → `Oid` | Сгенерировать новый уникальный OID |
| `$` | `Oid` → `string` | Сериализовать в 24-символьную строчную hex-строку |
| `parseOid` | `cstring` → `Oid` | Разобрать 24-символьную hex-строку обратно в OID |
| `generatedTime` | `Oid` → `Time` | Извлечь временну́ю метку создания (точность до секунды) |
| `==` | `Oid, Oid` → `bool` | True, если все три компонента идентичны |
| `hash` | `Oid` → `Hash` | Хеш для использования в `Table`, `HashSet` и т.д. |
| `hexbyte` | `char` → `int` | Преобразовать один hex-символ в целое число (0–15) |
