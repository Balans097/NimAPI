# std/tables — справочное руководство

## Общее описание

Модуль `tables` предоставляет три семейства обобщённых хеш-таблиц. Каждое семейство существует в двух вариантах — **тип-значение** и **ссылочный тип** — итого шесть конкретных типов.

| Семейство | Тип-значение | Ссылочный тип | Особенность |
|---|---|---|---|
| Обычная хеш-таблица | `Table[A, B]` | `TableRef[A, B]` | Неупорядоченная, O(1) в среднем |
| Таблица с сохранением порядка | `OrderedTable[A, B]` | `OrderedTableRef[A, B]` | Помнит порядок вставки |
| Таблица подсчёта | `CountTable[A]` | `CountTableRef[A]` | Сопоставляет элементы с числом вхождений |

### Семантика значений против семантики ссылок

Это самая важная концепция модуля. **Типы-значения** (`Table`, `OrderedTable`, `CountTable`) ведут себя как `int` или `string`: присваивание одной таблицы другой переменной создаёт полностью независимую копию. Изменение копии не затрагивает оригинал.

**Ссылочные типы** (`TableRef`, `OrderedTableRef`, `CountTableRef`) ведут себя как `ref object`: при присваивании копируется только указатель, обе переменные смотрят на одни и те же данные. Изменение через одну переменную немедленно видно через другую.

```nim
# Семантика значений — независимая копия
var a = {"x": 1, "y": 2}.toTable
var b = a          # глубокая копия
b["z"] = 3
assert "z" notin a # a не изменилась

# Семантика ссылок — общие данные
var c = {"x": 1, "y": 2}.newTable
var d = c          # тот же объект
d["z"] = 3
assert "z" in c    # c тоже изменилась
```

Используйте типы-значения по умолчанию. Ссылочные типы нужны, когда требуется разделить одну таблицу между несколькими переменными или передавать её без дорогостоящего копирования.

### Пользовательские типы ключей

Любой тип, используемый как ключ, должен иметь:
- процедуру `hash`, возвращающую `Hash` (из `std/hashes`)
- оператор `==`, согласованный с `hash` (стандартный `==` для объектов уже выполняет поэлементное сравнение)

```nim
import std/hashes
type Point = object
  x, y: int

proc hash(p: Point): Hash =
  result = p.x.hash !& p.y.hash
  result = !$result

var t = initTable[Point, string]()
t[Point(x: 1, y: 2)] = "около начала координат"
```

### Запрет на изменение таблицы во время итерации

Все итераторы проверяют, что длина таблицы не изменилась. Вставка или удаление во время итерации вызовет аварийное завершение (assertion failure). Если нужно изменить таблицу в процессе обхода, сначала соберите ключи в отдельный seq.

## Table / TableRef — Обычная хеш-таблица

### Создание

#### `initTable`

```nim
proc initTable*[A, B](initialSize = defaultInitialSize): Table[A, B]
```

Создаёт пустую таблицу типа `Table`. Начиная с Nim v0.20, переменная типа `Table`, объявленная как `var`, уже инициализируется нулями, поэтому явный вызов `initTable` необязателен. Он полезен, когда нужно заранее зарезервировать место и избежать ранних перераспределений памяти.

```nim
var t = initTable[string, int]()        # ёмкость по умолчанию
var t2 = initTable[string, int](128)    # подсказка: ~128 элементов до rehash
```

#### `toTable`

```nim
proc toTable*[A, B](pairs: openArray[(A, B)]): Table[A, B]
```

Преобразует массив или последовательность кортежей `(ключ, значение)` в новую таблицу за один вызов. Самый удобный способ создать заранее заполненную таблицу из литерала.

```nim
let colours = {"red": 0xFF0000, "green": 0x00FF00, "blue": 0x0000FF}.toTable
assert colours["red"] == 0xFF0000
```

#### `newTable` (TableRef)

```nim
proc newTable*[A, B](initialSize = defaultInitialSize): TableRef[A, B]
proc newTable*[A, B](pairs: openArray[(A, B)]): TableRef[A, B]
```

Ссылочные аналоги `initTable` и `toTable`. Возвращают таблицу, размещённую в куче за `ref`. Две переменные, присвоенные из одного `newTable`, разделяют одни данные.

```nim
let t = newTable([("a", 1), ("b", 2)])
let u = t            # тот же объект, не копия
u["c"] = 3
assert t["c"] == 3  # видно и через t
```

#### `newTableFrom`

```nim
proc newTableFrom*[A, B, C](collection: A, index: proc(x: B): C): TableRef[C, B]
```

Индексирует коллекцию с помощью функции извлечения ключа. Каждый элемент `collection` становится значением, а функция `index` вычисляет ключ. Удобно, когда есть список объектов и нужно искать их по какому-нибудь полю.

```nim
type User = object
  id: int
  name: string

let users = @[User(id: 1, name: "Alice"), User(id: 2, name: "Bob")]
let byId = newTableFrom(users, proc(u: User): int = u.id)
assert byId[1].name == "Alice"
```

### Чтение значений

#### `[]` — оператор индексирования

```nim
proc `[]`*[A, B](t: Table[A, B], key: A): lent B          # иммутабельный
proc `[]`*[A, B](t: var Table[A, B], key: A): var B        # мутабельный
```

Возвращает значение, хранящееся под ключом `key`. Если ключ отсутствует, выбрасывается `KeyError`. Используйте этот оператор, когда вы уверены в наличии ключа или хотите, чтобы ошибка распространилась выше.

```nim
let t = {"a": 10}.toTable
echo t["a"]          # 10
# echo t["z"]        # выбросит KeyError: key not found: z
```

Перегрузка для `var` возвращает мутабельную ссылку, позволяя изменять хранимое значение напрямую: `t["a"] += 5`.

#### `getOrDefault` — безопасное чтение с запасным значением

```nim
proc getOrDefault*[A, B](t: Table[A, B], key: A): B
proc getOrDefault*[A, B](t: Table[A, B], key: A, def: B): B
```

Возвращает значение по ключу, а если ключ отсутствует — запасное значение. Одноаргументная форма возвращает нулевое значение типа `B` (0 для целых, `""` для строк и т. д.). Двухаргументная позволяет указать любое запасное значение.

```nim
let t = {"a": 5}.toTable
assert t.getOrDefault("a") == 5
assert t.getOrDefault("z") == 0         # нулевое значение для int
assert t.getOrDefault("z", 99) == 99    # своё запасное значение
```

`getOrDefault` никогда ничего не вставляет в таблицу — это чистое чтение.

### Проверка наличия ключа

#### `hasKey` / `contains` / `in`

```nim
proc hasKey*[A, B](t: Table[A, B], key: A): bool
proc contains*[A, B](t: Table[A, B], key: A): bool
```

`hasKey` проверяет наличие ключа. `contains` — псевдоним, обеспечивающий идиоматичный синтаксис `key in t` (оператор `in` в Nim вызывает `contains`).

```nim
let t = {"a": 1, "b": 2}.toTable
assert t.hasKey("a")
assert "b" in t
assert "z" notin t
```

### Вставка и обновление

#### `[]=` — вставить или заменить

```nim
proc `[]=`*[A, B](t: var Table[A, B], key: A, val: sink B)
```

Основная операция записи. Если ключа нет, создаётся новая запись. Если есть, существующее значение заменяется. После использования `[]=` в таблице всегда не более одной записи на ключ.

```nim
var t = initTable[string, int]()
t["x"] = 1   # вставка
t["x"] = 2   # замена — теперь t["x"] == 2
```

#### `hasKeyOrPut` — вставить если отсутствует, сообщить о наличии

```nim
proc hasKeyOrPut*[A, B](t: var Table[A, B], key: A, val: B): bool
```

Возвращает `true`, если ключ уже был (таблица не меняется). Возвращает `false` и вставляет `(key, val)`, если ключа не было. Объединяет проверку существования и условную вставку в один вызов — нет риска случайно перезаписать только что установленное значение.

```nim
var t = {"a": 1}.toTable
assert t.hasKeyOrPut("a", 99) == true   # уже есть, 99 игнорируется
assert t.hasKeyOrPut("b", 99) == false  # вставлено со значением 99
assert t["b"] == 99
```

#### `mgetOrPut` — получить или вставить с мутабельным результатом

```nim
proc mgetOrPut*[A, B](t: var Table[A, B], key: A, val: B): var B
proc mgetOrPut*[A, B](t: var Table[A, B], key: A): var B
```

Возвращает мутабельную ссылку на значение по ключу. Если ключ отсутствует, сначала вставляется `val` (или нулевое значение в форме без аргумента). Основной сценарий использования — инициализация при первом обращении: `t.mgetOrPut(key, @[]).add(item)` позволяет накапливать элементы в список по ключу.

**Важный подводный камень:** присвоение результата в `var` создаёт копию для value-типов вроде `seq` или `string`. Всегда цепочите вызов напрямую, если хотите изменить хранимое значение.

```nim
var t = initTable[string, seq[int]]()

# Правильно: цепочка — изменяет seq в таблице
t.mgetOrPut("nums", @[]).add(42)
assert t["nums"] == @[42]

# Неправильно: создаёт копию — оригинал не изменится
var copy = t.mgetOrPut("nums2", @[])
copy.add(99)
assert t["nums2"] == @[]  # всё ещё пуст!
```

### Удаление записей

#### `del` — тихое удаление

```nim
proc del*[A, B](t: var Table[A, B], key: A)
```

Удаляет запись с данным ключом. Если ключ не существует, вызов — это no-op, ошибка не возникает. Используйте, когда отсутствие ключа — допустимое условие.

```nim
var t = {"a": 1, "b": 2}.toTable
t.del("a")
t.del("z")   # no-op
assert "a" notin t
```

#### `pop` — удалить и получить значение

```nim
proc pop*[A, B](t: var Table[A, B], key: A, val: var B): bool
```

Удаляет запись и, если она была найдена, помещает значение в `val` и возвращает `true`. Возвращает `false` и не меняет `val`, если ключ отсутствовал. Используйте `pop`, когда нужно атомарно «потребить» значение — извлечь из таблицы и использовать.

```nim
var t = {"a": 10}.toTable
var v: int
assert t.pop("a", v) == true and v == 10
assert t.pop("z", v) == false          # v по-прежнему 10
```

#### `take` — псевдоним для `pop`

```nim
proc take*[A, B](t: var Table[A, B], key: A, val: var B): bool
```

Идентичен `pop`. Предоставлен как альтернативное название, которое в некоторых контекстах читается естественнее.

#### `clear` — очистить таблицу

```nim
proc clear*[A, B](t: var Table[A, B])
```

Удаляет все записи, обнуляя счётчик. Внутренний массив слотов сохраняется, но все слоты помечаются как пустые. Ёмкость остаётся, таблицу можно переиспользовать без перераспределения памяти.

```nim
var t = {"a": 1, "b": 2}.toTable
t.clear()
assert t.len == 0
```

### Доступ на месте через `withValue`

#### `withValue` — двухветочная форма (только если ключ найден)

```nim
template withValue*[A, B](t: var Table[A, B], key: A, value, body: untyped)
template withValue*[A, B](t: Table[A, B], key: A, value, body: untyped)
```

Ищет `key`. Если найден, внедряет ссылку на хранимое значение как `value` и выполняет `body`. Если ключ отсутствует, `body` тихо пропускается. Это лучший способ прочитать и при необходимости обновить значение без его копирования из таблицы, особенно для типов, которые нельзя копировать.

```nim
var t = {"a": @[1, 2, 3]}.toTable
t.withValue("a", v):
  v[].add(4)       # изменяет seq прямо в таблице, без копирования
assert t["a"] == @[1, 2, 3, 4]
```

#### `withValue` — трёхветочная форма (найден / не найден)

```nim
template withValue*[A, B](t: var Table[A, B], key: A, value, body1, body2: untyped)
template withValue*[A, B](t: Table[A, B], key: A, value, body1, body2: untyped)
```

То же, что двухветочная форма, но принимает блок `do:` в качестве `body2`, который выполняется когда ключ отсутствует.

```nim
var t = initTable[string, int]()
t.withValue("x", v):
  echo "нашли: ", v[]
do:
  t["x"] = 0    # не найден: инициализируем
```

### Итераторы

Все итераторы для `Table` и `TableRef` обходят записи в **неопределённом (хешированном) порядке**. Не полагайтесь на конкретный порядок.

#### `pairs` / `mpairs`

```nim
iterator pairs*[A, B](t: Table[A, B]): (A, B)
iterator mpairs*[A, B](t: var Table[A, B]): (A, var B)
```

`pairs` выдаёт каждый кортеж `(ключ, значение)`. `mpairs` выдаёт мутабельные ссылки на значения, позволяя изменять их во время итерации.

```nim
var t = {"a": 1, "b": 2}.toTable
for k, v in t.mpairs:
  v *= 10
# t теперь {"a": 10, "b": 20}
```

#### `keys`

```nim
iterator keys*[A, B](t: Table[A, B]): lent A
```

Выдаёт только ключи.

#### `values` / `mvalues`

```nim
iterator values*[A, B](t: Table[A, B]): lent B
iterator mvalues*[A, B](t: var Table[A, B]): var B
```

`values` выдаёт иммутабельные ссылки на хранимые значения. `mvalues` — мутабельные.

---

### Операторы

#### `==`

```nim
proc `==`*[A, B](s, t: Table[A, B]): bool
```

Две таблицы равны, если содержат в точности одинаковые пары ключ-значение, независимо от порядка вставки. Для `TableRef` два значения `nil` считаются равными.

#### `$`

```nim
proc `$`*[A, B](t: Table[A, B]): string
```

Преобразует таблицу в человекочитаемую строку, например `{'a': 1, 'b': 2}`. Вызывается автоматически при `echo`.

#### `hash`

```nim
proc hash*[K, V](s: Table[K, V]): Hash
```

Вычисляет хеш всей таблицы (XOR хешей всех пар). Не зависит от порядка, поэтому две равные таблицы всегда дают одинаковый хеш. Позволяет использовать `Table` как ключ в другой таблице.

## OrderedTable / OrderedTableRef — Таблица с сохранением порядка

`OrderedTable` ведёт себя идентично `Table` для всех операций поиска и модификации. Единственное отличие: итерация всегда происходит в порядке первой вставки ключей. Внутренне хранится связный список индексов вставки рядом с хеш-таблицей.

### Ключевые отличия от `Table`

**`==` сравнивает порядок.** Два `OrderedTable` равны, только если содержат одинаковые пары _в одном и том же порядке вставки_.

```nim
let a = {'x': 1, 'y': 2}.toOrderedTable
let b = {'y': 2, 'x': 1}.toOrderedTable
assert a != b   # одни пары, разный порядок вставки
```

**`sort` переставляет порядок обхода.** После вызова `sort` все итераторы посещают записи в отсортированном порядке. `sort` помечает таблицу как переупорядоченную, но не уничтожает её. Повторные сортировки разрешены.

**`del` имеет сложность O(n).** Удаление из `OrderedTable` требует перевязки цепочки вставки, что в худшем случае занимает линейное время. Для частых удалений `Table` быстрее.

### Создание

#### `initOrderedTable` / `toOrderedTable`

```nim
proc initOrderedTable*[A, B](initialSize = defaultInitialSize): OrderedTable[A, B]
proc toOrderedTable*[A, B](pairs: openArray[(A, B)]): OrderedTable[A, B]
```

Семантика та же, что у их аналогов для `Table`. Пары вставляются в порядке массива, который и становится порядком итерации.

```nim
let ot = [('z', 1), ('y', 2), ('x', 3)].toOrderedTable
assert $ot == "{'z': 1, 'y': 2, 'x': 3}"  # z первый, как вставили
```

#### `newOrderedTable` (OrderedTableRef)

```nim
proc newOrderedTable*[A, B](initialSize = defaultInitialSize): OrderedTableRef[A, B]
proc newOrderedTable*[A, B](pairs: openArray[(A, B)]): OrderedTableRef[A, B]
```

Ссылочная версия. При присваивании данные разделяются.

---

### Сортировка

#### `sort` — переупорядочить цепочку вставки

```nim
proc sort*[A, B](t: var OrderedTable[A, B],
                 cmp: proc(x, y: (A, B)): int,
                 order = SortOrder.Ascending)
```

Переставляет записи так, чтобы последующая итерация посещала их в порядке, заданном `cmp`. Функция сравнения получает полные кортежи `(ключ, значение)`, давая полный контроль: сортировка по значению, по ключу, по комбинации и т. д.

**Предупреждение:** после сортировки таблицу нельзя изменять. Сортировка перевязывает внутренний связный список, но не запрещает последующие `[]=` или `del` — такие операции молча испортят порядок итерации.

```nim
var t = {"banana": 3, "apple": 1, "cherry": 2}.toOrderedTable
t.sort(proc(a, b: (string, int)): int = cmp(a[1], b[1]))
# Теперь итерирует по возрастанию значений: apple(1), cherry(2), banana(3)
for k, v in t:
  echo k, ": ", v
```

### Все остальные операции

`[]`, `[]=`, `hasKey`, `contains`, `getOrDefault`, `mgetOrPut`, `hasKeyOrPut`, `del`, `pop`, `clear`, `len`, `withValue`, `pairs`, `mpairs`, `keys`, `values`, `mvalues`, `$`, `==` — работают идентично аналогам в `Table`, с добавлением того, что `pairs`, `mpairs`, `keys`, `values` и `mvalues` посещают записи **в порядке вставки**.

## CountTable / CountTableRef — Таблица подсчёта

`CountTable[A]` — специализированная хеш-таблица, которая сопоставляет элементы типа `A` с целочисленными счётчиками. Тип значения всегда `int`, указывать его отдельно не нужно. Оптимизирован для типичного паттерна «сколько раз встречается каждый элемент». В отличие от `Table`, использует ноль как признак отсутствия, поэтому `t["отсутствующий"]` возвращает 0, а не выбрасывает исключение.

### Создание

#### `initCountTable` / `toCountTable`

```nim
proc initCountTable*[A](initialSize = defaultInitialSize): CountTable[A]
proc toCountTable*[A](keys: openArray[A]): CountTable[A]
```

`initCountTable` создаёт пустой счётчик. `toCountTable` подсчитывает вхождения каждого элемента в переданный массив или последовательность.

```nim
let freq = toCountTable("abracadabra")
# freq: {'a': 5, 'b': 2, 'r': 2, 'c': 1, 'd': 1}
assert freq['a'] == 5
assert freq['z'] == 0   # отсутствующий ключ даёт 0, не исключение
```

#### `newCountTable` (CountTableRef)

```nim
proc newCountTable*[A](initialSize = defaultInitialSize): CountTableRef[A]
proc newCountTable*[A](keys: openArray[A]): CountTableRef[A]
```

Ссылочная версия. `newCountTable(keys)` подсчитывает вхождения в `keys`, аналогично `toCountTable`.

---

### Чтение

#### `[]`

```nim
proc `[]`*[A](t: CountTable[A], key: A): int
```

Возвращает счётчик для `key`, или **0**, если ключ никогда не встречался. В отличие от `Table`, это не выбрасывает `KeyError` — ноль является естественным значением «ещё не видели».

#### `getOrDefault`

```nim
proc getOrDefault*[A](t: CountTable[A], key: A; def: int = 0): int
```

Как `[]`, но позволяет указать произвольное значение по умолчанию (вместо 0).

---

### Вставка и подсчёт

#### `[]=` — установить точный счётчик

```nim
proc `[]=`*[A](t: var CountTable[A], key: A, val: int)
```

Устанавливает счётчик `key` равным точно `val`. Значение должно быть неотрицательным (`assert val >= 0`). Установка в 0 удаляет ключ (так как 0 означает отсутствие).

#### `inc` — увеличить счётчик

```nim
proc inc*[A](t: var CountTable[A], key: A, val = 1)
```

Увеличивает счётчик `key` на `val` (по умолчанию 1). Если ключ отсутствует, он создаётся со значением `val`. Если после увеличения (при отрицательном `val`) счётчик становится 0, запись автоматически удаляется. Это основной способ накапливать статистику по одному элементу за раз.

```nim
var freq = initCountTable[char]()
for c in "hello world":
  freq.inc(c)
assert freq['l'] == 3
```

---

### Запросы о крайних значениях

#### `smallest` / `largest`

```nim
proc smallest*[A](t: CountTable[A]): tuple[key: A, val: int]  # O(n)
proc largest*[A](t: CountTable[A]): tuple[key: A, val: int]   # O(n)
```

Возвращают запись с минимальным или максимальным счётчиком соответственно. Оба просматривают всю таблицу за O(n). Если нужно запрашивать это многократно, вызовите `sort` один раз и итерируйте с начала или конца.

```nim
let freq = toCountTable("abracadabra")
assert freq.largest == ('a', 5)
assert freq.smallest.val == 1   # одна из: 'c', 'd'
```

---

### Сортировка

#### `sort` — отсортировать по счётчику

```nim
proc sort*[A](t: var CountTable[A], order = SortOrder.Descending)
```

Переставляет записи по значениям счётчиков. Порядок по умолчанию — убывающий (самый частый первым), что естественно для задачи «топ-N». После сортировки вызов `[]`, `inc`, `del` и любой мутации вызывает ошибку — внутренний флаг `isSorted` установлен. Сортировка необратима без очистки таблицы.

```nim
var freq = toCountTable("abracadabra")
freq.sort()
for k, v in freq:
  echo k, ": ", v
# a: 5
# b: 2
# r: 2
# c: 1
# d: 1
```

### Слияние

#### `merge` — сложить счётчики из другой таблицы

```nim
proc merge*[A](s: var CountTable[A], t: CountTable[A])
```

Добавляет все счётчики из `t` в `s`. Для каждого ключа из `t` его счётчик прибавляется к существующему в `s` (или вставляется, если отсутствует). Эквивалентно повторной вставке всех исходных элементов `t` в `s`.

```nim
var a = toCountTable("aab")   # a:2, b:1
let b = toCountTable("bc")    # b:1, c:1
a.merge(b)
assert a == toCountTable("aabbc")  # a:2, b:2, c:1
```

---

### Все остальные операции

`del`, `pop`, `clear`, `hasKey`, `contains`, `len`, `$`, `==`, `pairs`, `mpairs`, `keys`, `values`, `mvalues` — работают так же, как в `Table`, со счётчиками в качестве значений. Итераторы пропускают записи со счётчиком 0 (которые представляют отсутствующие ключи).

## Быстрая шпаргалка: выбор правильного типа

| Потребность | Используйте |
|---|---|
| Простое отображение ключ→значение, порядок не важен | `Table` |
| Нужно разделить таблицу / передать без копирования | `TableRef` |
| Важен порядок вставки (конфиги, меню, журналы) | `OrderedTable` |
| Нужна итерация в пользовательском порядке сортировки | `OrderedTable` + `sort` |
| Подсчёт вхождений элементов | `CountTable` |
| Топ-N самых частых элементов | `CountTable` + `sort` + итерация |

## Быстрая шпаргалка: безопасное получение значения

| Ситуация | Рекомендуемый вызов |
|---|---|
| Ключ точно существует | `t[key]` |
| Ключ может отсутствовать, нулевое умолчание подходит | `t.getOrDefault(key)` |
| Ключ может отсутствовать, своё умолчание | `t.getOrDefault(key, def)` |
| Нужно изменить на месте без копирования | `t.withValue(key, v): v[]...` |
| Вставить умолчание если отсутствует, потом изменить | `t.mgetOrPut(key, default).modify(...)` |
