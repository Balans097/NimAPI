# `cellsets.nim` — справочник модуля

> **Часть стандартной библиотеки времени выполнения Nim (внутренности сборщика мусора)**  
> Внутренний модуль GC — пользовательский код его не импортирует.

---

## Общее описание

`cellsets.nim` реализует `CellSet` — **центральную структуру данных сборщика мусора Nim**. `CellSet` — это множество указателей на память (называемых *ячейками*), которое GC использует для отслеживания всех живых аллокаций в куче. Каждый раз, когда GC нужно проверить, принадлежит ли указатель управляемой куче, или перечислить все управляемые объекты — он обращается к `CellSet`.

Определяющее требование — экстремальная производительность. GC, тратящий слишком много времени на отслеживание аллокаций, бесполезен. Поэтому модуль реализует специализированную гибридную структуру, а не использует контейнер общего назначения.

### Гибридная структура: хеш-таблица битовых множеств

Конструкция описана во вступительном комментарии модуля и заслуживает понимания перед чтением отдельных процедур:

```
Хеш-таблица
 ┌──────────────────────────────┐
 │ [слот 0] → PageDesc          │   PageDesc
 │ [слот 1] → nil               │  ┌──────────────────────────┐
 │ [слот 2] → PageDesc ─────────┼─►│ key   = базовый адрес    │
 │ ...                          │  │ bits  = [64 целых чисел]  │
 └──────────────────────────────┘  │ next  = →PageDesc (список)│
                                    └──────────────────────────┘
```

Память разделена на фиксированные **страницы** (определяемые `PageSize` / `PageShift` из `bitmasks.nim`). Для каждой страницы, содержащей хотя бы один управляемый объект, создаётся `PageDesc`. `PageDesc` содержит **битовый вектор**: один бит на каждую возможную позицию ячейки в пределах страницы. Если бит *k* установлен, то ячейка в позиции *k* — живой управляемый объект.

Такой дизайн одновременно даёт три ценных свойства:
- **O(1) для вставки, удаления и поиска** — указатель находится за три шага: поиск страницы в хеш-таблице, затем две битовые операции.
- **Компактное хранение** — страница с 512 ячейками требует лишь 512 бит (64 байта) битового хранилища, а не 512 указателей.
- **Быстрый обход** — итерация всех живых объектов — это плотный цикл по словам каждого битового вектора с мгновенным пропуском нулевых слов.

### Варианты времени компиляции

Модуль адаптируется к активному GC:

- **ARC/ORC/AtomicArc**: определяется псевдоним типа `PCell = Cell`, а `bitmasks.nim` включается для констант страниц.
- **Классический GC (refc и др.)**: вместо него включается `cellseqs_v1`, предоставляющий дополнительное отслеживание ячеек на основе последовательностей, используемое старым GC.

## Определения типов

### `PageDesc`

```nim
type
  PageDesc {.final, pure.} = object
    next: PPageDesc    # интрузивный связный список — все дескрипторы образуют цепь
    key:  uint         # базовый адрес страницы (адрес ячейки >> PageShift)
    bits: array[BitIndex, int]  # один бит на каждую ячейку в этой странице
```

Основная запись данных. Каждый `PageDesc` представляет одну страницу памяти ОС и хранит информацию о том, какие адреса ячеек в пределах этой страницы в данный момент живы. Массив `bits` — это само битовое множество: каждое слово размером `sizeof(int)*8` бит содержит один бит на каждую потенциальную позицию ячейки.

Указатель `next` используется для **интрузивного связного списка** (`head` в `CellSet`), позволяющего итератору `elements` обходить все дескрипторы страниц без сканирования всей хеш-таблицы.

`BitIndex` — это диапазон `[0..UnitsPerPage-1]`, ограничивающий доступ к массиву допустимыми индексами.

---

### `CellSet`

```nim
type
  CellSet {.final, pure.} = object
    counter: int        # количество записей PageDesc в хеш-таблице
    max:     int        # ёмкость хеш-таблицы минус единица (всегда 2^n - 1)
    head:    PPageDesc  # голова интрузивного связного списка всех PageDesc
    data:    PPageDescArray  # сама хеш-таблица (открытая адресация)
```

Контейнер верхнего уровня. Его хеш-таблица (`data`) отображает адреса страниц на записи `PageDesc`. Связный список `head` обеспечивает быстрый обход всех дескрипторов без сканирования пустых слотов.

Ёмкость всегда является степенью двойки (`max = 2^n - 1`), что делает модульную индексацию единственной операцией побитового AND.

## Процедуры жизненного цикла

### `init`

```nim
proc init(s: var CellSet)
```

Инициализирует `CellSet` для использования. Выделяет начальную хеш-таблицу из `InitCellSetSize` (1024) слотов, все обнулённые. Устанавливает `counter` в ноль, `head` — в nil.

Должна вызываться перед любой другой операцией над `CellSet`. Начальный размер таблицы в 1024 слота — степень двойки, выбранная для предотвращения раннего перехеширования в типичных программах: у большинства программ с управлением GC одновременно живых страниц значительно меньше тысячи.

```nim
var cs: CellSet
init(cs)
# cs теперь готов к использованию
```

---

### `deinit`

```nim
proc deinit(s: var CellSet)
```

Освобождает всю память, принадлежащую `CellSet`. Проходит по интрузивному связному списку от `head` и вызывает `dealloc` для каждого `PageDesc`, затем освобождает массив хеш-таблицы `data`. Сбрасывает все поля в безопасные нулевые/nil-значения.

Вызывается при завершении работы GC или при перестройке его структур отслеживания. Дескрипторы страниц *никогда* не удаляются из таблицы по отдельности в ходе нормальной работы (см. архитектурное примечание выше); `deinit` — единственная точка, где они освобождаются.

```nim
deinit(cs)
# Вся память возвращена аллокатору; cs нельзя использовать
# без повторного вызова init.
```

## Внутренние процедуры хеш-таблицы

Эти процедуры реализуют механику самой хеш-таблицы. Они не являются частью публичного интерфейса `CellSet` — это двигатель под капотом.

---

### `nextTry`

```nim
proc nextTry(h, maxHash: int): int {.inline.}
```

Вычисляет **следующую позицию зондирования** при разрешении коллизий с открытой адресацией. Использует формулу `(5*h + 1) and maxHash`.

Это хорошо известная последовательность зондирования. Когда `maxHash` равен `2^n - 1`, начиная с любого начального `h` и многократно применяя эту формулу, посещается **каждый слот таблицы ровно один раз** перед возвратом к `h`. Это свойство полного цикла гарантирует завершение вставки (пока есть хотя бы один пустой слот) и завершение поиска (как только обнаружится nil-слот).

Константа `5` выбрана потому, что она взаимно проста с любой степенью двойки — это математическое требование для покрытия полного цикла.

```
# Визуализация для таблицы из 8 слотов (maxHash = 7):
# Начало с h=3: 3 → 0 → 1 → 6 → 7 → 4 → 5 → 2 → 3 (полный цикл)
```

---

### `cellSetGet`

```nim
proc cellSetGet(t: CellSet, key: uint): PPageDesc
```

**Поиск** — находит `PageDesc` для адреса страницы `key` в хеш-таблице. Возвращает дескриптор, если найден, или `nil`, если страница не отслеживается.

Цикл зондирования завершается на `nil` (слот пуст, значит ключ никогда не вставлялся). Один и тот же слот никогда не посещается дважды благодаря свойству полного цикла `nextTry`.

Вызывается в `contains`, `excl` и `containsOrIncl` — в любой операции, которой нужно найти существующую страницу без создания новой.

```nim
// Используется внутри, например:
let desc = cellSetGet(cs, pageAddress)
if desc != nil:
  # эта страница известна — проверяем отдельные биты
```

---

### `cellSetRawInsert`

```nim
proc cellSetRawInsert(t: CellSet, data: PPageDescArray, desc: PPageDesc)
```

**Низкоуровневая вставка** — помещает существующий `PageDesc` (с уже установленным `desc.key`) в заданный массив хеш-таблицы `data`. Не изменяет `t.counter` и указатели списка. Использует `sysAssert` для проверки инварианта: дескриптор ещё не присутствует в таблице, а выбранный слот действительно пуст.

Используется исключительно в `cellSetEnlarge` при перехешировании таблицы, когда дескрипторы из старой таблицы нужно вставить в новую, большую.

---

### `cellSetEnlarge`

```nim
proc cellSetEnlarge(t: var CellSet)
```

**Перехеширование** — удваивает ёмкость хеш-таблицы, когда она становится слишком заполненной. Выделяет новую таблицу вдвое большего размера, вставляет все существующие записи `PageDesc` с помощью `cellSetRawInsert`, затем освобождает старую таблицу.

Порог изменения размера проверяется в `cellSetPut`: таблица расширяется, когда выполняется одно из условий:
- Количество записей превышает 2/3 ёмкости (коэффициент заполнения > 0,67), или
- Остаётся менее 4 свободных слотов.

Это обеспечивает достаточную разреженность хеш-таблицы, чтобы цепочки зондирования оставались короткими и сохранялась ожидаемая сложность O(1) для поиска.

---

### `cellSetPut`

```nim
proc cellSetPut(t: var CellSet, key: uint): PPageDesc
```

**Найти или создать** — основной путь записи. Находит `PageDesc` для страницы `key` и возвращает его. Если дескриптора для этой страницы не существует, выделяет новый обнулённый `PageDesc`, связывает его в список `head`, вставляет в хеш-таблицу, увеличивает `counter` и возвращает его.

Это единственное место, где создаются новые объекты `PageDesc`. С точки зрения вызывающего кода все пять шагов происходят атомарно:
1. Проверить, существует ли ключ → вернуть его при наличии.
2. Проверить, нужно ли расширить таблицу → расширить при необходимости.
3. Выделить новый `PageDesc` (обнуление через `alloc0`).
4. Добавить в начало интрузивного связного списка `head`.
5. Вставить в хеш-таблицу.

## Процедуры операций над множеством

Это содержательный высокоуровневый интерфейс `CellSet`. Они выполняют декомпозицию адреса — разбивая указатель на номер страницы и индекс бита — и делегируют работу слою хеш-таблицы.

Декомпозиция адреса для любого указателя ячейки `cell` выглядит так:

```
ключ_страницы  = cast[uint](cell) >> PageShift         → какой PageDesc искать
смещение_бита  = (cast[uint](cell) mod PageSize) div MemAlign  → какой бит в странице
индекс_слова   = смещение_бита >> IntShift             → какой int в bits[]
индекс_бита    = смещение_бита and IntMask             → какой бит внутри этого int
```

---

### `contains`

```nim
proc contains(s: CellSet, cell: PCell): bool
```

Возвращает `true`, если `cell` в данный момент является членом множества. Запрашивает хеш-таблицу для дескриптора страницы, затем проверяет конкретный бит, соответствующий смещению `cell` в пределах этой страницы. Немедленно возвращает `false`, если для страницы нет дескриптора (то есть ни одна ячейка с этой страницы никогда не вставлялась).

Это чистый запрос — он никогда не изменяет множество.

```nim
if contains(liveSet, somePointer):
  # этот указатель отслеживается GC
```

---

### `incl`

```nim
proc incl(s: var CellSet, cell: PCell)
```

Добавляет `cell` в множество. Использует `cellSetPut` для поиска или создания дескриптора страницы, затем **устанавливает** соответствующий бит. Если дескриптор страницы уже существовал, меняется только один бит. Если он новый — весь дескриптор выделяется и обнуляется, затем устанавливается единственный бит.

Имя `incl` отражает стандартную операцию Nim над множеством.

```nim
incl(rootSet, newlyAllocatedPointer)
# Указатель теперь отслеживается
```

---

### `excl`

```nim
proc excl(s: var CellSet, cell: PCell)
```

Удаляет `cell` из множества, **сбрасывая** его бит. Если для страницы ячейки нет дескриптора, вызов является безопасной no-op — никакого исключения, никакого краша.

Важно: `excl` **никогда не удаляет дескрипторы страниц**, даже если все биты в дескрипторе становятся нулевыми. В архитектурном разделе объясняется почему: GC периодически перестраивает свои структуры, поэтому накладные расходы на удаление пустых дескрипторов не оправдывают сложности.

```nim
excl(rootSet, freedPointer)
# Бит сброшен; PageDesc остаётся
```

---

### `containsOrIncl`

```nim
proc containsOrIncl(s: var CellSet, cell: PCell): bool
```

**Атомарное тестирование и установка.** Возвращает `true`, если `cell` уже находился в множестве; возвращает `false`, если не находился (и в этом случае добавляет его). Объединяет проверку членства и условную вставку в единую эффективную операцию.

Это эффективнее, чем отдельный вызов `contains` с последующим `incl`, потому что когда дескриптор страницы уже существует, бит одновременно тестируется и устанавливается за один проход через `cellSetGet` — второй поиск по хешу не нужен.

Эта операция является центральной для фазы маркировки GC: в ходе маркировки каждый достижимый указатель должен быть посещён ровно один раз. `containsOrIncl` предоставляет точную семантику «пометить, если ещё не помечено, сообщить, если уже помечено», которая делает это эффективным.

```nim
if not containsOrIncl(markedSet, reachablePtr):
  # Первый раз видим этот указатель — помещаем его дочерние в очередь сканирования
  pushChildren(reachablePtr)
# Если вернул true — уже обрабатывали этот указатель, пропускаем
```

## Итераторы

### `elements`

```nim
iterator elements(t: CellSet): PCell {.inline.}
```

Возвращает по одному каждый указатель ячейки, находящийся в данный момент в множестве. Обходит все дескрипторы страниц через связный список `head`, затем для каждого дескриптора сканирует все слова массива `bits`. Для каждого ненулевого слова сканирует все установленные биты и восстанавливает исходный адрес ячейки.

Восстановление адреса является обратным к декомпозиции:

```
адрес_ячейки = (ключ_страницы << PageShift) | (индекс_слова << IntShift + индекс_бита) * MemAlign
```

**Критическое ограничение:** *изменение множества во время итерации является неопределённым поведением.* Итератор делает снимок каждого слова `bits` (`var w = r.bits[i]`), а не читает живую память — это одновременно оптимизация и мера безопасности, но она не защищает от структурных изменений в самом списке дескрипторов страниц.

Этот итератор используется GC для перечисления всех живых корней и всех достижимых объектов в ходе цикла сборки.

```nim
for cell in elements(liveSet):
  # обрабатываем каждый отслеживаемый указатель ровно один раз
  markAndScan(cell)
```

---

### `elementsExcept`

```nim
iterator elementsExcept(t, s: CellSet): PCell {.inline.}
```

Возвращает по одному каждый указатель ячейки, который есть в множестве `t`, но **отсутствует** в множестве `s`. Это итератор разности множеств: `t \ s`.

Реализация эффективна: она не строит временное множество разности. Для каждого дескриптора страницы в `t` ищется соответствующий дескриптор в `s`. Если он найден, биты дескриптора `s` вычитаются из бит `t` с помощью побитового AND-NOT (`w and not ss.bits[i]`), получая слово только с битами «есть в `t`, но нет в `s`». Если в `s` нет дескриптора для этой страницы, возвращаются все биты дескриптора `t`.

Используется GC для поиска объектов, выделенных с момента последнего цикла GC — фактически вычисляя `выделеноВТекущемЦикле \ ужеОтслеживаемое`.

```nim
for newCell in elementsExcept(currentAllocs, previousAllocs):
  # Этот указатель новый — добавляем его в корневое множество
  incl(rootSet, newCell)
```

## Мёртвый код: `CellSetIter` и `next`

```nim
when false:
  type CellSetIter = object ...
  proc next(it: var CellSetIter): PCell = ...
  proc init(it: var CellSetIter; t: CellSet): PCell = ...
```

Этот блок охраняется `when false` и никогда не компилируется. Он представляет **итератор с явным состоянием** — альтернативу синтаксису `iterator`, позволяющую хранить состояние итерации в объекте и продвигаться на один шаг за раз. Это полезно в контекстах, где цикл `for` использовать нельзя (например, в корутинах или при ручном чередовании двух итераций).

Скорее всего, он сохранён как справочный материал для будущего использования — возможно, для бэкенда NIR или другого контекста, где форма с `iterator` неприменима.

---

## Внутренняя константа: `InitCellSetSize`

```nim
const InitCellSetSize = 1024
```

Начальное количество слотов в массиве хеш-таблицы. Обязательно должно быть степенью двойки (это подчёркнуто в комментарии). Выбрано достаточно большим, чтобы типичные программы не нуждались в раннем перехешировании, но достаточно малым для избежания расточительной аллокации в краткоживущих программах.
