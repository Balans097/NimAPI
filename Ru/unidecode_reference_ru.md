# Модуль Unidecode — Справочник

> **Происхождение модуля:** стандартная библиотека Nim. Портирована из Python-библиотеки [Unidecode](https://pypi.org/project/Unidecode/) авторства Томаша Сольца, которая, в свою очередь, основана на Perl-модуле [Text::Unidecode](https://metacpan.org/pod/Text::Unidecode) Шона М. Бёрка.

---

## Что делает этот модуль

`unidecode` решает конкретную практическую задачу: как представить Unicode-строку — китайские иероглифы, немецкие умляуты, арабские буквы, кириллический текст — используя только 128 символов ASCII?

Модуль отвечает на этот вопрос, **транслитерируя** каждый Unicode-символ в ближайший ASCII-эквивалент. Преобразование **необратимо** (одностороннее) и **детерминировано**: одинаковый ввод всегда даёт одинаковый вывод, однако разные Unicode-строки могут давать один и тот же ASCII-результат. Несмотря на потерю информации, человек, как правило, способен восстановить исходное слово по контексту.

Для работы используется заранее собранная таблица перевода (`unidecode.dat`), которая отображает каждую кодовую точку Unicode до `0xFFFF` на соответствующий ASCII-аналог. По умолчанию этот файл **встраивается прямо в бинарник** при компиляции, поэтому распространять его вместе с программой не нужно.

## Экспортируемые процедуры

### `unidecode`

```nim
proc unidecode*(s: string): string
```

#### Описание

Принимает строку `s` в кодировке UTF-8 и возвращает новую строку, содержащую только символы ASCII (кодовые точки 0–127). Функция перебирает каждый Unicode-руну (символ) во входной строке:

- Если кодовая точка руны **уже находится в диапазоне ASCII** (≤ 127), она копируется в результат без изменений.
- Если кодовая точка **больше 127**, функция ищет соответствие в таблице перевода и добавляет к результату найденную ASCII-последовательность. Если для данной кодовой точки соответствия нет (например, некоторые редкие или символы частного пользования), эта руна **молча пропускается** — никакой ошибки не возникает.

Результат всегда является корректной строкой в чистом ASCII.

#### Важные особенности

- **Необратимость.** Обратной функции не существует. `"Ä"` → `"A"`, но программно восстановить `"Ä"` из `"A"` невозможно.
- **Пробелы в транслитерациях.** Некоторые письменности (например, китайская) используют пробелы как разделители между слогами в ASCII-представлении. Это может приводить к лишним или завершающим пробелам в выводе.
- **Молчаливое отбрасывание.** Символы, для которых нет транслитерации, удаляются без вывода ошибки. Нельзя рассчитывать на то, что длина результата будет пропорциональна длине ввода.
- **Только символы BMP.** Таблица охватывает кодовые точки Unicode до `0xFFFF` (Основная многоязычная плоскость). Кодовые точки выше этого диапазона молча игнорируются.

#### Параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `s` | `string` | Входная строка в кодировке UTF-8, любой длины. |

#### Возвращаемое значение

`string`, содержащая только символы ASCII.

#### Примеры

```nim
import unidecode

# Латинские буквы с диакритикой → ближайшие ASCII-буквы
doAssert unidecode("Äußerst") == "Ausserst"
# "Ä" → "A", "ü" → "u", "ß" → "ss", остальные буквы — без изменений

# Китайские иероглифы → пиньинь с пробелами
doAssert unidecode("北京") == "Bei Jing "
# Каждый иероглиф отображается в свой слог пиньиня; обратите внимание на пробел в конце

# Чистый ASCII возвращается без изменений
doAssert unidecode("Hello, world!") == "Hello, world!"

# Кириллица
echo unidecode("Привет")   # → "Privet"

# Японская хирагана
echo unidecode("にほんご")  # → "nihongo"

# Арабский
echo unidecode("مرحبا")    # → "mrHb'" (приближённая романизация)

# Смешанные письменности
echo unidecode("Zürich 北京 Москва")
# → "Zurich Bei Jing  Moskva"
```

#### Когда использовать

- Генерация **URL-слагов** из Unicode-заголовков.
- Создание **имён файлов**, безопасных в ASCII-файловых системах.
- **Индексирование и поиск** текста в системах без поддержки Unicode-сортировки.
- Создание **читаемых идентификаторов** из иноязычного ввода.

### `loadUnidecodeTable`

```nim
proc loadUnidecodeTable*(datafile = "unidecode.dat")
```

#### Описание

Загружает таблицу перевода из внешнего файла на диске, вместо использования версии, скомпилированной в бинарник. Эта процедура **имеет смысл** (и **производит какой-либо эффект**) только тогда, когда программа скомпилирована с флагом:

```
--define:noUnidecodeTable
```

Без этого флага таблица перевода встраивается на этапе компиляции как константа, и вызов `loadUnidecodeTable` является **пустой операцией** — вызов компилируется и выполняется без ошибки, но не производит никакого эффекта.

Когда флаг `--define:noUnidecodeTable` **установлен**, таблица не встраивается и модуль стартует с пустой последовательностью. В этом случае **необходимо** вызвать `loadUnidecodeTable` до любого обращения к `unidecode`, иначе `unidecode` будет молча возвращать пустые строки для всех не-ASCII символов.

#### Требование к многопоточности

`loadUnidecodeTable` должна вызываться из **главного потока** до того, как любой другой поток вызовет `unidecode`. Таблица перевода — это переменная уровня модуля, общая для всех потоков. Инициализация её из рабочего потока, пока другие потоки могут читать её, была бы гонкой данных. Nim намеренно помечает переменную как `shared`, чтобы разрешить безопасное конкурентное чтение после начальной загрузки.

#### Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `datafile` | `string` | `"unidecode.dat"` | Путь к файлу данных. Может быть относительным или абсолютным. |

#### Возвращаемое значение

Отсутствует (`void`).

#### Примеры

```nim
# --- Компиляция: nim c --define:noUnidecodeTable myapp.nim ---

import unidecode

# Обязательный вызов один раз при старте программы, в главном потоке:
loadUnidecodeTable()                         # путь по умолчанию "unidecode.dat"
# или с явным путём:
loadUnidecodeTable("/opt/data/unidecode.dat")

echo unidecode("Привет")  # → "Privet"  (работает только после загрузки таблицы)
```

```nim
# Если скомпилировано БЕЗ --define:noUnidecodeTable, вызов безвреден, но бесполезен:
import unidecode

loadUnidecodeTable()  # пустая операция; таблица уже встроена в бинарник
echo unidecode("Ångström")  # → "Angstrom"  (работает в любом случае)
```

#### Когда использовать

Используйте `loadUnidecodeTable` совместно с `--define:noUnidecodeTable`, когда:

- Нужно **уменьшить размер бинарника**, не встраивая таблицу данных (~1 МБ).
- Требуется **распространять или обновлять** таблицу перевода независимо от исполняемого файла.
- Строится система, в которой путь к файлу данных определяется **во время выполнения** (например, из конфигурационного файла).

## Флаги компиляции

| Флаг | Эффект |
|------|--------|
| *(не задан)* | Таблица перевода встраивается в бинарник. Внешний файл не нужен. `loadUnidecodeTable` — пустая операция. |
| `--define:noUnidecodeTable` | Таблица перевода **не встраивается**. Необходимо вызвать `loadUnidecodeTable` перед использованием `unidecode`. |

---

## Сводка ограничений

- Охватываются только кодовые точки Unicode до `U+FFFF`.
- Преобразование строго одностороннее.
- Символы без записи в таблице транслитерации молча отбрасываются.
- Транслитерации приближённые — результат читаем человеком, но лингвистически неточен.
- Длина результата непредсказуема относительно длины ввода.
