# effecttraits — справочник модуля

## Обзор

Модуль `effecttraits` предоставляет авторам макросов **программный доступ к метаданным системы эффектов**, которые компилятор Nim выводит для каждой процедуры, функции, конвертера, итератора или метода. С его помощью можно задавать — во время компиляции, прямо внутри тела макроса — такие вопросы:

- *Какие исключения может бросить эта функция?*
- *Какие теги эффектов она несёт?*
- *Какие теги она явно запрещает своим вызовам?*
- *Безопасна ли она для GC (gcsafe)?*
- *Является ли она чистой функцией без побочных эффектов?*

Это открывает целый класс мощных метапрограммных приёмов: генерацию обёрток, автоматически наследующих эффекты оригинала; применение собственных политик эффектов; построение интроспективных утилит — и всё это без изменения инспектируемых процедур.

**Доступность:** Nim 1.4 и выше. На месте вызова можно использовать защитный блок:

```nim
when defined(nimHasEffectTraitsModule):
  import std/effecttraits
```

**Критическое ограничение, общее для всех пяти процедур:** Аргумент `fn` должен быть **узлом разрешённого символа** (`nnkSym`). Это означает, что вызывающий макрос обязан принимать параметры с типом **`typed`**, а не `untyped`. Параметр `typed` указывает компилятору разрешить аргумент — найти его символ в таблице символов — до передачи в макрос. Передача неразрешённого `untyped`-узла вызовет падение compile-time-проверки.

**Замечание о бэкенде:** Все пять процедур делегируют вызов внутренним VM-операциям компилятора, определённым в `compiler/vmops.nim`. Их тела на уровне Nim — намеренно недостижимые заглушки; настоящая реализация живёт внутри компилятора и вызывается только в процессе разворачивания макроса.

## `getRaisesList`

```nim
proc getRaisesList*(fn: NimNode): NimNode
```

### Описание

Возвращает **список эффектов `.raises`**, который компилятор вывел (или который был явно аннотирован) для процедуры `fn`. Возвращаемый узел — это `NimNode`, представляющий скобочный список типов исключений — ту же структуру данных, что появляется в секции прагм процедуры при записи `.raises: [IOError, ValueError]`.

Если компилятор установил, что `fn` ничего не бросает, возвращается пустой список. Если эффекты raises не ограничены (процедура никогда не аннотировалась, а вывод не сумел их сузить), список отражает это неограниченное состояние.

Главный сценарий применения — генерация обёрток, которые должны нести тот же `.raises`-контракт, что и обёртываемая функция, или compile-time-проверка, что коллбэк, переданный в обобщённый фреймворк, не бросает определённые исключения.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `fn` | `NimNode` | Узел разрешённого символа (`nnkSym`) инспектируемой процедуры |

### Возвращаемое значение

`NimNode` — конкретно узел, который можно вставить прямо в генерируемую прагму или использовать для дальнейшей инспекции AST. Его форма зеркалит то, что компилятор хранит внутри для аннотаций `.raises`.

### Пример

```nim
import std/macros
import std/effecttraits

# Процедура с ограниченным списком raises
proc riskyOp() {.raises: [IOError].} =
  raise newException(IOError, "disk full")

# Процедура, которая ничего не бросает
proc safeOp() {.raises: [].} =
  discard

macro inspectRaises(fn: typed): untyped =
  let raises = getRaisesList(fn)
  echo "AST списка raises: ", raises.treeRepr
  result = newEmptyNode()

inspectRaises(riskyOp)
# Вывод (во время компиляции):
# AST списка raises: BracketExpr
#   Sym "IOError"

inspectRaises(safeOp)
# Вывод (во время компиляции):
# AST списка raises: BracketExpr   ← пустые скобки, нет потомков
```

Более практичный пример — проверка, что коллбэк безопасен перед встраиванием в критическую секцию:

```nim
macro requireNoRaises(fn: typed): untyped =
  let raises = getRaisesList(fn)
  if raises.len > 0:
    error("Коллбэк не должен бросать исключений, однако raises: " & raises.repr, fn)
  result = newEmptyNode()

proc safe() {.raises: [].} = discard
proc unsafe() {.raises: [IOError].} = discard

requireNoRaises(safe)    # OK
requireNoRaises(unsafe)  # Ошибка компиляции
```

## `getTagsList`

```nim
proc getTagsList*(fn: NimNode): NimNode
```

### Описание

Возвращает **список эффектов `.tags`**, ассоциированный с `fn`. Теги — это пользовательские или стандартные маркеры эффектов, помечающие процедуру как обладающую определёнными наблюдаемыми поведениями — например `ReadIOEffect`, `WriteIOEffect`, `ReadDirEffect` или любой тег, определённый пользователем. Они объявляются через `.tags: [ТегA, ТегB]` в прагмах процедуры.

Теги служат механизмом документирования и контроля: вызывающая сторона, аннотированная ограниченным набором тегов, не может вызвать процедуру, вносящую тег вне этого набора. `getTagsList` позволяет макросам инспектировать этот набор и принимать решения на его основе.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `fn` | `NimNode` | Узел разрешённого символа (`nnkSym`) инспектируемой процедуры |

### Возвращаемое значение

`NimNode` со списком типов тегов — по структуре аналогичен списку `.raises`.

### Пример

```nim
import std/macros
import std/effecttraits

proc readConfig() {.tags: [ReadIOEffect].} =
  discard

proc pureCompute() {.tags: [].} =
  discard

macro checkTags(fn: typed): untyped =
  let tags = getTagsList(fn)
  echo fn.repr, " имеет ", tags.len, " тег(ов): ", tags.repr
  result = newEmptyNode()

checkTags(readConfig)
# readConfig имеет 1 тег(ов): [ReadIOEffect]

checkTags(pureCompute)
# pureCompute имеет 0 тег(ов): []
```

Практический шаблон — принудительная проверка, что в таблицу обработчиков «только для чтения» регистрируются лишь подходящие процедуры:

```nim
macro registerReadOnly(fn: typed): untyped =
  let tags = getTagsList(fn)
  for tag in tags:
    if tag.repr == "WriteIOEffect":
      error("Обработчик только для чтения не должен иметь WriteIOEffect", fn)
  # далее — генерация кода регистрации...
  result = newEmptyNode()
```

## `getForbidsList`

```nim
proc getForbidsList*(fn: NimNode): NimNode
```

### Описание

Возвращает **список эффектов `.forbids`** процедуры `fn`. Прагма `.forbids` является противоположностью `.tags`: если `.tags` объявляет, какие эффекты процедура *вносит*, то `.forbids` объявляет, какие эффекты она *запрещает в своих вызовах*. Процедура, аннотированная `.forbids: [WriteIOEffect]`, вызовет ошибку компилятора при попытке вызвать любую процедуру, несущую `WriteIOEffect` в своём списке тегов.

`getForbidsList` позволяет коду макроса прочитать этот список и принять метарешения: например, убедиться, что оборачиваемый коллбэк подчиняется тем же ограничениям, что и собственная политика обёртки.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `fn` | `NimNode` | Узел разрешённого символа (`nnkSym`) инспектируемой процедуры |

### Возвращаемое значение

`NimNode` со списком запрещённых типов эффектов.

### Пример

```nim
import std/macros
import std/effecttraits

proc strictReader() {.forbids: [WriteIOEffect].} =
  discard

proc relaxedReader() =
  discard

macro showForbids(fn: typed): untyped =
  let forbids = getForbidsList(fn)
  echo fn.repr, " запрещает: ", forbids.repr
  result = newEmptyNode()

showForbids(strictReader)
# strictReader запрещает: [WriteIOEffect]

showForbids(relaxedReader)
# relaxedReader запрещает: []
```

Совместная инспекция — проверка совместимости плагина с хостом, запрещающим запись:

```nim
macro ensureCompatible(plugin: typed): untyped =
  # Хост запрещает WriteIOEffect; плагин не должен его вносить
  let tags = getTagsList(plugin)
  for tag in tags:
    if tag.repr == "WriteIOEffect":
      error("Плагин вносит WriteIOEffect, который хост запрещает", plugin)
  result = newEmptyNode()
```

## `isGcSafe`

```nim
proc isGcSafe*(fn: NimNode): bool
```

### Описание

Возвращает `true`, если компилятор установил, что `fn` является **GC-безопасной** — то есть не обращается к управляемой GC памяти через глобальные или замкнутые переменные способом, который был бы небезопасен в многопоточном контексте. Это соответствует прагме `{.gcsafe.}`.

GC-безопасность — необходимое условие для передачи процедуры новому потоку через `createThread` или `spawn`. Инспекция этого свойства в макросе позволяет обусловить генерацию потокового кода фактической безопасностью переданного коллбэка, выдав понятную ошибку компиляции — вместо труднодиагностируемой ошибки компоновщика или рантайма.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `fn` | `NimNode` | Узел разрешённого символа (`nnkSym`) инспектируемой процедуры |

### Возвращаемое значение

`true`, если `fn` является GC-безопасной; `false` в противном случае.

### Пример

```nim
import std/macros
import std/effecttraits

proc threadSafe() {.gcsafe.} =
  discard

proc notThreadSafe() =
  var x = @[1, 2, 3]  # захватывает кучу GC; не gcsafe в общем случае
  discard x

macro spawnSafe(fn: typed): untyped =
  if not isGcSafe(fn):
    error("Невозможно запустить в потоке: процедура не является GC-безопасной", fn)
  # генерация вызова spawn...
  result = newEmptyNode()

spawnSafe(threadSafe)     # OK
# spawnSafe(notThreadSafe) # ← ошибка компиляции
```

Особенно полезно во фреймворках, абстрагирующих `createThread` или работающих с пулами потоков: небезопасные коллбэки отклоняются заблаговременно с понятным сообщением, а не через собственные, менее точные ошибки компилятора.

## `hasNoSideEffects`

```nim
proc hasNoSideEffects*(fn: NimNode): bool
```

### Описание

Возвращает `true`, если `fn` помечена или выведена как **не имеющая побочных эффектов** — то есть для одинаковых входных данных она всегда производит одинаковый результат и не изменяет никакого наблюдаемого внешнего состояния. Это соответствует прагме `{.noSideEffect.}` и именно этим `func` отличается от `proc` в Nim.

Знание во время разворачивания макроса о том, является ли функция чистой, открывает оптимизационные и корректностные гарантии: можно генерировать мемоизирующую обёртку только для чистых функций, требовать, чтобы математические комбинаторы принимали лишь чистые аргументы, или строить compile-time-таблицы значений для чистых функций на небольшом домене.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `fn` | `NimNode` | Узел разрешённого символа (`nnkSym`) инспектируемой процедуры |

### Возвращаемое значение

`true`, если `fn` не имеет побочных эффектов; `false` в противном случае.

### Пример

```nim
import std/macros
import std/effecttraits

func square(x: int): int = x * x          # func → noSideEffect
proc printSquare(x: int) = echo x * x     # proc → имеет побочные эффекты

macro memoize(fn: typed): untyped =
  if not hasNoSideEffects(fn):
    error("memoize требует чистой функции (noSideEffect)", fn)
  # генерация мемоизирующей обёртки...
  result = newEmptyNode()

memoize(square)        # OK — square чистая
# memoize(printSquare) # ← ошибка компиляции
```

Ещё один сценарий — генерация compile-time-таблицы поиска путём вызова чистой функции на диапазоне входных значений:

```nim
macro buildTable(fn: typed): untyped =
  if not hasNoSideEffects(fn):
    error("Построение таблицы требует чистой функции", fn)
  # Безопасно вызывать fn во время компиляции для каждого входного значения
  # и запечь результаты в константный массив.
  result = newEmptyNode()
```

## `typed` vs `untyped` — подробнее

Каждая процедура модуля первым делом вызывает `expectKind fn, nnkSym`. Эта проверка срабатывает, если `fn` — что угодно, кроме узла разрешённого символа. Единственный надёжный способ гарантировать разрешённый символ — объявить параметр макроса как `typed`:

```nim
# ПРАВИЛЬНО: fn разрешается до запуска тела макроса
macro inspect(fn: typed): untyped =
  echo getRaisesList(fn).repr
  result = newEmptyNode()

# НЕПРАВИЛЬНО: fn приходит как сырой AST-идентификатор, не символ
macro inspect(fn: untyped): untyped =
  echo getRaisesList(fn).repr  # ← падение runtime-проверки при разворачивании
  result = newEmptyNode()
```

Параметр `typed` заставляет компилятор выполнить **семантический анализ** аргумента — разрешение имён, разрешение перегрузок, проверку типов — прежде чем макрос его увидит. Результат — узел `nnkSym`, несущий полную запись символа, заполненную компилятором, из которой и читается информация об эффектах.
