# Справочник модуля `strbasics`

> **Назначение модуля:** `strbasics` предоставляет небольшой набор **высокопроизводительных операций над строками, работающих на месте (in-place)**. Все экспортируемые процедуры изменяют строку непосредственно в памяти, не выделяя новых объектов в куче. Это делает их пригодными для горячих путей выполнения и кода, критичного по производительности.
>
> ⚠️ **Экспериментальный API.** Интерфейс может измениться в будущих версиях Nim.

---

## Основная идея: мутация на месте

Стандартный модуль `strutils` как правило *возвращает новые строки* — удобно, но затратно при вызове в плотных циклах. `strbasics` придерживается противоположного подхода: каждая экспортируемая процедура напрямую изменяет свой первый аргумент (`var string`). Лишних выделений памяти не происходит.

Этот подход отлично сочетается с хелпером `dup` из `std/sugar`: когда всё же нужна изменённая *копия*, `dup` создаёт её, применяет мутацию и возвращает результат, не трогая оригинал (примеры ниже).

---

## Внутренняя константа: `whitespaces`

```nim
const whitespaces = {' ', '\t', '\v', '\r', '\l', '\f'}
```

Это набор символов по умолчанию, считающихся «пробельными» для функции `strip`. Константа не экспортируется, но используется как значение по умолчанию параметра `chars`. Охватывает:

| Символ | Описание |
|---|---|
| `' '` | Пробел |
| `'\t'` | Горизонтальная табуляция |
| `'\v'` | Вертикальная табуляция |
| `'\r'` | Возврат каретки |
| `'\l'` | Перевод строки (`\n`) |
| `'\f'` | Перевод страницы |

## Справочник по процедурам

---

### `add`

```nim
proc add*(x: var string, y: openArray[char])
```

#### Что делает

Дописывает все символы из `y` в конец `x`, расширяя `x` на месте. Это перегрузка встроенного `add`, принимающая `openArray[char]` в качестве источника — то есть она работает со строками, последовательностями символов, срезами массивов и любыми другими совместимыми коллекциями без лишних преобразований.

#### Замечание о производительности

Процедура написана так, чтобы в будущем можно было применить оптимизацию на основе `memcpy`. Ключевое ограничение, делающее эту оптимизацию возможной — **правило отсутствия псевдонимов (no-alias)**: `y` не должен быть видом (view) внутрь самого `x`. Дописывать срез строки в саму себя — неопределённое поведение.

#### Описание параметров

| Параметр | Тип | Описание |
|---|---|---|
| `x` | `var string` | Строка, в конец которой дописываются символы (мутируется на месте) |
| `y` | `openArray[char]` | Символы для дописывания; не должны перекрываться с `x` |

#### Примеры

```nim
import strbasics

# Дописать строковый литерал (неявно преобразуется в openArray[char])
var s = "Hello"
s.add(", world!")
assert s == "Hello, world!"

# Дописать seq[char]
var t = "Nim"
let suffix: seq[char] = @[' ', 'r', 'o', 'c', 'k', 's']
t.add(suffix)
assert t == "Nim rocks"

# Дописать срез другой строки (безопасно — разные переменные, нет перекрытия)
var base = "foo"
let extra = "barBaz"
base.add(extra.toOpenArray(0, 2))  # дописывает "bar"
assert base == "foobar"
```

#### Когда предпочесть `add` вместо `&=`

Встроенный `&=` принимает справа только `string`. `add` из `strbasics` принимает любой `openArray[char]`, поэтому можно дописывать срезы, массивы или `seq[char]` напрямую, без создания промежуточной строки.

### `setSlice`

```nim
func setSlice*(s: var string, slice: Slice[int])
```

#### Что делает

Заменяет содержимое `s` подстрокой, определённой диапазоном `slice` — **на месте**, без выделения новой строки. Думайте об этом как о деструктивном, безаллокационном аналоге `substr` или `s[a..b]`.

После вызова `s` содержит ровно те символы, которые находились на позициях от `slice.a` до `slice.b` включительно.

#### Предусловия (проверяются через `assert`)

| Условие | Что происходит при нарушении |
|---|---|
| `slice.a >= 0` | `AssertionDefect` во время выполнения |
| `slice.b <= s.high` | `AssertionDefect` во время выполнения |

#### Особый случай: пустой результат

Если `slice.a > slice.b` (инвертированный или нулевой диапазон), `s` становится пустой строкой (`""`) без ошибки. Это позволяет условно очищать строку в одно действие.

#### Описание параметров

| Параметр | Тип | Описание |
|---|---|---|
| `s` | `var string` | Строка, над которой выполняется срез на месте |
| `slice` | `Slice[int]` | Включительный диапазон индексов, который нужно оставить |

#### Примеры

```nim
import strbasics, std/sugar

var a = "Hello, Nim!"

# Оставить только "Nim" (индексы 7, 8, 9)
doAssert a.dup(setSlice(7 .. 9)) == "Nim"

# Оставить один символ
doAssert a.dup(setSlice(0 .. 0)) == "H"

# Оставить всю строку (фактически ничего не меняет)
doAssert a.dup(setSlice(0 .. 10)) == "Hello, Nim!"

# Инвертированный диапазон → пустая строка (без ошибки)
doAssert a.dup(setSlice(1 .. 0)).len == 0

# Индекс выходит за границы → AssertionDefect
doAssertRaises(AssertionDefect):
  discard a.dup(setSlice(1 .. 11))
```

#### Использование `dup` для неразрушающих копий

`dup` (из `std/sugar`) копирует строку, применяет мутацию к копии и возвращает её — оригинал остаётся нетронутым. Это идиоматический способ использовать `setSlice` (и `strip`), когда нужен новый результат без потери исходного значения.

```nim
import std/sugar, strbasics

var original = "Hello, Nim!"
let shortened = original.dup(setSlice(7 .. 9))
assert original == "Hello, Nim!"  # не изменился
assert shortened == "Nim"
```

#### Детали реализации

На нативных платформах с `nimSeqsV2` `setSlice` использует `moveMem` (по сути `memmove`) для сдвига нужных байтов в начало буфера, а затем вызывает `setLen` для укорачивания длины. Операция выполняется за **O(n)** от длины оставляемого среза без выделения памяти.

### `strip`

```nim
func strip*(a: var string, leading = true, trailing = true,
            chars: set[char] = whitespaces) {.inline.}
```

#### Что делает

Удаляет символы с начала и/или конца строки `a` **на месте**. По умолчанию удаляет все пробельные символы с обеих сторон. Поведение (только с начала, только с конца, с обеих сторон) управляется булевыми параметрами.

Это in-place аналог `strutils.strip`, который возвращает новую строку. `strbasics.strip` мутирует оригинал и не выделяет памяти.

#### Описание параметров

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `a` | `var string` | — | Строка для обрезки (мутируется на месте) |
| `leading` | `bool` | `true` | Удалять совпадающие символы с начала строки |
| `trailing` | `bool` | `true` | Удалять совпадающие символы с конца строки |
| `chars` | `set[char]` | `whitespaces` | Набор символов для удаления |

Если оба флага `leading` и `trailing` равны `false`, строка остаётся полностью неизменной.

#### Примеры

```nim
import strbasics

# По умолчанию: удалить пробелы с обеих сторон
var a = "  vhellov   "
strip(a)
assert a == "vhellov"

# Удалить только завершающие пробелы
a = "  vhellov   "
a.strip(leading = false)
assert a == "  vhellov"

# Удалить только ведущие пробелы
a = "  vhellov   "
a.strip(trailing = false)
assert a == "vhellov   "

# Удалить кастомный набор символов с обеих сторон
var c = "blaXbla"
c.strip(chars = {'b', 'a'})
assert c == "laXbl"

# Расширить кастомный набор
c = "blaXbla"
c.strip(chars = {'b', 'a', 'l'})
assert c == "X"
```

#### Использование `dup` для неразрушающей обрезки

```nim
import std/sugar, strbasics

let original = "  hello  "
let stripped = original.dup(strip)
assert original == "  hello  "  # не изменился
assert stripped == "hello"
```

#### Как работает внутри

`strip` — тонкая обёртка с `{.inline.}`, которая вызывает приватный хелпер `stripSlice` для вычисления диапазона, который нужно оставить, а затем передаёт его в `setSlice` для выполнения обрезки на месте. Никакие символы не перезаписываются — функция лишь сдвигает начальный указатель и уменьшает длину, что очень дружелюбно к кэшу процессора.

## Приватный хелпер: `stripSlice` (не экспортируется)

```nim
func stripSlice(s: openArray[char], leading = true, trailing = true,
                chars: set[char] = whitespaces): Slice[int]
```

Не является частью публичного API, но полезен для понимания внутренней архитектуры модуля. Вычисляет и возвращает `Slice[int]` той части входных данных, которая остаётся после обрезки — не изменяя ничего. `strip` вызывает его для определения диапазона, который затем передаётся в `setSlice`.

```nim
assert stripSlice(" abc  ") == 1 .. 3
# Позиции: 0=' ', 1='a', 2='b', 3='c', 4=' ', 5=' '
# Диапазон после обрезки: индексы 1..3 → "abc"
```

## Сводная таблица

| Процедура | На месте | Выделяет память | Назначение |
|---|---|---|---|
| `add` | ✅ | Нет (расширяет существующий буфер) | Дописать `openArray[char]` к строке |
| `setSlice` | ✅ | Нет | Усечь строку до заданного диапазона индексов |
| `strip` | ✅ | Нет | Удалить ведущие/завершающие символы |

## Типичные паттерны использования

### Паттерн 1 — Высокопроизводительная построчная обработка

```nim
import strbasics

proc processLines(data: string) =
  var line = ""
  for ch in data:
    if ch == '\n':
      line.strip()          # обрезка на месте, без выделения памяти
      if line.len > 0:
        echo line
      line.setLen(0)        # переиспользуем буфер
    else:
      line.add([ch])
```

Буфер `line` выделяется один раз и повторно используется для каждой строки — никаких лишних аллокаций в цикле.

### Паттерн 2 — Сборка строки из частей без промежуточных объектов

```nim
import strbasics

var result = ""
for part in ["Hello", ", ", "world", "!"]:
  result.add(part.toOpenArray(0, part.high))
assert result == "Hello, world!"
```

### Паттерн 3 — Неразрушающее использование с `dup`

```nim
import strbasics, std/sugar

let raw = "  |data|  "
let clean = raw.dup do (s: var string):
  s.strip()
  s.setSlice(1 .. s.high - 1)  # убрать обрамляющие символы '|'
assert clean == "data"
assert raw == "  |data|  "     # оригинал не изменился
```

## Важные замечания

- Все процедуры принимают `var string` — необходима мутируемая переменная; строковые литералы напрямую передать нельзя.
- `setSlice` использует `assert` (а не `doAssert` или более мягкую проверку), поэтому нарушение предусловий всегда прерывает выполнение — как в отладочном, так и в релизном режиме, если только `assert`-ы явно не отключены.
- Пометка `{.inline.}` на `strip` означает, что компилятор как правило встроит вызов, снижая накладные расходы на вызов функции в циклах.
- Модуль входит в стандартную библиотеку Nim, но помечен **экспериментальным** — в стабильном библиотечном коде рекомендуется зафиксировать версию Nim перед использованием.
