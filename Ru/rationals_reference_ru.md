# Модуль `rationals` для Nim — полный справочник

## Обзор

Модуль `rationals` реализует **точную рациональную арифметику** для Nim. Рациональное число — это дробь из двух целых чисел: числителя и знаменателя — и может представлять такие значения, как ½, −⅗ или 8/15 **без каких-либо ошибок округления**, присущих числам с плавающей точкой.

Модуль построен вокруг обобщённого типа `Rational[T]`, где `T` может быть любым целочисленным типом (`int`, `int64`, `int32` и т. д.). Все операции автоматически сокращают дробь до несократимого вида, поэтому вам никогда не нужно думать о том, равны ли `2/4` и `1/2` — они всегда приводятся к одному виду.

### Зачем использовать рациональные числа вместо float?

Арифметика с плавающей точкой быстра, но принципиально неточна. Например:

```nim
echo 0.1 + 0.2   # выводит 0.30000000000000004, а не 0.3
```

С рациональными числами:

```nim
import std/rationals
echo 1 // 10 + 2 // 10   # выводит 3/10 — точно
```

### Быстрый пример

```nim
import std/rationals

let r1 = 1 // 2
let r2 = -3 // 4

doAssert r1 + r2 == -1 // 4
doAssert r1 - r2 ==  5 // 4
doAssert r1 * r2 == -3 // 8
doAssert r1 / r2 == -2 // 3
```

---

## Тип `Rational[T]`

```nim
type Rational*[T] = object
  num*, den*: T
```

Основной тип данных. Содержит два публичных поля:

| Поле | Описание |
|------|---------|
| `num` | Числитель — верхняя часть дроби |
| `den` | Знаменатель — нижняя часть дроби; никогда не должен быть 0 |

К полям можно обращаться напрямую для чтения, но почти всегда лучше использовать предоставленные функции — они гарантируют правильное сокращение.

## Создание и инициализация

### `reduce`

```nim
func reduce*[T: SomeInteger](x: var Rational[T])
```

Сокращает дробь `x` на месте так, чтобы числитель и знаменатель не имели общих делителей, кроме 1. Знак всегда переносится в числитель — знаменатель после сокращения всегда положителен.

Эта функция вызывается **автоматически** всеми арифметическими операциями и при создании через `initRational` / `//`, поэтому вручную вызывать её обычно не нужно. Она полезна, если вы вручную задаёте поля `.num` и `.den`.

Выбрасывает `DivByZeroDefect`, если знаменатель равен 0.

```nim
import std/rationals

var r = Rational[int](num: 6, den: -9)   # создано вручную
reduce(r)
doAssert r.num == 2
doAssert r.den == 3   # знак перенесён; den всегда положителен
```

---

### `initRational`

```nim
func initRational*[T: SomeInteger](num, den: T): Rational[T]
```

Создаёт новое `Rational[T]` из явно заданных числителя и знаменателя, немедленно сокращает дробь и возвращает результат. Это канонический конструктор.

`den` не должен быть 0 (проверяется через `assert` при включённых утверждениях).

```nim
import std/rationals

let r = initRational(3, 6)
doAssert r.num == 1   # автоматически сокращено до 1/2
doAssert r.den == 2
```

---

### `//` (оператор двойной косой черты)

```nim
func `//`*[T](num, den: T): Rational[T]
```

Лаконичная операторная альтернатива `initRational`. Читается естественно как дробь: `1 // 3` означает «одна треть». Это то, что вы будете использовать в большинстве случаев.

```nim
import std/rationals

let r = 1 // 3 + 1 // 5
doAssert r == 8 // 15
```

Отрицательные дроби: знак по соглашению помещается в числитель.

```nim
let r = -3 // 4
doAssert r.num == -3
doAssert r.den ==  4
```

---

## Функции преобразования

### `$` — в строку

```nim
func `$`*[T](x: Rational[T]): string
```

Возвращает строку вида `"числитель/знаменатель"`. Полезно для вывода и отладки.

```nim
import std/rationals

doAssert $(1 // 2)  == "1/2"
doAssert $(-3 // 4) == "-3/4"
```

---

### `toRational` (из целого числа)

```nim
func toRational*[T: SomeInteger](x: T): Rational[T]
```

Оборачивает целое число в рациональное со знаменателем 1. Любое целое число является допустимым рациональным (например, 5 = 5/1).

```nim
import std/rationals

doAssert toRational(42) == 42 // 1
doAssert toRational(0)  ==  0 // 1
```

---

### `toRational` (из числа с плавающей точкой)

```nim
func toRational*(x: float, n: int = ...): Rational[int]
```

Находит **наилучшее рациональное приближение** числа с плавающей точкой `x`, знаменатель которого не превышает `n`. По умолчанию `n` очень велико, что обеспечивает максимальную точность.

Алгоритм основан на **цепных дробях** (Дэвид Эппстайн, UC Irvine, 1993) и гарантированно находит простейшую дробь, которая при обратном преобразовании в `float` совпадает с исходным значением.

```nim
import std/rationals

let r = (1.2).toRational
doAssert r.toFloat == 1.2   # точное обратное преобразование

doAssert (0.5).toRational == 1 // 2
doAssert (0.1).toRational.toFloat == 0.1
```

Ограничение знаменателя даёт более простые, но менее точные результаты:

```nim
import std/rationals

let approx = toRational(3.14159, 100)
echo approx   # например 22/7 — простое приближение π
```

---

### `toFloat`

```nim
func toFloat*[T](x: Rational[T]): float
```

Преобразует рациональное число в `float` выполнением деления `num / den`. Может потерять точность для дробей, которые не могут быть точно представлены в формате с плавающей точкой.

```nim
import std/rationals

doAssert (1 // 4).toFloat == 0.25
doAssert (1 // 3).toFloat == 0.3333333333333333   # не точно в float
```

---

### `toInt`

```nim
func toInt*[T](x: Rational[T]): int
```

Преобразует рациональное число в целое выполнением целочисленного деления `num div den`, которое **усекает к нулю** (отбрасывает дробную часть).

```nim
import std/rationals

doAssert (7 // 2).toInt  ==  3   # 3.5 → 3
doAssert (-7 // 2).toInt == -3   # -3.5 → -3 (к нулю)
doAssert (1 // 2).toInt  ==  0
```

---

## Арифметические операции

Все бинарные операторы существуют в трёх вариантах:
- `рац. OP рац.`
- `рац. OP целое`
- `целое OP рац.`

Все операторы присваивания (`+=`, `-=`, `*=`, `/=`) изменяют левый операнд на месте.

Каждая операция, возвращающая рациональное число, автоматически вызывает `reduce`, поэтому результаты всегда в несократимой форме.

---

### Сложение: `+`, `+=`

```nim
func `+`*[T](x, y: Rational[T]): Rational[T]
func `+`*[T](x: Rational[T], y: T): Rational[T]
func `+`*[T](x: T, y: Rational[T]): Rational[T]
func `+=`*[T](x: var Rational[T], y: Rational[T])
func `+=`*[T](x: var Rational[T], y: T)
```

Складывает два значения. При сложении дробей с разными знаменателями реализация находит наименьшее общее кратное (НОК), чтобы избежать излишнего роста числителя и знаменателя.

```nim
import std/rationals

doAssert 1 // 4 + 1 // 4 == 1 // 2   # одинаковые знаменатели
doAssert 1 // 3 + 1 // 4 == 7 // 12  # разные → НОК(3,4) = 12
doAssert 1 // 2 + 1      == 3 // 2   # рац. + целое

var r = 1 // 6
r += 1 // 3
doAssert r == 1 // 2
```

---

### Смена знака (унарный минус): `-`

```nim
func `-`*[T](x: Rational[T]): Rational[T]
```

Возвращает аддитивную обратную величину `x` — меняет знак числителя.

```nim
import std/rationals

doAssert -(3 // 4)  == -3 // 4
doAssert -(-1 // 5) ==  1 // 5
```

---

### Вычитание: `-`, `-=`

```nim
func `-`*[T](x, y: Rational[T]): Rational[T]
func `-`*[T](x: Rational[T], y: T): Rational[T]
func `-`*[T](x: T, y: Rational[T]): Rational[T]
func `-=`*[T](x: var Rational[T], y: Rational[T])
func `-=`*[T](x: var Rational[T], y: T)
```

Вычитает второе значение из первого. Как и `+`, использует НОК знаменателей.

```nim
import std/rationals

doAssert 3 // 4 - 1 // 4 == 1 // 2
doAssert 3 // 4 - 1      == -1 // 4  # рац. - целое
doAssert 1 - 3 // 4      ==  1 // 4  # целое - рац.

var r = 5 // 6
r -= 1 // 3
doAssert r == 1 // 2
```

---

### Умножение: `*`, `*=`

```nim
func `*`*[T](x, y: Rational[T]): Rational[T]
func `*`*[T](x: Rational[T], y: T): Rational[T]
func `*`*[T](x: T, y: Rational[T]): Rational[T]
func `*=`*[T](x: var Rational[T], y: Rational[T])
func `*=`*[T](x: var Rational[T], y: T)
```

Перемножает числители и знаменатели, затем сокращает результат.

```nim
import std/rationals

doAssert 2 // 3 * 3 // 4 == 1 // 2   # (2·3)/(3·4) = 6/12 → 1/2
doAssert 3 // 5 * 2      == 6 // 5   # рац. * целое
doAssert 4 * (1 // 8)    == 1 // 2   # целое * рац.

var r = 1 // 3
r *= 3
doAssert r == 1 // 1
```

---

### `reciprocal` — обратная дробь

```nim
func reciprocal*[T](x: Rational[T]): Rational[T]
```

Возвращает `1/x` — меняет местами числитель и знаменатель. Если `x` равен нулю, выбрасывает `DivByZeroDefect`. Знак обрабатывается корректно: `reciprocal(-2/3)` возвращает `-3/2`.

```nim
import std/rationals

doAssert reciprocal(2 // 3)  == 3 // 2
doAssert reciprocal(-2 // 3) == -3 // 2
```

---

### Деление: `/`, `/=`

```nim
func `/`*[T](x, y: Rational[T]): Rational[T]
func `/`*[T](x: Rational[T], y: T): Rational[T]
func `/`*[T](x: T, y: Rational[T]): Rational[T]
func `/=`*[T](x: var Rational[T], y: Rational[T])
func `/=`*[T](x: var Rational[T], y: T)
```

Делит `x` на `y`. Деление на рациональное эквивалентно умножению на его обратную величину.

```nim
import std/rationals

doAssert (1 // 2) / (3 // 4) == 2 // 3
doAssert (3 // 4) / 3        == 1 // 4   # рац. / целое
doAssert 3 / (3 // 4)        == 4 // 1   # целое / рац.

var r = 3 // 4
r /= 3
doAssert r == 1 // 4
```

---

### `div` — целочисленное деление с усечением

```nim
func `div`*[T: SomeInteger](x, y: Rational[T]): T
```

Вычисляет целую часть `x / y`, усекая к нулю. Результат — **целое число** типа `T`, а не рациональное. Это зеркало встроенного `div` для целых чисел.

```nim
import std/rationals

doAssert  (7 // 2) div (1 // 1) ==  3   #  3.5 → 3
doAssert (-7 // 2) div (1 // 1) == -3   # -3.5 → -3 (к нулю)
```

---

### `mod` — остаток от деления с усечением

```nim
func `mod`*[T: SomeInteger](x, y: Rational[T]): Rational[T]
```

Возвращает остаток от деления с усечением, то есть `x - (x div y) * y`. Результат — рациональное число. Знак результата совпадает со знаком делимого `x`.

```nim
import std/rationals

doAssert (7 // 4) mod (1 // 2) == 1 // 4
# 7/4 ÷ 1/2 = 3.5; trunc = 3; остаток = 7/4 - 3·(1/2) = 7/4 - 6/4 = 1/4
```

---

### `floorDiv` — целочисленное деление с округлением вниз

```nim
func floorDiv*[T: SomeInteger](x, y: Rational[T]): T
```

Вычисляет `floor(x / y)` — округляет **вниз** к минус бесконечности, в отличие от `div`, который округляет к нулю. Для неотрицательных результатов `floorDiv` и `div` совпадают. Разница возникает при отрицательном результате.

```nim
import std/rationals

doAssert floorDiv( 7 // 2, 1 // 1) ==  3  # как div
doAssert floorDiv(-7 // 2, 1 // 1) == -4  # div даёт -3; floorDiv даёт -4
```

---

### `floorMod` — остаток после floor-деления

```nim
func floorMod*[T: SomeInteger](x, y: Rational[T]): Rational[T]
```

Возвращает остаток после floor-деления, то есть `x - floorDiv(x, y) * y`. Знак результата всегда совпадает со знаком **делителя** `y` — это то же поведение, что у оператора `%` в Python.

```nim
import std/rationals

# Положительный делитель → неотрицательный остаток
doAssert floorMod(-7 // 4, 1 // 2) == 1 // 4

# Для сравнения: mod при отрицательном делимом даёт отрицательный остаток
doAssert (-7 // 4) mod (1 // 2) == -1 // 4
```

---

### `^` — возведение в степень

```nim
func `^`*[T: SomeInteger](x: Rational[T], y: T): Rational[T]
```

Возводит рациональное `x` в целую степень `y`. Работает как для положительных, так и для отрицательных показателей. Отрицательный показатель переворачивает дробь (эквивалентно возведению обратной дроби в положительную степень). Показатель 0 всегда возвращает `1/1`.

```nim
import std/rationals

doAssert (-3 // 5) ^ 0  == 1 // 1    # любое^0 = 1
doAssert (-3 // 5) ^ 1  == -3 // 5
doAssert (-3 // 5) ^ 2  == 9 // 25   # (-3)²/5² = 9/25
doAssert (-3 // 5) ^ -2 == 25 // 9   # обратная дробь в квадрате
```

Поскольку все степени уже сокращённой дроби тоже сокращены, `reduce` внутри `^` **не вызывается** — операция безопасна и эффективна.

---

### `abs` — абсолютное значение

```nim
func abs*[T](x: Rational[T]): Rational[T]
```

Возвращает абсолютное значение рационального числа: числитель и знаменатель становятся неотрицательными.

```nim
import std/rationals

doAssert abs( 1 // 2) == 1 // 2
doAssert abs(-1 // 2) == 1 // 2
doAssert abs(-3 // 4) == 3 // 4
```

## Функции сравнения

### `cmp`

```nim
func cmp*(x, y: Rational): int
```

Трёхстороннее сравнение. Возвращает отрицательное число, если `x < y`; ноль, если `x == y`; положительное число, если `x > y`. Полезно при сортировке рациональных чисел или ветвлении по порядку.

```nim
import std/rationals

doAssert cmp(1 // 3, 1 // 2) < 0   # 1/3 < 1/2
doAssert cmp(1 // 2, 1 // 2) == 0
doAssert cmp(2 // 3, 1 // 2) > 0   # 2/3 > 1/2
```

---

### `<`, `<=`, `==`

```nim
func `<`*(x, y: Rational): bool
func `<=`*(x, y: Rational): bool
func `==`*(x, y: Rational): bool
```

Стандартные операторы отношения. Все три реализованы через вычитание двух рациональных чисел и проверку знака числителя результата — сравнение всегда точное, без ошибок округления.

Равенство работает корректно для разных представлений одного и того же значения, поскольку перед сравнением обе части сокращаются.

```nim
import std/rationals

doAssert 1 // 3 <  1 // 2
doAssert 1 // 2 <= 1 // 2
doAssert 2 // 4 == 1 // 2   # разная форма, одно значение
doAssert not (1 // 3 == 1 // 4)
```

## Утилиты

### `hash`

```nim
func hash*[T](x: Rational[T]): Hash
```

Вычисляет хеш рационального числа, пригодный для использования в качестве ключа в `Table` или элемента `HashSet`. Принципиально важно, что функция сначала **сокращает** `x` перед хешированием, что гарантирует: математически равные числа (например `2/4` и `1/2`) всегда дают **одинаковый хеш**.

```nim
import std/[rationals, tables]

var t = initTable[Rational[int], string]()
t[1 // 2] = "одна половина"

doAssert t[2 // 4] == "одна половина"   # 2/4 == 1/2, одинаковый хеш
```

## Практические примеры

### Точная дробная арифметика

```nim
import std/rationals

# Сумма гармонического ряда: 1 + 1/2 + 1/3 + 1/4
var sum = 0 // 1
for i in 1..4:
  sum += 1 // i
doAssert sum == 25 // 12
```

### Обратное преобразование float → rational → float

```nim
import std/rationals

for x in [0.1, 0.5, 1.25, 3.14159]:
  let r = x.toRational
  doAssert r.toFloat == x
  echo x, " ≈ ", r
```

### Рациональные числа как ключи таблицы

```nim
import std/[rationals, tables]

var fractions = initTable[Rational[int], string]()
fractions[1 // 2] = "половина"
fractions[1 // 3] = "треть"
fractions[1 // 4] = "четверть"

doAssert fractions[2 // 4] == "четверть"   # 2/4 → 1/4, тот же ключ
```

### Возведение в степень и обратная дробь

```nim
import std/rationals

let base = 2 // 3
doAssert base ^ 3  == 8 // 27
doAssert base ^ -3 == 27 // 8
doAssert reciprocal(base ^ 3) == base ^ -3
```

## Условия возникновения ошибок

| Ситуация | Выбрасываемая ошибка |
|----------|---------------------|
| `reduce` при `den == 0` | `DivByZeroDefect` |
| `initRational` / `//` с `den == 0` | `AssertionDefect` (при включённых утверждениях) |
| `reciprocal` от нуля | `DivByZeroDefect` |
