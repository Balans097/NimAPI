# enumerate — справочник по модулю

## Обзор

Модуль `enumerate` предоставляет один одноимённый макрос, привносящий в Nim перебор с индексами в стиле Python. Вместо того чтобы вручную вести переменную-счётчик рядом с переменной цикла, `enumerate` автоматически сопровождает каждый элемент его порядковым номером, возвращая пары `(индекс, значение)` на каждой итерации.

Под капотом `enumerate` — это **макрос, переписывающий цикл `for` на этапе компиляции**. Он не создаёт новый итератор в рантайме и не выделяет дополнительной памяти; он просто преобразует специальный синтаксис `for … in enumerate(…)` в обычный `for`-цикл с добавленной счётчиковой переменной. Преобразование невидимо для программиста, однако производит ровно тот же машинный код, что и ручной подход со счётчиком.

**Доступность:** Nim 1.3 и выше.

---

## `enumerate`

```nim
macro enumerate*(x: ForLoopStmt): untyped
```

### Описание

`enumerate` оборачивает любой итерируемый объект — массивы, последовательности, строки, пользовательские итераторы — и делает одновременно доступными в теле цикла и текущий **индекс** (счётчик `int`, по умолчанию начинающийся с `0`), и текущее **значение**.

Макрос принимает необязательный первый аргумент для задания **произвольного начального значения** счётчика. Это удобно при нужде в нумерации с единицы, при желании продолжить счёт с точки останова предыдущего цикла, или когда числовые значения индексов несут предметный смысл (например, коды ASCII, номера строк и т.д.).

Поскольку `enumerate` является макросом вида `ForLoopStmt` — особого рода макросом в Nim, перехватывающим синтаксис `for`-цикла, — он вызывается исключительно через оператор `for` и никогда как обычный вызов функции. Его результат нельзя сохранить в переменную; вместо этого пишется `for`-цикл, итерируемым выражением которого является `enumerate(…)`.

### Синтаксис

```nim
# Счётчик по умолчанию начинается с 0
for i, значение in enumerate(коллекция):
  …

# Произвольное начальное значение счётчика
for i, значение in enumerate(начало, коллекция):
  …

# Форма с распаковкой кортежа (тоже допустима)
for (i, значение) in enumerate(коллекция):
  …

for (i, значение) in enumerate(начало, коллекция):
  …
```

### Параметры (внутри оператора `for`)

| Позиция | Роль | Описание |
|---|---|---|
| Первая переменная `for` | Индекс | Получает текущее значение счётчика. Всегда `int`. |
| Остальные переменные `for` | Значение(я) | Получают элементы, выдаваемые базовым итератором. |
| Первый аргумент `enumerate` (необязательный) | Начальное значение | Целочисленное выражение, задающее начало счётчика. По умолчанию `0`. |
| Последний аргумент `enumerate` | Коллекция | Итерируемый объект: массив, seq, строка, итератор и т.д. |

### Как макрос преобразует цикл

Для точного понимания поведения полезно увидеть, во что разворачивается `enumerate`. Дано:

```nim
for i, x in enumerate(коллекция):
  тело
```

макрос генерирует эквивалент:

```nim
block:
  var i = -1          # инициализируется значением начало − 1
  for x in коллекция:
    inc i             # увеличивается до начального значения перед телом
    тело
```

Счётчик предварительно уменьшается на единицу, а затем увеличивается в самом начале каждой итерации. Такой подход позволяет счётчиковой переменной хранить своё финальное значение после завершения цикла, что важно для совместимости с другими макросами, инспектирующими состояние цикла.

### Требование к распаковке

Макрос **требует** привязки имён и для индекса, и для значения. Написать `for i in enumerate(коллекция)` с одной переменной нельзя — некуда поместить значение. Макрос выдаст ошибку компиляции (`"Missing second for loop variable"`), если это попытаться. Используйте либо форму с запятой (`for i, x in`), либо форму с распаковкой кортежа (`for (i, x) in`).

### Примеры

**Базовое использование — перебор последовательности с индексами:**

```nim
import std/enumerate

let fruits = @["яблоко", "банан", "вишня"]
for i, fruit in enumerate(fruits):
  echo i, ": ", fruit
# 0: яблоко
# 1: банан
# 2: вишня
```

**Перебор строки посимвольно:**

```nim
import std/enumerate

for i, ch in enumerate("hello"):
  echo i, " → ", ch
# 0 → h
# 1 → e
# 2 → l
# 3 → l
# 4 → o
```

**Произвольное начальное значение — нумерация с единицы:**

```nim
import std/enumerate

let tasks = ["Написать тесты", "Исправить баг", "Ревью PR"]
for i, task in enumerate(1, tasks):
  echo "Задача ", i, ": ", task
# Задача 1: Написать тесты
# Задача 2: Исправить баг
# Задача 3: Ревью PR
```

**Произвольное начальное значение — коды ASCII в качестве индексов:**

```nim
import std/enumerate

for code, ch in enumerate(97, "abcd"):
  echo ch, " = ", code
# a = 97
# b = 98
# c = 99
# d = 100
```

**Форма с распаковкой кортежа:**

```nim
import std/enumerate

let values = [100, 200, 300]
for (i, v) in enumerate(values):
  echo "(", i, ", ", v, ")"
# (0, 100)
# (1, 200)
# (2, 300)
```

**Сбор пар индекс–значение в последовательность:**

```nim
import std/enumerate

let letters = ['x', 'y', 'z']
var indexed: seq[(int, char)]
for i, c in enumerate(letters):
  indexed.add((i, c))
assert indexed == @[(0, 'x'), (1, 'y'), (2, 'z')]
```

**Использование enumerate с пользовательским итератором:**

```nim
import std/enumerate

iterator evens(n: int): int =
  var x = 0
  while x < n:
    yield x
    inc x, 2

for i, v in enumerate(evens(10)):
  echo i, ": ", v
# 0: 0
# 1: 2
# 2: 4
# 3: 6
# 4: 8
```

**Значение счётчика доступно после цикла:**

Поскольку счётчиковая переменная объявляется в окружающем блоке, её финальное значение доступно после завершения цикла — полезно, чтобы узнать, сколько элементов было реально обработано:

```nim
import std/enumerate

let data = @[10, 20, 30, 40]
var count: int
for count, v in enumerate(data):
  discard
# После цикла count == 3 (индекс последнего элемента)
```

### Типичные ошибки и как их избежать

**Ошибка 1 — единственная переменная цикла (нет привязки значения):**

```nim
# НЕПРАВИЛЬНО — ошибка компиляции: "Missing second for loop variable"
for i in enumerate(mySeq):
  discard

# ПРАВИЛЬНО
for i, x in enumerate(mySeq):
  discard
```

**Ошибка 2 — перепутан порядок аргументов:**

Необязательное начальное значение всегда идёт **перед** коллекцией, а не после.

```nim
# НЕПРАВИЛЬНО — пытается итерировать целое число 1, принимая "hello" за начало
for i, ch in enumerate("hello", 1):  # ошибка типов или рантайм
  discard

# ПРАВИЛЬНО
for i, ch in enumerate(1, "hello"):
  discard
```

**Ошибка 3 — использование `enumerate` вне `for`-цикла:**

`enumerate` — макрос вида `ForLoopStmt`. Вне оператора `for` он не имеет смысла и не может быть сохранён в переменную или передан как аргумент.

```nim
# НЕПРАВИЛЬНО — не компилируется
let e = enumerate(mySeq)

# ПРАВИЛЬНО — использовать исключительно в for-операторе
for i, x in enumerate(mySeq):
  discard
```

### Замечания о дизайне

- **Нулевые накладные расходы.** Преобразование чисто синтаксическое. Генерируемый код идентичен ручному написанию цикла со счётчиком.
- **Работает с любым итерируемым объектом.** Поскольку макрос просто переносит выражение коллекции в обычный `for`-цикл, поддерживается всё, что поддерживает `for` в Nim, — включая пользовательские итераторы, `items`, `pairs`, диапазоны — без каких-либо дополнительных настроек.
- **Блочная область видимости.** Макрос оборачивает сгенерированный код в оператор `block`, поэтому счётчиковая переменная не просачивается в окружающую область видимости. Это согласуется с тем, как в Nim ограничены переменные `for`-цикла.
- **Вид макроса `ForLoopStmt`.** В отличие от обычных макросов, макрос `ForLoopStmt` получает весь `for`-оператор в виде единственного аргумента. Именно это позволяет `enumerate` перехватывать синтаксис цикла прозрачно, не требуя никакого специального синтаксиса вызова.
