# Справочник модуля NimScript

> **Модуль:** `nimscript`  
> **Часть:** Стандартная библиотека Nim  
> **Назначение:** Предоставляет стандартный API для конфигурационных скриптов `.nims` и файлов пакетов Nimble. Позволяет автоматизировать сборку, управлять файловой системой, управлять настройками компилятора и определять задачи — всё это на языке Nim.

## Константы

### `buildOS`

```nim
const buildOS*: string
```

**Что это такое:** Операционная система, на которой *выполняется сама сборка* — машина, которая компилирует ваш код. Значение может отличаться от `system.hostOS` при кросс-компиляции (например, когда вы собираете бинарник для Windows, находясь на машине с Linux).

**Зачем нужно:** Используйте эту константу в `.nims`-скриптах, чтобы применять флаги компилятора или пути, специфичные для платформы *на которой идёт сборка*, а не для платформы, на которой будет запускаться результат.

```nim
# Применить Linux-специфичные флаги только при сборке на Linux
if buildOS == "linux":
  switch("passC", "-march=native")
```

---

### `buildCPU`

```nim
const buildCPU*: string
```

**Что это такое:** Архитектура процессора сборочной машины. Как и `buildOS`, может отличаться от `system.hostCPU` при кросс-компиляции.

```nim
if buildCPU == "amd64":
  switch("passC", "-mavx2")
```

## Режим работы скрипта

### `ScriptMode` (тип)

```nim
type ScriptMode* {.pure.} = enum
  Silent   # Молчаливый режим — никакого вывода об операциях
  Verbose  # Подробный режим — печатает каждую операцию
  Whatif   # Режим «что если» — печатает операции, но НЕ выполняет их
```

**Что это такое:** Перечисление, управляющее поведением файловых функций (`mkDir`, `rmDir`, `cpFile` и др.) — будут ли они молчать, выводить информацию или имитировать выполнение.

### `mode` (переменная)

```nim
var mode*: ScriptMode
```

**Что это такое:** Глобальная переменная, изменяя которую вы управляете режимом логирования всех файловых операций NimScript. По умолчанию — `Silent`.

```nim
# Смотреть, что именно делает скрипт
mode = ScriptMode.Verbose
mkDir "build/output"
# выводит: [NimScript] mkDir: build/output

# Протестировать скрипт, не трогая файловую систему
mode = ScriptMode.Whatif
rmDir "dist"
# выводит: [NimScript] rmDir: dist  (но ничего НЕ удаляет)
```

> **Совет:** Режим `Whatif` незаменим для отладки деструктивных операций вроде `rmDir` или `mvFile` — сначала убедитесь, что скрипт сделает то, что нужно, и только потом запускайте по-настоящему.

## Управление компилятором

### `switch`

```nim
proc switch*(key: string, val = "")
```

**Что это такое:** Основной способ передавать флаги компилятора из `.nims`-скрипта. Любой флаг командной строки, который вы обычно передаёте `nim c`, можно установить программно здесь. `key` — имя флага, `val` — его значение.

```nim
switch("opt", "speed")          # оптимизировать под скорость
switch("checks", "on")          # включить все проверки времени выполнения
switch("define", "myFeature")   # объявить символ
```

---

### `--` (шаблон, две формы)

```nim
template `--`*(key, val: untyped)
template `--`*(key: untyped)
```

**Что это такое:** Синтаксический сахар для `switch`. Вместо `switch("key", "value")` пишется `--key:value`. Благодаря этому `.nims`-файлы читаются почти как обычный `nim.cfg`, но с полной мощью языка Nim вокруг.

```nim
--opt:speed
--path:"./src"
--hint:"[Conf]:off"   # отключить подсказку Conf
--listCmd             # форма без значения
```

---

### `warning`

```nim
proc warning*(name: string; val: bool)
```

**Что это такое:** Включает или выключает именованное предупреждение компилятора. `true` — включить, `false` — заглушить. Внутри вызывает `switch` с суффиксом `"on"` / `"off"`.

```nim
warning("UnusedImport", false)   # убрать предупреждения о неиспользуемых импортах
warning("Deprecated", true)      # убедиться, что предупреждения об устаревании видны
```

---

### `hint`

```nim
proc hint*(name: string; val: bool)
```

**Что это такое:** То же самое, что `warning`, но для *подсказок* компилятора (информационные сообщения, менее серьёзные, чем предупреждения).

```nim
hint("Processing", false)   # скрыть сообщения "Обработка файла..."
hint("Conf", false)         # скрыть сообщения о загрузке конфигурации
```

---

### `patchFile`

```nim
proc patchFile*(package, filename, replacement: string)
```

**Что это такое:** Перенаправляет компилятор: вместо стандартного файла из пакета будет использован ваш файл-замена. Удобно для подстановки патченных версий модулей из stdlib или сторонних библиотек без изменения оригиналов.

**Важно:** Если `replacement` — относительный путь, он разрешается относительно самого `.nims`-файла, а не через `--path` компилятора.

```nim
# Заменить asyncdispatch локальной патченной версией
patchFile("stdlib", "asyncdispatch", "patches/asyncdispatch_patched")
```

---

### `getCommand`

```nim
proc getCommand*(): string
```

**Что это такое:** Возвращает команду компилятора Nim, которая сейчас выполняется — например `"c"`, `"js"`, `"cpp"`, `"build"`, `"help"`. Полезно, когда нужно вести себя по-разному в зависимости от того, как был вызван компилятор.

```nim
if getCommand() == "js":
  switch("define", "useNodeJs")
```

---

### `setCommand`

```nim
proc setCommand*(cmd: string; project = "")
```

**Что это такое:** Переопределяет команду компилятора, которая продолжится после завершения NimScript. Опционально можно указать файл `project` для компиляции. Именно так определения `task` меняют то, что компилятор делает дальше.

```nim
# Принудительно скомпилировать через бэкенд C
setCommand "c"

# Скомпилировать конкретный файл
setCommand("c", "src/main.nim")
```

---

### `cppDefine`

```nim
proc cppDefine*(define: string)
```

**Что это такое:** Сообщает Nim, что указанный идентификатор является C-препроцессорным `#define`. После этого Nim всегда будет правильно манглировать это имя при генерации кода C/C++, предотвращая конфликты имён.

```nim
cppDefine("SOME_C_MACRO")
```

## Конфигурационные ключи

Эти три процедуры читают и записывают внутреннее хранилище конфигурации Nim в формате «ключ–значение». Ключи используют точечную нотацию, как в `nim.cfg` (например, `"gcc.options.always"`).

### `put`

```nim
proc put*(key, value: string)
```

**Что это такое:** Устанавливает запись конфигурации. Эквивалентно строке в `nim.cfg`.

```nim
put("gcc.options.always", "-fno-strict-aliasing")
```

---

### `get`

```nim
proc get*(key: string): string
```

**Что это такое:** Читает запись конфигурации. Возвращает пустую строку, если ключ не существует.

```nim
let opts = get("gcc.options.always")
echo "Текущие опции GCC: ", opts
```

---

### `exists`

```nim
proc exists*(key: string): bool
```

**Что это такое:** Проверяет, установлен ли конфигурационный ключ. Полезно для условной логики перед чтением значения.

```nim
if exists("gcc.options.always"):
  echo "Установлены пользовательские опции GCC"
```

## Информация о проекте и путях

### `projectName`

```nim
proc projectName*(): string
```

**Что это такое:** Возвращает имя компилируемого проекта (имя главного `.nim`-файла без расширения и пути).

```nim
echo "Сборка: ", projectName()  # например, "myapp"
```

---

### `projectDir`

```nim
proc projectDir*(): string
```

**Что это такое:** Возвращает абсолютный путь к директории, содержащей главный файл проекта.

```nim
let outDir = projectDir() / "build"
mkDir outDir
```

---

### `projectPath`

```nim
proc projectPath*(): string
```

**Что это такое:** Возвращает полный абсолютный путь к главному `.nim`-файлу проекта, включая имя файла.

```nim
echo "Главный файл: ", projectPath()
# например: /home/user/myproject/src/main.nim
```

---

### `thisDir`

```nim
proc thisDir*(): string
```

**Что это такое:** Возвращает директорию, в которой находится текущий выполняемый `.nims`-скрипт. Это правильная точка отсчёта для построения путей относительно самого скрипта.

```nim
# Сослаться на вспомогательный файл рядом с .nims-скриптом
let helper = thisDir() / "helpers" / "codegen.nim"
```

---

### `nimcacheDir`

```nim
proc nimcacheDir*(): string
```

**Что это такое:** Возвращает путь к директории `nimcache`, где компилятор хранит сгенерированные файлы C/C++ и объектные файлы текущей сборки.

```nim
echo "Кэш находится в: ", nimcacheDir()
```

---

### `getCurrentDir`

```nim
proc getCurrentDir*(): string
```

**Что это такое:** Возвращает текущий рабочий каталог процесса на момент вызова. В отличие от `thisDir()` и `projectDir()`, отражает *рабочий* каталог времени выполнения, который можно менять с помощью `cd`.

```nim
echo "Работаем в: ", getCurrentDir()
```

---

### `findExe`

```nim
proc findExe*(bin: string): string
```

**Что это такое:** Ищет исполняемый файл по имени — сначала в текущем рабочем каталоге, затем во всех каталогах из переменной окружения `PATH`. Возвращает пустую строку `""`, если не нашёл.

```nim
let git = findExe("git")
if git == "":
  echo "git не установлен или отсутствует в PATH"
else:
  exec git & " pull"
```

## Операции с файловой системой

Все файловые операции учитывают переменную `mode`. В режиме `Verbose` они выводят, что делают; в режиме `Whatif` — выводят операцию, но не выполняют её.

### `listDirs`

```nim
proc listDirs*(dir: string): seq[string]
```

**Что это такое:** Возвращает список всех непосредственных поддиректорий внутри `dir`. Обход **нерекурсивный** — возвращается только первый уровень вложенности.

```nim
for d in listDirs("src"):
  echo "Найдена поддиректория: ", d
```

---

### `listFiles`

```nim
proc listFiles*(dir: string): seq[string]
```

**Что это такое:** Возвращает список всех файлов непосредственно внутри `dir`. Нерекурсивно.

```nim
for f in listFiles("assets"):
  echo "Ресурс: ", f
```

---

### `mkDir`

```nim
proc mkDir*(dir: string) {.raises: [OSError].}
```

**Что это такое:** Создаёт директорию `dir`, включая все промежуточные родительские директории (как `mkdir -p`). Если директория уже существует, ошибка не возникает — это делает вызов безопасным в любой ситуации.

```nim
mkDir "build/release/bin"   # создаст все три уровня при необходимости
```

---

### `rmDir`

```nim
proc rmDir*(dir: string, checkDir = false) {.raises: [OSError].}
```

**Что это такое:** Рекурсивно удаляет директорию со всем содержимым. Когда `checkDir = true`, выбрасывает `OSError`, если директория не существует; когда `false` (по умолчанию) — молча завершается при отсутствии директории.

```nim
rmDir "build"          # очистить директорию сборки; нет ошибки если её нет
rmDir("build", true)   # ошибка, если build не существует
```

---

### `rmFile`

```nim
proc rmFile*(file: string) {.raises: [OSError].}
```

**Что это такое:** Удаляет один файл. Выбрасывает `OSError` при неудаче.

```nim
rmFile "build/old_output.o"
```

---

### `cpFile`

```nim
proc cpFile*(`from`, to: string) {.raises: [OSError].}
```

**Что это такое:** Копирует файл из `from` в `to`. Целевая директория должна уже существовать.

```nim
cpFile("README.md", "dist/README.md")
```

---

### `cpDir`

```nim
proc cpDir*(`from`, to: string) {.raises: [OSError].}
```

**Что это такое:** Рекурсивно копирует всё дерево каталогов из `from` в `to`.

```nim
cpDir("templates", "build/templates")
```

---

### `mvFile`

```nim
proc mvFile*(`from`, to: string) {.raises: [OSError].}
```

**Что это такое:** Перемещает (переименовывает) файл. Если `from` и `to` находятся на разных файловых системах, файл копируется, а оригинал удаляется.

```nim
mvFile("app.log", "logs/app_2024.log")
```

---

### `mvDir`

```nim
proc mvDir*(`from`, to: string) {.raises: [OSError].}
```

**Что это такое:** Перемещает целую директорию. Удобно для переименования или перемещения результатов сборки.

```nim
mvDir("build/tmp", "build/release")
```

---

### `fileExists`

```nim
proc fileExists*(filename: string): bool {.tags: [ReadIOEffect].}
```

**Что это такое:** Возвращает `true`, если `filename` существует и является обычным файлом (не директорией).

```nim
if not fileExists("config.json"):
  echo "Внимание: config.json не найден, используются значения по умолчанию"
```

---

### `dirExists`

```nim
proc dirExists*(dir: string): bool {.tags: [ReadIOEffect].}
```

**Что это такое:** Возвращает `true`, если `dir` существует и является директорией.

```nim
if not dirExists("build"):
  mkDir "build"
```

## Навигация по каталогам

### `cd`

```nim
proc cd*(dir: string) {.raises: [OSError].}
```

**Что это такое:** Постоянно меняет текущий рабочий каталог на остаток выполнения скрипта. Для временного изменения с автоматическим возвратом используйте `withDir`.

```nim
cd "src"
exec "nim c main.nim"
cd ".."   # нужно вручную вернуться назад
```

---

### `withDir`

```nim
template withDir*(dir: string; body: untyped): untyped
```

**Что это такое:** Шаблон, который временно переходит в `dir`, выполняет `body`, а затем **автоматически восстанавливает** исходную директорию — даже если внутри `body` возникло исключение. Это идиоматичный и безопасный способ выполнить код в другой директории.

```nim
withDir "subproject":
  # текущая директория теперь subproject/
  exec "nim c src/main.nim"
# текущая директория восстановлена к прежней
```

> **Предпочитайте `withDir` вместо `cd`**, когда смена директории должна быть локальной для блока кода. Это предотвращает случайное «застревание» скрипта в неправильной директории после ошибки.

## Запуск внешних процессов

### `exec` (только команда)

```nim
proc exec*(command: string) {.raises: [OSError].}
```

**Что это такое:** Запускает внешнюю команду оболочки. Если команда завершается с ненулевым кодом выхода, немедленно выбрасывается `OSError`. Команда выполняется относительно текущего рабочего каталога.

> Если вам нужна версия `exec`, которая возвращает код выхода и/или захваченный вывод вместо исключения, используйте `system.gorgeEx` напрямую.

```nim
exec "nim c -r src/tests.nim"
exec "git add ."
exec "git commit -m 'automated release'"
```

---

### `exec` (команда + ввод + кэш)

```nim
proc exec*(command: string, input: string, cache = "") {.raises: [OSError].}
```

**Что это такое:** Как однааргументный `exec`, но дополнительно передаёт `input` на стандартный ввод процесса и поддерживает опциональный ключ `cache` для кэширования `gorgeEx`. Вывод команды выводится в stdout.

> **Внимание:** Этот вариант выполняется относительно пути модуля NimScript, что может влиять на разрешение относительных путей внутри команды. Предпочитайте `gorgeEx` напрямую, когда нужен тонкий контроль.

```nim
exec("python3 process.py", "некоторые входные данные")
```

---

### `selfExec`

```nim
proc selfExec*(command: string) {.raises: [OSError].}
```

**Что это такое:** Запускает команду компилятора Nim, используя **тот же самый исполняемый файл `nim`**, который сейчас выполняет этот скрипт. Это гарантирует вызов именно той версии компилятора, а не того `nim`, который первым встретится в PATH. `command` **не должна** начинаться с `"nim "`.

```nim
# Эквивалентно: nim doc src/mylib.nim
selfExec "doc src/mylib.nim"

# Сборка с конкретным бэкендом
selfExec "c -d:release src/main.nim"
```

## Переменные окружения

### `getEnv`

```nim
proc getEnv*(key: string; default = ""): string
```

**Что это такое:** Читает значение переменной окружения `key`. Если переменная не установлена, возвращает `default` (по умолчанию — пустая строка).

```nim
let buildType = getEnv("BUILD_TYPE", "debug")
if buildType == "release":
  switch("opt", "speed")
```

---

### `existsEnv`

```nim
proc existsEnv*(key: string): bool
```

**Что это такое:** Возвращает `true`, если переменная окружения `key` определена (даже если её значение — пустая строка).

```nim
if existsEnv("CI"):
  echo "Работаем в среде CI"
```

---

### `putEnv`

```nim
proc putEnv*(key, val: string)
```

**Что это такое:** Устанавливает переменную окружения `key` равной `val` для текущего процесса и всех дочерних процессов, которые он запустит.

```nim
putEnv("NIM_CACHE", nimcacheDir())
exec "some_tool_that_reads_NIM_CACHE"
```

---

### `delEnv`

```nim
proc delEnv*(key: string)
```

**Что это такое:** Удаляет переменную окружения `key` из окружения процесса.

```nim
delEnv("TMPDIR")   # заставить инструменты использовать системный каталог temp
```

## Стандартный ввод

### `readLineFromStdin`

```nim
proc readLineFromStdin*(): string {.raises: [IOError].}
```

**Что это такое:** Читает одну строку со стандартного ввода, блокируясь до получения символа переноса строки `\n` или EOF (закрытия stdin). При EOF выбрасывает `EOFError` (подтип `IOError`).

```nim
echo "Введите ваше имя:"
let name = readLineFromStdin()
echo "Привет, ", name
```

---

### `readAllFromStdin`

```nim
proc readAllFromStdin*(): string {.raises: [IOError].}
```

**Что это такое:** Читает всё со стандартного ввода до EOF и возвращает в виде одной строки. Полезно для обработки данных, переданных через канал (pipe).

```nim
let data = readAllFromStdin()
echo "Получено ", data.len, " байт"
```

## Строковые утилиты

### `cmpic`

```nim
proc cmpic*(a, b: string): int
```

**Что это такое:** Сравнивает две строки без учёта регистра. Возвращает `0` если равны, отрицательное число если `a < b`, положительное если `a > b`. Полезно для портабельных сравнений строк в скриптах, которые могут работать на файловых системах с учётом регистра (Linux) и без (Windows/macOS).

```nim
assert cmpic("Hello", "hello") == 0
assert cmpic("abc", "ABC") == 0

if cmpic(getCommand(), "release") == 0:
  switch("opt", "speed")
```

---

### `toExe`

```nim
proc toExe*(filename: string): string
```

**Что это такое:** На Windows добавляет `.exe` к `filename`. На всех остальных платформах возвращает `filename` без изменений. Используйте эту функцию всегда, когда строите путь к исполняемому файлу в кроссплатформенном скрипте.

```nim
let binary = "build" / toExe("myapp")
# На Windows: "build\myapp.exe"
# На Linux/macOS: "build/myapp"
exec binary
```

---

### `toDll`

```nim
proc toDll*(filename: string): string
```

**Что это такое:** Преобразует базовое имя библиотеки в платформенно-специфичное имя разделяемой библиотеки. На Windows: `name.dll`. На POSIX-системах: `libname.so`.

```nim
echo toDll("mylib")
# Windows: "mylib.dll"
# Linux:   "libmylib.so"
```

## Параметры командной строки

Полезны, когда `.nims`-файл вызывается как самостоятельный скрипт (например: `nim myScript.nims arg1 arg2`).

### `paramCount`

```nim
proc paramCount*(): int
```

**Что это такое:** Возвращает количество аргументов командной строки, переданных скрипту. Имя самого скрипта не считается.

```nim
echo "Получено аргументов: ", paramCount()
```

---

### `paramStr`

```nim
proc paramStr*(i: int): string
```

**Что это такое:** Возвращает `i`-й аргумент командной строки. Индекс `0` обычно ссылается на имя скрипта или на первый реальный аргумент в зависимости от контекста — уточните в документации Nim для конкретного способа вызова.

```nim
if paramCount() >= 1:
  let target = paramStr(1)
  echo "Сборка цели: ", target
```

## Задачи (Tasks)

### `task`

```nim
template task*(name: untyped; description: string; body: untyped): untyped
```

**Что это такое:** Определяет именованную задачу сборки, которую можно вызвать командой `nim <имя>` из командной строки. `description` отображается в выводе справки (`nim help`). Передайте пустую строку как `description`, чтобы создать скрытую задачу.

Для каждой задачи с именем `foo` генерируется процедура `fooTask()`, что позволяет вызывать одну задачу из другой.

```nim
task build, "Скомпилировать проект через бэкенд C":
  setCommand "c"

task clean, "Удалить артефакты сборки":
  rmDir "build"
  rmDir "nimcache"

task release, "Сборка оптимизированного релизного бинарника":
  switch("opt", "speed")
  switch("define", "release")
  buildTask()   # вызов задачи build из задачи release
```

Запуск:
```sh
nim build myproject.nims
nim clean myproject.nims
nim release myproject.nims
nim help myproject.nims    # выводит все задачи с описаниями
```

## Метаданные пакета Nimble

Эти переменные и процедуры используются внутри `.nimble`-файлов для описания пакета. Доступны только вне собственного окружения Nimble (у Nimble есть собственные реализации).

### Переменные

| Переменная | Тип | Назначение |
|---|---|---|
| `packageName` | `string` | Имя пакета (по умолчанию — имя `.nims`-файла) |
| `version` | `string` | Строка версии пакета (например, `"1.2.3"`) |
| `author` | `string` | Имя автора пакета |
| `description` | `string` | Краткое человекочитаемое описание |
| `license` | `string` | Идентификатор лицензии по SPDX (например, `"MIT"`) |
| `srcDir` | `string` | Директория с исходными файлами |
| `binDir` | `string` | Директория для скомпилированных бинарников |
| `backend` | `string` | Бэкенд компиляции по умолчанию (`"c"`, `"cpp"`, `"js"`) |
| `bin` | `seq[string]` | Список точек входа для компиляции |
| `skipDirs` | `seq[string]` | Директории, исключаемые из пакета |
| `skipFiles` | `seq[string]` | Файлы, исключаемые из пакета |
| `skipExt` | `seq[string]` | Расширения файлов для исключения |
| `installDirs` | `seq[string]` | Дополнительные директории при установке |
| `installFiles` | `seq[string]` | Дополнительные файлы при установке |
| `installExt` | `seq[string]` | Дополнительные расширения при установке |
| `requiresData` | `seq[string]` | Сырой список строк зависимостей (чтение/запись) |

### `requires`

```nim
proc requires*(deps: varargs[string])
```

**Что это такое:** Объявляет зависимости пакета. Каждая строка следует синтаксису ограничений версий Nimble.

```nim
# В .nimble-файле:
packageName = "mylib"
version     = "0.5.0"
author      = "Алиса"
description = "Полезная библиотека"
license     = "MIT"
srcDir      = "src"

requires "nim >= 1.6.0"
requires "jsony >= 1.1.5", "chronicles >= 0.10.0"
```
