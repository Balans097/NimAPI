# nimhcr.nim — полный справочник

> **Модуль**: `system/nimhcr` — стандартная библиотека Nim  
> **Полное название**: Nim Hot Code Reloading (HCR) Runtime — среда выполнения горячей перезагрузки кода  
> **Флаг компиляции**: `-d:hotcodereloading` (сторона пользователя) или `-d:createNimHcr` (сборка библиотеки)  
> **Поставляется как**: минимальная, *никогда не перезагружаемая* разделяемая библиотека (`nimhcr.dll` / `libnimhcr.so` / `libnimhcr.dylib`), к которой линкуются все перезагружаемые модули.

---

## Что такое горячая перезагрузка кода?

Горячая перезагрузка кода (HCR) позволяет **изменять, перекомпилировать и перезагружать отдельные модули запущенной Nim-программы без её перезапуска**. Состояние глобальных переменных сохраняется между перезагрузками; изменяются только тела функций и новые символы.

Области применения:
- Быстрая итерация над логикой игры / UI без остановки приложения.
- Рабочие процессы в стиле REPL.
- Оперативное обновление долго работающих серверных процессов.

## Архитектурный обзор

Для понимания отдельных функций необходимо усвоить трёхуровневую архитектуру.

```
┌──────────────────────────────────────────────────────────────────┐
│  Главный бинарный файл (никогда не перезагружается)              │
│   • Вызывает hcrInit() один раз при запуске                      │
│   • Вызывает hcrPerformCodeReload() по запросу пользователя      │
└────────────────────────┬─────────────────────────────────────────┘
                         │ загружает / управляет
┌────────────────────────▼─────────────────────────────────────────┐
│  nimhcr (этот файл — тоже никогда не перезагружается)            │
│   • Владеет таблицей переходов  (постоянные адреса функций)      │
│   • Владеет кучей глобалов  (постоянное хранилище переменных)    │
│   • Владеет графом импортов модулей                              │
└──────┬──────────────────────┬────────────────────────────────────┘
       │ регистрирует         │ регистрирует
┌──────▼──────┐        ┌──────▼──────┐   ...ещё перезагружаемые модули
│  module_A   │        │  module_B   │
│  .so/.dll   │        │  .so/.dll   │   (перестраиваются и
│ (перезагруж)│        │ (перезагруж)│    перезагружаются)
└─────────────┘        └─────────────┘
```

### Как адреса функций остаются стабильными (трамплины)

Каждый вызов функции между перезагружаемыми модулями проходит через *трамплин*:

1. Трамплин — это крошечный блок машинного кода (`LongJumpInstruction`), хранящийся в постоянно отображённом в память `jumpTable`.
2. При загрузке `hcrRegisterProc` записывает в трамплин инструкцию перехода на текущую реализацию `_actual`.
3. При перезагрузке `hcrRegisterProc` вызывается снова: он **обновляет существующий трамплин**, перенаправляя прыжок на новую `_actual`. Все вызовы, держащие указатель на трамплин, автоматически начинают вызывать новый код.

### Как глобальные переменные остаются стабильными

1. При первой загрузке `hcrRegisterGlobal` выделяет `size` байт в куче и записывает адрес в указатель-переменную модуля. Возвращает `true` → нужно инициализировать.
2. При перезагрузке тот же вызов находит существующую запись, записывает *тот же адрес* обратно и возвращает `false` → инициализацию пропустить. Значение переменной сохраняется.

## Ключевые типы

| Тип | Описание |
|-----|----------|
| `HcrProcGetter` | `proc(libHandle: pointer, procName: cstring): pointer` — обратный вызов, используемый модулями для разрешения символов из самой nimhcr-библиотеки |
| `HcrGcMarkerProc` | `proc() {.nimcall, raises: [].}` — беспараметрический коллбэк для GC-маркировки, регистрируемый для каждой глобальной переменной |
| `HcrModuleInitializer` | `proc() {.nimcall.}` — сигнатура сгенерированных точек входа `DatInit000`, `Init000`, `HcrCreateTypeInfos`, `HcrInit000` в каждом модуле-DLL |

### Внутренние типы (внутри `createNimHcr`)

| Тип | Описание |
|-----|----------|
| `ShortJumpInstruction` | Упакованный 5-байтный x86 относительный переход: опкод `0xE9` + 32-битное смещение |
| `LongJumpInstruction` | Упакованный 14-байтный x86-64 косвенный абсолютный переход: `0xFF 0x25` + 32-битное смещение + 64-битный целевой указатель |
| `ProcSym` | Запись в таблице функций модуля: `jump` (указатель в таблицу переходов) + `gen` (счётчик поколений) |
| `GlobalVarSym` | Запись в таблице глобалов модуля: `p` (указатель в кучу) + `markerProc` + `gen` |
| `ModuleDesc` | Полная запись модуля: таблица функций, таблица глобалов, список импортов, дескриптор DLL, хэш содержимого, поколение, время последней модификации, обработчики before/after |

## Важное глобальное состояние (внутри `createNimHcr`)

| Переменная | Тип | Описание |
|------------|-----|----------|
| `jumpTable` | `ReservedMemSeq[LongJumpInstruction]` | Постоянно отображённая в память исполняемая область, хранящая все трамплины. Ёмкость по умолчанию: 500 МБ на x86-64 (настраивается через `HOT_CODE_RELOADING_JUMP_TABLE_SIZE`). |
| `modules` | `Table[string, ModuleDesc]` | Отображает путь к файлу → дескриптор модуля. Авторитетный реестр модулей. |
| `root` | `string` | Путь к файлу главного модуля. |
| `system` | `string` | Путь к файлу модуля `system`. |
| `mainDatInit` | `HcrModuleInitializer` | Указатель на `DatInit000` главного модуля. Вызывается при каждой перезагрузке. |
| `generation` | `int` | Монотонно возрастающее целое. Увеличивается на 1 при каждом `hcrPerformCodeReload`. Модули/символы с `gen < generation` после прохода считаются устаревшими и удаляются. |
| `hashToModuleMap` | `Table[string, string]` | Отображает хэш содержимого модуля → путь к файлу. Используется `hcrHasModuleChanged`. |
| `currentModule` | `string` | Путь к файлу модуля, который в данный момент инициализируется. Устанавливается в `initGlobalScope`, чтобы `hcrAddEventHandler` регистрировал обработчики в правильном модуле. |
| `hcrDynlibHandle` | `pointer` | Дескриптор самой nimhcr-DLL, передаваемый каждому модулю для вызова `HcrProcGetter`. |
| `getProcAddr` | `HcrProcGetter` | Коллбэк для разрешения символов из nimhcr — устанавливается один раз в `hcrInit` и передаётся всем модулям в `initHcrData`. |

## Публичный API — функции

Все нижеперечисленные функции экспортируются из разделяемой библиотеки nimhcr. На *стороне библиотеки* (`-d:createNimHcr`) — это реализации; на *стороне пользователя* (`-d:hotcodereloading`) — импортируемые объявления.

---

### `hcrInit`

```nim
proc hcrInit*(moduleList: ptr pointer, main, sys: cstring,
              datInit: HcrModuleInitializer,
              handle: pointer, gpa: HcrProcGetter)
```

**Единственная точка входа, вызываемая главным модулем при запуске программы.** Запускает всю HCR-систему:

1. Сохраняет путь к главному модулю, путь к `system`, указатель `datInit`, дескриптор DLL и функцию разрешения символов.
2. Читает `moduleList` (нуль-завершённый массив `cstring`-указателей) для обнаружения прямых импортов главного модуля.
3. Вызывает `recursiveDiscovery` для DFS-обхода полного графа импортов с загрузкой всех модульных DLL.
4. Вызывает `initModules` для запуска четырёхпроходной последовательности инициализации (см. [Проходы инициализации](#проходы-инициализации)).
5. Устанавливает `currentModule = root`, чтобы код верхнего уровня главного модуля корректно регистрировал обработчики.

Сам главный модуль никогда не загружается как DLL (он уже является хост-процессом); загружаются только его импорты.

```nim
# Генерируется компилятором в C-коде главного модуля:
hcrInit(addr moduleList, "main", "system", datInit000, nimhcrHandle, getProcAddr)
```

---

### `hcrRegisterProc`

```nim
proc hcrRegisterProc*(module: cstring, name: cstring, fn: pointer): pointer
```

**Регистрирует (или обновляет) функцию в постоянной таблице переходов.** Вызывается из `DatInit000` каждого перезагружаемого модуля для каждой экспортируемой функции.

- **`module`**: путь к файлу владеющего модуля.
- **`name`**: искажённое (mangled) имя функции.
- **`fn`**: указатель на реализацию `_actual` в только что загруженной DLL.
- **Возвращает**: указатель на запись трамплина (стабилен между перезагрузками).

**Первая регистрация**: в `jumpTable` добавляется новый слот `LongJumpInstruction`; записывается инструкция перехода.  
**Последующие перезагрузки**: находится *существующая* запись перехода, её инструкция *перезаписывается* для указания на новый `fn`. Все существующие вызовы, держащие адрес трамплина, прозрачно начинают вызывать новый код.

```nim
# Сгенерированный C (упрощённо):
myFunc_ptr = (MyFuncType) hcrRegisterProc("mymodule.nim", "myFunc", myFunc_actual);
```

---

### `hcrGetProc`

```nim
proc hcrGetProc*(module: cstring, name: cstring): pointer
```

**Возвращает адрес трамплина для функции, зарегистрированной другим модулем.** Используется, когда модуль A нуждается в указателе на функцию, определённую в модуле B. В отличие от `hcrRegisterProc`, не выделяет новый трамплин — просто ищет существующий.

Возвращает `nil` (пустой `ProcSym().jump`), если символ ещё не существует (отложенный порядок инициализации).

```nim
# Сгенерированный C (упрощённо):
otherFunc_ptr = (OtherFuncType) hcrGetProc("othermodule.nim", "otherFunc");
```

---

### `hcrRegisterGlobal`

```nim
proc hcrRegisterGlobal*(module: cstring, name: cstring, size: Natural,
                        gcMarker: HcrGcMarkerProc, outPtr: ptr pointer): bool
```

**Регистрирует (или извлекает) постоянное место хранения глобальной переменной в куче.** Вызывается из `Init000` (или `DatInit000`) каждого перезагружаемого модуля для каждой переменной модульного уровня.

- **`module`**: путь к файлу владеющего модуля.
- **`name`**: искажённое имя глобала.
- **`size`**: `sizeof` типа переменной — используется для выделения памяти при первой регистрации.
- **`gcMarker`**: GC-трассировочный коллбэк для этого глобала; может быть `nil` для не-GC-типов.
- **`outPtr`**: выходной параметр — устанавливается в постоянный адрес в куче.
- **Возвращает**: `true` при первой регистрации (вызывающий должен выполнить код инициализации), `false` при последующих вызовах (глобал уже имеет значение — пропустить инициализацию).

Возврат `true`/`false` — это механизм, предотвращающий повторный запуск инициализации при каждой перезагрузке:

```nim
# Концептуальный Nim-код (генерируется компилятором):
var myGlobal_ptr: ptr MyType
if hcrRegisterGlobal("mymodule.nim", "myGlobal", sizeof(MyType), nil,
                     cast[ptr pointer](addr myGlobal_ptr)):
  myGlobal_ptr[] = MyType(field: initialValue)  # запускается только один раз
```

---

### `hcrGetGlobal`

```nim
proc hcrGetGlobal*(module: cstring, name: cstring): pointer
```

**Возвращает адрес в куче для глобала, зарегистрированного другим модулем.** Возвращённый указатель остаётся действительным в течение всего времени работы программы (глобал никогда не перемещается). Используется, когда модуль A обращается к глобалу, определённому в модуле B.

```nim
# Сгенерированный C (упрощённо):
otherGlobal = (OtherType*) hcrGetGlobal("othermodule.nim", "otherGlobal");
```

---

### `hcrHasModuleChanged`

```nim
proc hcrHasModuleChanged*(moduleHash: string): bool
```

**Возвращает `true`, если DLL на диске для модуля с данным хэшем содержимого новее, чем при последней загрузке.** Ищет путь к модулю в `hashToModuleMap`, затем сравнивает `lastModification` с `getLastModificationTime`.

`moduleHash` — хэш содержимого исходника модуля, встраиваемый компилятором при сборке. Косвенность через хэши делает запрос стабильным даже при изменении путей между сборками.

```nim
if hcrHasModuleChanged(myModuleHash):
  echo "myModule был пересобран"
```

---

### `hcrReloadNeeded`

```nim
proc hcrReloadNeeded*(): bool
```

**Возвращает `true`, если *хотя бы один* зарегистрированный модуль изменился на диске.** Перебирает `hashToModuleMap` и вызывает `hcrHasModuleChanged` для каждой записи. Прерывается на первом изменённом модуле.

Пользовательское имя (через стандартный модуль `hotcodereloading`) — `hasAnyModuleChanged`.

```nim
# Типичный паттерн игрового цикла:
while running:
  update()
  render()
  if hasAnyModuleChanged():
    performCodeReload()
```

---

### `hcrPerformCodeReload`

```nim
proc hcrPerformCodeReload*()
```

**Главная точка входа перезагрузки.** Вызывайте из главного модуля (или любого неперезагружаемого кода), когда хотите применить изменения. **Никогда не вызывайте, пока на стеке вызовов есть функции из перезагружаемых модулей.**

Последовательность операций:

1. Если `hcrReloadNeeded()` возвращает `false` — немедленный возврат.
2. **Отключение GC** (`GC_disable` или `GC_disableOrc`) на время перезагрузки, чтобы GC не трассировал устаревшие дескрипторы типов или маркеры в процессе замены. Повторное включение — через `defer`.
3. Увеличение `generation`.
4. Выполнение всех обработчиков *перед перезагрузкой* в порядке DFS (сначала листья, затем корень).
5. Повторный запуск `recursiveDiscovery` + `initModules` для загрузки изменённых DLL и повторной регистрации их символов.
6. Выполнение всех обработчиков *после перезагрузки* в том же порядке DFS.
7. Очистка устаревших модулей и символов (тех, у кого `gen < generation`).

```nim
# Обёртка в модуле hotcodereloading:
proc performCodeReload*() =
  hcrPerformCodeReload()
```

### `hcrAddEventHandler`

```nim
proc hcrAddEventHandler*(isBefore: bool, cb: proc () {.nimcall.})
```

**Регистрирует обратный вызов до или после перезагрузки для текущего инициализируемого модуля.** Флаг `isBefore` разграничивает две фазы:

- `isBefore = true` → коллбэк запускается непосредственно *перед* перезагрузкой модуля (удобно для сохранения состояния).
- `isBefore = false` → коллбэк запускается *после* перезагрузки (удобно для восстановления состояния).

Обработчики хранятся в `modules[currentModule].handlers` и автоматически очищаются в начале каждой перезагрузки (затем модуль заново регистрирует их при выполнении своего кода верхнего уровня).

Пользовательский API — через модуль `hotcodereloading`:

```nim
import hotcodereloading

beforeCodeReload:
  echo "сохраняем состояние перед перезагрузкой..."
  saveMyState()

afterCodeReload:
  echo "восстанавливаем состояние после перезагрузки..."
  restoreMyState()
```

Эти макросы разворачиваются в вызовы `hcrAddEventHandler`.

---

### `hcrAddModule`

```nim
proc hcrAddModule*(module: cstring)
```

**Гарантирует существование записи модуля в таблице `modules`.** Создаёт пустой `ModuleDesc` для `module`, если он ещё не зарегистрирован. Вызывается кодом, сгенерированным компилятором, для модулей, которые импортированы, но ещё не загружены (отложенное обнаружение).

---

### `hcrGeneration`

```nim
proc hcrGeneration*(): int
```

**Возвращает текущий счётчик поколений перезагрузки.** Начинается с 0, увеличивается на 1 при каждом `hcrPerformCodeReload`. Полезен в диагностике и тестах для проверки факта перезагрузки.

```nim
let before = hcrGeneration()
performCodeReload()
assert hcrGeneration() == before + 1
```

---

### `hcrMarkGlobals`

```nim
proc hcrMarkGlobals*() {.compilerproc, exportc, dynlib, nimcall, gcsafe.}
```

**GC-маркер корней для всех глобалов, управляемых HCR.** Вызывается GC для предотвращения сборки HCR-управляемой памяти глобалов. Перебирает каждый `GlobalVarSym` в каждом зарегистрированном модуле и вызывает его `markerProc` (если не `nil`).

Эта функция регистрируется в среде выполнения Nim как маркер глобальных корней через `nimRegisterGlobalMarker`. Работает только в главном потоке.

## Внутренние функции

Не экспортируются. Документируются здесь для объяснения жизненного цикла перезагрузки.

---

### `writeJump(jumpTableEntry, targetFn)`

Записывает **короткий относительный переход** (5 байт, `0xE9`) или **длинный косвенный абсолютный переход** (14 байт, `0xFF 0x25`) в `jumpTableEntry`, в зависимости от того, находится ли адрес назначения в диапазоне ±2 ГБ от трамплина.

- Короткий переход: используется, когда DLL и таблица переходов расположены близко в памяти (часто при отключённом ASLR или в небольших программах).
- Длинный переход: необходим на x86-64, когда DLL загружается далеко от таблицы переходов.

На **x86**: записывает абсолютный адрес поля `absoluteAddr` напрямую.  
На **x86-64**: записывает `offset = 0`, чтобы процессор читал указатель из соседнего поля `absoluteAddr` через RIP-относительную адресацию.

Поддержка ARM (8 или 16 байт) описана, но ещё не реализована.

---

### `loadDll(name)`

Загружает разделяемую библиотеку модуля:

1. Выгружает старый дескриптор (если есть).
2. Создаёт копию DLL с суффиксом `.copy.{dllExt}` — именно *эта копия* и загружается. Это предотвращает блокировку файла на Windows и гарантирует, что оригинал может быть перезаписан системой сборки.
3. Вызывает `loadLib` для копии.
4. Читает `HcrGetImportedModules` из DLL для получения списка импортов.
5. Читает `HcrGetSigHash` для получения хэша содержимого; обновляет `hashToModuleMap`.
6. Очищает обработчики событий модуля (они будут повторно зарегистрированы кодом верхнего уровня).

---

### `initHcrData(name)`

Вызывает точку входа `HcrInit000` DLL названного модуля. Этот проход передаёт модулю `hcrDynlibHandle` и `getProcAddr`, чтобы он мог обратно вызывать функции nimhcr-библиотеки.

---

### `initTypeInfoGlobals(name)`

Вызывает `HcrCreateTypeInfos` в DLL модуля. Этот проход выделяет глобалы с информацией о типах (например, дескрипторы `TNimType`) до запуска `DatInit`, поскольку `DatInit` может ссылаться на эти дескрипторы.

---

### `initPointerData(name)`

Вызывает `DatInit000` в DLL модуля. Здесь происходят вызовы `hcrRegisterProc` и `hcrGetProc` для всех функций модуля.

---

### `initGlobalScope(name)`

Вызывает `Init000` в DLL модуля. Здесь:
- Происходят вызовы `hcrRegisterGlobal` (первый вызов → инициализировать, повторный → пропустить).
- Выполняются Nim-инструкции верхнего уровня (только при первой загрузке, так как `hcrRegisterGlobal` при перезагрузке возвращает `false`).
- Повторно регистрируются обработчики before/after через `hcrAddEventHandler`.

Устанавливает `currentModule` в `name`, чтобы `hcrAddEventHandler` знал, к какому модулю прикреплять обработчики.

---

### `recursiveDiscovery(dlls)`

DFS-обход графа импортов модулей, начиная со списка прямых импортов. Для каждого модуля:

- Уже на текущем поколении → пропустить (уже обработан).
- DLL на диске не изменилась → обновить поколение (сохранить живым), рекурсивно обойти его импорты.
- Изменилась (или новая) → вызвать `loadDll`, рекурсировать, добавить в `modulesToInit` и `allModulesOrderedByDFS`.

Эта процедура заполняет две последовательности, управляющие `initModules`.

---

### `initModules()`

Запускает четырёхпроходную последовательность инициализации для всех модулей в `modulesToInit`:

| Проход | Точка входа | Что делает |
|--------|------------|------------|
| 1 | `HcrInit000` | Связывает указатели на HCR-функции в модуле |
| 2 | `HcrCreateTypeInfos` | Регистрирует глобалы с информацией о типах |
| 3 | `DatInit000` | Регистрирует трамплины функций |
| 4 | `Init000` | Регистрирует глобалы, выполняет код верхнего уровня |

Модуль `system` всегда полностью инициализируется (проходы 3 и 4) раньше любого другого, так как другие модули могут зависеть от его глобалов (например, состояние аллокатора).

---

### `cleanupSymbols(module)`

Удаляет устаревшие символы (функции и глобалы) из таблиц модуля — те, чей `gen` меньше текущего `generation`. Для каждого устаревшего глобала вызывается `dealloc` для освобождения памяти кучи. Для функций удаляется только запись в таблице (слот трамплина пока не освобождается — см. TODO про free-list в исходнике).

---

### `cleanupGlobal(module, name)`

Удаляет один символ глобала из таблицы модуля и вызывает `dealloc` для его указателя в куче.

---

### `unloadDll(name)`

Вызывает `unloadLib` для дескриптора DLL модуля (если загружена). После этого трамплины всё ещё существуют, но указывают на недействительный код — перед вызовом они должны быть перезаписаны.

---

## Проходы инициализации

Каждая загружаемая DLL-модуля предоставляет четыре сгенерированные точки входа, которые nimhcr вызывает в фиксированной последовательности:

```
HcrInit000         ← Проход 1: связать указатели на nimhcr-функции
HcrCreateTypeInfos ← Проход 2: выделить глобалы с инф. о типах
DatInit000         ← Проход 3: зарегистрировать функции (RegisterProc/GetProc)
Init000            ← Проход 4: зарегистрировать глобалы + выполнить код верхнего уровня
```

Последовательность спроектирована так, чтобы более поздние проходы могли безопасно использовать символы, зарегистрированные в более ранних.

---

## Переменные окружения

| Переменная | По умолчанию | Эффект |
|------------|-------------|--------|
| `HOT_CODE_RELOADING_JUMP_TABLE_SIZE` | `500` (МБ на x86-64, `50` на x86) | Управляет максимальным количеством слотов трамплинов. Каждый слот занимает `sizeof(LongJumpInstruction)` байт. Большие значения резервируют больше виртуального адресного пространства. |

## Жизненный цикл перезагрузки: шаг за шагом

```
Пользователь пересобрал module_B.nim
          │
          ▼
Пользователь вызывает performCodeReload()
          │
          ├─ hcrReloadNeeded() ── false → ранний возврат
          │
          ├─ true → GC_disable()
          │
          ├─ inc(generation)   # generation = N+1
          │
          ├─ recursiveExecuteHandlers(isBefore=true, root)
          │    └─ DFS: обработчики C → B → A
          │         └─ каждый: call cb()
          │
          ├─ recursiveDiscovery(root.imports)
          │    └─ module_B новее → loadDll(module_B)
          │         └─ копирует module_B.so → module_B.so.copy.so
          │         └─ загружает копию
          │         └─ обновляет hashToModuleMap
          │
          ├─ initModules()   # 4-проходная инит. для изменённых модулей
          │    ├─ Проход 1 (HcrInit000)         ← связать HCR-указатели
          │    ├─ Проход 2 (HcrCreateTypeInfos)  ← глобалы типов
          │    ├─ Проход 3 (DatInit000)          ← обновить трамплины
          │    └─ Проход 4 (Init000)             ← повторная регистрация глобалов
          │         └─ hcrRegisterGlobal → false → пропустить init
          │         └─ hcrAddEventHandler → перерегистрировать обработчики
          │
          ├─ recursiveExecuteHandlers(isBefore=false, root)
          │    └─ DFS: обработчики C → B → A
          │
          └─ очистка устаревших модулей и символов (gen < N+1)
               └─ dealloc удалённых глобалов
               └─ unloadLib старых копий DLL
               └─ GC_enable()
```

## Ограничения и подводные камни

**Никаких перезагружаемых вызовов на стеке во время перезагрузки**  
Старые DLL выгружаются в процессе `hcrPerformCodeReload`. Если в этот момент на стеке вызовов присутствует функция из перезагружаемого модуля, возврат из неё перейдёт на освобождённую память и приведёт к краху. Всегда запускайте перезагрузку из кода главного (неперезагружаемого) модуля.

**Инициализаторы глобалов выполняются только один раз**  
`hcrRegisterGlobal` возвращает `false` при каждой перезагрузке, поэтому код инициализации никогда не перезапускается. Чтобы изменить значение глобала при перезагрузке, используйте обработчики `beforeCodeReload` / `afterCodeReload`.

**Удаление глобала с последующим его возвратом сбрасывает значение**  
Если удалить глобал, перезагрузить (глобал освобождается через `dealloc`), затем добавить его снова и снова перезагрузить — он будет заново инициализирован. Это единственный способ сбросить значение глобала.

**Код верхнего уровня выполняется только один раз**  
Инструкции верхнего уровня Nim (вне функций) выполняются только при первой загрузке модуля, но не при перезагрузках. Используйте `afterCodeReload` для кода, который должен выполняться при каждой перезагрузке.

**Главный модуль никогда не перезагружается**  
Перезагружаться могут только импортированные модули. Главный модуль является хост-процессом. Поэтому вызовы из главного модуля не могут быть обновлены через HCR.

**ARM пока не поддерживается**  
Код записи трамплинов обрабатывает только x86 и x86-64. Константы для ARM/ARM64 определены, но `writeJump` для них не реализован.

**Таблица переходов имеет фиксированную максимальную ёмкость**  
Каждая регистрация функции занимает один слот постоянно (рекуперация через free-list — TODO). Программам с очень большим количеством уникальных функций может потребоваться увеличить `HOT_CODE_RELOADING_JUMP_TABLE_SIZE`.
