# Модуль `marshal` для Nim — полный справочник

## Обзор

Модуль `marshal` реализует **сериализацию** (преобразование структур данных Nim в строку) и **десериализацию** (восстановление структур данных из строки) с использованием **JSON** в качестве формата передачи данных.

Ключевое отличие от большинства аналогов — поддержка работы как в **рантайме**, так и на этапе **компиляции** (`nimvm`): функции `$$` и `to` можно применять внутри `static:`-блоков и макросов.

### Важные ограничения

- **Информация о типе в вывод не включается.** Если рантаймный тип объекта отличается от компайлтаймного (полиморфизм через наследование `ref`), сериализуются только поля, видимые через тип компайлтаймной переменной.
- **Не поддерживается на платформах JavaScript и NimScript.** При компиляции для этих целей модуль выдаёт ошибку; используйте сторонние пакеты.
- Значения `ptr`, `proc` и `cstring` сериализуются как целочисленные адреса памяти — они **непереносимы** между запусками программы.

---

## Экспортируемый API

Модуль экспортирует **четыре** публичных символа:

| Символ | Вид | Назначение |
|--------|-----|-----------|
| `load` | процедура | Десериализация из `Stream` в существующую переменную |
| `store` | процедура | Сериализация значения в `Stream` |
| `$$` | оператор (процедура) | Сериализация значения в строку `string` |
| `to` | процедура | Десериализация строки `string` в новое значение типа `T` |

---

## `load`

```nim
proc load*[T](s: Stream, data: var T)
```

### Что делает

Читает JSON из потока `s` и **заполняет** переменную `data` декодированными значениями. Переменная должна существовать заранее; `load` изменяет её на месте. При ошибке чтения выбрасывает `IOError`, при несоответствии структуры JSON ожидаемому типу — ошибку парсинга.

### Когда использовать

Используйте `load`, когда у вас уже есть `Stream` — например, файловый поток `openFileStream` или `newStringStream`, обёртывающий строку, полученную по сети. Это потоковый аналог `to`.

### Параметры

| Параметр | Описание |
|----------|---------|
| `s` | Любой `Stream` (файловый, строковый и т. д.), установленный на начало корректного JSON |
| `data` | Переменная (`var`) любого поддерживаемого типа Nim; её поля перезаписываются десериализованными значениями |

### Пример

```nim
import std/[marshal, streams]

var s = newStringStream("[1, 3, 5]")
var a: array[3, int]
load(s, a)
assert a == [1, 3, 5]
```

Чтение объекта из JSON-файла на диске:

```nim
import std/[marshal, streams]

type Person = object
  name: string
  age:  int

var f = openFileStream("person.json")
var p: Person
load(f, p)
f.close()
echo p.name, " — ", p.age, " лет"
```

### Важные замечания

- Позиция в потоке **не сбрасывается** автоматически перед чтением. Убедитесь, что она указывает на начало JSON (при необходимости вызовите `s.setPosition(0)`).
- `data` передаётся как `var` — то есть изменяется напрямую.

## `store`

```nim
proc store*[T](s: Stream, data: sink T)
```

### Что делает

Сериализует `data` в JSON и **записывает** результат в поток `s`. Это зеркальная операция по отношению к `load`: всё, что было записано через `store`, можно впоследствии восстановить через `load`.

`data` объявлен как `sink` — право владения может быть передано для оптимизации. При использовании ARC/ORC не следует обращаться к `data` после вызова.

### Когда использовать

Используйте `store`, когда нужно записать сериализованные данные куда-либо помимо простой строки: в файл, сетевой буфер или любой пользовательский поток.

### Параметры

| Параметр | Описание |
|----------|---------|
| `s` | Записываемый `Stream` |
| `data` | Значение для сериализации; передаётся как `sink` (владение может быть передано) |

### Пример

```nim
import std/[marshal, streams]

var s = newStringStream("")
var a = [1, 3, 5]
store(s, a)
s.setPosition(0)
assert s.readAll() == "[1, 3, 5]"
```

Запись объекта в файл:

```nim
import std/[marshal, streams]

type Config = object
  host: string
  port: int

let cfg = Config(host: "localhost", port: 8080)
var f = openFileStream("config.json", fmWrite)
store(f, cfg)
f.close()
# config.json теперь содержит: {"host": "localhost", "port": 8080}
```

### Важные замечания

- После вызова `store` позиция в потоке находится **в конце** записанных данных. Перед повторным чтением из того же потока вызовите `s.setPosition(0)`.
- Поскольку `data` — `sink`, при необходимости сохранить оригинальное значение передайте копию (на GC-режимах, отличных от ARC).

## `$$` (оператор двойного доллара)

```nim
proc `$$`*[T](x: sink T): string
```

### Что делает

Преобразует **любое** значение Nim `x` в его JSON-представление в виде строки. Это наиболее удобная точка входа для сериализации: не нужно создавать поток — функция просто возвращает `string`.

По сути, это высокоуровневая обёртка над `store`: внутри создаётся временный `StringStream`, вызывается `storeAny`, результат возвращается как строка.

**Работает на этапе компиляции** (`static:`-блоки, макросы). Путь компиляции обрабатывается VM-операциями компилятора (`loadVM`), а не рантаймным потоком.

### Параметры

| Параметр | Описание |
|----------|---------|
| `x` | Любое значение Nim для сериализации; передаётся как `sink` |

### Возвращаемое значение

Строка `string`, содержащая JSON-представление `x`.

### Пример — примитивные типы

```nim
import std/marshal

assert $$(42)      == "42"
assert $$(3.14)    == "3.14"
assert $$("hello") == "\"hello\""
assert $$(true)    == "true"
```

### Пример — объект

```nim
import std/marshal

type Foo = object
  id:  int
  bar: string

let x = Foo(id: 1, bar: "baz")
let y = $$x
assert y == """{"id": 1, "bar": "baz"}"""
```

### Пример — последовательности и массивы

```nim
import std/marshal

let nums = @[10, 20, 30]
assert $$nums == "[10, 20, 30]"

let pair = (name: "Alice", score: 99)
assert $$pair == """{"Field0": "Alice", "Field1": 99}"""  # анонимные поля кортежа
```

### Пример — использование во время компиляции

```nim
import std/marshal

static:
  type Point = object
    x, y: int
  const p = Point(x: 3, y: 7)
  const s = $$p
  assert s == """{"x": 3, "y": 7}"""
```

### Предостережение о полиморфизме

```nim
import std/marshal

type
  A = object of RootObj
  B = object of A
    f: int

let a: ref A = new(B)
assert $$a[] == "{}"   # компайлтаймный тип — A, поле f невидимо
```

Поле `f` принадлежит `B`, но указатель типизирован как `ref A`, поэтому маршаллер видит только часть `A`.

### Отличие от модуля `json`

`$$x` — **не то же самое**, что `%x` из `std/json`. Оператор `%` возвращает дерево `JsonNode` и встраивает метаданные типов для дискриминированных объединений. Используйте `$$` для быстрой двусторонней сериализации с `to`; используйте `%` / `jsonutils.toJson`, когда нужен стандартный, человекочитаемый JSON.

## `to`

```nim
proc to*[T](data: string): T
```

### Что делает

Разбирает JSON-строку и **конструирует** новое значение типа `T`. Это обратная операция по отношению к `$$`. Тип `T` нужно указать явно — через синтаксис `[:T]` или через контекст вывода.

Как и `$$`, работает **на этапе компиляции**.

### Параметры

| Параметр | Описание |
|----------|---------|
| `data` | JSON-строка, как правило полученная через `$$` или `store` |

### Параметр типа

| Параметр | Описание |
|----------|---------|
| `T` | Целевой тип Nim для декодирования; структура должна соответствовать JSON |

### Возвращаемое значение

Новое значение типа `T` с полями, заполненными из JSON.

### Пример — круговое преобразование объекта

```nim
import std/marshal

type Foo = object
  id:  int
  bar: string

let json = """{"id": 1, "bar": "baz"}"""
let z = json.to[:Foo]

assert z.id  == 1
assert z.bar == "baz"
```

### Пример — массив и последовательность

```nim
import std/marshal

let z = "[4, 5, 6]".to[:array[3, int]]
assert z == [4, 5, 6]

let s = "[1, 2, 3]".to[:seq[int]]
assert s == @[1, 2, 3]
```

### Пример — перечисление

```nim
import std/marshal

type Direction = enum North, South, East, West

let d = to[Direction]("""\"North\"""")
assert d == North
```

### Пример — вложенные объекты

```nim
import std/marshal

type
  Address = object
    city: string
  Person = object
    name:    string
    address: Address

let json = """{"name": "Alice", "address": {"city": "Oslo"}}"""
let p = json.to[:Person]
assert p.address.city == "Oslo"
```

### Пример — использование во время компиляции

```nim
import std/marshal

static:
  type Point = object
    x, y: int
  const p = """{"x": 10, "y": 20}""".to[:Point]
  assert p.x == 10
```

### Обработка ошибок

Если структура JSON не соответствует ожидаемому типу, выбрасывается `JsonParsingError` (подтип `CatchableError`). В продакшн-коде при обработке ненадёжного ввода всегда оборачивайте `to` в блок `try/except`.

```nim
import std/marshal

try:
  let bad = "не JSON".to[:int]
except CatchableError as e:
  echo "Ошибка разбора: ", e.msg
```

## Таблица поддерживаемых типов

| Тип Nim | Сериализуется как |
|---------|------------------|
| `bool` | `true` / `false` |
| `char` (ASCII) | JSON-строка `"a"` |
| `char` (не-ASCII) | целое число — порядковый номер символа |
| `int`, `uint`, все варианты | JSON-целое |
| `float`, все варианты | JSON-число с плавающей точкой |
| `string` (корректный UTF-8) | JSON-строка `"…"` |
| `string` (некорректный UTF-8) | JSON-массив байтовых порядковых номеров |
| `array`, `seq` | JSON-массив `[…]` |
| `object`, `tuple` | JSON-объект `{…}` |
| `set` | JSON-массив порядковых номеров |
| `enum` | JSON-строка с именем элемента перечисления |
| `ref`, `ptr` (не nil, первое появление) | пара `[адрес, значение]` |
| `ref`, `ptr` (не nil, повторное) | целое — адрес |
| `ref`, `ptr` (nil) | `null` |
| `proc`, `pointer`, `cstring` | целое — адрес |

---

## Полный пример кругового преобразования

```nim
import std/marshal

type
  Tag  = enum tRed, tGreen, tBlue
  Item = object
    name:  string
    value: float
    tags:  seq[Tag]

let original = Item(name: "widget", value: 3.14, tags: @[tRed, tBlue])

# Сериализация
let json = $$original
echo json
# {"name": "widget", "value": 3.14, "tags": ["tRed", "tBlue"]}

# Десериализация
let restored = json.to[:Item]
assert restored.name  == original.name
assert restored.value == original.value
assert restored.tags  == original.tags
```
