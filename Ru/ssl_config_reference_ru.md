# ssl_config — справочник по модулю

## Обзор

Модуль `ssl_config` предоставляет три строковые константы времени компиляции, содержащие **списки TLS-шифров в формате OpenSSL**, взятые непосредственно из [рекомендаций Mozilla по серверной стороне TLS](https://wiki.mozilla.org/Security/Server_Side_TLS) (файл конфигурации версии 5.4, сгенерирован 2020-06-03).

Единственная цель модуля — дать Nim-программам удобную и авторитетную отправную точку для настройки TLS, чтобы не приходилось вручную копировать и поддерживать «сырые» строки с шифрами. Все три константы можно передавать напрямую туда, где ожидается строка со списком шифров OpenSSL — например, в `newContext` из модулей Nim `net` или `asyncnet`.

### Три уровня совместимости

Mozilla определяет три именованных профиля совместимости, каждый из которых отражает осознанный компромисс между безопасностью и охватом клиентов:

| Константа | Профиль | Старейшие поддерживаемые клиенты |
|---|---|---|
| `CiphersModern` | **Modern** — максимальная безопасность, узкий охват | Firefox 63, Chrome 70, Android 10, Java 11 |
| `CiphersIntermediate` | **Intermediate** — высокая безопасность, широкий охват современных клиентов | Firefox 27, Chrome 31, Android 4.4.2, Java 8u31 |
| `CiphersOld` | **Old** — максимальный охват устаревших клиентов при приемлемом уровне риска | Firefox 1, IE 8 на XP, Android 2.3, Java 6 |

Профили накапливаются: `CiphersOld` является надмножеством `CiphersIntermediate`, который, в свою очередь, является надмножеством `CiphersModern`. Каждый шаг вниз добавляет шифры, поддерживающие более старые клиенты, но несущие повышенную угрозу безопасности.

## Экспортируемые константы

---

### `CiphersModern`

```nim
const CiphersModern* = "TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:TLS_CHACHA20_POLY1305_SHA256"
```

Наиболее строгий профиль. Содержит **исключительно наборы шифров TLS 1.3**, что неявно требует TLS 1.3 — соединения по TLS 1.0, 1.1 и 1.2 с использованием только этих шифров невозможны.

**Что означают эти шифры:**

- `TLS_AES_128_GCM_SHA256` — AES-128 в режиме Galois/Counter с SHA-256 для аутентификации. Быстро работает на оборудовании с поддержкой AES-NI.
- `TLS_AES_256_GCM_SHA384` — AES-256 в режиме GCM с SHA-384. Более высокая стойкость ключа при умеренных дополнительных затратах.
- `TLS_CHACHA20_POLY1305_SHA256` — потоковый шифр ChaCha20 с кодом аутентификации Poly1305. Отличная производительность на устройствах без аппаратного ускорения AES (мобильные устройства, встраиваемые системы).

**Старейшие поддерживаемые клиенты:**

Firefox 63, Android 10.0, Chrome 70, Edge 75, Java 11, OpenSSL 1.1.1, Opera 57, Safari 12.1.

**Когда использовать:** публичные сервисы, где вы контролируете аудиторию или точно знаете его состав — API для современных приложений, внутренние микросервисы, инструменты разработки. Не подходит, если нужна поддержка клиентов, выпущенных до конца 2018 года.

**Пример:**

```nim
import net, ssl_config

let ctx = newContext(cipherList = CiphersModern)
```

### `CiphersIntermediate`

```nim
const CiphersIntermediate* = "TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:" &
  "TLS_CHACHA20_POLY1305_SHA256:ECDHE-ECDSA-AES128-GCM-SHA256:" &
  "ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:" &
  "ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:" &
  "ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:" &
  "DHE-RSA-AES256-GCM-SHA384"
```

*(Для удобства чтения разбито на строки; фактическая константа — единая строка, разделённая двоеточиями.)*

**Рекомендованный профиль по умолчанию** для большинства серверов. К наборам шифров TLS 1.3 из `CiphersModern` добавляет шифры ECDHE и DHE для TLS 1.2, что позволяет согласовывать соединения с клиентами без поддержки TLS 1.3, но при этом исключает все слабые, экспортные и static-RSA шифры.

**Ключевые семейства шифров, добавленные по сравнению с Modern:**

- `ECDHE-{ECDSA,RSA}-AES{128,256}-GCM-SHA{256,384}` — эфемерный обмен ключами на эллиптических кривых Диффи-Хеллмана с AES-GCM. Обеспечивает **упреждающую секретность (forward secrecy)**: прошлые сессии нельзя расшифровать, даже если долгосрочный ключ сервера будет скомпрометирован в будущем.
- `ECDHE-{ECDSA,RSA}-CHACHA20-POLY1305` — ECDHE с ChaCha20-Poly1305. Хорошая производительность на мобильных клиентах.
- `DHE-RSA-AES{128,256}-GCM-SHA{256,384}` — классический (не-EC) эфемерный обмен Диффи-Хеллмана с AES-GCM. Для клиентов с проблемами с ECDHE.

**Старейшие поддерживаемые клиенты:**

Firefox 27, Android 4.4.2, Chrome 31, Edge (все версии), IE 11 на Windows 7, Java 8u31, OpenSSL 1.0.1, Opera 20, Safari 9.

**Когда использовать:** правильный выбор для подавляющего большинства публичных веб-серверов, REST API и сетевых сервисов. Сами Mozilla описывают этот профиль как *«рекомендованную конфигурацию для сервера общего назначения»*.

**Пример:**

```nim
import net, ssl_config

let ctx = newContext(cipherList = CiphersIntermediate)
```

### `CiphersOld`

```nim
const CiphersOld* = "TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:" &
  "TLS_CHACHA20_POLY1305_SHA256:ECDHE-ECDSA-AES128-GCM-SHA256:..." &
  "...:DES-CBC3-SHA"
```

*(Показано в сокращении; фактическая константа содержит 28 записей, разделённых двоеточиями.)*

Наиболее разрешительный профиль. Добавляет широкий спектр старых шифров TLS 1.2, а также TLS 1.0/1.1, включая шифры без упреждающей секретности (`AES128-SHA`, `AES256-SHA`, `DES-CBC3-SHA`) и наборы с SHA-1 для аутентификации. Единственный профиль, допускающий соединения от очень старых клиентов — IE 8 на Windows XP или Android 2.3.

**Дополнительные семейства шифров по сравнению с Intermediate:**

- `DHE-RSA-CHACHA20-POLY1305` — DHE (не-EC) с ChaCha20-Poly1305.
- `ECDHE-{ECDSA,RSA}-AES{128,256}-SHA{256,384}` — ECDHE с HMAC на SHA-1/SHA-2 вместо GCM (режим CBC в TLS 1.2).
- `ECDHE-{ECDSA,RSA}-AES{128,256}-SHA` — ECDHE с аутентификацией SHA-1.
- `DHE-RSA-AES{128,256}-SHA256` — DHE с SHA-2 HMAC.
- `AES{128,256}-{GCM-SHA256,SHA256,SHA}` — статический обмен ключами RSA (без упреждающей секретности).
- `DES-CBC3-SHA` — Triple-DES в режиме CBC с SHA-1. Очень старый шифр, включён исключительно ради IE 8 на Windows XP.

**Старейшие поддерживаемые клиенты:**

Firefox 1, Android 2.3, Chrome 1, Edge 12, IE 8 на Windows XP, Java 6, OpenSSL 0.9.8, Opera 5, Safari 1.

**Когда использовать:** только при наличии конкретного бизнес-требования обслуживать устаревшие клиенты, которые нельзя обновить. Mozilla предупреждает, что этот профиль имеет *«известные уязвимости»* и должен применяться лишь тогда, когда нет альтернативы. Наличие `DES-CBC3-SHA` и static-RSA шифров — это намеренная уступка совместимости, а не рекомендация с точки зрения безопасности.

**Пример:**

```nim
import net, ssl_config

# Интранет-приложение, обязанное поддерживать очень старых встраиваемых клиентов:
let ctx = newContext(cipherList = CiphersOld)
```

## Выбор подходящего профиля

Дерево решений простое:

1. **Все ваши клиенты поддерживают TLS 1.3?** → Используйте `CiphersModern`.
2. **Среди ваших клиентов есть что-либо из последних ~10 лет?** → Используйте `CiphersIntermediate`. Это правильный выбор по умолчанию для почти любого публичного сервиса.
3. **Есть жёсткое требование поддерживать клиентов выпуска до 2013 года (IE 8, Android 2.x, Java 6)?** → Используйте `CiphersOld` и задокументируйте причину.

При любых сомнениях **`CiphersIntermediate` всегда является наиболее безопасной отправной точкой**. Он полностью покрывает современных клиентов, обеспечивает упреждающую секретность для всех соединений и исключает действительно слабые шифры, которые добавляет `CiphersOld`.

## Практический пример — HTTPS-сервер с выбором профиля

```nim
import asynchttpserver, asyncnet, net, ssl_config

proc makeServerContext(profile: string): SslContext =
  let ciphers = case profile
    of "modern":       CiphersModern
    of "old":          CiphersOld
    else:              CiphersIntermediate   # безопасный выбор по умолчанию
  newContext(
    certFile   = "server.crt",
    keyFile    = "server.key",
    cipherList = ciphers
  )

let ctx = makeServerContext("intermediate")
```

## Важные замечания

- **Константы зафиксированы в снимке от 2020-06-03** рекомендаций Mozilla (версия 5.4). Mozilla периодически обновляет свои рекомендации; для продакшн-системы с высокими требованиями к безопасности проверяйте актуальную версию на [ssl-config.mozilla.org](https://ssl-config.mozilla.org) и обновляйте список шифров при необходимости.
- Строки используют **формат списка шифров OpenSSL**. Они могут не работать напрямую с другими TLS-библиотеками (например, mbedTLS, wolfSSL) без трансляции.
- Все три профиля **включают шифры TLS 1.3** (`TLS_AES_*` и `TLS_CHACHA20_*`). В OpenSSL 1.1.1+ шифры TLS 1.3 управляются отдельно от шифров TLS 1.2, поэтому их присутствие в строке безвредно и гарантирует, что TLS 1.3 будет использоваться при наличии поддержки.
- Модуль **не содержит процедур** — это чисто декларативный модуль констант с нулевыми накладными расходами во время выполнения.
