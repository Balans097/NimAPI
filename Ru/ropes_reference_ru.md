# ropes — справочник по модулю

> **Стандартная библиотека Nim** | `import std/ropes`

## Что такое канат (rope)?

**Канат (rope)** — это структура данных в форме дерева для хранения строк. Вместо одного непрерывного массива символов она строит бинарное дерево, листья которого хранят обычные Nim-строки. Внутренние узлы содержат только суммарную длину — символов они не хранят.

```
       [внутренний, len=11]
      /                    \
[лист "Hello, "]        [лист "Nim!"]
```

Ключевая идея: **конкатенация выполняется за O(1)**. Объединение двух канатов создаёт единственный новый внутренний узел, указывающий на оба дерева — символы не копируются. Строка-результат материализуется только при вызове `$` или записи в файл: дерево обходится ровно один раз за O(n).

Это делает канаты идеальным инструментом для **постепенного построения больших строк** — генераторов кода, шаблонных движков, форматировщиков журналов и любого другого кода, где сотни или тысячи фрагментов склеиваются вместе. С обычными строками каждый `&` копирует левую часть, что в итоге даёт O(n²). С канатами та же работа выполняется за O(n).

**Компромисс:** Доступ к символу по индексу (`r[i]`) обходит дерево и в худшем случае занимает O(n), тогда как у обычных строк это O(1). Канат — не универсальная замена строке, а специализированный инструмент.

**`nil` — пустой канат.** Отдельного значения-пустышки нет: ссылка `nil` типа `Rope` означает пустую строку во всём API.

## Типы

### `Rope`

```nim
type Rope* {.acyclic.} = ref object
```

Сам канат. Это ссылочный тип — присваивание копирует ссылку, а не данные. Поскольку структура неизменяема, поддеревья можно безопасно разделять между несколькими значениями `Rope` без копирования.

**Пустой канат — это `nil`.** Все процедуры модуля корректно обрабатывают `nil` (например, `len(nil) == 0`, `nil & r == r`).

## Конструкторы

### `rope(string)`

```nim
proc rope*(s: string = ""): Rope
```

Преобразует Nim-строку в листовой узел каната. Пустая строка возвращает `nil` (пустой канат).

Когда кэш листьев включён (см. [`enableCache`](#enablecache)), одинаковые строки используют один и тот же листовой узел в памяти вместо создания нового.

```nim
let r = rope("Привет, мир!")
echo $r  # Привет, мир!

let empty = rope("")
echo empty == nil  # true — пустой канат есть nil
```

---

### `rope(int)`

```nim
proc rope*(i: BiggestInt): Rope
```

Преобразует целое число в его десятичное строковое представление и оборачивает в канат. Принимает любой целочисленный тип, помещающийся в `BiggestInt`.

```nim
let r = rope(2024)
echo $r  # 2024

# Удобно при построении строк, смешивающих текст и числа:
let msg = rope("Год: ") & rope(2024)
echo $msg  # Год: 2024
```

---

### `rope(float)`

```nim
proc rope*(f: BiggestFloat): Rope
```

Преобразует число с плавающей точкой в его строковое представление и оборачивает в канат. Форматирование соответствует стандартному `$` Nim для вещественных чисел.

```nim
let r = rope(3.14159)
echo $r  # 3.14159

let line = rope("Пи ≈ ") & rope(3.14159)
echo $line  # Пи ≈ 3.14159
```

## Конкатенация

### `&` (Rope, Rope)

```nim
proc `&`*(a, b: Rope): Rope
```

Создаёт новый внутренний узел, левым потомком которого является `a`, а правым — `b`. **Символы не копируются** — это чисто операция над указателями, выполняется за O(1). Если один из операндов равен `nil`, возвращается другой операнд без выделения нового узла.

```nim
let r = rope("Привет, ") & rope("мир!")
echo $r  # Привет, мир!

# nil — нейтральный элемент для &
let r2 = nil & rope("Nim")
echo $r2  # Nim
```

---

### `&` (Rope, string)

```nim
proc `&`*(a: Rope, b: string): Rope
```

Удобная перегрузка — преобразует `b` в канат, затем выполняет конкатенацию. Эквивалентно `a & rope(b)`.

```nim
let r = rope("Привет, ") & "мир!"
echo $r  # Привет, мир!
```

---

### `&` (string, Rope)

```nim
proc `&`*(a: string, b: Rope): Rope
```

Зеркальная перегрузка — преобразует `a` в канат, затем выполняет конкатенацию. Эквивалентно `rope(a) & b`.

```nim
let r = "Привет, " & rope("мир!")
echo $r  # Привет, мир!
```

---

### `&` (openArray[Rope])

```nim
proc `&`*(a: openArray[Rope]): Rope
```

Объединяет массив или последовательность канатов слева направо в один канат. Для вызова этой перегрузки используется синтаксис префиксного оператора `&` (без левого операнда).

```nim
let parts = [rope("один"), rope(", "), rope("два"), rope(", "), rope("три")]
let r = &parts
echo $r  # один, два, три
```

---

### `add(Rope, Rope)`

```nim
proc add*(a: var Rope, b: Rope)
```

Изменяет `a` на месте, дописывая к нему `b`. Эквивалентно `a = a & b`. Удобно при постепенном построении каната в цикле без введения отдельной переменной-результата.

```nim
var r = rope("Строки:\n")
for i in 1..3:
  r.add(rope("  строка ") & rope(i) & rope("\n"))
echo $r
# Строки:
#   строка 1
#   строка 2
#   строка 3
```

---

### `add(Rope, string)`

```nim
proc add*(a: var Rope, b: string)
```

То же, что `add(Rope, Rope)`, но принимает обычную строку в качестве правого операнда. Внутренне конвертирует `b` в канат.

```nim
var r = rope("Привет")
r.add(", мир!")
echo $r  # Привет, мир!
```

## Форматирование

### `%` (строка формата)

```nim
proc `%`*(frmt: string, args: openArray[Rope]): Rope
```

Оператор подстановки — работает как строковый `%` в Nim, но производит `Rope`. Поддерживает три стиля заполнителей:

| Синтаксис | Значение |
|---|---|
| `$1`, `$2`, … | Позиционный: подставить `args[0]`, `args[1]`, … |
| `$#` | Последовательный: каждый `$#` берёт следующий аргумент по порядку |
| `${1}`, `${2}`, … | Скобочный позиционный (то же, что `$1`, но без неоднозначности) |
| `$$` | Буквальный знак доллара `$` |

> **Примечание:** Синтаксисы `$identifier` и `${identifier}` (именованные переменные) **не поддерживаются**. Работают только числовые индексы.

```nim
# Позиционный
let r1 = "$1 + $2 = $3" % [rope("1"), rope("2"), rope("3")]
echo $r1  # 1 + 2 = 3

# Последовательный — читается чище, когда порядок совпадает
let r2 = "$# $# $#" % [rope("Nim"), rope("—"), rope("отличный язык")]
echo $r2  # Nim — отличный язык

# Буквальный $ вместе с позиционным
let r3 = "$$1 стоит $1 руб." % [rope("500")]
echo $r3  # $1 стоит 500 руб.
```

---

### `addf`

```nim
proc addf*(c: var Rope, frmt: string, args: openArray[Rope])
```

Сокращение для `c.add(frmt % args)`. Добавляет отформатированный результат прямо к существующему канату без создания именованной промежуточной переменной. Удобно внутри циклов и построителей.

```nim
var report = rope("")
report.addf("Пользователь: $1\n", [rope("Алиса")])
report.addf("Оценка: $1\n", [rope("95")])
echo $report
# Пользователь: Алиса
# Оценка: 95
```

## Доступ и инспекция

### `len`

```nim
proc len*(a: Rope): int
```

Возвращает суммарную длину каната в символах — сумму длин всех листовых строк. Это значение кэшируется в каждом узле, поэтому операция выполняется за **O(1)**. `len(nil) == 0`.

```nim
let r = rope("Привет") & rope(", ") & rope("мир!")
echo r.len  # 12
echo len(nil)  # 0
```

---

### `[]` (символ по индексу)

```nim
proc `[]`*(r: Rope, i: int): char
```

Возвращает символ на позиции `i` (нумерация с нуля). Возвращает `'\0'`, если `i` выходит за границы (`i < 0` или `i >= r.len`) либо если канат равен `nil`.

**Предупреждение о производительности:** Операция обходит дерево и в худшем случае занимает **O(n)**. Если нужно читать много отдельных символов, сначала преобразуйте канат в строку через `$`, а затем индексируйте строку.

```nim
let r = rope("Привет, Nim!")
echo r[0]   # П
echo r[8]   # N
echo r[100] # '\0'  (за пределами диапазона)
```

---

### `leaves` (итератор)

```nim
iterator leaves*(r: Rope): string
```

Последовательно возвращает каждую листовую строку каната слева направо, не материализуя всю строку целиком. Это наиболее эффективный способ обработки содержимого каната по частям — при записи в сокет, хешировании или потоковом выводе.

Количество листьев равно числу вызовов `rope(string)`, использованных при построении, за вычетом узлов, которые были переиспользованы кэшем.

```nim
let r = rope("Привет") & rope(", ") & rope("мир!")
for leaf in r.leaves:
  stdout.write(leaf)  # записывает листовые строки по одной
echo ""
# Вывод: Привет, мир!

# Полезно для хеширования без выделения полной строки:
import std/md5
var ctx: MD5Context
md5Init(ctx)
for leaf in r.leaves:
  md5Update(ctx, leaf)
```

---

### `items` (итератор)

```nim
iterator items*(r: Rope): char
```

Последовательно возвращает каждый отдельный символ каната. Реализован как итерация по листьям, а затем по символам внутри каждого листа. Удобен для пословного прохода, но создаёт больше накладных расходов, чем работа с листьями напрямую.

```nim
let r = rope("абв") & rope("где")
var count = 0
for ch in r:
  if ch == 'а' or ch == 'е': inc count
echo count  # 2
```

## Вывод

### `$` (в строку)

```nim
proc `$`*(r: Rope): string
```

Разворачивает канат в обычную Nim-строку. Обходит все листья по порядку и конкатенирует их. Это единственный момент в жизни каната, когда происходит выделение O(n) памяти и копирование. Результирующая строка содержит в точности то же содержимое, как если бы все фрагменты были склеены обычным `&`.

```nim
let r = rope("Привет") & rope(", ") & rope("мир!")
let s: string = $r
echo s       # Привет, мир!
echo s.len   # 12
```

---

### `write(File, Rope)`

```nim
proc write*(f: File, r: Rope)
```

Записывает содержимое каната в открытый файловый дескриптор `File`, лист за листом, без выделения всей объединённой строки. Наиболее экономный по памяти способ сохранить большой канат в файл.

```nim
let output = rope("Строка 1\n") & rope("Строка 2\n") & rope("Строка 3\n")
var f = open("output.txt", fmWrite)
write(f, output)  # записывает все листья без построения полной строки
close(f)
```

---

### `write(Stream, Rope)`

```nim
proc write*(s: Stream, r: Rope)
```

Записывает содержимое каната в любой `Stream` (файловый поток, строковый поток, сетевой поток и т.д.), лист за листом.

```nim
import std/streams
let r = rope("часть1") & rope("часть2")
var strm = newStringStream()
write(strm, r)
strm.setPosition(0)
echo strm.readAll()  # часть1часть2
```

## Сравнение с файлом

Эти процедуры недоступны на платформах JavaScript и NimScript.

### `equalsFile(Rope, File)`

```nim
proc equalsFile*(r: Rope, f: File): bool
```

Возвращает `true`, если байтовое содержимое уже открытого файла `f` идентично содержимому каната `r`. Сравнение выполняется инкрементально с буфером 1 КБ — ни полная строка каната, ни полный файл не загружаются в память одновременно. Эффективно для проверки соответствия сгенерированного каната эталонному файлу.

```nim
let r = rope("ожидаемое содержимое\n")
var f = open("эталон.txt", fmRead)
if equalsFile(r, f):
  echo "Совпадение!"
close(f)
```

---

### `equalsFile(Rope, string)`

```nim
proc equalsFile*(r: Rope, filename: string): bool
```

Удобная перегрузка — открывает файл по пути `filename`, сравнивает, затем закрывает. Возвращает `false`, если файл не существует.

```nim
let generated = rope("строка один\n") & rope("строка два\n")
if generated.equalsFile("ожидаемый.txt"):
  echo "Вывод совпадает с эталонным файлом."
else:
  echo "Несовпадение — перезаписываем эталон."
  var f = open("ожидаемый.txt", fmWrite)
  write(f, generated)
  close(f)
```

## Кэш листьев

Кэш — это дерево splay, хранящее листовые узлы каната, отдельное для каждого потока. Когда кэш включён, вызов `rope("некоторая строка")` сначала ищет в кэше существующий лист с теми же данными. Если такой найден, возвращается уже существующий узел без выделения памяти. Это снижает потребление памяти, когда одни и те же строки многократно преобразуются в листья — например, повторяющиеся ключевые слова в генераторе кода.

**Компромисс:** Операции с кэшем требуют обхода дерева и сравнения строк, поэтому пропускная способность снижается. Включайте кэш только тогда, когда экономия памяти важнее скорости.

### `enableCache`

```nim
proc enableCache*()
```

Включает кэш листьев для текущего потока. После этого вызова `rope(string)` будет искать и при возможности переиспользовывать существующие листовые узлы.

```nim
enableCache()
let a = rope("ключевое_слово")
let b = rope("ключевое_слово")  # переиспользует тот же листовой узел, что и `a`
# a и b указывают на один и тот же объект Rope
```

---

### `disableCache`

```nim
proc disableCache*()
```

Отключает кэш и устанавливает дерево кэша в `nil`, позволяя сборщику мусора освободить занятую им память. Последующие вызовы `rope(string)` снова будут выделять новые узлы.

```nim
disableCache()
# GC теперь может собрать все ранее кэшированные листовые узлы
```

## Полный пример

Следующий пример демонстрирует построение небольшой HTML-страницы в виде каната, её инспекцию и запись в файл — без создания полной промежуточной строки.

```nim
import std/ropes

proc tag(name: string, content: Rope): Rope =
  rope("<" & name & ">") & content & rope("</" & name & ">")

proc li(text: string): Rope =
  tag("li", rope(text))

var body: Rope
body.add(tag("h1", rope("Демо канатов")))

var list: Rope
for item in ["яблоки", "бананы", "вишни"]:
  list.add(li(item))
body.add(tag("ul", list))

let page = tag("html", tag("body", body))

echo "Длина: ", page.len

# Запись прямо в файл — без выделения полной строки
var f = open("page.html", fmWrite)
write(f, page)
close(f)

# Проверка
if page.equalsFile("page.html"):
  echo "Файл записан корректно."

# Инспекция структуры через листья
var n = 0
for _ in page.leaves: inc n
echo "Количество листьев: ", n
```

## Краткая сводка

| Символ | Сигнатура | Стоимость | Описание |
|---|---|---|---|
| `rope` | `rope(string)` → `Rope` | O(1) | Обернуть строку в лист каната |
| `rope` | `rope(int)` → `Rope` | O(1) | Целое число → канат |
| `rope` | `rope(float)` → `Rope` | O(1) | Вещественное число → канат |
| `len` | `len(Rope)` → `int` | **O(1)** | Суммарная длина (кэшируется) |
| `&` | `Rope & Rope` → `Rope` | **O(1)** | Конкатенация двух канатов |
| `&` | `Rope & string` → `Rope` | **O(1)** | Конкатенация каната и строки |
| `&` | `string & Rope` → `Rope` | **O(1)** | Конкатенация строки и каната |
| `&` | `&openArray[Rope]` → `Rope` | O(k) | Конкатенация массива канатов |
| `add` | `add(var Rope, Rope)` | **O(1)** | Дописать канат на месте |
| `add` | `add(var Rope, string)` | **O(1)** | Дописать строку на месте |
| `%` | `string % openArray[Rope]` → `Rope` | O(fmt) | Форматирование с подстановкой |
| `addf` | `addf(var Rope, string, openArray[Rope])` | O(fmt) | Форматировать и дописать |
| `[]` | `Rope[int]` → `char` | **O(n)** | Символ по индексу |
