# enumutils — справочник по модулю

## Обзор

Модуль `enumutils` расширяет встроенную поддержку перечислений в Nim утилитами для работы с **дырявыми перечислениями** (enum с непоследовательными целочисленными значениями), **интроспекцией символов** (запрос позиции поля и его имени в исходном коде независимо от порядкового значения) и **инфраструктурой разбора строк** (макрос для генерации эффективных `case`-операторов на основе сопоставления строк).

Перед изучением конкретных символов необходимо понять два понятия о типах перечислений в Nim:

- **`OrdinalEnum`** — перечисление, чьи значения образуют непрерывный целочисленный диапазон начиная от наименьшего члена. Компилятор может использовать порядковое значение напрямую как индекс массива. Пример: `enum { a, b, c }` или `enum { a = 10, b, c }`.
- **`HoleyEnum`** — перечисление с пробелами в целочисленных значениях. Значения не могут использоваться как индексы массива без дополнительного отображения. Пример: `enum { a = 2, b = 4, c = 8 }`.

Ряд процедур модуля работает исключительно с `HoleyEnum`, поскольку для `OrdinalEnum` уже есть эффективная встроенная поддержка через порядковое значение.

## `items` (итератор)

```nim
iterator items*[T: HoleyEnum](E: typedesc[T]): T
```

### Описание

Перебирает все объявленные члены дырявого перечисления типа `E`, выдавая их в порядке объявления. Это восполняет пробел, оставленный встроенным итератором `items`, работающим только с `OrdinalEnum`. Для дырявого перечисления нет безопасного способа обойти значения, просто увеличивая целочисленный счётчик — при этом обход пройдёт через пробелы и произведёт значения, не соответствующие ни одному объявленному полю. Данный итератор решает задачу, опираясь на исчерпывающее знание компилятора о членах перечисления, сгенерированное во время компиляции через внутренний макрос `enumFullRange`.

Итератор вызывается на **типе**, а не на значении: `for x in MyEnum: …` — это зеркалит синтаксис, используемый для ординальных перечислений.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `E` | `typedesc[T]` | Дырявый тип перечисления для итерации |

### Что выдаёт

Каждый объявленный член `T` в порядке объявления, как значение типа `T`.

### Примеры

**Базовая итерация дырявого перечисления:**

```nim
import std/enumutils

type Color = enum
  Red   = 1
  Green = 3
  Blue  = 7

for c in Color:
  echo c, " = ", c.ord
# Red = 1
# Green = 3
# Blue = 7
# (Значения 2, 4, 5, 6 никогда не выдаются — это пробелы)
```

**Сбор всех членов в последовательность с помощью `toSeq`:**

```nim
import std/enumutils
from std/sequtils import toSeq

type Priority = enum
  Low    = 10
  Medium = 20
  High   = 50

let all = Priority.toSeq
assert all == [Low, Medium, High]
```

**Обобщённый параметр типа перечисления:**

```nim
import std/enumutils
from std/sequtils import toSeq

type Container[T] = enum
  Empty  = 0
  Single = 5
  Many   = 9

assert Container[string].toSeq == [Container[string].Empty,
                                   Container[string].Single,
                                   Container[string].Many]
```

**Зачем это нужно — опасность наивной итерации:**

```nim
# НЕПРАВИЛЬНО: для дырявого перечисления это итерирует недопустимые ординалы
# for i in Low.ord .. High.ord:
#   echo Priority(i)  # падает на i=11..19, 21..49 и т.д.

# ПРАВИЛЬНО
for p in Priority:
  echo p  # всегда безопасно, никогда не попадает в пробел
```

## `symbolRank`

```nim
template symbolRank*[T: enum](a: T): int
```

### Описание

Возвращает **позицию с нулевым началом** значения перечисления `a` в списке объявленных членов типа `T` — считая от первого объявленного члена с рангом `0`. Это принципиально отличается от `ord(a)`, возвращающего целочисленное значение, присвоенное полю. `symbolRank` считает, *сколько полей объявлено в исходном коде до данного*, вне зависимости от того, какие целочисленные значения им присвоены.

Для типов `OrdinalEnum` результат вычисляется тривиально во время компиляции как `ord(a) - T.low.ord` — единственное вычитание без затрат в рантайме. Для типов `HoleyEnum` реализация использует таблицу поиска, генерируемую во время компиляции, для небольших перечислений (охват ≤ 255) — это даёт O(1) в рантайме, а для крупных перечислений — линейный проход по объявленным членам.

Если `a` содержит целочисленное значение, не соответствующее ни одному объявленному полю (что возможно через небезопасное приведение типов), `symbolRank` возбуждает `IndexDefect`.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `a` | `T` (любое перечисление) | Значение перечисления, чей ранг запрашивается |

### Возвращаемое значение

Неотрицательный `int` в диапазоне `0 ..< T.enumLen`, представляющий позицию `a` в порядке объявления.

### `symbolRank` vs `ord` — ключевое различие

| | `ord(a)` | `a.symbolRank` |
|---|---|---|
| Что возвращает | Целочисленное значение, присвоенное полю | Позиция в списке объявлений (с нуля) |
| `enum { a=10, b, c }`, значение `c` | `12` | `2` |
| `enum { a=-3, b=10, c=11 }`, значение `c` | `11` | `2` |
| `enum { a, b, c }`, значение `c` | `2` | `2` (совпадает для непрерывных) |

### Примеры

**Непрерывное (ординальное) перечисление — ранг равен смещению ординала:**

```nim
import std/enumutils

type Direction = enum
  North  # ord = 0, ранг = 0
  East   # ord = 1, ранг = 1
  South  # ord = 2, ранг = 2
  West   # ord = 3, ранг = 3

assert South.symbolRank == 2
assert South.ord == 2   # для непрерывных совпадает
```

**Дырявое перечисление, начинающееся с ненулевого значения:**

```nim
import std/enumutils

type Status = enum
  Pending  = 100
  Active   = 200
  Closed   = 500

assert Pending.symbolRank == 0   # первый объявленный → ранг 0
assert Active.symbolRank  == 1   # второй объявленный → ранг 1
assert Closed.symbolRank  == 2   # третий объявленный → ранг 2

assert Pending.ord == 100        # ординал — 100, а не 0
assert Closed.ord  == 500        # ординал — 500, а не 2
```

**Перечисление с отрицательными и непоследовательными значениями:**

```nim
import std/enumutils

type A = enum
  a0 = -3
  a1 = 10
  a2       # = 11 (автоинкремент от предыдущего)
  a3 = (20, "f3Alt")   # ординал 20, пользовательское строковое представление

assert a0.symbolRank == 0
assert a2.symbolRank == 2
assert a3.symbolRank == 3
assert a2.ord == 11
assert a3.ord == 20
```

**`IndexDefect` для недопустимого ординала:**

```nim
import std/enumutils

type Sparse = enum
  X = 1
  Y = 3

var bad = 2.Sparse   # небезопасно: 2 — пробел, не объявленное поле
doAssertRaises(IndexDefect):
  discard bad.symbolRank
```

## `symbolName`

```nim
func symbolName*[T: enum](a: T): string
```

### Описание

Возвращает **имя идентификатора в исходном коде** поля перечисления `a` — имя в том виде, в котором оно написано в объявлении `type` — в виде строки. Это отличается от `$a` (оператора `$`), возвращающего *строковое представление* поля. В Nim полю перечисления можно задать пользовательское строковое представление, отличающееся от имени идентификатора, написав `имяПоля = (ординал, "пользовательскаяСтрока")` или `имяПоля = "пользовательскаяСтрока"`. В этом случае `$a` возвращает пользовательскую строку, тогда как `symbolName(a)` всегда возвращает идентификатор.

Это полезно в любом случае, когда нужен именно Nim-идентификатор — для кодогенерации, протоколов сериализации, использующих идентификаторы, а не отображаемые имена, для отладки или документации.

Внутренне `symbolName` использует `symbolRank` для индексации в массив строк идентификаторов, генерируемый во время компиляции, поэтому характеристики производительности зеркалят `symbolRank`.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `a` | `T` (любое перечисление) | Значение перечисления, чьё имя идентификатора запрашивается |

### Возвращаемое значение

`string`, содержащая имя идентификатора поля `a` в исходном коде.

### `symbolName` vs `$` — ключевое различие

| Объявление поля | `$a` | `a.symbolName` |
|---|---|---|
| `b0 = (10, "kb0")` | `"kb0"` | `"b0"` |
| `b1 = "kb1"` | `"kb1"` | `"b1"` |
| `b2` (без пользовательской строки) | `"b2"` | `"b2"` (совпадает) |

### Примеры

**Перечисление с пользовательскими строковыми представлениями:**

```nim
import std/enumutils

type B = enum
  b0 = (10, "kb0")   # ординал 10, отображается как "kb0"
  b1 = "kb1"         # ординал 11 (авто), отображается как "kb1"
  b2                 # ординал 12 (авто), отображается как "b2"

let x = B.low   # = b0

assert x.symbolName == "b0"    # имя идентификатора
assert $x        == "kb0"      # пользовательская строка отображения

assert B.high.symbolName == "b2"
assert $B.high   == "b2"       # нет пользовательской строки → совпадает с идентификатором
```

**Дырявое перечисление — имя идентификатора всегда совпадает с объявленным:**

```nim
import std/enumutils

type C = enum
  c0 = -3
  c1 = 4
  c2 = 20

assert c1.symbolName == "c1"
assert c1.ord == 4
```

**Использование `symbolName` для сериализации, требующей обхода через идентификатор:**

```nim
import std/enumutils

type Severity = enum
  Debug   = (0, "DBG")
  Info    = (1, "INF")
  Warning = (2, "WRN")
  Error   = (3, "ERR")

# $sev даёт краткую форму отображения; symbolName — канонический идентификатор
for sev in Severity:
  echo "id=", sev.symbolName, "  отображение=", $sev
# id=Debug    отображение=DBG
# id=Info     отображение=INF
# id=Warning  отображение=WRN
# id=Error    отображение=ERR
```

**Использование во время компиляции:**

```nim
import std/enumutils

type Flag = enum FlagA, FlagB, FlagC

static:
  assert FlagC.symbolName == "FlagC"
```

## `genEnumCaseStmt`

```nim
macro genEnumCaseStmt*(typ: typedesc, argSym: typed, default: typed,
    userMin, userMax: static[int],
    normalizer: static[proc(s: string): string]): untyped
```

### Описание

Генерирует во время компиляции `case`-оператор, преобразующий **нормализованную строку** в значение перечисления типа `typ`. Это фундаментальный строительный блок для реализации процедур в стиле `parseEnum` с произвольной нормализацией строк — например, для регистронезависимого разбора.

Макрос обходит AST объявления перечисления `typ`, применяет `normalizer` к строковому представлению каждого поля и генерирует ветку `of` для каждой отдельной нормализованной строки, отображающей её в соответствующее значение перечисления. Если входная строка не совпадает ни с одной ветвью, макрос либо возбуждает `ValueError` (когда `default` равен `nil`), либо переходит к предоставленному вызывающим значению по умолчанию.

Этот макрос предназначен для **авторов библиотек и фреймворков**, которым необходимо строить обобщённые парсеры «строка → перечисление». Типичный прикладной код вместо него использует высокоуровневый `parseEnum` из `std/strutils`.

**Важные ограничения:**
- `argSym` должен быть разрешённым символом (`typed`), содержащим строку для сопоставления.
- `default` — либо `nil` (возбудить исключение при отсутствии совпадения), либо `typed`-символ, называющий запасное значение перечисления.
- `userMin` / `userMax` позволяют ограничить диапазон ординалов полей, включаемых в генерируемый `case`.
- `normalizer` должна быть `static`-процедурой, поскольку вызывается во время компиляции для нормализации строк полей, запекаемых в ветки `of`.
- Неоднозначные перечисления — те, в которых два поля нормализуются к одной строке — отклоняются с ошибкой компиляции.

### Параметры

| Имя | Тип | Описание |
|---|---|---|
| `typ` | `typedesc` | Тип перечисления, для которого генерируется `case`-оператор |
| `argSym` | `typed` | Символ (переменная), содержащий входную строку |
| `default` | `typed` | `nil` — возбудить исключение при отсутствии совпадения; или символ запасного значения |
| `userMin` | `static[int]` | Минимальный ординал включаемых полей |
| `userMax` | `static[int]` | Максимальный ординал включаемых полей (включительно) |
| `normalizer` | `static[proc(s:string):string]` | Функция нормализации строк, применяемая как при компиляции, так и в рантайме |

### Возвращаемое значение

`NimNode` с `case`-оператором, который можно напрямую использовать как тело генерируемой процедуры.

### Пример

Ниже показан минимальный пользовательский `parseEnum`, построенный поверх `genEnumCaseStmt`:

```nim
import std/enumutils
import std/macros

type Direction = enum
  North = "north"
  East  = "east"
  South = "south"
  West  = "west"

func myNormalize(s: string): string =
  # простая нормализация к нижнему регистру
  var r = s
  for c in mitems(r): c = c.toLowerAscii
  r

macro parseDirection(input: typed): Direction =
  genEnumCaseStmt(Direction, input, nil,
                  Direction.low.ord, Direction.high.ord,
                  myNormalize)

let d = parseDirection("NORTH")
assert d == North

# Вызов с недопустимой строкой возбуждает ValueError в рантайме:
# let bad = parseDirection("diagonal")  # → ValueError: "Invalid enum value: diagonal"
```

**С запасным значением по умолчанию вместо исключения:**

```nim
import std/enumutils
import std/macros

type Color = enum
  Red, Green, Blue

func id(s: string): string = s   # нормализатор-тождество

macro tryParseColor(input: typed): Color =
  let fallback = bindSym"Red"    # по умолчанию — Red при отсутствии совпадения
  genEnumCaseStmt(Color, input, fallback,
                  Color.low.ord, Color.high.ord, id)

assert tryParseColor("Green") == Green
assert tryParseColor("Purple") == Red   # нет совпадения → значение по умолчанию
```

## Связи между экспортируемыми символами

```
symbolRank(a)        → позиция в списке объявлений (с нуля)
    │
    └─ используется в symbolName(a) → строка-идентификатор в исходном коде

items(E)             → безопасная итерация HoleyEnum
    │
    └─ внутренне использует enumFullRange (приватный макрос) для получения всех членов

genEnumCaseStmt(…)   → построитель case-оператора во время компиляции для разбора строка→перечисление
    │
    └─ основа для утилит в стиле parseEnum; внутренне опирается на концепции symbolRank
```

## Сводная таблица

| Символ | Вид | Работает с | Назначение |
|---|---|---|---|
| `items(E)` | итератор | `HoleyEnum` | Безопасный перебор всех объявленных членов |
| `symbolRank(a)` | шаблон | любое перечисление | Позиция значения в порядке объявления |
| `symbolName(a)` | func | любое перечисление | Имя идентификатора значения в исходном коде |
| `genEnumCaseStmt(…)` | макрос | любое перечисление | Генерация `case`-оператора строка→перечисление |
