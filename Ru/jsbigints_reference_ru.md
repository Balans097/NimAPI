# Справочник модуля `jsbigints`

> **Модуль:** `std/jsbigints` (стандартная библиотека Nim)  
> **Цель компиляции:** Только JavaScript — использование с нативным бэкендом является фатальной ошибкой компиляции.  
> **Назначение:** Целочисленная арифметика произвольной точности для Nim-программ, компилируемых в JavaScript, через нативный тип `BigInt`.

---

## Контекст: зачем этот модуль нужен

Стандартный тип JavaScript `number` — это 64-битное значение с плавающей точкой по IEEE 754. Это означает, что целые числа, превышающие 2⁵³ − 1 (`Number.MAX_SAFE_INTEGER`), не могут быть представлены точно — арифметика над ними молча теряет точность. Это реальная проблема в криптографии, хешировании, финансовых вычислениях и везде, где используются 64-битные идентификаторы или временны́е метки.

JavaScript решил это в 2020 году, введя `BigInt`: нативный тип целых чисел произвольной точности с собственным синтаксисом литералов (`42n`) и полным набором операторов. Nim-модуль `jsbigints` открывает эту возможность для Nim-кода, компилируемого с `--backend:js`, отображая идиоматичный Nim-синтаксис напрямую на операции JavaScript `BigInt`.

**Ключевые характеристики `JsBigInt`:**
- Нет фиксированной верхней или нижней границы — ограничение только по доступной памяти.
- Все операции точны (без округлений с плавающей точкой).
- Нельзя смешивать с JavaScript `number` в арифметике — необходимо явное преобразование.
- Недоступен во время компиляции Nim (`nimvm`) и в `static`-контекстах.
- Значение по умолчанию — `0`.

---

## Тип `JsBigInt`

```nim
type JsBigInt* = distinct JsBigIntImpl
```

`JsBigInt` — `distinct`-тип, оборачивающий JavaScript-примитив `bigint`. Слово `distinct` означает, что он не взаимозаменяем с Nim-типом `int` на уровне типов — нельзя случайно передать `JsBigInt` туда, где ожидается `int`, и наоборот. Это предотвращает молчаливую потерю точности при смешении.

Значение любой неинициализированной переменной `JsBigInt` равно `0`:

```nim
var x: JsBigInt
doAssert x == big"0"
doAssert x == JsBigInt.default
```

## Экспортируемые символы

---

### Конструкторы

Существует три способа создать значение `JsBigInt` — все они компилируются в `BigInt(...)` на JavaScript:

---

#### `big` (из целого числа)

```nim
func big*(integer: SomeInteger): JsBigInt
```

**Что делает**

Преобразует любой Nim-целочисленный тип (`int`, `int32`, `int64`, `uint` и др.) в `JsBigInt`. Аргумент передаётся напрямую в JavaScript-конструктор `BigInt()`. Это естественный путь, когда у вас уже есть Nim-целое во время выполнения.

**Пример**

```nim
import std/jsbigints

let a = big(1234567890)
let b = 999.big          # синтаксис вызова метода тоже работает

doAssert a == big"1234567890"
doAssert 0b1111100111.big == 0o1747.big and 0o1747.big == b
```

---

#### `` `'big` `` (суффикс литерала)

```nim
func `'big`*(num: cstring): JsBigInt
```

**Что делает**

Пользовательский суффикс числового литерала, позволяющий записывать `JsBigInt`-литералы прямо в коде с суффиксом `'big` — аналогично `'f32` для float. Суффикс поддерживает все числовые основания Nim: десятичную, двоичную (`0b`), восьмеричную (`0o`) и шестнадцатеричную (`0x`).

Это наиболее эргономичный способ задать большие константы, выходящие за диапазон нативных Nim-целых: число передаётся как строка в `BigInt()` и никогда не проходит через Nim-целочисленные типы.

**Пример**

```nim
import std/jsbigints

doAssert -1'big == 1'big - 2'big
doAssert 12'big == 12.big
doAssert 0b101'big == 0b101.big
doAssert 0o701'big == 0o701.big
doAssert 0xdeadbeaf'big == 0xdeadbeaf.big

# Числа, выходящие за 64-битный диапазон — представлены точно:
doAssert 0xffffffffffffffff'big == (1'big shl 64'big) - 1'big
```

Обратите внимание: `static(12'big)` не компилируется — `JsBigInt` нельзя использовать в контекстах времени компиляции.

---

#### `big` (из строки)

```nim
func big*(integer: cstring): JsBigInt
```

**Что делает**

Псевдоним для суффикса `'big`, но вызываемый с явным аргументом типа `cstring`. Удобен, когда нужно создать `JsBigInt` из строковой переменной или строкового выражения, вычисленного во время выполнения. Строка передаётся напрямую в JavaScript `BigInt()` — она должна быть корректным представлением целого числа (десятичным, шестнадцатеричным с префиксом `0x` и т. д.).

**Пример**

```nim
import std/jsbigints

let s: cstring = "99999999999999999999"
let x = big(s)
doAssert x == big"99999999999999999999"

doAssert big"-12" == -12'big
```

### Функции преобразования

---

#### `toCstring` (с основанием)

```nim
func toCstring*(this: JsBigInt; radix: 2..36): cstring
```

**Что делает**

Преобразует `JsBigInt` в строковое представление в заданной системе счисления. `radix` должен быть от 2 до 36 включительно — то же ограничение, что у JavaScript `BigInt.prototype.toString(radix)`, которую эта функция и оборачивает. Основание 2 даёт двоичную запись, 16 — шестнадцатеричную, 10 — десятичную; основания от 11 до 36 используют цифры `0–9` и буквы `a–z`. Знак сохраняется: отрицательные значения получают ведущий `-`.

**Пример**

```nim
import std/jsbigints

doAssert big"2147483647".toCstring(2) == "1111111111111111111111111111111".cstring
doAssert big"255".toCstring(16) == "ff".cstring
doAssert big"8".toCstring(2) == "1000".cstring
doAssert big"-10".toCstring(2) == "-1010".cstring
```

---

#### `toCstring` (десятичная)

```nim
func toCstring*(this: JsBigInt): cstring
```

**Что делает**

Краткая форма `toCstring(this, 10)` — преобразует `JsBigInt` в десятичную строку. Оборачивает JavaScript `BigInt.prototype.toString()` без аргумента.

**Пример**

```nim
import std/jsbigints

let x = big"123456789012345678901234567890"
doAssert x.toCstring() == "123456789012345678901234567890".cstring
```

---

#### `$`

```nim
func `$`*(this: JsBigInt): string
```

**Что делает**

Возвращает Nim-строку (`string`) с представлением `JsBigInt`. В выводе присутствует суффикс `n` — в соответствии с синтаксисом JavaScript-литералов `BigInt` (например, `42n`). Это сразу показывает при печати, что значение является `BigInt`, а не обычным числом. Внутри вызывает `toCstring` и добавляет `'n'`.

**Пример**

```nim
import std/jsbigints

doAssert $big"1024" == "1024n"
doAssert $big"-7" == "-7n"
doAssert $(2'big ** 64'big) == "18446744073709551616n"
```

---

#### `toNumber`

```nim
func toNumber*(this: JsBigInt): int
```

**Что делает**

Преобразует `JsBigInt` в JavaScript `Number` (Nim `int`). Оборачивает JavaScript-конструктор `Number()`. **Проверки границ нет** — если значение превышает диапазон безопасных целых JavaScript (±2⁵³ − 1), результат будет неточным. Это заведомо потенциально-теряющее преобразование; используйте его только когда уверены, что значение вписывается в диапазон.

**Пример**

```nim
import std/jsbigints

doAssert toNumber(big"2147483647") == 2147483647.int

# Опасно — значение превышает диапазон безопасных целых:
# toNumber(big"99999999999999999999")  # результат будет неточным
```

---

#### `wrapToInt`

```nim
func wrapToInt*(this: JsBigInt; bits: Natural): JsBigInt
```

**Что делает**

Приводит значение к знаковому диапазону `bits`-битного целого в дополнительном коде: от `−2^(bits−1)` до `2^(bits−1) − 1`. Биты выше заданной разрядности отбрасываются, результат интерпретируется как знаковый. Оборачивает JavaScript `BigInt.asIntN(bits, value)`.

Полезно для имитации арифметики фиксированной разрядности со знаком — например, поведения переполнения `int32` или модульной арифметики в знаковой области.

**Параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `this` | `JsBigInt` | Приводимое значение. |
| `bits` | `Natural` | Разрядность целевого знакового целого типа. |

**Пример**

```nim
import std/jsbigints

# 3 + 2^66 — огромное число, но при приведении к 13 битам получим 3
# (потому что 2^66 mod 2^13 == 0):
doAssert (big"3" + big"2" ** big"66").wrapToInt(13) == big"3"

# Имитация переполнения int8:
doAssert big"200".wrapToInt(8) == big"-56"  # 200 в int8 оборачивается в -56
```

---

#### `wrapToUint`

```nim
func wrapToUint*(this: JsBigInt; bits: Natural): JsBigInt
```

**Что делает**

Приводит значение к беззнаковому диапазону `bits`-битного целого: от `0` до `2^bits − 1`. Биты выше заданной разрядности отбрасываются. Оборачивает JavaScript `BigInt.asUintN(bits, value)`. Результат всегда неотрицателен. Удобно для имитации беззнаковой арифметики фиксированной разрядности или вычислений по модулю степени двойки.

**Параметры**

| Параметр | Тип | Описание |
|----------|-----|----------|
| `this` | `JsBigInt` | Приводимое значение. |
| `bits` | `Natural` | Разрядность целевого беззнакового целого типа. |

**Пример**

```nim
import std/jsbigints

# 3 + 2^66 при приведении к 66 битам равно 3 (2^66 mod 2^66 == 0):
doAssert (big"3" + big"2" ** big"66").wrapToUint(66) == big"3"

# Значение по модулю 2^8 (то есть как uint8):
doAssert big"300".wrapToUint(8) == big"44"  # 300 mod 256 == 44
```

### Арифметические операторы

Все бинарные арифметические операторы требуют, чтобы **оба операнда были `JsBigInt`** — смешивать `JsBigInt` с `int` напрямую нельзя. Сначала преобразуйте с помощью `big()` или `toNumber()`.

---

#### `+` — сложение

```nim
func `+`*(x, y: JsBigInt): JsBigInt
```

```nim
doAssert big"9" + big"1" == big"10"
doAssert big"99999999999999999999" + big"1" == big"100000000000000000000"
```

---

#### `-` — вычитание

```nim
func `-`*(x, y: JsBigInt): JsBigInt
```

```nim
doAssert big"9" - big"1" == big"8"
doAssert big"0" - big"1" == big"-1"
```

---

#### `*` — умножение

```nim
func `*`*(x, y: JsBigInt): JsBigInt
```

```nim
doAssert big"42" * big"9" == big"378"
doAssert big"1000000000" * big"1000000000" == big"1000000000000000000"
```

---

#### `div` — целочисленное деление (с усечением к нулю)

```nim
func `div`*(x, y: JsBigInt): JsBigInt
```

**Что делает**

Целочисленное деление с усечением результата к нулю. Соответствует семантике нативного `div` Nim (и оператора `BigInt /` JavaScript). Знак результата: если ровно один из операндов отрицателен — результат отрицателен.

```nim
doAssert big"13"  div big"3"  == big"4"
doAssert big"-13" div big"3"  == big"-4"   # к нулю, не к полу
doAssert big"13"  div big"-3" == big"-4"
doAssert big"-13" div big"-3" == big"4"
```

---

#### `mod` — остаток от деления

```nim
func `mod`*(x, y: JsBigInt): JsBigInt
```

**Что делает**

Возвращает остаток от деления со знаком **делимого** (левого операнда). Это поведение оператора `%` JavaScript для `BigInt`. Обратите внимание: это отличается от математического остатка по модулю, когда делимое отрицательно.

```nim
doAssert big"13"  mod big"3"  == big"1"
doAssert big"-13" mod big"3"  == big"-1"   # знак следует за делимым
doAssert big"13"  mod big"-3" == big"1"
doAssert big"-13" mod big"-3" == big"-1"
```

---

#### `**` — возведение в степень

```nim
func `**`*(x, y: JsBigInt): JsBigInt
```

**Что делает**

Возводит `x` в степень `y`. Показатель должен быть неотрицательным — передача отрицательного показателя вызывает JavaScript `RangeError` во время выполнения (не ошибку компиляции). Результат может быть астрономически большим: `BigInt` не имеет верхней границы.

```nim
doAssert big"2"  ** big"64" == big"18446744073709551616"
doAssert big"-2" ** big"3"  == big"-8"
doAssert -big"2" ** big"2"  == big"4"   # разбирается как (-2n) ** 2n
doAssert big"0"  ** big"0"  == big"1"   # граничный случай: 0^0 = 1

var ok = false
try: discard big"2" ** big"-1"  # RangeError: отрицательный показатель
except: ok = true
doAssert ok
```

---

#### `-` — унарное отрицание

```nim
func `-`*(this: JsBigInt): JsBigInt
```

Меняет знак значения. Положительное становится отрицательным, отрицательное — положительным, ноль остаётся нулём.

```nim
doAssert -(big"10101010101") == big"-10101010101"
doAssert -(big"-7") == big"7"
doAssert -(big"0") == big"0"
```

**Замечание об унарном `+`:** Унарный `+` намеренно отключён (помечен `{.error.}`), потому что спецификация JavaScript `BigInt` явно запрещает его из соображений совместимости с asm.js. Попытка использовать `+big"5"` — ошибка компиляции.

---

### Операторы сравнения

Все операторы сравнения возвращают `bool`.

---

#### `<`, `<=`, `==`

```nim
func `<`*(x, y: JsBigInt): bool
func `<=`*(x, y: JsBigInt): bool
func `==`*(x, y: JsBigInt): bool
```

Стандартные операторы упорядочения и равенства. Операторы `>`, `>=` и `!=` Nim выводит из них автоматически.

```nim
doAssert big"2"  < big"9"
doAssert big"1"  <= big"5"
doAssert big"42" == big"42"

let a = big"2147483647"
let b = big"666"
doAssert a != b
doAssert a > b
doAssert a >= b
doAssert b < a
doAssert b <= a
```

### Побитовые операторы

Побитовые операторы работают с представлением значения в дополнительном коде. Для отрицательных чисел используется бесконечно-прецизионный дополнительный код (все старшие биты концептуально равны `1`). Результаты согласуются с тем, что дают побитовые операторы JavaScript `BigInt`.

---

#### `and` — побитовое И

```nim
func `and`*(x, y: JsBigInt): JsBigInt
```

```nim
doAssert (big"555" and big"2") == big"2"
# 555 = 0b1000101011, 2 = 0b10 → AND = 0b10 = 2
```

---

#### `or` — побитовое ИЛИ

```nim
func `or`*(x, y: JsBigInt): JsBigInt
```

```nim
doAssert (big"555" or big"2") == big"555"
# в 555 уже установлен бит 1, поэтому ИЛИ с 2 даёт всё те же 555
```

---

#### `xor` — побитовое исключающее ИЛИ

```nim
func `xor`*(x, y: JsBigInt): JsBigInt
```

```nim
doAssert (big"555" xor big"2") == big"553"
# переключает бит 1 в числе 555
```

---

#### `shl` — сдвиг влево

```nim
func `shl`*(a, b: JsBigInt): JsBigInt
```

Сдвигает `a` влево на `b` позиций — эквивалентно умножению на `2^b`. В отличие от целых фиксированной ширины, переполнения нет — результат просто растёт.

```nim
doAssert big"999" shl big"2" == big"3996"         # 999 * 4
doAssert big"1"   shl big"64" == big"18446744073709551616"
```

---

#### `shr` — сдвиг вправо (арифметический)

```nim
func `shr`*(a, b: JsBigInt): JsBigInt
```

Сдвигает `a` вправо на `b` позиций — эквивалентно целочисленному делению на `2^b` (деление с округлением вниз для положительных чисел). Для отрицательных `BigInt` — арифметический сдвиг (знакорасширяющий).

```nim
doAssert big"999" shr big"2" == big"249"   # 999 / 4, усечение
doAssert big"8"   shr big"1" == big"4"
```

---

### Операторы изменения на месте

Все операторы изменения на месте модифицируют переменную напрямую, как их аналоги для `int`.

---

#### `inc` (на 1)

```nim
func inc*(this: var JsBigInt)
```

Увеличивает значение на 1.

```nim
var x = big"1"
inc x
doAssert x == big"2"
```

---

#### `inc` (на заданное значение)

```nim
func inc*(this: var JsBigInt; amount: JsBigInt)
```

Увеличивает значение на `amount`.

```nim
var x = big"1"
inc x, big"2"
doAssert x == big"3"
```

---

#### `dec` (на 1)

```nim
func dec*(this: var JsBigInt)
```

Уменьшает значение на 1.

```nim
var x = big"2"
dec x
doAssert x == big"1"
```

---

#### `dec` (на заданное значение)

```nim
func dec*(this: var JsBigInt; amount: JsBigInt)
```

Уменьшает значение на `amount`.

```nim
var x = big"1"
dec x, big"2"
doAssert x == big"-1"
```

---

#### `+=`, `-=`, `*=`, `/=`

```nim
func `+=`*(x: var JsBigInt; y: JsBigInt)
func `-=`*(x: var JsBigInt; y: JsBigInt)
func `*=`*(x: var JsBigInt; y: JsBigInt)
func `/=`*(x: var JsBigInt; y: JsBigInt)
```

Операторы составного присваивания. `/=` выполняет целочисленное деление с усечением (аналогично `div`).

```nim
var a = big"1";  a += big"2";  doAssert a == big"3"
var b = big"1";  b -= big"2";  doAssert b == big"-1"
var c = big"2";  c *= big"4";  doAssert c == big"8"
var d = big"11"; d /= big"2";  doAssert d == big"5"   # усечение
```

---

### Намеренно отключённые операции

Три операции навсегда отключены с помощью прагм `{.error.}` — их использование является **ошибкой компиляции** по замыслу:

| Операция | Причина |
|----------|---------|
| Унарный `+big"x"` | Спецификация JavaScript `BigInt` явно запрещает унарный `+` из соображений совместимости с asm.js. |
| `low(JsBigInt)` | Целые произвольной точности не имеют минимального значения. |
| `high(JsBigInt)` | Целые произвольной точности не имеют максимального значения. |

## Типичные паттерны и подводные камни

**Смешивание `JsBigInt` с `int` — ошибка компиляции:**

```nim
# let x = big"5" + 3    # Error: type mismatch
let x = big"5" + big(3) # правильно — сначала преобразовать
```

**`JsBigInt` нельзя использовать во время компиляции:**

```nim
# static: discard big"5"  # Error: JsBigInt cannot be used in static context
```

**Проверка делимости:**

```nim
let n = big"1000000000000000000"
if n mod big"2" == big"0":
  echo "чётное"
```

**Вычисление больших факториалов:**

```nim
func factorial(n: JsBigInt): JsBigInt =
  result = big"1"
  var i = big"2"
  while i <= n:
    result *= i
    inc i

echo $factorial(big"50")  # точный результат, без переполнения
```
