# Справочник модуля `wordwrap`

> Алгоритм переноса Unicode-строк по границам слов.  
> Часть стандартной библиотеки Nim (`std/wordwrap`).

---

## Что такое перенос по словам

Перенос по словам — это разбиение длинной строки на несколько строк так, чтобы ни одна из них не превышала заданной ширины. Главная сложность — делать это *аккуратно*: разрывать по пробелам или другим разделителям везде, где это возможно, и только в крайнем случае (или по явному запросу) разрывать слово посередине.

Модуль корректно работает с **Unicode-текстом** — ширина строк измеряется в *графемных кластерах* (видимых символах), а не в байтах. Буква с акцентом вроде `é` или эмодзи вроде `🌍` считаются за **одну** единицу ширины независимо от того, сколько байт они занимают.

---

## Экспортируемые символы

### `wrapWords` — перенести Unicode-строку под заданную ширину

```nim
proc wrapWords*(
  s: string,
  maxLineWidth = 80,
  splitLongWords = true,
  seps: set[char] = Whitespace,
  newLine = "\n"
): string {.noSideEffect.}
```

**Что делает:** Просматривает `s` слева направо, отслеживает, сколько графемных столбцов занято на текущей строке, и вставляет разрывы `newLine` в нужных местах, удерживая каждую строку в пределах `maxLineWidth` столбцов. Возвращает новую строку; исходная не изменяется.

---

## Параметры — подробно

### `s: string`

Входной текст для переноса. Может содержать любой корректный UTF-8, включая многобайтовые символы, комбинирующие знаки и эмодзи. Подсчёт ширины ведётся в графемных кластерах через `std/unicode.graphemeLen`, так что решения принимаются по *визуальной* ширине, а не по числу байт.

---

### `maxLineWidth: int = 80`

Максимально допустимое число графемных столбцов в строке. Значение по умолчанию 80 соответствует традиционной ширине терминала.

Разделитель, попавший точно на границу, сам по себе не вызывает разрыва — разрыв происходит только тогда, когда следующее *слово* (непрерывный токен без разделителей) вышло бы за предел.

```nim
import std/wordwrap

# Ширина по умолчанию 80 — короткое предложение умещается в одну строку
echo "Привет, мир!".wrapWords()
# → "Привет, мир!"

# Ширина 10 — нужен разрыв
echo "Привет, мир!".wrapWords(10)
# → "Привет,\nмир!"

# Ширина 20 — длинная строка цифр разбивается
echo "123456789012345678901234567890".wrapWords(20)
# → "12345678901234567890\n1234567890"
```

---

### `splitLongWords: bool = true`

Определяет, что происходит, когда одно слово (непрерывный токен без разделителей) само по себе *длиннее* `maxLineWidth`.

- **`true` (по умолчанию):** Слово разрывается посередине графемного кластера: `newLine` вставляется каждый раз, когда счётчик столбцов достигает нуля. Слово никогда не «свисает» за край строки.
- **`false`:** Слово-гигант помещается на отдельную строку и ему позволяют превысить `maxLineWidth`. Разрывов посередине слова нет ни при каких условиях.

```nim
import std/wordwrap

let longWord = "superlongwordthatexceedsthewidth"

# splitLongWords = true (по умолчанию): разрывается безжалостно посередине
echo longWord.wrapWords(10)
# → "superlongw\nordthatexc\needsthewid\nth"

# splitLongWords = false: помещается на отдельную строку целиком
echo longWord.wrapWords(10, splitLongWords = false)
# → "superlongwordthatexceedsthewidth"
# (разрыва нет вообще — слово просто переполняет строку)

# Реальный случай: обычное предложение, перенос по словам
echo "Hello Bob. Hello John.".wrapWords(13, false)
# → "Hello Bob.\nHello John."
```

---

### `seps: set[char] = Whitespace`

Набор символов, которые считаются *разделителями* — границами, где разрешено вставлять переносы строк. По умолчанию это `std/strutils.Whitespace`: пробел, табуляция, перевод строки (`\n`), возврат каретки (`\r`), перевод страницы и вертикальная табуляция.

**Как обрабатываются разделители:**

- Символы перевода строки и возврата каретки внутри последовательности разделителей *отбрасываются* — они не попадают в результат. Остальные символы-разделители (пробелы, табуляции) сохраняются и воспроизводятся перед следующим словом, если только вместо них не вставляется разрыв строки.
- Если последовательность разделителей состоит исключительно из `\n`/`\r` (ни пробелов, ни табуляций), синтезируется один пробел, который вычитается из бюджета строки.

Можно передать собственный набор, чтобы считать другие символы (`;`, `/`, `-`) допустимыми точками разрыва:

```nim
import std/wordwrap

# Точка с запятой — единственный разделитель
echo "Hello Bob. Hello John.".wrapWords(13, true, {';'})
# → "Hello Bob. He\nllo John."
# (пробел внутри "Hello Bob." НЕ является разделителем,
#  поэтому алгоритм вынужден разорвать слово посередине)

# Пробел и дефис оба служат точками разрыва
echo "long-hyphenated-compound-word".wrapWords(10, false, {' ', '-'})
# → "long-\nhyphenated-\ncompound-\nword"
```

---

### `newLine: string = "\n"`

Строка, вставляемая в каждой точке переноса. По умолчанию — Unix-перевод строки `"\n"`. Замените на `"\r\n"` для окончаний строк в стиле Windows или на любую другую строку (например, `"<br>"` для HTML-вывода).

```nim
import std/wordwrap

echo "Hello world from Nim".wrapWords(11, newLine = "\r\n")
# → "Hello world\r\nfrom Nim"

echo "Hello world from Nim".wrapWords(11, newLine = "<br>")
# → "Hello world<br>from Nim"
```

## Разбор алгоритма

Понимание алгоритма помогает предсказать поведение в граничных случаях:

1. Строка просматривается слева направо, чередуя *последовательности разделителей* и *последовательности символов слова*.
2. Для каждой **последовательности разделителей**: символы `\n` и `\r` отбрасываются; остальные сохраняются в `lastSep`. Если вся последовательность состоит только из `\n`/`\r` — синтезируется один пробел. Сохранённые разделители вычитаются из `spaceLeft`.
3. Для каждой **последовательности слова** шириной `wlen` в графемных столбцах:
   - Если `wlen ≤ spaceLeft`: добавляются `lastSep`, затем слово; `wlen` вычитается из `spaceLeft`.
   - Если `wlen > spaceLeft` и (`splitLongWords = false` или `wlen ≤ maxLineWidth`): вставляется `newLine`, слово помещается на новую строку, `spaceLeft = maxLineWidth - wlen`.
   - Если `wlen > spaceLeft` и `splitLongWords = true` и `wlen > maxLineWidth`: слово разбивается по графемным кластерам, `newLine` вставляется каждый раз, когда `spaceLeft` обнуляется.

**Важное следствие:** Разделитель, стоящий непосредственно перед словом, вызвавшим перенос строки, *не выводится* — вместо него вставляется только `newLine`. Это означает, что в конце строк не будет висящих пробелов.

## Корректность Unicode

Ширина измеряется функцией `graphemeLen` из `std/unicode`, которая соблюдает границы графемных кластеров Unicode. Это означает:

- Комбинирующий акцент (например, `e` + комбинирующая острая черта = `é`) считается за **1** столбец.
- Последовательность с объединителем нулевой ширины (например, эмодзи семьи 👨‍👩‍👧) считается за **1** столбец.
- Иероглиф CJK считается за **1** столбец (обратите внимание: модуль *не* реализует учёт двойной ширины восточноазиатских символов — если это нужно, используйте специализированную библиотеку).

```nim
import std/wordwrap

# Кириллица — каждая буква один графемный кластер = один столбец
echo "Привет мир".wrapWords(7)
# → "Привет\nмир"
```

## Практические примеры

```nim
import std/wordwrap

# ── Вывод в терминал ─────────────────────────────────────────────────────────
let longText = "The quick brown fox jumps over the lazy dog near the riverbank."
echo longText.wrapWords(30)
# → "The quick brown fox jumps over\nthe lazy dog near the riverbank."

# ── HTML-абзац ───────────────────────────────────────────────────────────────
let html = longText.wrapWords(40, newLine = "<br>\n")
echo "<p>" & html & "</p>"

# ── Окончания строк в стиле Windows ─────────────────────────────────────────
let winText = longText.wrapWords(40, newLine = "\r\n")

# ── Поведение с уже имеющимися переносами ────────────────────────────────────
# Если во входной строке уже есть переводы строк, они трактуются как
# разделители и нормализуются (сворачиваются в один пробел).
# Исходная разбивка НЕ сохраняется.
echo "Строка первая\nСтрока вторая".wrapWords(40)
# → "Строка первая Строка вторая"   (\n превратился в пробел)

# ── Идентификатор пути с косыми чертами ─────────────────────────────────────
echo "some/very/long/path/that/overflows".wrapWords(12, false, {'/'})
# → "some/\nvery/\nlong/\npath/\nthat/\noverflows"
```

## Быстрая шпаргалка

| Параметр | По умолчанию | Действие |
|----------|-------------|---------|
| `maxLineWidth` | `80` | Максимальное число графемных столбцов в строке |
| `splitLongWords` | `true` | Разрывать слова длиннее `maxLineWidth` посередине графемы |
| `seps` | `Whitespace` | Символы, служащие допустимыми точками разрыва |
| `newLine` | `"\n"` | Строка, вставляемая при каждом переносе |

| Поведение | Примечания |
|-----------|-----------|
| Единица ширины | Графемный кластер (не байты, не кодовые точки) |
| Разделитель в конце строки | Отбрасывается — висящих пробелов нет |
| Переводы строк во входных данных | Трактуются как разделители, нормализуются в пробел |
| Переполнение при `splitLongWords=false` | Слово помещается на отдельную строку, может превысить `maxLineWidth` |
| Побочные эффекты | Отсутствуют (`{.noSideEffect.}`) |
