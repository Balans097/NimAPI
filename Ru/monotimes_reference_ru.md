# std/monotimes — полный справочник модуля

> **Входит в стандартную библиотеку Nim** (`std/monotimes`).

## Обзор

Модуль `monotimes` предоставляет один тип — `MonoTime` — и небольшой набор операций, позволяющих **измерять прошедшее время с высокой точностью и надёжностью**.

Ключевая гарантия модуля — *монотонность*: временна́я метка, полученная позже, всегда больше или равна той, что была получена раньше. Это звучит очевидно, но стандартный тип `times.Time` из Nim **не** даёт такой гарантии — показания системных часов могут скакнуть назад из-за корректировок NTP, перехода на летнее время или ручного изменения времени. `MonoTime` защищён от всего этого, потому что ведёт отсчёт от произвольной внутрисистемной точки отсчёта, а не от календарной эпохи.

**Компромисс:** именно по той же причине `MonoTime` нельзя преобразовать в читаемую дату или время. Он имеет смысл только при сравнении с другим `MonoTime`. Если нужна отображаемая метка времени — используйте `times.Time`; если нужно надёжно измерить продолжительность — используйте `MonoTime`.

Внутри хранилище — 64-битное целое со знаком в **наносекундном разрешении**. Фактическая аппаратная точность зависит от платформы (см. примечания к [`getMonoTime`](#getmonotime)).

---

## Тип

### `MonoTime`

```nim
type MonoTime* = object
  ticks: int64   # наносекунды с момента произвольного начала отсчёта
```

Непрозрачный тип-значение, представляющий точку монотонного времени. Внутреннее поле `ticks` закрыто; для его чтения используйте `ticks(t)`, а для работы с самими метками — арифметические операторы. Два значения `MonoTime` имеют смысл только относительно друг друга: их можно вычитать (получая `Duration`), сравнивать с помощью `<` / `<=` / `==` и смещать на `Duration` с помощью `+` / `-`.

## Процедуры

---

### `getMonoTime`

```nim
proc getMonoTime*(): MonoTime {.tags: [TimeEffect].}
```

**Возвращает текущую монотонную временну́ю метку.**

Это главная точка входа в модуль — процедура, которую вы вызываете, чтобы зафиксировать «сейчас» на монотонных часах. Каждый вызов возвращает значение, гарантированно большее или равное любому предыдущему на той же системе.

**Платформенные детали:**

| Платформа | Используемый механизм |
|---|---|
| Linux / BSD (POSIX) | `clock_gettime(CLOCK_MONOTONIC)` |
| macOS | `mach_absolute_time()` + пересчёт timebase |
| Windows | `QueryPerformanceCounter` / `QueryPerformanceFrequency` |
| Zephyr RTOS | `k_uptime_ticks()` + `k_ticks_to_ns_floor64()` |
| JS (Node.js) | `process.hrtime()` |
| JS (Браузер) | `window.performance.now()` |

Независимо от платформы возвращаемое значение всегда использует наносекунды как единицу измерения. *Точность* (наименьший измеримый шаг) зависит от платформы, но *единица измерения* всегда — наносекунды.

**Пример — базовое измерение прошедшего времени:**

```nim
import std/monotimes, std/times

let start = getMonoTime()
# ... выполнить какую-то работу ...
let finish = getMonoTime()

let elapsed: Duration = finish - start
echo "Прошло: ", elapsed
```

**Пример — гарантия монотонности:**

```nim
let a = getMonoTime()
let b = getMonoTime()
assert a <= b  # всегда истинно, без исключений
```

### `ticks`

```nim
proc ticks*(t: MonoTime): int64
```

**Возвращает «сырое» количество наносекундных тиков, хранящихся внутри `MonoTime`.**

Результат — обычный `int64`, который можно сохранить, записать в лог, сериализовать или напрямую использовать в арифметике. Абсолютного смысла это значение не имеет — оно становится полезным только при сравнении с `ticks` другого `MonoTime`, полученного на той же системе. Разность двух значений `ticks` — это продолжительность в наносекундах.

**Пример:**

```nim
let t = getMonoTime()
let ns: int64 = t.ticks
echo "Тики (нс): ", ns

# Ручное вычисление длительности в наносекундах без использования Duration:
let before = getMonoTime()
# ... работа ...
let after = getMonoTime()
let diffNs = after.ticks - before.ticks
echo "Прошло наносекунд: ", diffNs
```

---

### `$`

```nim
proc `$`*(t: MonoTime): string
```

**Преобразует `MonoTime` в его строковое представление.**

Результат — просто десятичное число внутреннего счётчика тиков (наносекунды). Полезно для быстрой отладки или логирования. Это не читаемая метка времени — для этого нужен `times.Time`.

**Пример:**

```nim
let t = getMonoTime()
echo t    # выведет что-то вроде: 123456789012345
echo $t   # идентично, явное преобразование
```

### `-` (MonoTime − MonoTime)

```nim
proc `-`*(a, b: MonoTime): Duration
```

**Вычисляет время, прошедшее между двумя метками `MonoTime`, возвращая значение типа `Duration`.**

Это центральная операция, которую вы будете использовать после снятия двух временны́х меток. Результат — значение `times.Duration` (из `std/times`), которое затем можно выразить в любых нужных единицах: миллисекунды, микросекунды, наносекунды и т.д.

Обратите внимание: если `b` позже `a`, результат будет отрицательным `Duration`. В большинстве сценариев измерения нужно вычитать более раннюю метку из более поздней.

**Пример:**

```nim
import std/monotimes, std/times

let t1 = getMonoTime()
for i in 1..1_000_000:
  discard i * i  # имитация работы
let t2 = getMonoTime()

let elapsed: Duration = t2 - t1
echo "Миллисекунды: ", elapsed.inMilliseconds
echo "Микросекунды: ", elapsed.inMicroseconds
echo "Наносекунды:  ", elapsed.inNanoseconds
```

### `+` (MonoTime + Duration)

```nim
proc `+`*(a: MonoTime, b: Duration): MonoTime
```

**Возвращает новый `MonoTime`, сдвинутый вперёд на заданную `Duration`.**

Позволяет вычислить будущую (или, при отрицательной `Duration`, прошлую) метку времени относительно существующей. Полезно для планирования, тайм-аутов и работы с дедлайнами — можно заранее вычислить `deadline` и затем сравнивать с `getMonoTime()`, чтобы понять, не истёк ли срок.

**Пример:**

```nim
import std/monotimes, std/times

let now = getMonoTime()
let deadline = now + initDuration(milliseconds = 500)

# ... выполнить работу ...

if getMonoTime() > deadline:
  echo "Дедлайн превышен!"
else:
  echo "Укладываемся в срок."
```

---

### `-` (MonoTime − Duration)

```nim
proc `-`*(a: MonoTime, b: Duration): MonoTime
```

**Возвращает новый `MonoTime`, сдвинутый назад на заданную `Duration`.**

Это обратная операция по отношению к `+`. Даёт метку времени, которая на `b` раньше `a`. Полезно, когда нужно выразить точку в прошлом относительно известного ориентира — например, проверить, произошло ли событие в пределах последних N секунд.

**Пример:**

```nim
import std/monotimes, std/times

let now = getMonoTime()
let oneSecondAgo = now - initDuration(seconds = 1)

# Проверка: было ли зафиксированное событие "недавним" (в пределах 1 секунды):
let recorded: MonoTime = getSomeTimestamp()
if recorded >= oneSecondAgo:
  echo "Событие произошло в течение последней секунды."
```

### `<`

```nim
proc `<`*(a, b: MonoTime): bool
```

**Возвращает `true`, если метка `a` строго предшествует метке `b`.**

Сравнивает два значения `MonoTime` по их тикам. Поскольку часы монотонны, «меньше» напрямую означает «произошло раньше».

**Пример:**

```nim
let t1 = getMonoTime()
let t2 = getMonoTime()
assert t1 < t2 or t1 == t2  # t1 не может быть больше t2
```

---

### `<=`

```nim
proc `<=`*(a, b: MonoTime): bool
```

**Возвращает `true`, если `a` предшествует `b` или они представляют один и тот же момент.**

Это нестрогое сравнение. На некоторых платформах последовательные вызовы `getMonoTime()` могут возвращать одинаковые значения (если разрешение часов грубее, чем время между вызовами), поэтому `<=` в утверждениях об упорядоченности надёжнее, чем `<`.

**Пример:**

```nim
let a = getMonoTime()
let b = getMonoTime()
assert a <= b  # задокументированная гарантия самого модуля
```

---

### `==`

```nim
proc `==`*(a, b: MonoTime): bool
```

**Возвращает `true`, если два значения `MonoTime` имеют одинаковый счётчик тиков.**

На практике два независимо полученных значения `MonoTime` редко оказываются `==` (это означало бы, что часы вообще не продвинулись между вызовами), но это возможно на системах с низким разрешением монотонных часов. Равенство также значимо при сравнении сохранённой метки с самой собой или при построении структур данных, индексируемых по `MonoTime`.

**Пример:**

```nim
let t = getMonoTime()
let copy = t
assert t == copy   # одно и то же значение, очевидно равны

let later = getMonoTime()
echo t == later    # скорее всего false, но возможно true на часах с низким разрешением
```

### `high`

```nim
proc high*(typ: typedesc[MonoTime]): MonoTime
```

**Возвращает наибольшее представимое значение `MonoTime`.**

Соответствует `high(int64)` наносекундам — астрономически большой временно́й метке, которая на практике никогда не будет достигнута. Полезно в качестве сентинел-значения «плюс бесконечность» в алгоритмах, которым нужен начальный максимум — например, при поиске самой ранней метки среди набора кандидатов.

**Пример:**

```nim
var earliest = MonoTime.high  # начинаем с «плюс бесконечности»
for ts in collectedTimestamps:
  if ts < earliest:
    earliest = ts
echo "Тики самой ранней метки: ", earliest.ticks
```

---

### `low`

```nim
proc low*(typ: typedesc[MonoTime]): MonoTime
```

**Возвращает наименьшее представимое значение `MonoTime`.**

Соответствует `low(int64)` наносекундам — отрицательному значению, глубоко в прошлом, которого ни одна реальная метка никогда не достигнет. Полезно в качестве сентинел-значения «минус бесконечность» в алгоритмах, которым нужен начальный минимум.

**Пример:**

```nim
var latest = MonoTime.low   # начинаем с «минус бесконечности»
for ts in collectedTimestamps:
  if ts > latest:
    latest = ts
echo "Тики самой поздней метки: ", latest.ticks
```

## Сводная таблица

| Символ | Сигнатура | Назначение |
|--------|-----------|------------|
| `getMonoTime()` | `() → MonoTime` | Снять текущую метку монотонных часов |
| `ticks(t)` | `MonoTime → int64` | Прочитать «сырой» счётчик наносекунд |
| `$t` | `MonoTime → string` | Преобразовать счётчик тиков в строку |
| `a - b` | `(MonoTime, MonoTime) → Duration` | Вычислить прошедшее время между двумя метками |
| `a + d` | `(MonoTime, Duration) → MonoTime` | Сдвинуть метку вперёд на длительность |
| `a - d` | `(MonoTime, Duration) → MonoTime` | Сдвинуть метку назад на длительность |
| `a < b` | `(MonoTime, MonoTime) → bool` | Строго раньше? |
| `a <= b` | `(MonoTime, MonoTime) → bool` | Раньше или одновременно? |
| `a == b` | `(MonoTime, MonoTime) → bool` | Одновременно? |
| `MonoTime.high` | `typedesc → MonoTime` | Максимальное представимое значение (сентинел) |
| `MonoTime.low` | `typedesc → MonoTime` | Минимальное представимое значение (сентинел) |

## Когда использовать `MonoTime`, а когда `times.Time`

| Задача | Используйте |
|--------|------------|
| Измерить, сколько выполнялся код | `MonoTime` |
| Реализовать тайм-ауты и дедлайны | `MonoTime` |
| Надёжно упорядочить события без риска скачка часов | `MonoTime` |
| Показать пользователю дату и время | `times.Time` |
| Зафиксировать, когда именно произошло событие | `times.Time` |
| Сериализовать/десериализовать метку переносимо | `times.Time` |
