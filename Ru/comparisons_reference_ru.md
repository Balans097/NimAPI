# Справочник модуля comparisons

> **Модуль:** `comparisons`  
> **Часть:** Стандартная библиотека Nim  
> **Назначение:** Определяет все операторы сравнения и связанные утилиты для встроенных типов Nim. Это единый источник истины для `==`, `<`, `<=`, `!=`, `>`, `>=` применительно к целым числам, вещественным числам, строкам, символам, булевым значениям, перечислениям, множествам, указателям, ссылкам, процедурам, массивам, последовательностям и открытым массивам — а также для `min`, `max` и `clamp`.

---

## Основные концепции

### Семейства операторов

Модуль предоставляет шесть операторов сравнения. Четыре из них *примитивны* (реализованы магией компилятора или напрямую), а два — *производные* шаблоны:

| Оператор | Производный от | Примечания |
|---|---|---|
| `==` | примитив | одна перегрузка на тип |
| `<` | примитив | одна перегрузка на тип |
| `<=` | примитив | одна перегрузка на тип |
| `!=` | `not (x == y)` | универсальный шаблон |
| `>` | `y < x` | универсальный шаблон |
| `>=` | `y <= x` | универсальный шаблон |

Поскольку `!=`, `>` и `>=` — шаблоны, делегирующие к `==`, `<` и `<=`, **определив `==` и `<` для пользовательского типа, вы автоматически получаете все шесть операторов бесплатно**.

### Операторы беззнакового переинтерпретирования (`<%`, `<=%`, `>%`, `>=%`)

Отдельное семейство операторов трактует *знаковые* целые числа как *беззнаковые*. Они несут суффикс `%`, чтобы сделать переинтерпретирование явным в месте вызова. Это критически важно для низкоуровневых битовых манипуляций, эмуляции арифметики указателей и разбора протоколов, где знаковый бит является данными.

## Равенство: `==`

### Целые числа со знаком и без знака

```nim
proc `==`*(x, y: int): bool
proc `==`*(x, y: int8): bool
proc `==`*(x, y: int16): bool
proc `==`*(x, y: int32): bool
proc `==`*(x, y: int64): bool

proc `==`*(x, y: uint): bool
proc `==`*(x, y: uint8): bool
proc `==`*(x, y: uint16): bool
proc `==`*(x, y: uint32): bool
proc `==`*(x, y: uint64): bool
```

**Что это такое:** Побитовое равенство значений для всех разрядностей целых чисел — знаковых и беззнаковых. Каждая перегрузка обрабатывается магией `"EqI"` компилятора, которая напрямую отображается на инструкцию сравнения целевого CPU без каких-либо накладных расходов.

```nim
assert 42 == 42
assert 0'u8 == 0'u8
assert not (1'i32 == 2'i32)
```

---

### Вещественные числа

```nim
proc `==`*(x, y: float32): bool
proc `==`*(x, y: float): bool   # float64
```

**Что это такое:** Равенство по стандарту IEEE 754. Важная оговорка стандарта: `NaN != NaN` — значение `NaN` **не равно самому себе**. Для проверки на `NaN` используйте `math.isNaN`.

```nim
assert 1.0 == 1.0
assert not (0.1 + 0.2 == 0.3)   # классическая неточность вещественной арифметики
let nan = 0.0 / 0.0
assert nan != nan                # NaN никогда не равен самому себе
```

---

### Строки

```nim
proc `==`*(x, y: string): bool
```

**Что это такое:** Лексикографическое побайтовое равенство двух строк. Строки равны тогда и только тогда, когда они имеют одинаковую длину и каждый соответствующий байт идентичен. Сравнение чувствительно к регистру.

```nim
assert "hello" == "hello"
assert not ("Hello" == "hello")   # разный регистр → не равно
assert "" == ""
```

---

### Символы

```nim
proc `==`*(x, y: char): bool
```

**Что это такое:** Сравнивает числовые (ASCII / порядковые значения Unicode) коды двух символов. Эквивалентно `ord(x) == ord(y)`.

```nim
assert 'a' == 'a'
assert not ('A' == 'a')   # 65 ≠ 97
```

---

### Булевы значения

```nim
proc `==`*(x, y: bool): bool
```

**Что это такое:** Стандартное булево равенство. `true == true` и `false == false`; смешанные значения не равны.

---

### Перечисления

```nim
proc `==`*[Enum: enum](x, y: Enum): bool
```

**Что это такое:** Сравнивает два значения *одного и того же* типа перечисления по их базовому целочисленному значению. Что принципиально важно, оператор **типобезопасен**: сравнение значений двух разных типов перечисления является ошибкой времени компиляции, даже если их порядковые значения случайно совпадают.

```nim
type
  Color = enum red = 0, green = 1
  Status = enum ok = 0, fail = 1

assert red == red
assert not (red == green)
# assert red == ok  # ← ошибка компиляции: нельзя сравнивать разные типы перечислений
```

Пример из исходного кода: два значения *разных* перечислений с одинаковым порядковым значением можно сравнить лишь через явное преобразование:

```nim
type
  Enum1 = enum field1 = 3, field2
  Enum2 = enum place1, place2 = 3

var e1 = field1
var e2 = place2.ord.Enum1   # явное переинтерпретирование
assert e1 == e2             # одинаковое базовое значение (3)
assert not compiles(e1 == place2)  # прямое сравнение разных типов запрещено
```

---

### Множества

```nim
proc `==`*[T](x, y: set[T]): bool
```

**Что это такое:** Два множества равны, если содержат ровно одни и те же элементы. Дублирование элементов в литерале множества несущественно — по определению множество хранит каждое значение не более одного раза.

```nim
assert {1, 2, 3} == {3, 1, 2}     # порядок не важен
assert {1, 2, 2, 3} == {1, 2, 3}  # дубликаты игнорируются
assert not ({1, 2} == {1, 2, 3})
```

---

### Ссылки (`ref T`)

```nim
proc `==`*[T](x, y: ref T): bool
```

**Что это такое:** Проверяет, указывают ли две переменные типа `ref` на **один и тот же объект в куче** — то есть сравнение идентичности, а не структурное равенство. Два разных объекта с одинаковым содержимым **не** равны по этому оператору.

```nim
type Node = ref object
  val: int

let a = Node(val: 5)
let b = Node(val: 5)
let c = a

assert a == c        # один и тот же объект
assert not (a == b)  # разные объекты, одинаковое содержимое
```

> Если вам нужно структурное (глубокое) равенство для `ref`-типов, необходимо определить собственную перегрузку `==`.

---

### Сырые указатели (`ptr T`, `pointer`)

```nim
proc `==`*[T](x, y: ptr T): bool
proc `==`*(x, y: pointer): bool
```

**Что это такое:** Числовое равенство адресов — два указателя равны тогда и только тогда, когда содержат один и тот же адрес памяти. `nil` и указатель, приведённый из `0`, считаются равными.

```nim
var x = 42
let p1 = addr x
let p2 = addr x
assert p1 == p2   # одинаковый адрес

let null1 = cast[pointer](0)
let null2 = cast[pointer](nil)
assert null1 == null2
```

---

### Процедуры и итераторы

```nim
proc `==`*[T: proc | iterator](x, y: T): bool
```

**Что это такое:** Сравнивает две *переменные* типа процедуры (замыкания или указатели на функции) на идентичность — они равны, если ссылаются на одну и ту же базовую процедуру. Захваченное состояние не сравнивается; два замыкания, созданные из одного `proc`, в общем случае не равны.

---

### Массивы

```ником
proc `==`*[I, T](x, y: array[I, T]): bool
```

**Что это такое:** Поэлементное равенство для массивов фиксированного размера. Возвращает `true` только если каждая соответствующая пара элементов сравнивается как равная с помощью оператора `==` типа элемента. Оба массива должны иметь одинаковый тип индекса `I` и тип элемента `T`.

```nim
assert [1, 2, 3] == [1, 2, 3]
assert not ([1, 2, 3] == [1, 2, 4])
```

---

### Открытые массивы

```nim
proc `==`*[T](x, y: openArray[T]): bool
```

**Что это такое:** Поэлементное равенство для любого непрерывного представления последовательности — одинаково работает с массивами, последовательностями и срезами. Сначала проверяет длины, затем сравнивает поэлементно.

```nim
proc check(a, b: openArray[int]): bool = a == b

assert check([1, 2, 3], @[1, 2, 3])    # массив vs seq: равны
assert not check([1, 2], [1, 2, 3])    # разные длины
```

---

### Последовательности

```nim
proc `==`*[T](x, y: seq[T]): bool
```

**Что это такое:** Поэлементное равенство для динамически изменяемых последовательностей. Включает важный быстрый путь: если обе последовательности разделяют один и тот же указатель на базовую память (то есть одна была присвоена из другой без изменений), немедленно возвращается `true` без итерации.

```nim
assert @[1, 2, 3] == @[1, 2, 3]
assert not (@[1, 2] == @[1, 2, 3])

var a = @[1, 2, 3]
var b = a           # b разделяет указатель с a (поверхностное копирование)
assert a == b       # срабатывает быстрый путь через сравнение указателей
```

## Строгое меньше: `<`

### Целые числа (знаковые и беззнаковые)

```nim
proc `<`*(x, y: int): bool     # также int8, int16, int32, int64
proc `<`*(x, y: uint): bool    # также uint8, uint16, uint32, uint64
```

**Что это такое:** Числовое строгое меньше. Знаковые варианты используют знаковое сравнение; беззнаковые — беззнаковое. Смешивание знаковых и беззнаковых требует явного приведения типов.

```nim
assert 1 < 2
assert not (2 < 2)
assert 0'u8 < 255'u8
```

---

### Вещественные числа

```nim
proc `<`*(x, y: float32): bool
proc `<`*(x, y: float): bool
```

**Что это такое:** Строгое меньше по IEEE 754. Любое сравнение с участием `NaN` возвращает `false`.

```nim
assert 1.0 < 2.0
let nan = 0.0 / 0.0
assert not (nan < 1.0)   # сравнения с NaN всегда ложны
assert not (1.0 < nan)
```

---

### Строки

```nim
proc `<`*(x, y: string): bool
```

**Что это такое:** Лексикографическое побайтовое строгое меньше. Сравнение ведётся посимвольно; первый отличающийся байт определяет результат. Если все символы совпадают до длины более короткой строки, более короткая строка считается меньшей. Заглавные ASCII-буквы (65–90) имеют меньшее порядковое значение, чем строчные (97–122), поэтому `"Z" < "a"` — истина.

```nim
assert "abc" < "abd"
assert "abc" < "abcd"   # "abc" является префиксом "abcd" и короче
assert "Z" < "a"        # заглавная < строчная в ASCII
assert not ("abc" < "abc")
```

---

### Символы

```nim
proc `<`*(x, y: char): bool
```

**Что это такое:** Порядковое (ASCII / кодовая точка Unicode) строгое меньше для символов. Применяется то же ASCII-упорядочивание: цифры < заглавные < строчные.

```nim
assert 'a' < 'b'
assert 'A' < 'a'
assert not ('z' < 'a')
```

---

### Перечисления

```nim
proc `<`*[Enum: enum](x, y: Enum): bool
```

**Что это такое:** Сравнивает два значения перечисления по их базовому целочисленному порядку. Для стандартного перечисления без явных значений порядок объявления определяет порядковый номер: первый вариант — 0, следующий — 1 и так далее.

```nim
type Priority = enum low, medium, high

assert low < high
assert medium < high
assert not (high < low)
```

---

### Множества

```nim
proc `<`*[T](x, y: set[T]): bool
```

**Что это такое:** Проверка **строгого (собственного) подмножества**. Возвращает `true`, если каждый элемент `x` также входит в `y`, и `y` содержит хотя бы один элемент, которого нет в `x`. Множество **не** является строгим подмножеством самого себя.

```nim
assert {1, 2} < {1, 2, 3}    # {1,2} ⊂ {1,2,3}
assert not ({1, 2} < {1, 2}) # множество не является собственным подмножеством себя
assert not ({4} < {1, 2, 3}) # 4 не входит в правое множество
```

---

### Указатели и ссылки

```nim
proc `<`*[T](x, y: ref T): bool
proc `<`*[T](x, y: ptr T): bool
proc `<`*(x, y: pointer): bool
```

**Что это такое:** Числовое сравнение адресов — `x` имеет меньший адрес памяти, чем `y`. Используется преимущественно в арифметике указателей, пользовательских аллокаторах и структурах данных над сырыми блоками памяти.

## Меньше или равно: `<=`

### Целые числа (знаковые и беззнаковые)

```nim
proc `<=`*(x, y: int): bool    # также int8, int16, int32, int64
proc `<=`*(x, y: uint): bool   # также uint8, uint16, uint32, uint64
```

**Что это такое:** Числовое меньше-или-равно для всех разрядностей целых чисел.

```nim
assert 2 <= 2
assert 1 <= 2
assert not (3 <= 2)
```

---

### Вещественные числа

```nim
proc `<=`*(x, y: float32): bool
proc `<=`*(x, y: float): bool
```

**Что это такое:** Меньше-или-равно по IEEE 754. Сравнения с `NaN` возвращают `false`.

---

### Строки

```nim
proc `<=`*(x, y: string): bool
```

**Что это такое:** Лексикографическое меньше-или-равно. Возвращает `true`, если `x < y` или `x == y`.

```nim
assert "abc" <= "abc"
assert "abc" <= "abd"
assert not ("abd" <= "abc")
```

---

### Символы

```nim
proc `<=`*(x, y: char): bool
```

**Что это такое:** Порядковое меньше-или-равно для символов.

```nim
assert 'a' <= 'a'
assert 'a' <= 'b'
assert not ('b' <= 'a')
```

---

### Перечисления

```nim
proc `<=`*[Enum: enum](x, y: Enum): bool
```

**Что это такое:** Порядковое меньше-или-равно для значений перечисления. Значение перечисления `<=` самому себе.

---

### Множества

```nim
proc `<=`*[T](x, y: set[T]): bool
```

**Что это такое:** Проверка **подмножества или равенства**. Возвращает `true`, если каждый элемент `x` входит в `y`. В отличие от `<`, возвращает `true` когда `x == y`.

```nim
let a = {3, 5}
let b = {1, 3, 5, 7}

assert a <= b    # {3,5} ⊆ {1,3,5,7}
assert a <= a    # множество является подмножеством самого себя
assert not ({2} <= a)
```

---

### Булевы значения, ссылки и указатели

```nim
proc `<=`*(x, y: bool): bool
proc `<=`*[T](x, y: ref T): bool
proc `<=`*(x, y: pointer): bool
```

Для `bool`: `false <= true`, `false <= false`, `true <= true`. Для ссылок и указателей: числовой порядок адресов.

## Производные операторы: `!=`, `>`, `>=`

Эти три оператора — универсальные шаблоны, работающие с любым типом, для которого определены `==` и `<` / `<=`. Они несут прагму `{.callsite.}`, сохраняющую информацию о местоположении в исходном коде для сообщений об ошибках.

### `!=`

```nim
template `!=`*(x, y: untyped): untyped
```

**Что это такое:** Сокращение для `not (x == y)`. Поскольку это шаблон, компилятор раскрывает его в месте вызова — никаких накладных расходов на вызов функции.

```nim
assert 1 != 2
assert "hello" != "world"
assert not (42 != 42)
```

---

### `>`

```nim
template `>`*(x, y: untyped): untyped
```

**Что это такое:** Раскрывается в `y < x`. Определив `<` для пользовательского типа, вы автоматически получаете `>`.

```nim
assert 3 > 2
assert "b" > "a"
assert not (2 > 3)
```

---

### `>=`

```nim
template `>=`*(x, y: untyped): untyped
```

**Что это такое:** Раскрывается в `y <= x`. Определив `<=` для пользовательского типа, вы автоматически получаете `>=`.

```nim
assert 3 >= 3
assert 3 >= 2
assert not (2 >= 3)
```

---

## Беззнаковое переинтерпретирование: `<%`, `<=%`, `>%`, `>=%`

Это семейство операторов предназначено для ситуаций, когда вы держите *знаковые* целые числа, но должны сравнивать их как *беззнаковые*. Типичные случаи применения: зондирование хэш-таблиц, манипуляции с битовыми полями, проверки диапазонов памяти и разбор протоколов.

Суффикс `%` — мнемоника: «модульная» или «битовый образец» семантика, та же условная метка, что используется в Nim для `+%`, `*%` и т. д.

### `<%`

```nim
proc `<%`*(x, y: int): bool    # также int8, int16, int32, int64
```

**Что это такое:** Переинтерпретирует оба значения `x` и `y` как их беззнаковые аналоги и выполняет беззнаковое строгое меньше. Битовый образец не изменяется — меняется только интерпретация.

```nim
assert 1 <% 2      # то же, что беззнаковое: 1 < 2
assert -1 >% 0     # -1 как uint — это MaxUInt, что больше 0
assert not (0 <% 0)
```

Классический случай применения: проверка вхождения знакового целого числа в диапазон `[0, n)` одним беззнаковым сравнением вместо двух знаковых:

```nim
proc inRange(i, n: int): bool = i <% n
# Эквивалентно: i >= 0 and i < n
# -1 <% 10 → false (MaxUInt не < 10)
#  5 <% 10 → true
# 15 <% 10 → false
```

---

### `<=%`

```nim
proc `<=%`*(x, y: int): bool   # также int8, int16, int32, int64
```

**Что это такое:** Беззнаковое меньше-или-равно через переинтерпретирование.

```nim
assert 0 <=% 0
assert 0 <=% 5
assert not (-1 <=% 0)   # -1 как uint больше 0
```

---

### `>%` и `>=%`

```nim
template `>%`*(x, y: untyped): untyped    # раскрывается в y <% x
template `>=%`*(x, y: untyped): untyped   # раскрывается в y <=% x
```

**Что это такое:** Аналоги «больше» и «больше-или-равно» для беззнакового переинтерпретирования. Оба — шаблоны, производные от `<%` и `<=%`.

```nim
assert -1 >% 0    # MaxUInt > 0
assert 10 >=% 5
```

## Минимум и максимум: `min`, `max`

### `min` и `max` для двух значений

```nim
# Целые числа (специализированы для каждой разрядности):
proc min*(x, y: int): int
proc min*(x, y: int8): int8
# ... аналогично для int16, int32, int64

proc max*(x, y: int): int
# ... аналогично для int8, int16, int32, int64

# Вещественные числа (с распространением NaN):
proc min*(x, y: float32): float32
proc min*(x, y: float64): float64
proc max*(x, y: float32): float32
proc max*(x, y: float64): float64

# Обобщённый запасной вариант для любого не-float типа с <=:
proc min*[T: not SomeFloat](x, y: T): T
proc max*[T: not SomeFloat](x, y: T): T
```

**Что это такое:** Возвращает меньшее (или большее) из двух значений. Целочисленные варианты реализованы магией компилятора для максимальной эффективности. Варианты для вещественных чисел обрабатывают особый случай `NaN`: если `y` является `NaN`, возвращается `x` — это намеренный выбор, согласующийся со многими числовыми библиотеками (если один из операндов `NaN`, возвращается не-NaN; если оба `NaN`, возвращается первый операнд).

```nim
assert min(3, 7) == 3
assert max(3, 7) == 7
assert min(1.5, 2.5) == 1.5

let nan = 0.0 / 0.0
assert min(1.0, nan) == 1.0   # NaN отбрасывается
assert min(nan, 1.0) == nan   # когда x — NaN, возвращается именно x
```

> **Оговорка о NaN для вещественных чисел:** `min(nan, 1.0)` возвращает `nan`, потому что проверка `if x <= y or y != y: x else: y` — когда `y` является NaN, `y != y` истинно, и мы возвращаем `x` (NaN); когда же `x` является NaN, `x <= y` ложно и `y != y` тоже ложно, поэтому возвращается `y` (не-NaN). Поведение несимметрично — учитывайте это в численном коде.

### `min` и `max` для коллекции

```nim
proc min*[T](x: openArray[T]): T
proc max*[T](x: openArray[T]): T
```

**Что это такое:** Сканирует всю коллекцию и возвращает наименьший (или наибольший) элемент. Требует, чтобы тип элемента имел оператор `<`. Работает с массивами, последовательностями и любым другим типом, удовлетворяющим `openArray[T]`.

```nim
assert min([5, 2, 8, 1, 9]) == 1
assert max([5, 2, 8, 1, 9]) == 9
assert min(@["банан", "яблоко", "вишня"]) == "банан"
```

> **Внимание:** Вызов `min` или `max` на пустой коллекции вызовет ошибку выхода за границы массива во время выполнения, поскольку реализация безусловно обращается к `x[0]`.

## Ограничение диапазона: `clamp`

```nim
proc clamp*[T](x, a, b: T): T
```

**Что это такое:** Ограничивает значение `x` замкнутым интервалом `[a, b]`. Если `x < a`, возвращает `a`. Если `x > b`, возвращает `b`. В противном случае возвращает `x` без изменений. Работает для любого типа, у которого определены `<` и `>`.

> **Допущение:** Предполагается, что `a <= b`, и это не проверяется. Передача `a > b` даёт неопределённое поведение.

Семантически эквивалентно `max(a, min(b, x))`, но реализовано двумя прямыми сравнениями вместо двух вызовов функций — что чуть быстрее.

```nim
assert clamp(0.5, 0.0, 1.0) == 0.5   # в диапазоне, без изменений
assert clamp(1.4, 0.0, 1.0) == 1.0   # выше максимума, ограничено до 1.0
assert clamp(-0.3, 0.0, 1.0) == 0.0  # ниже минимума, ограничено до 0.0
assert clamp(4, 1, 3) == 3
assert clamp(0, 1, 3) == 1

# Практическое применение: нормализация пользовательского ввода
let volume = clamp(userInput, 0, 100)
```

## Защиты времени компиляции

### `isNil` для строк (процедура-ошибка)

```nim
proc isNil*(x: string): bool {.error: "'isNil' is invalid for 'string'".}
```

**Что это такое:** Намеренная ошибка времени компиляции, срабатывающая при случайном вызове `isNil` для `string`. В текущей модели памяти Nim строки являются типами-значениями и никогда не могут быть `nil` — вызов `isNil(s)` всегда является ошибкой программиста. Эта перегрузка существует исключительно для выдачи понятного сообщения об ошибке вместо запутанного несоответствия типов.

```nim
var s = "hello"
# if isNil(s): ...  ← ошибка компиляции: 'isNil' is invalid for 'string'
```

### Сравнение строк с nil (процедуры-ошибки)

```nim
proc `==`*(x: string; y: typeof(nil) | typeof(nil)): bool {.error: "'nil' is invalid for 'string'".}
proc `==`*(x: typeof(nil) | typeof(nil); y: string): bool {.error: "'nil' is invalid for 'string'".}
```

**Что это такое:** Защитные перегрузки, предотвращающие прямое сравнение `string` с `nil`. В Nim строки никогда не бывают `nil`; такое сравнение всегда является логической ошибкой. Эти процедуры существуют, чтобы перехватить ошибку во время компиляции (а не молча вернуть `false` из-за неожиданного неявного преобразования).

```nim
var s = "hello"
# if s == nil: ...   ← ошибка компиляции: 'nil' is invalid for 'string'
# if nil == s: ...   ← ошибка компиляции: 'nil' is invalid for 'string'
```
