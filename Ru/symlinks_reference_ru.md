# Справочник модуля `symlinks`

> **Назначение модуля:** `symlinks` предоставляет тонкую, типобезопасную обёртку над низкоуровневыми операциями с символическими ссылками, представляя их через строго типизированную абстракцию `Path` из `std/paths`. Модуль покрывает три фундаментальные операции: проверку существования ссылки, её создание и получение пути, на который она указывает.

---

## Что такое символическая ссылка?

Символическая ссылка (симлинк) — это особый элемент файловой системы, действующий как указатель на другой файл или директорию. Он хранит путь в виде строки, а не реальные данные. Операционная система прозрачно переходит по ссылке при обычном обращении к файлу — но специальные операции позволяют инспектировать или изменять саму ссылку.

```
/home/user/config  →  /etc/myapp/config.json   (симлинк → цель)
```

Симлинки могут указывать на файлы, директории, другие симлинки или даже на пути, которых ещё не существует (*висящие* ссылки). Модуль корректно обрабатывает все эти случаи.

---

## Используемый тип

### `Path`

Все процедуры модуля работают с `Path` из `std/paths`, а не с обычным `string`. `Path` — это отдельный тип, оборачивающий строку, который на этапе компиляции предотвращает случайное смешение путей с произвольными строками. Создаётся через `Path("some/path")`, исходная строка извлекается через `.string`.

## Справочник по процедурам

---

### `symlinkExists`

```nim
proc symlinkExists*(link: Path): bool
  {.inline, tags: [ReadDirEffect], sideEffect.}
```

#### Что делает

Проверяет, существует ли символическая ссылка по пути `link`, и возвращает `true`, если да. Ключевая особенность: процедура проверяет **саму ссылку**, а не её цель. Это означает:

- Если ссылка существует, но её цель удалена (*висящая* или *битая* ссылка), `symlinkExists` всё равно вернёт `true`.
- Если по данному пути находится обычный файл или директория (не симлинк), вернётся `false`.
- Если пути вообще не существует, вернётся `false`.

Это ключевое отличие от `fileExists` и `dirExists`, которые следуют по ссылке и проверяют цель. `symlinkExists` останавливается на самой ссылке.

#### Теги

| Тег | Значение |
|---|---|
| `ReadDirEffect` | Процедура читает метаданные директории из файловой системы |
| `sideEffect` | Результат может различаться между вызовами (состояние ФС может меняться) |

#### Примеры

```nim
import std/symlinks, std/paths

# Создаём симлинк для демонстрации
createSymlink(Path("/etc/hosts"), Path("/tmp/my_hosts_link"))

# Сама ссылка существует
assert symlinkExists(Path("/tmp/my_hosts_link")) == true

# Обычный файл — не симлинк
assert symlinkExists(Path("/etc/hosts")) == false

# Несуществующий путь
assert symlinkExists(Path("/tmp/no_such_link")) == false
```

Обнаружение битой (висящей) ссылки — то, что `fileExists` не умеет:

```nim
import std/symlinks, std/paths, std/os

createSymlink(Path("/nonexistent/target"), Path("/tmp/broken_link"))

# symlinkExists: видит саму ссылку → true
assert symlinkExists(Path("/tmp/broken_link")) == true

# fileExists: следует по ссылке, цель не найдена → false
assert not fileExists("/tmp/broken_link")
```

#### Когда использовать

Используйте `symlinkExists`, когда вам важно знать, **является ли путь симлинком**, независимо от того, доступна ли цель. Например: очистка устаревших ссылок, аудит директории на наличие симлинков, или принятие решения о вызове `expandSymlink`.

### `createSymlink`

```nim
proc createSymlink*(src, dest: Path) {.inline.}
```

#### Что делает

Создаёт новую символическую ссылку по пути `dest`, указывающую на `src`. После вызова обращение к `dest` через обычный файловый ввод-вывод будет прозрачно перенаправляться на `src`.

`src` не обязан существовать в момент создания ссылки — ОС сохраняет строку пути и разрешает её лениво при обращении. Это позволяет создавать ссылки на будущие или опциональные ресурсы.

#### Условия ошибки

Процедура выбросит `OSError`, если:

- По пути `dest` уже существует файл, директория или симлинк.
- У вызывающего процесса недостаточно прав (см. примечание о Windows ниже).
- Родительская директория `dest` не существует.

#### ⚠️ Ограничение Windows

На большинстве Unix-подобных систем любой пользователь может свободно создавать симлинки. На **Windows** создание символических ссылок требует одного из:

- Запуска от имени Администратора, или
- Включённого **Режима разработчика** в настройках системы (доступно с Windows 10 версии 1703).

При невыполнении ни одного из условий вызов завершится ошибкой доступа. Это политика операционной системы, а не ограничение Nim.

#### Описание параметров

| Параметр | Тип | Описание |
|---|---|---|
| `src` | `Path` | Цель, на которую будет указывать симлинк (не обязана существовать) |
| `dest` | `Path` | Путь, по которому будет создан новый симлинк |

#### Примеры

```nim
import std/symlinks, std/paths

# Создать симлинк: /tmp/link_to_hosts → /etc/hosts
createSymlink(Path("/etc/hosts"), Path("/tmp/link_to_hosts"))

assert symlinkExists(Path("/tmp/link_to_hosts"))

# Теперь чтение /tmp/link_to_hosts — то же самое, что чтение /etc/hosts
```

Симлинк на директорию:

```nim
import std/symlinks, std/paths

# Удобный псевдоним для длинного пути
createSymlink(Path("/var/log/myapp/current"), Path("/tmp/myapp_log"))

# Файлы внутри доступны через симлинк как через настоящую директорию
```

Создание ссылки на несуществующую пока цель:

```nim
import std/symlinks, std/paths

# Успешно, даже если /mnt/data/store ещё не существует
createSymlink(Path("/mnt/data/store"), Path("/tmp/store"))
# Как только /mnt/data/store будет создана или смонтирована, ссылка заработает
```

Защита от повторного создания:

```nim
import std/symlinks, std/paths

let dest = Path("/tmp/my_link")
if not symlinkExists(dest):
  createSymlink(Path("/some/target"), dest)
else:
  echo "Ссылка уже существует, пропускаем"
```

### `expandSymlink`

```nim
proc expandSymlink*(symlinkPath: Path): Path {.inline.}
```

#### Что делает

Читает символическую ссылку по пути `symlinkPath` и возвращает путь, на который она указывает — то есть значение, хранящееся *внутри* ссылки, а не разрешённый конечный файл. Иногда это называют «чтением» симлинка.

Возвращаемый путь может быть:

- **Абсолютным** (`/etc/hosts`) — если ссылка создавалась с абсолютной целью.
- **Относительным** (`../config/settings.json`) — если ссылка создавалась с относительной целью. В таком случае путь интерпретируется относительно *директории, содержащей ссылку*, а не текущей рабочей директории.

Процедура **не проверяет** существование цели и не следует по цепочкам симлинков. Она просто возвращает то, что хранит ссылка.

#### Поведение на Windows

На Windows `expandSymlink` фактически является **заглушкой (no-op)**: входной `symlinkPath` возвращается без изменений. Это известное ограничение. Симлинки Windows требуют специальных вызовов API, которые могут быть недоступны в различных контекстах; чтобы избежать скрытых ошибок, функция просто возвращает входной путь как есть. Кроссплатформенный код, зависящий от раскрытия симлинков, должен явно учитывать этот случай.

#### Описание параметров

| Параметр | Тип | Описание |
|---|---|---|
| `symlinkPath` | `Path` | Путь симлинка, который нужно прочитать |

**Возвращает:** `Path` — цель, хранящаяся внутри симлинка (или неизменённый вход на Windows).

#### Примеры

```nim
import std/symlinks, std/paths

# Создаём известный симлинк
createSymlink(Path("/etc/hosts"), Path("/tmp/my_link"))

# Читаем его
let target = expandSymlink(Path("/tmp/my_link"))
assert target == Path("/etc/hosts")
```

Относительные симлинки:

```nim
import std/symlinks, std/paths

# Если ссылка была создана как: ln -s ../config/app.cfg /tmp/links/app
let target = expandSymlink(Path("/tmp/links/app"))
# target может быть Path("../config/app.cfg")
# Чтобы получить абсолютный путь, нужно объединить с директорией ссылки:
# /tmp/links/ + ../config/app.cfg = /tmp/config/app.cfg
```

Безопасное чтение с предварительной проверкой:

```nim
import std/symlinks, std/paths

let link = Path("/tmp/maybe_link")
if symlinkExists(link):
  let target = expandSymlink(link)
  echo "Ссылка указывает на: ", target.string
else:
  echo "По этому пути симлинка нет"
```

Учёт заглушки на Windows:

```nim
import std/symlinks, std/paths

let link = Path("/some/link")
let expanded = expandSymlink(link)
when defined(windows):
  assert expanded == link  # no-op на Windows
```

## Как три процедуры работают вместе

Типичный рабочий сценарий использует все три процедуры совместно:

```nim
import std/symlinks, std/paths, std/os

proc ensureSymlink(src, dest: Path) =
  if symlinkExists(dest):
    let current = expandSymlink(dest)
    if current == src:
      return  # ссылка уже правильная, ничего делать не нужно
    else:
      removeFile(dest.string)  # устаревшая ссылка — удаляем

  createSymlink(src, dest)
  echo "Создана ссылка: ", dest.string, " → ", src.string
```

| Шаг | Процедура | Назначение |
|---|---|---|
| 1 | `symlinkExists` | Проверить, есть ли ссылка по данному пути |
| 2 | `expandSymlink` | Убедиться, что существующая ссылка указывает на правильную цель |
| 3 | `createSymlink` | Создать ссылку, если её нет или она указывает не туда |

## Типичные ошибки

**Путаница `symlinkExists` с `fileExists`/`dirExists`.** Последние две следуют по ссылке и проверяют цель. Используйте `symlinkExists`, когда вас интересует сама ссылка, а не то, куда она ведёт.

**Забыть, что `expandSymlink` может вернуть относительный путь.** Если ссылка создавалась с относительной целью, результат нужно объединить с родительской директорией ссылки для получения абсолютного пути.

**Режим разработчика на Windows.** Если ваш код создаёт симлинки и должен работать на Windows без прав администратора, обязательно документируйте требование к Режиму разработчика или предусматривайте запасной вариант (например, копирование файла вместо создания ссылки).

**Перезапись существующей ссылки.** `createSymlink` не перезаписывает — при существующем `dest` выбрасывается `OSError`. Всегда проверяйте через `symlinkExists`, если операция может выполняться повторно.
