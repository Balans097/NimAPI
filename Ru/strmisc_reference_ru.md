# Справочник по модулю `strmisc`

> **Модуль:** `std/strmisc`  
> **Язык:** Nim  
> **Назначение:** Набор редко используемых, но полезных утилит для работы со строками, дополняющих стандартный модуль `strutils`.

---

## Обзор

Модуль `strmisc` содержит три экспортируемые функции для специфических задач обработки строк. Они не настолько универсальны, чтобы попасть в основной `strutils`, но незаменимы, когда нужны:

- **`expandTabs`** — заменяет символы табуляции пробелами с учётом колоночного выравнивания
- **`partition`** — разбивает строку на три части по разделителю
- **`rpartition`** — то же самое, но поиск разделителя ведётся с правого конца строки

Все три функции объявлены как `func` — чистые функции без побочных эффектов. Они всегда возвращают одинаковый результат для одинаковых входных данных и не изменяют исходную строку.

---

## `expandTabs`

```nim
func expandTabs*(s: string, tabSize: int = 8): string
```

### Что делает

Функция проходит по строке и заменяет каждый символ табуляции (`\t`) ровно таким количеством пробелов, которое нужно для достижения **следующей позиции табуляции**. Результирующая строка при отображении в моноширинном шрифте выглядит так же, как оригинал, но содержит только пробелы.

Ключевое понятие здесь — **позиции табуляции** (tab stops): столбцы с номерами, кратными `tabSize`. Когда функция встречает `\t`, она не вставляет ровно `tabSize` пробелов — вместо этого она вставляет ровно столько пробелов, сколько нужно, чтобы колонка «перепрыгнула» на следующую позицию табуляции. Именно так работают текстовые редакторы и терминалы.

### Как отслеживается текущий столбец

- Счётчик `pos` хранит номер текущего столбца, начиная с `0`.
- Каждый символ (включая вставленные пробелы) увеличивает `pos` на 1.
- Символ перевода строки (`\n`) сбрасывает `pos` обратно в `0`, начиная новую строку.
- При встрече табуляции: `количество_пробелов = tabSize - (pos mod tabSize)`

Это означает, что табуляция в начале строки добавит больше пробелов, чем табуляция в середине интервала между позициями.

### Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `s` | `string` | — | Входная строка, может содержать `\t` и `\n` |
| `tabSize` | `int` | `8` | Ширина между позициями табуляции. Если `≤ 0`, трактуется как `1` |

### Возвращаемое значение

Новая строка, в которой все `\t` заменены пробелами. Исходная строка не изменяется.

### Примеры

```nim
import std/strmisc

# Одиночный таб в столбце 0, tabSize=4 → 4 пробела
doAssert expandTabs("\t", 4) == "    "

# Таб перед "foo": 4 пробела (стоп в столбце 4)
# Таб после "foo": "foo" = 3 символа, следующий стоп = 4, вставляем 1 пробел
doAssert expandTabs("\tfoo\t", 4) == "    foo "

# Несколько строк: перевод строки сбрасывает счётчик столбца
doAssert expandTabs("a\tb\n\txy\t", 3) == "a  b\n   xy "
# Строка 1: "a" в столбце 1, таб → 2 пробела до столбца 3, потом "b"
# Строка 2: столбец сбрасывается, таб → 3 пробела, "xy" в столбцах 3–4, таб → 1 пробел

# По умолчанию tabSize = 8
doAssert expandTabs("a\tb") == "a       b"
#                              "a" + 7 пробелов + "b"
```

### Типичное применение

Нормализация исходного кода или конфигурационных файлов, в которых смешаны табуляции и пробелы, перед их дальнейшей обработкой, сравнением (diff) или отображением.

## `partition`

```nim
func partition*(s: string, sep: string, right: bool = false): (string, string, string)
```

### Что делает

`partition` ищет подстроку `sep` внутри `s` и разбивает строку ровно на **три части**: часть до разделителя, сам разделитель и часть после него. Результат — всегда кортеж из трёх строк, независимо от того, найден ли разделитель.

Функция аналогична методам `str.partition()` и `str.rpartition()` в Python, объединённым в одну через флаг `right`.

### Почему именно кортеж из трёх элементов?

Ключевое решение в дизайне: разделитель сохраняется в середине кортежа. Это позволяет:

1. **Узнать, был ли разделитель найден** — средний элемент будет пустой строкой, если нет.
2. **Восстановить исходную строку** — `before & sep & after == s` всегда верно.

### Направление поиска

- `right = false` (по умолчанию): ищет **первое** вхождение `sep` (слева направо)
- `right = true`: ищет **последнее** вхождение `sep` (справа налево)

### Если `sep` не найден

| Значение `right` | Возвращаемое значение |
|-----------------|----------------------|
| `false` | `(s, "", "")` — вся строка в первом элементе |
| `true` | `("", "", s)` — вся строка в третьем элементе |

Пустой средний элемент — надёжный признак того, что разделитель отсутствует.

### Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|-------------|----------|
| `s` | `string` | — | Строка для разбиения |
| `sep` | `string` | — | Строка-разделитель |
| `right` | `bool` | `false` | Если `true`, поиск с конца (последнее вхождение) |

### Возвращаемое значение

Кортеж `(beforeSep, sep, afterSep)`. Все три элемента — строки.

### Примеры

```nim
import std/strmisc

# Разбиение по первому ":"
doAssert partition("foo:bar:baz", ":") == ("foo", ":", "bar:baz")
#                                          ^^^    sep   ^^^^^^^
#                                          до           после

# Разбиение по последнему ":" с помощью right = true
doAssert partition("foo:bar:baz", ":", right = true) == ("foo:bar", ":", "baz")

# Разделитель не найден (right = false): вся строка в первом элементе, середина — ""
doAssert partition("foobar", ":") == ("foobar", "", "")

# Разделитель не найден (right = true): вся строка в третьем элементе, середина — ""
doAssert partition("foobar", ":", right = true) == ("", "", "foobar")

# Практика: разбор строки "ключ=значение"
let line = "host=localhost"
let (key, _, value) = partition(line, "=")
echo key    # "host"
echo value  # "localhost"

# Проверка наличия разделителя
let (_, sep, _) = partition("строка-без-знака-равно", "=")
if sep == "":
  echo "разделитель не найден"
```

### Типичное применение

- Разбор строк формата `ключ=значение` или `ключ:значение`
- Извлечение расширения файла разбиением по последней точке
- Разбиение URL на протокол и остальную часть по `://`

## `rpartition`

```nim
func rpartition*(s: string, sep: string): (string, string, string)
```

### Что делает

`rpartition` — удобная обёртка над `partition(s, sep, right = true)`. Она ищет **последнее** вхождение `sep` в строке `s` и возвращает кортеж из трёх элементов в том же формате.

Отдельная функция делает код читабельнее: когда вам нужен поиск справа, намерение понятно без именованного аргумента — название говорит само за себя.

### Если `sep` не найден

Возвращает `("", "", s)` — вся строка оказывается в третьем элементе, средний пустой.

### Параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `s` | `string` | Строка для разбиения |
| `sep` | `string` | Разделитель (ищется последнее вхождение) |

### Возвращаемое значение

Кортеж `(beforeSep, sep, afterSep)`.

### Примеры

```nim
import std/strmisc

# Находит ПОСЛЕДНЕЕ ":", отделяя "baz"
doAssert rpartition("foo:bar:baz", ":") == ("foo:bar", ":", "baz")

# Не найден: всё в третьем элементе
doAssert rpartition("foobar", ":") == ("", "", "foobar")

# Практика: получить расширение файла (последняя ".")
let filename = "archive.tar.gz"
let (base, _, ext) = rpartition(filename, ".")
echo base  # "archive.tar"
echo ext   # "gz"

# Практика: получить последний сегмент URL
let url = "https://example.com/docs/api/reference"
let (_, _, page) = rpartition(url, "/")
echo page  # "reference"
```

### `rpartition` против `partition` наглядно

```
Входная строка: "foo:bar:baz"   разделитель: ":"

partition  → ("foo",     ":", "bar:baz")   ← первое  ":"
rpartition → ("foo:bar", ":", "baz")       ← последнее ":"
```

### Типичное применение

- Извлечение финального расширения файла (`.gz`, `.nim` и т.д.)
- Получение последнего сегмента пути URL или файловой системы
- Любая ситуация, где важен *самый правый* разделитель

| `rpartition(s, sep)` | Справа (последнее) | — | `("", "", s)` | Псевдоним для правостороннего разбиения |

## Примечания

- Все функции объявлены как `func` (чистые функции) — безопасно использовать в контекстах времени компиляции.
- Ни одна из функций не изменяет входную строку; каждая возвращает новое значение.
- `partition` и `rpartition` всегда возвращают кортеж из трёх элементов; проверяйте средний на `""`, чтобы определить, был ли разделитель найден.
- `expandTabs` с `tabSize ≤ 0` молча использует `1`, чтобы избежать деления на ноль.
