# Модуль `fenv` для Nim — полный справочник

## Обзор

Модуль `fenv` — это привязка Nim к заголовочному файлу стандартной библиотеки C `<fenv.h>`, которая предоставляет прямой доступ к управлению **средой операций с плавающей точкой** (floating-point environment) вашего процесса. Эта среда состоит из двух независимых частей:

1. **Флаги исключений** — «липкие» биты состояния, которые аппаратура устанавливает при возникновении особых ситуаций во время операций с плавающей точкой (деление на ноль, переполнение, неточное округление и т. д.). Они накапливаются молча, пока вы сами не проверите или не сбросите их.
2. **Режим округления** — управляющая настройка, которая говорит FPU (блоку обработки чисел с плавающей точкой), как округлять результаты, не поддающиеся точному представлению. По умолчанию: «округление к ближайшему чётному» (round half to even).

Помимо управления в рантайме, модуль также экспортирует **константы времени компиляции** и **шаблоны-запросы характеристик**, описывающие числовые пределы типов `float32` и `float64`.

### Когда использовать этот модуль?

- **Численный анализ и научные вычисления** — обнаружение того, что алгоритм породил промежуточное переполнение или антипереполнение, хотя итоговый результат не выглядит очевидно неверным.
- **Реализация корректно-округлённой арифметики** — временное переключение в другой режим округления (например, `FE_UPWARD`) для вычисления границ интервальной арифметики.
- **Защитное программирование** — очистка флагов исключений перед блоком вычислений и их проверка по окончании для подтверждения отсутствия аномалий.
- **Написание обобщённого числового кода** — запрос `epsilon`, `maximumPositiveValue` и т. д. вместо жёстко зашитых магических чисел.

### Платформенные замечания

- Модуль является тонкой обёрткой над C `<fenv.h>`. Доступен на всех POSIX-платформах и Windows (с MSVC или MinGW C runtime). На POSIX (кроме macOS и Genode) `-lm` подключается автоматически.
- Точные значения `FE_DIVBYZERO`, `FE_OVERFLOW` и т. д. определяются платформой на уровне C; Nim импортирует их из заголовка C при компиляции, поэтому они всегда корректны для целевой платформы.
- Манипуляции со средой плавающей точки — **глобальная, локальная для потока** операция. Изменения влияют на все операции с плавающей точкой в текущем потоке.

## Экспортируемый API — краткий обзор

### Константы флагов исключений

| Константа | Значение |
|-----------|---------|
| `FE_DIVBYZERO` | Произошло деление на ноль |
| `FE_INEXACT` | Результат был округлён (не точный) |
| `FE_INVALID` | Недопустимая операция (например, `0.0 / 0.0`, `sqrt(-1)`) |
| `FE_OVERFLOW` | Результат слишком велик для представления (стал ±Inf) |
| `FE_UNDERFLOW` | Результат слишком мал для представления (стал нулём или субнормальным) |
| `FE_ALL_EXCEPT` | Побитовое ИЛИ всех исключений выше — удобная маска для «всего» |

### Константы режимов округления

| Константа | Поведение при округлении |
|-----------|------------------------|
| `FE_TONEAREST` | К ближайшему представимому значению (по умолчанию, при равноудалении — к чётному) |
| `FE_DOWNWARD` | К −∞ (всегда вниз) |
| `FE_UPWARD` | К +∞ (всегда вверх) |
| `FE_TOWARDZERO` | К нулю (усечение, как целочисленное деление) |

### Константа среды

| Константа | Значение |
|-----------|---------|
| `FE_DFL_ENV` | Дескриптор среды плавающей точки по умолчанию |

### Типы

| Тип | Значение |
|-----|---------|
| `Tfenv` | Непрозрачный объект, хранящий полную сохранённую среду FP |
| `Tfexcept` | Непрозрачный объект, хранящий снимок флагов исключений |

### Процедуры для флагов исключений

| Процедура | Назначение |
|-----------|-----------|
| `feclearexcept` | Сброс (очистка) выбранных флагов исключений |
| `fegetexceptflag` | Сохранение текущих флагов в `Tfexcept` |
| `fesetexceptflag` | Восстановление флагов из сохранённого `Tfexcept` |
| `feraiseexcept` | Искусственная установка флагов исключений |
| `fetestexcept` | Проверка, какие флаги в данный момент установлены |

### Процедуры для режима округления

| Процедура | Назначение |
|-----------|-----------|
| `fegetround` | Чтение текущего режима округления |
| `fesetround` | Изменение режима округления |

### Процедуры для полной среды

| Процедура | Назначение |
|-----------|-----------|
| `fegetenv` | Сохранение всей среды FP |
| `fesetenv` | Восстановление ранее сохранённой среды FP |
| `feholdexcept` | Сохранение среды, сброс флагов, включение non-stop режима |
| `feupdateenv` | Слияние сохранённой среды с текущими отложенными исключениями |

### Шаблоны числовых свойств (float32 и float64)

`mantissaDigits`, `digits`, `minExponent`, `maxExponent`, `min10Exponent`, `max10Exponent`, `minimumPositiveValue`, `maximumPositiveValue`, `epsilon`, `fpRadix`

## Константы флагов исключений

Флаги исключений — это отдельные биты. Для адресации нескольких исключений одновременно они объединяются побитовым ИЛИ.

### `FE_DIVBYZERO`

Устанавливается, когда операция с плавающей точкой делит конечное число на ноль и результат равен ±Inf. Обычное целочисленное деление на ноль — другой механизм, не связанный с FPU.

```nim
import std/fenv

discard feclearexcept(FE_ALL_EXCEPT)
let x = 1.0 / 0.0           # IEEE 754: результат +Inf, флаг установлен
doAssert fetestexcept(FE_DIVBYZERO) != 0
```

### `FE_INEXACT`

Устанавливается каждый раз, когда истинный математический результат операции не может быть представлен точно и должен быть округлён. Этот флаг срабатывает крайне часто — почти каждое умножение или деление его устанавливает. Он полезен в высокоточных численных вычислениях, когда нужно убедиться, что критический шаг был точным.

```nim
import std/fenv

discard feclearexcept(FE_INEXACT)
let _ = 1.0 / 3.0   # 1/3 не представима точно
doAssert fetestexcept(FE_INEXACT) != 0
```

### `FE_INVALID`

Устанавливается, когда операция не имеет чётко определённого результата в вещественной арифметике: `0.0 / 0.0`, `sqrt` от отрицательного числа, `Inf - Inf` и т. п. Результат — NaN.

```nim
import std/fenv
import std/math

discard feclearexcept(FE_INVALID)
let _ = sqrt(-1.0)
doAssert fetestexcept(FE_INVALID) != 0
```

### `FE_OVERFLOW`

Устанавливается, когда математический результат конечен, но слишком велик по модулю для представления в типе с плавающей точкой; результат становится ±Inf.

```nim
import std/fenv

discard feclearexcept(FE_OVERFLOW)
let _ = maximumPositiveValue(float32) * 2.0'f32
doAssert fetestexcept(FE_OVERFLOW) != 0
```

### `FE_UNDERFLOW`

Устанавливается, когда математический результат ненулевой, но слишком мал по модулю для представления как нормальное число. Результат — ноль или субнормальное (денормализованное) число с пониженной точностью.

```nim
import std/fenv

discard feclearexcept(FE_UNDERFLOW)
let _ = minimumPositiveValue(float32) / 1000.0'f32
doAssert fetestexcept(FE_UNDERFLOW) != 0
```

### `FE_ALL_EXCEPT`

Побитовое ИЛИ всех флагов исключений, поддерживаемых платформой. Удобное сокращение для ситуаций, когда нужно работать сразу со всеми исключениями.

```nim
import std/fenv

discard feclearexcept(FE_ALL_EXCEPT)   # сброс всего разом

let anyRaised = fetestexcept(FE_ALL_EXCEPT)   # проверка: было ли хоть что-то
```

## Константы режимов округления

IEEE 754 требует, чтобы каждая операция с плавающей точкой давала правильно округлённый результат для активного режима. Изменение режима влияет на все последующие операции в потоке.

### `FE_TONEAREST`

Режим по умолчанию практически на всех системах. Когда истинный результат лежит ровно посередине между двумя представимыми значениями, выбирается то, у которого последний бит чётный («округление к чётному» или «банковское округление»). Это минимизирует статистическое смещение при большом числе операций.

```nim
import std/fenv

doAssert fegetround() == FE_TONEAREST   # истинно в свежем процессе
```

### `FE_DOWNWARD`

Каждый результат округляется к −∞. `2.9` становится `2.0`, `−2.1` становится `−3.0`. Используется в интервальной арифметике для вычисления гарантированной нижней границы.

### `FE_UPWARD`

Каждый результат округляется к +∞. `2.1` становится `3.0`, `−2.9` становится `−2.0`. Используется в интервальной арифметике для вычисления гарантированной верхней границы.

### `FE_TOWARDZERO`

Каждый результат округляется к нулю — эквивалентно усечению. `2.9` становится `2.0`, `−2.9` становится `−2.0`.

## Типы

### `Tfenv`

```nim
type Tfenv {.importc: "fenv_t", ...} = object
```

Непрозрачный объект, который захватывает **полную** среду плавающей точки: и режим округления, и все флаги исключений. Используется как цель `fegetenv` / `fesetenv` / `feholdexcept` / `feupdateenv`. Инспектировать отдельные поля нельзя — воспринимайте как непрозрачный снимок состояния.

### `Tfexcept`

```nim
type Tfexcept {.importc: "fexcept_t", ...} = object
```

Непрозрачный объект, который захватывает только **флаги исключений** из среды FP. Используется с `fegetexceptflag` и `fesetexceptflag`. Как и `Tfenv`, воспринимайте как непрозрачный — платформа может хранить дополнительное состояние рядом с флагами.

---

## Процедуры для флагов исключений

### `feclearexcept`

```nim
proc feclearexcept*(excepts: cint): cint
```

Сбрасывает (обнуляет) флаги исключений, указанные в `excepts`. Возвращает ноль при успехе. Вызывайте это перед вычислительным блоком, когда хотите проверить, не возникло ли в именно этом блоке каких-либо исключений.

```nim
import std/fenv

discard feclearexcept(FE_ALL_EXCEPT)   # начинаем с чистого листа
# ... выполняем вычисления ...
if fetestexcept(FE_OVERFLOW or FE_INVALID) != 0:
  echo "Что-то пошло не так!"
```

---

### `fegetexceptflag`

```nim
proc fegetexceptflag*(flagp: ptr Tfexcept, excepts: cint): cint
```

Сохраняет снимок флагов исключений, указанных в `excepts`, в объект `Tfexcept`, на который указывает `flagp`. Возвращает ноль при успехе. Это более низкоуровневое сохранение, чем `fegetenv` — захватываются только запрошенные флаги, но не режим округления.

```nim
import std/fenv

var saved: Tfexcept
discard fegetexceptflag(addr saved, FE_ALL_EXCEPT)
# ... делаем что-то, что может изменить флаги ...
discard fesetexceptflag(addr saved, FE_ALL_EXCEPT)   # восстанавливаем
```

---

### `fesetexceptflag`

```nim
proc fesetexceptflag*(flagp: ptr Tfexcept, excepts: cint): cint
```

Восстанавливает флаги исключений, указанные в `excepts`, из снимка в `flagp`. Важно: эта функция **не генерирует аппаратные исключения** (сигналы/ловушки) — она просто устанавливает биты флагов. Если нужно действительно вызвать аппаратное исключение, используйте `feraiseexcept`. Возвращает ноль при успехе.

```nim
import std/fenv

var saved: Tfexcept
discard fegetexceptflag(addr saved, FE_OVERFLOW)
# сбрасываем флаг, выполняем работу, восстанавливаем исходное состояние:
discard feclearexcept(FE_OVERFLOW)
# ... работа ...
discard fesetexceptflag(addr saved, FE_OVERFLOW)
```

---

### `feraiseexcept`

```nim
proc feraiseexcept*(excepts: cint): cint
```

Искусственно вызывает (генерирует) исключения плавающей точки, указанные в `excepts`, как если бы соответствующие исключительные арифметические операции реально произошли. Может вызвать обработчик сигнала, если исключение не замаскировано. Возвращает ноль при успехе.

Главным образом полезно для тестирования и в библиотечном коде, который должен распространить состояние исключений через секцию, где исключения временно подавлялись (например, после `feholdexcept`).

```nim
import std/fenv

discard feclearexcept(FE_ALL_EXCEPT)
discard feraiseexcept(FE_DIVBYZERO or FE_OVERFLOW)
doAssert fetestexcept(FE_DIVBYZERO) != 0
doAssert fetestexcept(FE_OVERFLOW)  != 0
```

---

### `fetestexcept`

```nim
proc fetestexcept*(excepts: cint): cint
```

Проверяет, какие из флагов, указанных в `excepts`, в данный момент установлены. Возвращает битовую маску установленных флагов (подмножество `excepts`). Возвращает ноль, если ни один из запрошенных флагов не установлен.

```nim
import std/fenv

discard feclearexcept(FE_ALL_EXCEPT)
let _ = 1.0 / 0.0

let flags = fetestexcept(FE_ALL_EXCEPT)
if (flags and FE_DIVBYZERO) != 0: echo "Произошло деление на ноль"
if (flags and FE_OVERFLOW)  != 0: echo "Произошло переполнение"
```

## Процедуры для режима округления

### `fegetround`

```nim
proc fegetround*(): cint
```

Возвращает идентификатор текущего активного режима округления. Возвращаемое значение — одна из констант `FE_*`. Полезно для сохранения режима перед его изменением, чтобы потом восстановить.

```nim
import std/fenv

let original = fegetround()
echo original == FE_TONEAREST   # как правило, истинно
```

---

### `fesetround`

```nim
proc fesetround*(roundingDirection: cint): cint
```

Устанавливает активный режим округления в `roundingDirection`. Возвращает ноль при успехе. **Всегда восстанавливайте исходный режим** после окончания работы — оставление нестандартного режима округления активным является распространённым источником незаметных ошибок.

```nim
import std/fenv

let saved = fegetround()

discard fesetround(FE_UPWARD)
let upper = 1.0 / 3.0   # округление к +Inf

discard fesetround(FE_DOWNWARD)
let lower = 1.0 / 3.0   # округление к -Inf

discard fesetround(saved)   # всегда восстанавливаем!
echo "Интервал: [", lower, ", ", upper, "]"
```

## Процедуры для полной среды

### `fegetenv`

```nim
proc fegetenv*(envp: ptr Tfenv): cint
```

Сохраняет **полную** среду плавающей точки (режим округления + все флаги исключений) в объект `Tfenv` по адресу `envp`. Возвращает ноль при успехе. Это наиболее «тяжёлая» операция сохранения — используйте её, когда нужно сохранить и восстановить всё.

```nim
import std/fenv

var env: Tfenv
discard fegetenv(addr env)
# ... изменяем среду ...
discard fesetenv(addr env)   # восстанавливаем всё
```

---

### `fesetenv`

```nim
proc fesetenv*(a1: ptr Tfenv): cint
```

Восстанавливает полную среду плавающей точки из объекта `Tfenv`. В отличие от `feupdateenv`, **отбрасывает** все флаги исключений, которые были подняты между сохранением и восстановлением. Возвращает ноль при успехе.

Специальную константу `FE_DFL_ENV` можно передать (приведя к `ptr Tfenv`) для сброса в среду по умолчанию (как при старте процесса).

```nim
import std/fenv

var env: Tfenv
discard fegetenv(addr env)
discard fesetround(FE_DOWNWARD)
# ... работа ...
discard fesetenv(addr env)   # флаги, поднятые во время работы, отбрасываются
```

---

### `feholdexcept`

```nim
proc feholdexcept*(envp: ptr Tfenv): cint
```

Комбинированная операция, которая:
1. Сохраняет текущую среду в `envp` (как `fegetenv`).
2. Очищает все флаги исключений.
3. Устанавливает «non-stop» режим, при котором исключения замаскированы (без сигналов/ловушек).

Возвращает ноль при успехе. Это стандартная преамбула для секции кода, которая хочет выполнять операции с плавающей точкой без прерываний и проверить флаги в конце. Используйте в паре с `feupdateenv` для распространения возникших исключений.

```nim
import std/fenv

var saved: Tfenv
discard feholdexcept(addr saved)   # сохраняем + сбрасываем + маскируем

let _ = 1.0 / 0.0   # флаг FE_DIVBYZERO устанавливается, но без сигнала

discard feupdateenv(addr saved)    # восстанавливаем среду и переподнимаем флаги
```

---

### `feupdateenv`

```nim
proc feupdateenv*(envp: ptr Tfenv): cint
```

Комбинированная операция, которая:
1. Временно сохраняет текущие отложенные флаги исключений.
2. Восстанавливает среду из `envp` (как `fesetenv`).
3. Повторно поднимает временно сохранённые флаги (как `feraiseexcept`).

Это дополнение к `feholdexcept`. Эффект: восстановить старую среду, но не потерять исключения, возникшие во время non-stop секции. Возвращает ноль при успехе.

```nim
import std/fenv

var saved: Tfenv
discard feholdexcept(addr saved)

# Вычисления — исключения накапливаются в флагах молча
let a = 1.0 / 0.0   # устанавливает FE_DIVBYZERO внутренне

# Восстанавливаем исходную среду; FE_DIVBYZERO поднимается в ней
discard feupdateenv(addr saved)
doAssert fetestexcept(FE_DIVBYZERO) != 0
```

## Шаблоны числовых свойств

Эти шаблоны вычисляются на этапе компиляции и возвращают числовые пределы типов с плавающей точкой. Они принимают `typedesc` (`float32` или `float64`) в качестве аргумента — это делает их обобщёнными и самодокументирующимися.

### `fpRadix`

```nim
template fpRadix*: int
```

Основание системы счисления (радикс), используемой для представления чисел с плавающей точкой на данной архитектуре. На всём современном «железе» это `2` (двоичная). Это свойство аппаратуры, а не конкретного типа — аргумент не принимает.

```nim
import std/fenv
doAssert fpRadix == 2
```

---

### `mantissaDigits`

```nim
template mantissaDigits*(T: typedesc[float32 | float64]): int
```

Количество цифр в основании `fpRadix` в мантиссе (значащей части). Для `float32` это 24 (один неявный ведущий бит «1» плюс 23 хранимых бита); для `float64` — 53. Говорит о точности типа в его родном основании.

```nim
import std/fenv
doAssert mantissaDigits(float32) == 24
doAssert mantissaDigits(float64) == 53
```

---

### `digits`

```nim
template digits*(T: typedesc[float32 | float64]): int
```

Количество **десятичных** цифр точности — сколько десятичных цифр можно сохранить в типе без потери информации при обратном преобразовании. `float32` даёт 6, `float64` — 15. Полезно при форматировании вывода чисел.

```nim
import std/fenv
doAssert digits(float32) == 6
doAssert digits(float64) == 15
```

### `minExponent` и `maxExponent`

```nim
template minExponent*(T: typedesc[float32 | float64]): int
template maxExponent*(T: typedesc[float32 | float64]): int
```

Минимальный и максимальный показатель степени нормализованного числа с плавающей точкой, выраженный как степень `fpRadix` (т. е. основания 2). Для `float32`: мин = −125, макс = 128. Для `float64`: мин = −1021, макс = 1024.

```nim
import std/fenv
doAssert minExponent(float32) == -125
doAssert maxExponent(float64) == 1024
```

### `min10Exponent` и `max10Exponent`

```nim
template min10Exponent*(T: typedesc[float32 | float64]): int
template max10Exponent*(T: typedesc[float32 | float64]): int
```

Те же границы показателей степени, выраженные в основании 10. Для `float32`: от −37 до 38. Для `float64`: от −307 до 308. Соответствует диапазону, выводимому в научной нотации.

```nim
import std/fenv
doAssert min10Exponent(float64) == -307
doAssert max10Exponent(float64) == 308
```

### `minimumPositiveValue`

```nim
template minimumPositiveValue*(T: typedesc[float32]): float32
template minimumPositiveValue*(T: typedesc[float64]): float64
```

Наименьшее **положительное нормализованное** число, представимое в типе. Значения меньше этого являются субнормальными (денормализованными) с пониженной точностью; значения меньше наименьшего субнормального становятся нулём (антипереполнение). Для `float32` это примерно `1.18e-38`; для `float64` — примерно `2.23e-308`.

```nim
import std/fenv
echo minimumPositiveValue(float32)   # 1.1754944e-38
echo minimumPositiveValue(float64)   # 2.2250738585072014e-308
```

---

### `maximumPositiveValue`

```nim
template maximumPositiveValue*(T: typedesc[float32]): float32
template maximumPositiveValue*(T: typedesc[float64]): float64
```

Наибольшее конечное положительное число, представимое в типе. Значения, превышающие это, переполняются до `+Inf`. Для `float32` — примерно `3.40e+38`; для `float64` — примерно `1.80e+308`.

```nim
import std/fenv
echo maximumPositiveValue(float32)   # 3.4028235e+38
echo maximumPositiveValue(float64)   # 1.7976931348623157e+308
```

---

### `epsilon`

```nim
template epsilon*(T: typedesc[float32]): float32
template epsilon*(T: typedesc[float64]): float64
```

**Машинный эпсилон**: разность между `1.0` и следующим большим представимым значением. Это наименьшее `e` такое, что `1.0 + e != 1.0`. Для `float32`: примерно `1.19e-7`; для `float64`: примерно `2.22e-16`.

Машинный эпсилон фундаментален в численном анализе — он определяет относительную точность типа и служит порогом при приближённом сравнении чисел с плавающей точкой на равенство.

```nim
import std/fenv

# Корректный способ сравнения чисел с плавающей точкой:
proc nearlyEqual(a, b: float64): bool =
  abs(a - b) < epsilon(float64) * max(abs(a), abs(b))

doAssert nearlyEqual(0.1 + 0.2, 0.3)
```

## Полные практические примеры

### Паттерн 1 — Проверка вычислительного блока на наличие исключений

```nim
import std/fenv
import std/math

proc safeCompute(x: float64): float64 =
  discard feclearexcept(FE_ALL_EXCEPT)

  result = sqrt(x) * log(x + 1.0)

  let flags = fetestexcept(FE_ALL_EXCEPT)
  if (flags and FE_INVALID)   != 0: echo "Внимание: недопустимая операция"
  if (flags and FE_DIVBYZERO) != 0: echo "Внимание: деление на ноль"
  if (flags and FE_OVERFLOW)  != 0: echo "Внимание: переполнение"

echo safeCompute(4.0)    # чисто
echo safeCompute(-1.0)   # вызывает FE_INVALID (log от отрицательного)
```

---

### Паттерн 2 — Интервальная арифметика с направленным округлением

```nim
import std/fenv

proc intervalAdd(aLo, aHi, bLo, bHi: float64): (float64, float64) =
  let saved = fegetround()

  discard fesetround(FE_DOWNWARD)
  let lo = aLo + bLo

  discard fesetround(FE_UPWARD)
  let hi = aHi + bHi

  discard fesetround(saved)
  (lo, hi)

let (lo, hi) = intervalAdd(0.1, 0.1, 0.2, 0.2)
echo "0.1 + 0.2 гарантированно в [", lo, ", ", hi, "]"
```

### Паттерн 3 — Non-stop секция с `feholdexcept` / `feupdateenv`

```nim
import std/fenv

var savedEnv: Tfenv
discard feholdexcept(addr savedEnv)   # заглушаем все исключения

# Этот блок может давать исключительные результаты, но без сигналов
var acc = 0.0
for i in 1..1000:
  acc += 1.0 / float(i)   # FE_INEXACT накапливается молча

discard feupdateenv(addr savedEnv)    # восстанавливаем и переподнимаем флаги

if fetestexcept(FE_INEXACT) != 0:
  echo "Результаты суммирования были округлены"
```

## Возвращаемые значения

Все процедуры `fe*` возвращают `cint`. Соглашение (унаследованное от C):

| Возвращаемое значение | Смысл |
|----------------------|-------|
| `0` | Успех |
| Ненулевое | Ошибка (операция не поддерживается или завершилась неудачей) |

В продакшн-коде всегда проверяйте возвращаемое значение. На большинстве платформ все операции успешны, но переносимость требует проверки.
